# Functional Description

Functional Description

Enables and disables the device identification. In order to facilitate identifying Sercos slaves in the machine, the parameter IdentifyDevice exists. The precondition for this functionality is an operational Sercos communication (Phases 2–4).

| Value | Data type | Meaning |
| --- | --- | --- |
| off / 0 | BOOL | Default value; device identification disabled. |
| on / 1 | BOOL | Device identification by the orange flashing of the S3 LED on the device is to be identified. |

NOTE: The parameter value is transferred from the master to the slave via the parameter channel of the Sercos at every access. Typically, this takes about 10 ms. However, times up to 1 s can occur if there is a lot of data transferred on the parameter channel.
