Functional Description

FeedbackCurrent only contains the torque-forming current (more information see Parameter [Current](RefActualValues-17.htm#XREF_D_SE_0071519_1)).

The parameter shows the effective value of the reference current portion at the output of the speed controller in milliampere (mA). The proportion from the current supply (CurrentFeedForward) is deducted from the overall reference current.

The parameter [Direction](../Mechanic_2/Mechanic_2-4.htm#XREF_D_SE_0071840_1)  must be taken into account when interpreting FeedbackCurrent.

A reference period cannot be specified as this parameter is not transferred via the real-time channel of the Sercos bus.

NOTE: The parameter value is transferred from the slave to the master via the parameter channel of the Sercos by every access. Typically, this takes about 10 ms. By a high capacity of the parameter channel, times up to 1 s can occur. If the Sercos bus is in phase 0 or 1, then a standard value is indicated here. If the Sercos bus is in phase 3 or 4, then the parameter value is transferred and indicated. In the Sercos phase 2, the parameter can be read through the application.

This parameter has no effect for asynchronous motors in open-loop V / f mode ([ControlMode](../ControlLoop_2/ControlLoop_2-2.htm#XREF_D_SE_0071561_1) = open-loop control / 1).
