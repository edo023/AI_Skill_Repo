The EncoderGearIn / [EncoderGearOut](Encoder_2-6.htm#XREF_D_SE_0071807_1) parameters describe a gear factor between motor and encoder. This gear factor can be different from the gear factor between motor and load (parameter [GearIn](../Mechanic_2/Mechanic_2-2.htm#XREF_D_SE_0071811_1) / [GearOut](../Mechanic_2/Mechanic_2-3.htm#XREF_D_SE_0071838_1)).

The parameter EncoderGearIn indicates the number of teeth on the motor side.

NOTE: The parameter is supported as of version V01.36.xx.00.

![G-SE-0061862.1.gif](images/G-SE-0061862.1.gif)

NOTE: Modifications to the parameter are only applied during the Sercos phase up (communication phase 0 => communication phase 4).

This parameter has no effect for asynchronous motors in open-loop V / f mode ([ControlMode](../ControlLoop_2/ControlLoop_2-2.htm#XREF_D_SE_0071561_1) = open-loop control / 1).

Synchronous Motors with EncoderGearIn > 1

If EncoderGearIn > 1 is set, then the commutation of the motor that is saved in the motor nameplate is not evaluated. Saving the commutation into the motor nameplate with the object parameter [MotorCommutationControl](../Motor_2/Motor_2-19.htm#XREF_D_SE_0071796_1) is not possible. The commutation has to be determined over again every time the drive is newly switched on or reset.
