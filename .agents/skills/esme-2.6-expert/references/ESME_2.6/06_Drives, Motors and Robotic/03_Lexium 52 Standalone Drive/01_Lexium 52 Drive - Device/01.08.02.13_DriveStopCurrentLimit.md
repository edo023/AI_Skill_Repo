# DriveStopCurrentLimit

![](images/logo.png)

DriveStopCurrentLimit

General

The parameter is used for the limitation of the peak current during a [device reaction](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Diagnostic&topicID=D_SE_0063389_1) BD1 or BD2 in the drive. If for this parameter, a value greater than 10 % of [MaxDrivePeakCurrent](General_2-6.htm#XREF_D_SE_0071528_1) and less than the MaxDrivePeakCurrent is entered, then this value is used as peak current during the drive reaction best possible stop (BD1) or user-defined stop (BD2).

The value zero disables this function. Then the drive is stopped with MaxDrivePeakCurrent. No additional limit is active during BD1 and BD2.

The parameter is written to the drive during phase up of the Sercos bus (Phase 0 -> 4).

The magnetizing current (asynchronous motors) is not limited (more information see [Current](../RefActualValues/RefActualValues-17.htm#XREF_D_SE_0071519_1) parameter).

NOTE:  A modified parameter value is only transferred with a new Sercos phase up (phase 0 -> 4).

This parameter modifies the limits of the parameters [ControllerStopDec](General_2-10.htm#XREF_D_SE_0071532_1) and [ControllerStopJerk](General_2-11.htm#XREF_D_SE_0071533_1). The corresponding stop profile is modified especially when the parameters ControllerStopDec or ControllerStopJerk are set to zero. This is because of the limited current during the stop profile that leads to a reduced acceleration.
