# AdditionalIDNListMDTLength

![](images/logo.png)

AdditionalIDNListMDTLength

General

|  |  |
| --- | --- |
| Type | AD |
| Devices supporting the parameter | Lexium LXM52 Drive, Lexium LXM52 Linear Drive,  Lexium LXM62 Drive, Lexium LXM62 Linear Drive,  Lexium ILM62 Drive Module,  Lexium LXM62 Power Supply |
| Traceable | No |

Indicates the length (in a string separated by semicolon) of the additional configured master data. The addresses of the master data can be calculated together with the parameter [AdditionalIDNListMDTAdr](SystemInternals_2-7.htm#XREF_D_SE_0071929_1).

Example: 2;

The standard value is ''. Currently there are no additional drive data in the real-time channel.

If there is an error detected while evaluating the IDN string, the following messages can occur instead of the numerical values:

o'ERROR read attribute (xxxx)' -> The length of the parameter could not be read. The exact detected error code is indicated in brackets. For example, (0x1001) = IDN is not supported by this device.

o'ERROR wrong attribute (xxxx)' -> The parameter has a non-supported data type. Only parameters with a static data length 2, 4 byte, and 8 byte are supported. The read out Sercos attribute is indicated in brackets.

o'ERROR parsing IDN-List!' -> The IDN string could not be evaluated correctly.

Numerical values and detected error messages can also occur in a mixed manner. In this case, only the IDNs are configured in the real-time data if the numerical values are identified as valid.
