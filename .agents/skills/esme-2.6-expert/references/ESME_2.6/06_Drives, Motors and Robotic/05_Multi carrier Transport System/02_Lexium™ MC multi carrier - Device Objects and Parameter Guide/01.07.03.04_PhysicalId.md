# PhysicalId

## General

|  |  |
| --- | --- |
| Type | AF |
| Offline editable | No |
| Devices supporting the parameter | Lexium MC Carrier |
| Traceable | Yes |

## Functional description

Smart Carriers are carriers that are provided with a pre-defined ID that is hard-coded in the hardware of the carrier.

The parameter PhysicalId indicates the ID of the Smart Carrier on the track(s).

| Value | Data type | Meaning |
| --- | --- | --- |
| (0...4294967295) | UDINT | Indicates the ID of the Smart Carrier on the track(s). |

The function FC\_ReadCarriersPhysicalIds allows to read the IDs of the carriers on a track. For Basic Carriers, the value of the PhysicalId is 0. For Smart Carriers, the hard-coded ID is indicated.

For more information on the function FC\_ReadCarriersPhysicalIds, refer to the [SystemInterface library](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.SystemInterface&topicID=ReadCarriersPhysicalId_Gen_420EB61F).

When you have configured or updated the [ConfiguredLogicalId](ConfigLogicId-9C6C7FF5.html#ConfigLogicId-9C6C7FF5), the function FC\_PairAllCarriers allows you to pair the logical carriers (carrier device tree objects) with the physical carriers on a Lexium™ MC multi carrier track by pairing the [LogicalId](LogicId-9C6BFB2C.html#LogicId-9C6BFB2C) to the modified ConfiguredLogicalId.

For more information on the function FC\_PairAllCarriers, refer to the [SystemInterface library](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.SystemInterface&topicID=FC_PairAllCarriers_Gen_4218A0D7).
