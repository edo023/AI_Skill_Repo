# ConfiguredLogicalId

## General

|  |  |
| --- | --- |
| Type | EF |
| Offline editable | Yes |
| Devices supporting the parameter | Lexium MC Carrier |
| Traceable | Yes |

## Functional description

The parameter represents the ID that is expected for the carrier on the track(s).

During the Sercos phase-up, the system connects the physical carrier to the object in the Devices tree according to the configured logical ID.

  

NOTE: For carriers in WorkingMode = real / 1, the possible values range from 1 to the number of physical carriers on the track. For multi-track configurations, you must assign unique ConfiguredLogicalIds to the carriers on all tracks. For example, for track 1 with 20 physical carriers, you must assign the ConfiguredLogicalIds 1 to 20. For track 2 and so forth, the ConfiguredLogicalIds must continue with the subsequent number, in this case number 21.

For more information on the assignment of carrier IDs, refer to [LogicalId](../../../../../api/crossBook?lang=en-US&virtualBookName=LogicId-9C6BFB2C.html#LogicId-9C6BFB2C) and to the [Assignment of Carrier ID](../../MLSLib&topicID=IntroMC_AssignCarrID_10C5290E) in the Multicarrier library.

| Value | Data type | Meaning |
