# RefVelocity

## General

|  |  |
| --- | --- |
| Type | AF |
| Offline editable | No |
| Devices supporting the parameter | Lexium MC Carrier |
| Traceable | Yes |

## Functional Description

The parameter represents the reference velocity, which is derived from the reference position RefPosition, in units per second.

NOTE: In PacDrive systems, the user coordinate system is represented in units. Depending on the setting of the FeedConstant (units/revolution or units/m), the user coordinate system is mapped to the physical dimensions. In a Lexium™ MC multi carrier transport system, one user unit is mapped to 1 mm (0.04 in) on the physical track. The setting of a FeedConstant is not supported.

Coordinate displacements with [FC\_SetposSingle()](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.SystemInterface&topicID=D_SE_0085319) do not affect the parameter RefVelocity.

The value of RefVelocity is calculated once per Sercos cycle (see CycleTime).   
For more information on the CycleTime, refer to the [LMC Pro Device Objects and Parameters Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LMCPro&topicID=D_SE_0073362).

NOTE: If EnableSet inside the user function CarrierState (see [CarrierState](CarrState-9D6C246C.html#CarrState-9D6C246C)) is disabled, the value of Position (see [Position](Position-9C72977E.html#Position-9C72977E)) is cyclically copied to RefPosition (see [RefPosition](RefPosition-9C732D91.html#RefPosition-9C732D91)). RefVelocity is calculated from this path of positions. Disabling EnableSet while the carrier is still in motion causes a jump in RefPosition. When calculating RefVelocity from RefPosition, this jump is indicated. For this reason, the derivation causes an errant value in RefVelocity. The carrier does not react to this errant value as it no longer follows the reference values when EnableSet is disabled.

| Data type | Meaning |
| --- | --- |
| LREAL | The reference velocity of the carrier, derived from the reference position RefPosition. |
