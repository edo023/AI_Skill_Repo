# CyberSecurityDeviceLockStatus

## General

|  |  |
| --- | --- |
| Type | AF |
| Offline editable | No |
| Devices supporting the parameter | Lexium MC Segment |
| Traceable | Yes |

## Functional description

The parameter CyberSecurityDeviceLockStatus indicates whether additional services can be used on the segment firmware.

The default value for this parameter is locked / 1.

For in-depth diagnosis, a cybersecurity device unlock/lock operation can be performed by authorized and qualified personnel (Schneider Electric field application engineer).

Segments are shipped by Schneider Electric in the cybersecurity device state locked.

If the cybersecurity device state of a segment is set to unlocked, the segment can potentially be accessed without authorization.

| WARNING | |
| --- | --- |
|  | UNAUTHENTICATED ACCESS AND SUBSEQUENT UNAUTHORIZED MACHINE OPERATION  Ensure that the cybersecurity device state of the segment is set back to “locked” before you put your system back into operation.  Failure to follow these instructions can result in death, serious injury, or equipment damage. |

| Value | Data type | Meaning |
| --- | --- | --- |
| unknown / 0 | DINT | The state of the device has not yet been read and therefore initialized. |
| locked / 1 | DINT | The default state of the device. No additional services can be used on the segment firmware. |
| unlocked / 2 | DINT | The state “unlocked” enables in-depth diagnosis by authorized and qualified personnel (Schneider Electric field application engineer). |
