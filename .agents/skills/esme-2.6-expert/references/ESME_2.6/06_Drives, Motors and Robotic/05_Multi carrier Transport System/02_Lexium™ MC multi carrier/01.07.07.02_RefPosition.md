# RefPosition

## General

|  |  |
| --- | --- |
| Type | AF |
| Offline editable | No |
| Devices supporting the parameter | Lexium MC Carrier |
| Traceable | Yes |

## Functional Description

The parameter represents the reference position that is generated by the reference position generator in units.

NOTE: In PacDrive systems, the user coordinate system is represented in units. Depending on the setting of the FeedConstant (units/revolution or units/m), the user coordinate system is mapped to the physical dimensions. In a Lexium™ MC multi carrier transport system, one user unit is mapped to 1 mm (0.04 in) on the physical track. The setting of a FeedConstant is not supported.

Coordinate displacements with [FC\_SetposSingle()](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.SystemInterface&topicID=D_SE_0085319) affect the parameter RefPosition.

NOTE: If EnableSet inside the user function CarrierState (see [CarrierState](CarrState-9D6C246C.html#CarrState-9D6C246C)) is disabled, the value of Position (see [Position](Position-9C72977E.html#Position-9C72977E)) is cyclically copied to RefPosition. If EnableSet is disabled while the carrier is still in motion, a jump occurs in RefPosition. The carrier does not react to this jump because it no longer follows the setpoint settings when EnableSet is disabled.

| Data type | Meaning |
| --- | --- |
| LREAL | The reference position of the carrier in units. |
