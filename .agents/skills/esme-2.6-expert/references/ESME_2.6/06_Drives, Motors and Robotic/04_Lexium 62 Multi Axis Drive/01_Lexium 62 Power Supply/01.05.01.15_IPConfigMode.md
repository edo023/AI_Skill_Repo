# IPConfigMode

## General

|  |  |
| --- | --- |
| Type | EF |
| Offline editable | Yes |
| Devices supporting the parameter | Lexium LXM52 Drive, Lexium LXM52 Linear Drive,  Lexium LXM62 Drive, Lexium LXM62 Linear Drive,  Lexium ILM62 Drive Module,  Lexium LXM62 Power Supply,  Sercos I/O,  TM5CSLCx00FS,  Bus Interface TM5NS31,  Logic Motion Controller Device |
| Traceable | Yes |

## Functional Description

This parameter configures the behavior of the IP address configuration.

| Value | Data type | Meaning |
| --- | --- | --- |
| Automatic IP addressing / 0 (default) | DINT | The slave settings are performed automatically.  The automatically generated configuration is displayed in IPAddress, Subnetmask, and Gateway.  This is the default value for the devices except for the Safety Logic Controller.  This value cannot be set for the Safety Logic Controller.  Access type: read/write |
| manual IP addressing (hardware) / 1 | DINT | The settings are read from the slave device and displayed in IPAddress, Subnetmask, and Gateway.  The settings must be performed on the hardware (for example, using switches) or via the drive assistant.  This value cannot be set for the Safety Logic Controller.  Access type: read/write |
| manual IP addressing (software) / 2 | DINT | The settings IPAddress, Subnetmask, and Gateway are transferred to the slave.  This value can be set for the Safety Logic Controller.  Access type: read/write |

During the setting of the configuration, the parameter is verified for the following:

* For double drives: The specified IPConfigMode is the same for both drives.

Also see diagnostic message `8967 NRT IP parameter device different`.
