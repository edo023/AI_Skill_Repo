# MotorTemp

## General

|  |  |
| --- | --- |
| Type | AS |
| Devices supporting the parameter | Lexium LXM52 Drive, Lexium LXM52 Linear Drive,  Lexium LXM62 Drive, Lexium LXM62 Linear Drive,  Lexium ILM62 Drive Module |
| Traceable | No |

## Functional Description

The parameter indicates the motor temperature in °C. Linear or nonlinear temperature sensors can be used in the motor.

## Linear Temperature Sensors

Linear temperature sensors can (in certain areas) detect the complete temperature range linear. When using a linear temperature sensor, the parameter can represent temperatures in the range of -10...999 °C.

## Nonlinear Temperature Sensors

Nonlinear switching temperature sensors can only present two conditions; for example, temperature too high / too low. When using a nonlinear temperature sensor (for example, a PTC temperature sensor), the parameter can only present two conditions.

| Condition | Meaning |
| --- | --- |
| 0 | The motor temperature is OK.  The motor can be operated. |
| 999 | The motor temperature is too high.  The motor cannot be operated. |

For further information on the temperature sensor, see the SH servo motor operator manual.

NOTE: The parameter value is transferred from the slave to the master via the parameter channel of the Sercos by every access. Typically, this takes about 10 ms. However, up to 1 s can be realized if large amounts of data are transferred on the parameter channel. If the Sercos bus is in phase 0 or 1, then a default value is indicated here. If the Sercos bus is in phase 3 or 4, then the parameter value is transferred and indicated. In the Sercos phase 2, the parameter can be read through the application.
