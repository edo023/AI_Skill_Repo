# ShaftRefDelay

## General

|  |  |
| --- | --- |
| Type | AD |
| Devices supporting the parameter | Lexium LXM52 Drive, Lexium LXM52 Linear Drive,  Lexium LXM62 Drive, Lexium LXM62 Linear Drive,  Lexium ILM62 Drive Module,  Sercos Drive |
| Traceable | Yes |

## Functional Description

ShaftRefDelay indicates the delay between the drive shaft and the reference values. This delay is negative because the reference values are already available before they take effect on the drive. The delay arises through the constant dead times which occur because of the processing and transfer of the signals via the Sercos to the drive.

The delay ShaftRefDelay is contained in the parameters RefPosition, MechRefPosition, RefVelocity, MechRefVelocity, and RefAcceleration. By the velocities RefVelocity and MechRefVelocity the dead times are reduced by *[CycleTime](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LMCEco&topicID=D_SE_0073362)*/2, by the acceleration RefAcceleration by *[CycleTime](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LMCEco&topicID=D_SE_0073362)*.

By the parameter ShaftMechRefPosition this dead time is compensated.
