# IO0\_Mode, IO1\_Mode

## General

|  |  |
| --- | --- |
| Type | EF |
| Offline editable | Yes |
| Devices supporting the parameter | OnBoard I/O |
| Traceable | Yes |

## Functional Description

The parameter defines the functionality of a field input.

| Value | Data type | Meaning |
| --- | --- | --- |
| Input / 0 | DINT | Only the static state of the input (Value parameter) can be queried. |
| Touchprobe / 1 | DINT | The touchprobe function is activated. This functionality latches a Position value by an edge at the field input with a high time resolution. See also the CaptureState, CaptureOk, CaptureValue, and SensorDelay parameters. |
| Fast counter / 2 | DINT | The impulse counter function is activated. This functionality is able to count pulses at a field input. See also CounterState, CounterValue, CounterStart, and CounterEdge. |

NOTE: Modifications to the parameter are only applied during the Sercos phase up (communication phase 0 => communication phase 4).

NOTE: The first measured input (TP\_0) is assigned to IO\_0 and the second is assigned to IO\_1.
