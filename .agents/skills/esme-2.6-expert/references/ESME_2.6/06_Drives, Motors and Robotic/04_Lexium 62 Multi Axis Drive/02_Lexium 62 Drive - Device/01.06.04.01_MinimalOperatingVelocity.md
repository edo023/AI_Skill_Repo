# MinimalOperatingVelocity

## General

|  |  |
| --- | --- |
| Type | AF |
| Devices supporting the parameter | Lexium LXM62 Drive |
| Device variants supporting the parameter | Standard Plus, Advanced Plus, Application defined |
| Traceable | Yes |

## Functional Description

The parameter represents the threshold velocity in Unit/s as of which the motor is switched between open-loop control mode and closed-loop control mode. Encoderless closed-loop control is only possible above this threshold velocity.

* When the MinimalOperatingVelocity limit is exceeded, the motor operates in encoderless closed-loop control mode (the speed and position controller is closed). (See [Encoderless Velocity Control User Function](D-SE-0090124.html#D-SE-0090124).)
* When the velocity is less than the limit, the motor runs in open-loop control (the speed and position controller is not closed). (See [Encoderless Velocity Control User Function](D-SE-0090124.html#D-SE-0090124).)

NOTE: The parameter value is transferred from the master to the slave via the parameter channel of the Sercos at every access. Typically, this takes about 10 ms. However, times up to 1 s may be realized if large amounts of data are transferred on the parameter channel.
