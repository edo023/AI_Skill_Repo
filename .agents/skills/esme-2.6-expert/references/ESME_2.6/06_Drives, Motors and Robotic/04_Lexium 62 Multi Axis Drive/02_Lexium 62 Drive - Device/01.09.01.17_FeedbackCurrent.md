# FeedbackCurrent

## General

|  |  |
| --- | --- |
| Type | AS |
| Devices supporting the parameter | Lexium LXM52 Drive, Lexium LXM52 Linear Drive,  Lexium LXM62 Drive, Lexium LXM62 Linear Drive,  Lexium ILM62 Drive Module |
| Traceable | No |

## Functional Description

The parameter represents the effective value of the reference current portion at the output of the speed controller in milliampere (mA). The proportion from the current supply (CurrentFeedForward) is deducted from the overall reference current.

FeedbackCurrent only contains the torque-forming current (more information see Parameter Current).

The parameter Direction  must be taken into account when interpreting FeedbackCurrent.

A reference period cannot be specified as this parameter is not transferred via the real-time channel of the Sercos bus.

NOTE: The parameter value is transferred from the slave to the master via the parameter channel of the Sercos by every access. Typically, this takes about 10 ms. However, up to 1 s can be realized if large amounts of data are transferred on the parameter channel. If the Sercos bus is in phase 0 or 1, then a default value is indicated here. If the Sercos bus is in phase 3 or 4, then the parameter value is transferred and indicated. In the Sercos phase 2, the parameter can be read through the application.

This parameter has no effect for asynchronous motors in open-loop V / f mode (ControlMode = open-loop control / 1) and synchronous motors without encoders used with the encoderless velocity control below the MinimalOperatingVelocity.
