# TorqueLimitationMode

## General

|  |  |
| --- | --- |
| Type | ES |
| Devices supporting the parameter | Lexium LXM62 Drive |
| Device variants supporting the parameter | Standard Plus,  Advanced Plus,  Application defined |
| Traceable | Yes |

## Functional Description

The parameter is used to set the mode for torque limitation on load side.

| Value | Meaning |
| --- | --- |
| No additional limitation / 0 | The torque limitation is deactivated.  The settings for the parameters AccelerationTorqueLimit, DecelerationTorqueLimit, and TorqueLimitationFilter are not taken into account. |
| Acceleration dependent torque limitation / 1 | The motor current is limited to the set torque limits depending on the present acceleration or deceleration to get an adjustable maximum absolute level of torque at the gearbox output.  The torque limits during acceleration and deceleration can be configured individually via the parameters AccelerationTorqueLimit and DecelerationTorqueLimit. When a limit value is set to zero, the corresponding torque limit is deactivated. |
| Mechanical overload protection / 2 | As long as the torque is below the set torque limits, full torque is available at the gearbox output. When the defined levels are exceeded (due to an overload), the torque transmission is interrupted and the motor is switched torque free and the diagnostic message `8191 Mechanical overload protection` is triggered.  A filter (set via the TorqueLimitationFilter parameter) can be activated to smooth the detected torque, thus allowing to set the sensibility. As long as the smoothed signal does not exceed a torque limit, full torque is available. When the smoothed signal reaches the limits, overload protection becomes active (torque transmission is interrupted and motor is switched torque free).  The torque limits for the acceleration and deceleration are set via the parameters AccelerationTorqueLimit and DecelerationTorqueLimit. |

NOTE: The parameter value is transferred from the master to the slave via the parameter channel of the Sercos at every access. Typically, this takes about 10 ms. However, up to 1 s can be realized due to the volume of data transferred on the parameter channel.
