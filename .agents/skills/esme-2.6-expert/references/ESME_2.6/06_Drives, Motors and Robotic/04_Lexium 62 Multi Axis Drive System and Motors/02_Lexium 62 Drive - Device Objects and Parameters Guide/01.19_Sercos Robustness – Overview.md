# Sercos Robustness – Overview

## Overview

The Sercos Robustness feature allows you to specify the Sercos response to an error detected at the device level in communication phase 4. If a device error is detected in communication phase 4, the default error response is phasing down to communication phase 0. With this feature, the Sercos bus can remain in communication phase 4 even if an error is detected at the device level and the affected device is no longer operative.

The internal error response of the affected device remains effective.

A Sercos master object is provided as a global parameter that contains the error response defined at the device level. It is used to indicate the specified response of a device in the diagnostics file.

A Sercos device object sets the type of error response using the following parameters:

* ConfiguredSystemReaction
* SystemReaction
* Valid

The specific response for a device is defined in the application.

## Activating Sercos Robustness

To activate the Sercos Robustness feature, display the Feature Configuration tab of the device and check the checkboxes  Enable Feature Configuration and  Sercos Robustness.

## Error Responses

The default error response to a device error is detected in communication phase 4 is phasing down to communication phase 0.

If the Sercos Robustness is activated, the following response is triggered, depending on the device type:

Drive:

* Sercos remains in communication phase 4.
* The value of the parameter WorkingState is set to Error (the working state is changed in the real-time process).
* The affected drive is treated as if it was a virtual device.
* Movements are not aborted by the system.
* The diagnostics message “8533 SERCOS Slave unresponsive” is generated.

NOTE: You need to implement an appropriate response in the application via the event handler.

IO device:

* Sercos remains in communication phase 4.
* The value of the parameter WorkingState is set to Error (the working state is changed in the real-time process).
* The value of the parameters OutputsActive and InputsValid are set to False.
* The diagnostics message “8533 SERCOS Slave unresponsive” is generated.
* The IO data remain unaffected.

NOTE: You need to implement an appropriate response in the application via the event handler.

## Event Handler

The Sercos Robustness feature provides the event sercos\_DeviceDisconnected. This event is triggered when the value of the parameter WorkingState of a device configured for a non-default error response set to Error. You can use this event to trigger a response in your application via the appropriate event handler.

The function is executed in the real-time process. Therefore, only use it to trigger the response in your application.

To add the required event handler, display the tab  System Events on the tab  Task Configuration of the device. Click the button  Add Event Handler. The dialog box  Add Event Handler opens. From the listbox  Event , select the list item  sercos\_DeviceDisconnected. Select the  Scope and the  Application language according to the requirements of your application.
