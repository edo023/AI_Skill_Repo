# CurrentLimitationState

## General

|  |  |
| --- | --- |
| Type | AS |
| Devices supporting the parameter | Lexium LXM62 Drive,  Lexium LXM62 Linear Drive |
| Traceable | Yes |

## Functional Description

The parameter represents the state of the effective current limitation, i.e., which current limitation has the lowest threshold. The effective current limitation threshold is indicated via the CurrentLimitationTreshold parameter.

| Value | Meaning |
| --- | --- |
| 0 | No value available |
| 1 | Current is limited to the drive peak current (see parameters MaxDrivePeakCurrent and UserDrivePeakCurrent). |
| 2 | Current is limited to the peak current (see DriveStopCurrentLimit parameter). |
| 3 | Current is limited to the reference current of the drive (see UserCurrentLimit parameter). |
| 4 | Current is limited dependent on the present acceleration/deceleration (see parameters AccelerationTorqueLimit and DecelerationTorqueLimit). |
| 5 | Speed dependent current reduction.  (See the diagnostic message “*[8133 Speed-dependent current reduction](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Diagnostic&topicID=D_SE_0063869)*”.) |

NOTE: The parameter value is transferred from the master to the slave via the parameter channel of the Sercos at every access. Typically, this takes about 10 ms. However, times up to 1 s may be realized if large amounts of data are transferred on the parameter channel.
