# Functional Description

Functional Description

The parameter is used to set the maximum velocity on the drive shaft (gear box outside) and is specified in [units/seconds].

With physical axes, the UserMaxVel can be set up to the equivalent value that corresponds to the maximum motor rotational speed [MaxSpeed](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM62Drive&topicID=D_SE_0071784_1)  [RPM]. With the UserMaxVel parameter, the maximum velocity [MaxVel](General_2-2.htm#XREF_D_SE_0071523_1), determined by the system, can be limited. This parameter defines the machine-specific value of MaxVel.

The default value of MaxVel is overwritten by virtual axes.

If UserMaxVel is set to 0, then the parameter MaxVel is determined by the system.

NOTE:  If the rated speed is exceeded, then the motor picks up less current by increasing rotational speed. The amplitude of the current depends on the momentary voltage in the DC bus. The current is reduced until a further increasing of the speed is not possible anymore. Depending on the induced voltage of the motor used, it is possible that MaxSpeedLinear cannot be reached (the typical DC bus voltage on a 3-phase 400 V network is 560 V, even though MaxSpeedLinear requires up to 780 V). Verify that enough voltage/current is available for the speed requirements of your application (see Drive sizing with Motion Builder).

UserMaxVel influences the threshold for triggering the diagnostic message [8122 Shutdown due to velocity limit](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Diagnostic&topicID=D_SE_0063862_3).

NOTE: Modifications to the parameter are only applied during the Sercos phase up (communication phase 0 => communication phase 4). The maximum user speed is not supervised for induction motors without encoder (in open-loop V / f control mode, [ControlMode](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM52Drive&topicID=D_SE_0071561_1) = open-loop control / 1).
