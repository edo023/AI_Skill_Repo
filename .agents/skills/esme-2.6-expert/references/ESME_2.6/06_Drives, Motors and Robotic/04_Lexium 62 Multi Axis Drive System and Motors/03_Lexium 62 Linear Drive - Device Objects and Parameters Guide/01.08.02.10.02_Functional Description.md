# Functional Description

Functional Description

ShaftDelay provides the delay between the drive shaft and the actual values. The delay is the result of the constant dead times, which are caused by the processing and the transfer of the signals from the drive via the Sercos.

The delay ShaftDelay is contained in the parameters [Position](../../../../../../api/crossBook?lang=en-US&virtualBookName=RefActualValues-7.htm#XREF_D_SE_0071521_1), [MechPosition](RefActualValues-8.htm#XREF_D_SE_0071499_1), [EncoderPosition](../Encoder_2/Encoder_2-2.htm#XREF_D_SE_0071802_1), [TrackingDeviation](RefActualValues-11.htm#XREF_D_SE_0071507_1), [Velocity](RefActualValues-14.htm#XREF_D_SE_0071516_1), [MechVelocity](RefActualValues-15.htm#XREF_D_SE_0071517_1), and [Current](RefActualValues-17.htm#XREF_D_SE_0071519_1). There are additional dead times of [CycleTime](../../../PD.Parameter.LMCEco&topicID=D_SE_0073362_1)/2 for the velocities Velocity and MechVelocity.

For the parameter [ShaftMechPosition](RefActualValues-10.htm#XREF_D_SE_0071503_1), this dead time is compensated (see [Ref-Actual Values](RefActualValues-1.htm#XREF_D_SE_0071489_1)).

This parameter has no meaning for asynchronous motors without encoder (in open-loop V / f control mode, [ControlMode](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM52Drive&topicID=D_SE_0071561_1) = open-loop control / 1).
