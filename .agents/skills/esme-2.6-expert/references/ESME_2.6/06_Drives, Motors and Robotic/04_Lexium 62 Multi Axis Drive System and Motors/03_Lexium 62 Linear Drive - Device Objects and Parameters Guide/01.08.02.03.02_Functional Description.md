# Functional Description

Functional Description

RefPosition represents the reference position that is generated by the reference position generator (for example, MasterCam, PosGenerator, …) in units. It is indicated on the drive shaft (gear box output side). The [YOffsetPosition](../YOffsetGenerator_2/YOffsetGenerator_2-4.htm#XREF_D_SE_0071850_1) from the Y offset generator is not taken into account. Therefore it is not the reference position that is transferred to the drive (see [Ref-Actual Values](RefActualValues-1.htm#XREF_D_SE_0071489_1)).

Coordinate displacements with SetPos ([FC\_SetposDual()](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.SystemInterface&topicID=D_SE_0085315_1), [FC\_SetposGroup()](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.SystemInterface&topicID=D_SE_0085317_1), [FC\_Setpos­Single()](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.SystemInterface&topicID=D_SE_0085319_1)) affect this parameter. The parameter [Direction](../../../../../../api/crossBook?lang=en-US&virtualBookName=Mechanic_2/Mechanic_2-2.htm#XREF_D_SE_0071840_1) has to be taken into account by the interpretation of RefPosition. The value of RefPosition is calculated once every Sercos cycle ([CycleTime](../../../PD.Parameter.LMCEco&topicID=D_SE_0073362_1)). Because the [ShaftRefDelay](RefActualValues-5.htm#XREF_D_SE_0071497_1) is negative, the following is defined: RefPosDelay = -ShaftRefDelay

In relation to the reference position for the drive shaft, the RefPosition is accelerated by the time RefPosDelay. Therefore, a reference position without [YOffsetPosition](../YOffsetGenerator_2/YOffsetGenerator_2-4.htm#XREF_D_SE_0071850_1) is indicated for the drive shaft after the time RefPosDelay.

NOTE:  If [ControllerEnable](../State_2/State_2-9.htm#XREF_D_SE_0071477_1) is disabled, the value of [Position](RefActualValues-7.htm#XREF_D_SE_0071521_1) is cyclically copied to RefPosition. Thus, the drive is automatically synchronized to the position for the next release. If Controller­Enable is switched off while the drive is still in motion, then a jump occurs in RefPosition. The drive does not react to this jump because it no longer follows the set-point settings when ControllerEnable is switched off.
