# DissipatedEnergyDrive

## General

|  |  |
| --- | --- |
| Type | AS |
| Devices supporting the parameter | Lexium LXM52 Drive, Lexium LXM52 Linear Drive,  Lexium LXM62 Drive, Lexium LXM62 Linear Drive,  Lexium ILM62 Drive Module,  Lexium LXM62 Power Supply |
| Traceable | No |
| Version | Valid as of V1.34.X.X |

## Functional Description

The parameter is used to indicate the energy loss in the motor winding.

The parameter has the data type LREAL. The default value is zero.

The default value is represented if the axis is virtual or the parameter was not yet read by the drive. The parameter can only be read if the Sercos bus is at least in phase 2.

Exception: If a controller firmware V1.34.X.X or higher and at the same time a drives firmware V1.31.X.X to V1.33.X.X is installed, then the parameter equals 0.

The dissipated power is measured every millisecond. The power is averaged over 1 s. The energy loss of the drive then results from integrating the power over the duration 1 s.

The energy loss has a positive preceding sign.

NOTE: The energy values are saved regularly. The calculation of the energy values is also continued after switching the device on and off until you stop the energy measurement independently (see EnergyMeasurementEnableSet); otherwise, the energy values are not reset to zero after restarting of the device, but will be continued to be measured.

NOTE: The parameter value is transferred from the slave to the master via the parameter channel of the Sercos by every access. Typically, this takes about 10 ms. However, up to 1 s can be realized if large amounts of data are transferred on the parameter channel. If the Sercos bus is in phase 0 or 1, then a default value is indicated here. If the Sercos bus is in phase 3 or 4, then the parameter value is transferred and indicated. In the Sercos phase 2, the parameter can be read through the application.
