# MotorIdentification

## General

|  |  |
| --- | --- |
| Type | EF |
| Offline editable | Yes |
| Devices supporting the parameter | Lexium LXM52 Drive, Lexium LXM52 Linear Drive,  Lexium LXM62 Drive, Lexium LXM62 Linear Drive,  Lexium ILM62 Drive Module |
| Traceable | Yes |

## Task

The end user custom-assembles a modular machine. Individual machine blocks are dismantled with the motor. The PacDrive controller and the servo amplifier remain central machine components. There is no change to the Sercos loop or bus addresses.

## Functional Description

The parameter MotorIdentification was introduced to support different motor configurations for a drive.

| NOTICE | |
| --- | --- |
|  | INOPERABLE EQUIPMENT  Always remove power from the drive when connecting or disconnecting the motor.  Failure to follow these instructions can result in equipment damage. |

| Value | Data type | Meaning |
| --- | --- | --- |
| Motor with type plate / 0 | DINT | Normal operation. The motor nameplate is saved in the encoder-EEPROM of the motor. |
| motorless / 1 | DINT | No motor is connected to the servo amplifier. Servo amplifier in an inactive status. No motion tasks can be performed. |
| Motor without type plate / 2 | DINT | The motor nameplate is saved in the servo amplifier.  Not supported in V1.31.10.0. Drive reports non-supported mode (service channel diagnostic message). |

NOTE: Modifications to the parameter are only applied during the Sercos phase up (communication phase 0 => communication phase 4).

NOTE: If the MotorIdentification parameter is not correctly configured, the diagnostic message `8997 MotorIdentification invalid` is triggered during the Sercos phase up.
