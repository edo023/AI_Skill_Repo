# Functional Description

Functional Description

ShaftRefDelay indicates the delay between the drive shaft and the displayed reference values. This delay is negative because the reference values are already available before they take effect on the drive. The delay arises through the constant dead times which occur because of the processing and transfer of the signals via the Sercos to the drive.

The delay ShaftRefDelay is contained in the parameters [RefPosition](../../../../../../api/crossBook?lang=en-US&virtualBookName=RefActualValues-2.htm#XREF_D_SE_0071490_1), [MechRefPosition](RefActualValues-4.htm#XREF_D_SE_0071492_1), [RefVelocity](RefActualValues-9.htm#XREF_D_SE_0071512_1), [MechRefVelocity](RefActualValues-11.htm#XREF_D_SE_0071515_1), and [RefAcceleration](RefActualValues-13.htm#XREF_D_SE_0071518_1). By the velocities RefVelocity and MechRefVelocity the dead times are reduced by [CycleTime](../../../PD.Parameter.LMCEco&topicID=D_SE_0073362_1)/2, by the acceleration RefAccel­eration by [CycleTime](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LMCEco&topicID=D_SE_0073362_1).

By the parameter [ShaftMechRefPosition](RefActualValues-17.htm#XREF_D_SE_0071498_1) this dead time is compensated.
