# Functional Description

Functional Description

ShaftMechRefPosition shows the reference position of the drive shaft (gear box output side) in units. The dead time [ShaftRefDelay](../../../../../../api/crossBook?lang=en-US&virtualBookName=RefActualValues-16.htm#XREF_D_SE_0071497_1) is compensated starting from [MechRefPosition](RefActualValues-4.htm#XREF_D_SE_0071492_1). The [YOffsetPosition](../YOffsetGenerator/YOffsetGenerator-4.htm#XREF_D_SE_0071850_1) from the Y offset generator is included. Coordinate displacements with SetPos ([FC\_SetposDual()](../../../PD.Lib.SystemInterface&topicID=D_SE_0085315_1), [FC\_SetposGroup()](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.SystemInterface&topicID=D_SE_0085317_1), [FC\_SetposSingle()](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.SystemInterface&topicID=D_SE_0085319_1)) affect this parameter. The parameter [Direction](../Motor_Mechanic/Motor_Mechanic-4.htm#XREF_D_SE_0071840_1) has to be considered during the interpretation of ShaftMechRefPosition.

The value of ShaftMechRefPosition is calculated once every Sercos cycle ([CycleTime](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LMCEco&topicID=D_SE_0073362_1)).

In relation to the reference position on the drive shaft, there is no delay. However, an acceleration-proportional detected error occurs during the compensation of the dead time.

NOTE: If [ControllerEnable](../State/State-8.htm#XREF_D_SE_0071477_1) is disabled, the value of [ShaftMechPosition](RefActualValues-15.htm#XREF_D_SE_0071503_1) is cyclically copied to ShaftMechRefPosition. Thus, the drive is automatically synchronized to the position for the next release. If ControllerEnable is switched off while the drive is still in motion a jump occurs in ShaftMechRefPosition. The drive does not react to this jump because it no longer follows the set-point settings when ControllerEnable is switched off.

NOTE: The parameter value is transferred from the slave to the master via the parameter channel of the Sercos by every access. Typically, this takes about 10 ms. By a high capacity of the parameter channel, times up to 1 s can occur. If the Sercos bus is in phase 0 or 1, then a standard value is indicated here. If the Sercos bus is in phase 3 or 4, then the parameter value is transferred and indicated. In the Sercos phase 2, the parameter can be read through the application.
