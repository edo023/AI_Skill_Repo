# Step 1: Insert Motor Data in ST_UserMotorDataACIM

![](images/logo.png)

Step 1: Insert Motor Data in ST\_UserMotorDataACIM

Overview

1.A variable of type MTP.ST\_UserMotorData must be created in the code:

VAR  
    stUserMotorData: MTP.ST\_UserMotorData;  
END\_VAR

2.This structure is then filled with the motor data.

etMotorType is set to RotaryACIM since a rotating asynchronous motor should be operated. Most of the data can be taken from the type plate of the asynchronous motor (for example rated speed, rated current, cos phi, rated frequency, and rated power).

In this example, the peak current is used as start-up current of the motor (in the data sheet of the motor the start-up current is indicated as 3.5 \* rated current).

The motor does not have an encoder, thus, the uiEncoderType is set to None.

Moreover, the motor does not have a brake (uiBrake =0) and has a bimetal switch as a temperature sensor (uiTempSensorType=1).

The parameters sMotorname, sMotorSerialNumber, and sMotorArticleNumber are freely definable texts that are displayed in the axis parameters MotorType, MotorSerial­Number, and MotorArticleNumber.

The parameters do not have to be allocated with values, however, they are helpful for identifying the motor.

stUserMotorData.etMotorType := MTP.ET\_MotorType.RotaryACIM;  
stUserMotorData.sMotorname := 'SEW-Eurodrive-Motor';  
stUserMotorData.sMotorArticleNumber := 'DRS71S4/FF/TH/AV1H';  
stUserMotorData.sMotorSerialNumber := '01.1710743001.0002X';  
stUserMotorData.stMotorDataACIM.uiNominalSpeed := 1380;  // rpm   
stUserMotorData.stMotorDataACIM.uiPolePair := 2;  
stUserMotorData.stMotorDataACIM.uiNominalFrequency := 50;  // Hz  
stUserMotorData.stMotorDataACIM.rNominalPower := 0.370;  // kW  
stUserMotorData.stMotorDataACIM.rNominalVoltage := 400;  // V  
stUserMotorData.stMotorDataACIM.rNominalCurrent := 1.14;  // A  
stUserMotorData.stMotorDataACIM.rPeakCurrent := 1.14 \* 3.5;  // A  
stUserMotorData.stMotorDataACIM.uiRotatingFieldDirection := 1;  
stUserMotorData.stMotorDataACIM.uiEncoderType := MTP.ET\_EncoderType.None;  
stUserMotorData.stMotorDataACIM.uiTempSensorType := 1;  // ptc temp sensor  
stUserMotorData.stMotorDataACIM.uiMaxSpeed := 1800;  // rpm  
stUserMotorData.stMotorDataACIM.uiBrake := 0;  // no brake

If the asynchronous motor is to be used in regulated operation, an encoder is required. The used encoder type (for example SinCos 1Vpp) must be entered in the uiEncoderType parameter.

NOTE: When entering the data, the unit of the individual parameters is to be considered. Specifications to the units can be found in the description of the structure ST\_UserMotor­DataACIM of the library UserMotorTypePlate.

Resistance and inductance values must be entered as phase-to-phase value.

Since the motor is equipped with a non-linear bimetal switch whose resistance increases from a very small value to a very large value when reaching the switch-off temperature. In this case, it is enough to adopt the default settings for the thermal monitoring in the type plate.

Generally, these values have to be adjusted. When using a PTC temperature sensor, it is required to determine the resistance value in the uiTempSensorResistanceOvertemp parameter, which defines the switch-off threshold. If there is a linear temperature sensor, motion points of a complete resistance temperature characteristic curve must be implemented instead (auiTempSensorCharacteristics).
