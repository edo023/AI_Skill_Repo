# Branch Management

## Introduction

Branch management allows you to create and independently manage different copies of your solution. The main branch is the default branch that stores the main version of your solution from which new branches can be created.

Creating a new branch allows you to work separately on your solution without affecting the other branches. Afterward, the work done on the separate branches can be merged together.

Click the ![](images/DD01428586.png) icon in the sidebar (upper left corner) of the [Projects Explorer](AutomationSolutionExplorer-4BEC913D.html) to access the Repository window.

The active branch is highlighted and indicates its status (for example HEAD).

You can click a branch to display its content in the form of files and folder.

## Creating a Branch

To create a branch, follow these steps:

| Step | Action |
| --- | --- |
| 1 | In the Repository window, click Create Branch. |
| 2 | In the Create branch window:  * Enter the Name of the branch. * Select source: you can create a branch from the main branch (available by default), or from another branch.  Then click Create branch.  **Result**: The new branch is created. |

The new branch is created from the content of the selected source branch.

You can perform modifications in the new branch without impacting the content of the source branch. The modifications are committed into the new branch.

Use the branch selection menu ![](images/DD01418060.JPG.jpg) in the [Projects Explorer](AutomationSolutionExplorer-4BEC913D.html) to switch the active working branch. To view the content of another branch without switching, select it in the Repository view.

## Merging Branches

For a specific solution, you can work on multiple branches and later merge the content from various branches as needed.

To merge two branches, follow these steps:

| Step | Action |
| --- | --- |
| 1 | In the Repository window, click  and select Merge from the menu. |
| 2 | Select the source branch and the target branch. You can display the list of differences between their contents.  Then click Merge branch.  NOTE: You can merge two branches only if these branches do not have any conflict. In case of conflict, first resolve the conflict(s) in the Control Application tool. |
| 3 | In the Merge branch window:  Select the Merge type:   * Squash merge: it allows you to take the changes from one branch and squash them into a single commit on top of the target branch. * Basic merge: it takes each commit in the source branch and add them to the History of the target branch based on the timestamp of when they were created.   Optionally, you can add comments in the Merge comment text box.  Optionally, you can also choose to delete the source branch after the merge by selecting the Delete source branch after merge check box.  Click Merge branch and then click Close.  **Result**: The two selected branches and their contents are merged. |

## Deleting a Branch

You can delete a branch which is no longer needed.

To delete a branch, follow these steps:

| Step | Action |
| --- | --- |
| 1 | In the Repository window, click  for the branch to delete and select Delete from the menu. |
| 2 | In the Delete branch window, confirm your choice by clicking Delete branch.  **Result**: The selected branch and its content are deleted. |
