# PROFINET Diagnosis

This chapter describes how the diagnosis mechanisms from PROFINET can be used in CODESYS.

The detection and correction of system-related malfunctions (for example, caused by firewalls) is described in the [Troubleshooting](_pnio_f_troubleshooting.html#_pnio_f_troubleshooting) chapter.

To prevent downtimes during the operation of a plant, the diagnosis is intended to allow for quick detection or even prevention of faults. To do this, PROFINET provides extensive diagnostic mechanisms which give information about the source and location of the fault. As a result, a device or module or even a specific I/O channel can be identified as the trigger of the fault. The diagnosis provides additional information about the severity of the error as well as details about the cause.
