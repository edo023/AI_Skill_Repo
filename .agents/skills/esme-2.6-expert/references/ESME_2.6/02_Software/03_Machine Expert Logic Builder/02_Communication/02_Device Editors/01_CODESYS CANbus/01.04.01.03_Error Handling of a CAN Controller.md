# Error Handling of a CAN Controller

Error handling is integrated into the CAN protocol and is extremely significant for the performance of a CAN system. Error handling aims at detecting errors in CAN messages so that the sender can resend a failed message. Every CAN controller tries to detect errors within a message. When an error is found, the detecting node sends an error flag and therefore disrupts the bus traffic. The other nodes will detect the error which was caused by the error flag (if they have not already detected the original error). They will take appropriate action and reject the current message.

These counters and also the bus state often provide initial insights into the cause of the error when diagnosing CAN errors. This information can be read in CODESYS by means of the user interface as well as the application.
