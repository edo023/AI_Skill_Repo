Issue: Random Access Memory Demand of the OPC Server Increases with Increasing Runtime up to the System Limit

Issue

With increasing runtime of the OPC server, the random access memory required by the OPC server is increased up to the system limit.

Cause 1

Clients write cyclically asynchronous values and the OPC server cannot process the queries in simultaneous speed. The effect is that increasingly more memory is required than can be released. The computer does not provide the capacities required for this application.

Solution 1

Use a computer with a higher capacity (RAM, CPU).

Cause 2

Cyclically asynchronous writing of item values.

Solution 2

Only write values if necessary.

If possible, write synchronously. This has the advantages that the sequence of queries is retained and the return of the query indicates that the value has been written to the controller.

NOTE: With synchronous reading, the client calls the method and waits for the return value. This may have the effect, that the client is blocked for quite a long time.
