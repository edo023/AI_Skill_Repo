# CANopen

CANopen is a protocol which is used in the automation of flows. A CANopen network consists of a manager and Devices that act as master and slave. In this case, a device can be structured modularly or non-modularly.

**Moreover, CODESYS distinguishes between remote devices and local devices.**

* **CANopen Remote Device**: In CODESYS, a CANopen Remote Device is a slave device that is inserted below a CANopen Manager (master) in the device tree of a project.
* **CANopen Local Device**: In CODESYS a CANopen Local Device is a slave device that is inserted directly below a CAN bus node in the device tree of a project. As a result, the CODESYS runtime can be used as a CANopen Slave.

You can download the CODESYS CANopen Manager SL in the [CODESYS store](https://store.codesys.com/en/codesys-canopen-manager-sl.html). There you will also find the data sheet for the product.
