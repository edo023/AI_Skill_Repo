# Commissioning

## Overview

The following sections describe the procedure of activating the Sercos communication phase CP4 via EcoStruxure Machine Expert Logic Builder. In this communication phase, the Sercos bus is operational.

This is typically done multiple times during the configuration of the Sercos devices and the creation of your motion application. The following sections do not describe the commissioning of the final machine/process.

## Logging in and Phasing up or Down

| Step | Action |
| --- | --- |
| 1 | In EcoStruxure Machine Expert Logic Builder, click the Login button or press the key combination Alt+F8 to log in to the Sercos master. |
| 2 | Confirm the message displayed if the corresponding prerequisites and conditions are met.  **Result**: The application and setup are downloaded to the controller. Detected errors, alerts, and other messages are displayed in the **Messages** window. |
| 3 | Display the **Device tree** by clicking View > Navigators > Device tree. |
| 4 | Double-click the node **SercosMaster (Sercos Master)**. |
| 5 | Display the tab **Parameters** of the Sercos master. |
| 6 | Expand the parameter group **SercosPhaseChanger** by clicking the corresponding plus sign. |
| 7 | For phase-up, at least one device must be physically connected to the Sercos master. The other devices can be run as [simulated devices](D-SE-0096550.html#D-SE-0096550). Set the parameter DesiredPhase to one of the following values:   * NRT / -1 (Ethernet communication available, no Sercos communication) * Phase 0 / 0 * Phase 1 / 1 * Phase 2 / 2 (service channel available for acyclic Sercos communication: reading and writing parameters via function blocks, execution of procedure commands) * Phase 3 / 3 * Phase 4 / 4 (real-time Sercos communication available, Sercos bus operational) |

## Logging In

In EcoStruxure Machine Expert Logic Builder, click the Login button or press the key combination Alt+F8 to log in to the Sercos master.
