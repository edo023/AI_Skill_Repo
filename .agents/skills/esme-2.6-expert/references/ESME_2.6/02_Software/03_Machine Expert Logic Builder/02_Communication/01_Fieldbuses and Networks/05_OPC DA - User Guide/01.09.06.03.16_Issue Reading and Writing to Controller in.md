# Issue: Reading and Writing to Controller in Multi-PLC Configuration in Constant Intervals Delayed

![](images/logo.png)

Issue: Reading and Writing to Controller in Multi-PLC Configuration in Constant Intervals Delayed

Issue

The reading and writing to a controller in a multi-controller configuration is delayed in constant intervals by approximately 5...6 seconds.

Cause 1

One controller that is part of the multi-controller configuration of the OPC server has been powered down. It is written to another controller.

The OPC server constantly tries to communicate with the unconnected controllers in defined intervals (also refer to the Reconnect time (s) [parameter](../OPC_Server_Configuration/OPC_Server_Configuration-2.htm#XREF_D_SE_0083829_12)). These attempts usually take 5...6 seconds.

Solution 1

oEnter only active controllers in the OPC configuration. Controllers that are not active can be deselected in the configuration.

oIncrease the value of the Reconnect time (s) [parameter](../OPC_Server_Configuration/OPC_Server_Configuration-2.htm#XREF_D_SE_0083829_12) to improve the behavior.
