# Issue: Message Writing values failed to <controller>, GatewayError-28 (affects gateway V2.3) is

Issue: Message Writing values failed to <controller>, GatewayError=28 (affects gateway V2.3) is Displayed

Cause 1: One controller that is part of the multi-controller configuration of the OPC server has been powered off. It is written to another controller.

The OPC server constantly tries to communicate with the unconnected controllers in defined intervals (also refer to the Reconnect time (s) [parameter](../OPC_Server_Configuration/OPC_Server_Configuration-2.htm#XREF_D_SE_0083829_12)). These attempts usually take 5...6 seconds.

A writing process is also delayed during this time.

Solution 1: Increase the value of the Reconnect time (s) [parameter](../OPC_Server_Configuration/OPC_Server_Configuration-2.htm#XREF_D_SE_0083829_12) to, for example 10,000 ms.
