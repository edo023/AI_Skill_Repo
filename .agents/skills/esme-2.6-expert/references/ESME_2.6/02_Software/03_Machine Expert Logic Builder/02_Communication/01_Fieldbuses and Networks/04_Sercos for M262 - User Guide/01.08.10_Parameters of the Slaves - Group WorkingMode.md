# Parameters of the Slaves - Group **WorkingMode**

## Overview

The parameter group **WorkingMode** comprises the parameters used for setting Sercos slaves to “simulated” or “activated” working mode.

## Parameter DesiredState

|  |  |
| --- | --- |
| Offline editable | Yes |
| Online editable | Yes |
| Traceable | Yes |

With this parameter, you can set the working mode of the slave. The parameter lets you configure a slave as a simulated device. A simulated device is treated like a device that is physically present and connected. However, the system does not perform any communication with a simulated device.

If the value of the parameter DesiredWorkingMode is modified in communication phase 4, the new value becomes effective only after the next full Sercos phase-up.

The parameter provides the following values:

| Value | Description |
| --- | --- |
| Simulated working mode | The working mode **Simulated** is intended for use during development, for example, for testing an application before the drive is connected to the logic/motion controller. |
| Activated working mode (default value) | The working mode **Activated** is intended for regular machine operation when the device is connected. |

Refer to the chapter [Configuring Devices as Simulated or Activated](D-SE-0096550.html#D-SE-0096550) for additional information.

## Parameter CurrentState

|  |  |
| --- | --- |
| Offline editable | No |
| Online editable | No |
| Traceable | Yes |

With this parameter, you can read the currently set working mode of the slave.

Refer to parameter DesiredState for the details on the values.
