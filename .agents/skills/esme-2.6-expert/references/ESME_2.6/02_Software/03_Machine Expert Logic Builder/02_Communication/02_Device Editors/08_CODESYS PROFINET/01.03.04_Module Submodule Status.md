# Module/Submodule Status

The `SubmoduleState` is a status word that indicates for each individual submodule the presence of diagnosis entries or other malfunctions such as a corrupt configuration.

In this way, when establishing a connection to a Device, the Controller also sends the list of modules and submodules that it will use for data exchange. If the specified/actual state between existing modules of a Device and those projected by the controller differs or a diagnosis exists, then flags are set in the status accordingly.
