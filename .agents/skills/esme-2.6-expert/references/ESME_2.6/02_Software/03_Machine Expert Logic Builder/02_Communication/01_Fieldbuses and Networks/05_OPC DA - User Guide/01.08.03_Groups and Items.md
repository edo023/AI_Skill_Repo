# Groups and Items

![](images/logo.png)

Groups and Items

Overview

As soon as a connection to the OPC server has been established, you can add client Groups.

Most OPC clients generate default names for these groups. However, it is a good practice to assign meaningful names to the Groups.

By default, the OPC server provides two Items that work independent of configured symbol files:

o\_\_CommState

o\_\_CommStateOk

\_\_CommState Item

\_\_CommState items can be analyzed by different OPC clients and can have the following states:

oSTATE\_TERMINATE = -1

oSTATE\_PLC\_NOT\_CONNECTED = 0

oSTATE\_PLC\_CONNECTED = 1

oSTATE\_NO\_SYMBOLS = 2

oSTATE\_SYMBOLS\_LOADED = 3

oSTATE\_RUNNING = 4

oSTATE\_DISCONNECT = 5

oSTATE\_NO\_CONFIGURATION = 6

\_\_CommStateOk Item

The \_\_CommStateOK item is a Boolean variable. TRUE indicates that communication with the OPC server is available. The suffix of the variable indicates the name of the controller.

FALSE indicates that communication is not possible, as for example, during a download.

In redundant groups, status variables are generated for each controller of the group. In order to differentiate them, the names are generated according to the following syntax:

o\_\_CommStateOK<ControllerName>

o\_\_CommState<ControllerName>
