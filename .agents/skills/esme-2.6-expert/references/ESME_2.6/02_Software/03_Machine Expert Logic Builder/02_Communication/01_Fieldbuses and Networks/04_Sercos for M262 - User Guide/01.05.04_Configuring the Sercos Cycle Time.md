# Configuring the Sercos Cycle Time

## Overview

The Sercos cycle time is a key factor in the performance of your system. The correct cycle time depends on a range of factors including devices on the Sercos bus, amount of data for cyclic data exchange, cycle time of the MAST task and other tasks. The Sercos cycle time triggers the Real-Time Motion Process. The values for the axes are calculated in the Sercos cycle time.

The Modicon M262 Motion Controller enforces a number of limits, depending on the reference of the controller, as well as the type and number of connected devices. Refer to the chapter [Device Limits and Cycle Time Limits M262 With Sercos](D-SE-0096553.html#D-SE-0096553) for details.

Refer to the chapter [Sercos Communication Cycle](D-SE-0096488.html#D-SE-0096488) for details on the communication cycle.

## Configuration of the Sercos Cycle Time

The cycle time can be set in communication state NRT and in communication phase CP0. The configured value becomes effective as of communication phase CP2.

| Step | Action |
| --- | --- |
| 1 | Display the **Device tree** by clicking View > Navigators > Device tree. |
| 2 | Double-click the node **SercosMaster (Sercos Master)**. |
| 3 | Display the tab **Parameters** of the Sercos master. |
| 4 | Expand the parameter group **SercosCycletimeConfig** by clicking the corresponding plus sign. |
| 5 | Set the parameter CycleTime to one of the following values (nanoseconds):   * 1000000 (= 1 ms) * 2000000 (= 2 ms) * 4000000 (= 4 ms) |

Refer to the chapter [Parameters of the Master - Group **SercosCycletimeConfig**](D-SE-0096458.html#D-SE-0096458) for additional information on the parameter.
