# Tab: CANopen Remote Device – SRDO CRC

This dialog is available for CANopen Safety Slaves only. It provides an overview of the current configured transmit and receive SRDOs with the determined checksums (CRC). The validation result is shown here.

|  |  |
| --- | --- |
| **Ignore mismatch** | : The error remains, but it is not evaluated anymore. |
| **Validation result** | * **CRC OK** * **CRC mismatch. Expected value: 16#xxxx**: A CFC mismatch occurs when the configuration of an SRDO has changed. The calculated value does not match the set value anymore and has to be fixed. |
| **Fix All CRC Mismatches** | All mismatches are fixed automatically. The CRC value is corrected. |

**A CRC mismatch can be handled as follows:**

* Select the **Ignore mismatch** check box.
* Execution of the **Fix All CRC Mismatches** command
* Modify the CRC manually (for example, by computing the CRC using a tool from the hardware manufacturer).
* Automatic repair with the "Check and Fix Configuration" command in the CANopen Manager SIL2.
