# Project Settings - Multicarrier Configuration Editor

## Overview

The Project > Project Settings > Multicarrier Configuration Editor dialog box provides core settings for the Multicarrier Configuration editor.

The settings will be applied to the open editor views as soon as they have been confirmed (with the dialog box closed) by clicking the OK button.

## General Tab

| Element | Description |
| --- | --- |
| Automatic Updates | If this option is enabled, modifications are updated as follows:   * If you make modifications in the Multicarrier Configuration editor, these modifications are automatically updated (Update To All) when another editor within the Logic Builder is activated. * If you make modifications to any multi carrier device or to the GVL\_MulticarrierConfiguration, these changes are automatically updated (update From Last Modified) to the Multicarrier Configuration editor when the Multicarrier Configuration editor is activated. |

NOTE: If you activate the Automatic Updates option, you must complete and verify any modifications you make with the Multicarrier Configuration editor / track editor. If modifications are left incomplete and you leave the Multicarrier Configuration editor / track editor, the automatic update is executed on incomplete or inconsistent data which may corrupt part of the multi carrier configuration.

## Device Generation Tab

| Element | Description |
| --- | --- |
| Track Base Name | Default name that will be used when creating track devices. An ascending index is automatically appended to each newly created track device (for example, \_1, \_2). |
| Segment Base Name | Default name that will be used when creating segment devices. An ascending index is automatically appended to each newly created segment device (for example, \_1, \_2). |
| Carrier Pool Base Name | Default name that will be used when creating carrier pools. An ascending index is automatically appended to each newly created carrier pool (for example, \_1, \_2). |
| Carrier Base Name | Default name that will be used when creating carrier devices. An ascending index is automatically appended to each newly created carrier device (for example, \_1, \_2). |

## Code Generation Tab

| Element | Description |
| --- | --- |
| Configuration  The configuration data contains information on the objects defined in the Multicarrier Configuration editor. The data is stored in the folder with the fixed name MulticarrierConfiguration. | |
| Folder Name | Name of the folder that is not editable. It is used when creating the GVL (Global Variable List) folder name. |
| Emulation  The emulation data contains information required for displaying your configured multi carrier track as a 3-D emulation with EcoStruxure Machine Expert Twin. The data is stored in the folder with the fixed name MulticarrierEmulation.  For further information, refer to the description of the Emulation tab in the  [Lexium™ MC multi carrier Configuration Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=MLSConfG&topicID=TPC_MLS_Config_Tab_Visualization_DA963F21). | |
| Create Emulation Data option | Select this option to generate emulation data. |
| Folder Name | Name of the folder that is not editable. It is used when creating the folder name for emulation data. |
| Station Code  You can create programming code for the multi carrier stations that are defined in the Multicarrier Configuration editor. The data is stored in the folder with the fixed name MulticarrierStations.  For further information, refer to the [Lexium™ MC multi carrierConfiguration Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=MLSConfG&topicID=UpdateToCodeCommand_43B0E772). | |
| Create Code for Stations option | Select this option to enable programming code generation for multi carrier stations. |
| Folder Name | Fixed name of the folder where station code is stored. |
