# Displaying as a Variant of Remote TargetVisu

You can display a visualization of a CODESYS project on a panel. To do this, you need the program `RemoteTargetVisu.exe`. This program is supplied with the CODESYS Control Win runtime and therefore available for Windows. For Linux, the feature is provided by device manufacturers on specific panels.

One of the uses of this feature is to run on a panel a specific visualization of a controller of the network. In addition, the feature allows all visualizations of the controllers of a network which support Remote TargetVisu to be displayed on one panel.

**Starting a Remote TargetVisu**

**Requirements**

* A controller with a visualization exists in the network and this controller is running.
* The start visualization has been defined for the Remote TargetVisu.
* The program `RemoteTargetVisu.exe` is located on the panel.

1. Start the program `RemoteTargetVisu.exe`. This program is supplied with the CODESYS Control Win runtime.

   * When the program is executed for the first time, a dialog opens prompting whether or not the configuration of the Remote TargetVisu should be started. When you click **No**, a CFG file with briefly explained settings is generated. In this way, the Remote TargetVisu can be configured without a dialog.
2. Click **OK**.

   If there are multiple applications with visualizations on the controller, then a dialog for selecting the desired application opens first. If the Remote TargetVisu is started via the CFG file, then that visualization of the application is started which is defined by the setting `VisuClient.VisuAppName=<application>`.

   * The visualization of the selected controller is displayed on the panel.
