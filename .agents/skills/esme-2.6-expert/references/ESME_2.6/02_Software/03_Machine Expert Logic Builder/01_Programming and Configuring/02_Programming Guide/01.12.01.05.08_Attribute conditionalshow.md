# `Attribute conditionalshow`

## Overview

The function block or variable that contains this attribute is not displayed in the user interface if the library is installed as *\*.compiled-library*.

If the library is available as source library (*\*.library*) in the repository, the function block or variable is visible in the user interface.

If you use `conditionalshowsymbols` as a command-line option, such as `LogicBuilder.exe --conditionalshowsymbols="Library_Developer"`, the function blocks or variables are visible in the user interface. This option may be useful for library developers when testing libraries. For more information, refer to [Command Line Arguments Activating Visibility](#D-SE-0107035__CommandLineArgumentsActivatingVisib-D8F5667D).

## Syntax

```
{attribute 'conditionalshow' := 'SomeText'}
```

or

```
{attribute 'conditionalshow'}
```

## Insert Location

Insert this pragma in the first line of the declaration part of the function block or before a variable.

## Example

Example for hiding a variable:

```
FUNCTION_BLOCK FB_DataManager
VAR
    {attribute 'conditionalshow' := 'Library_Developer'}
    iLocal : INT;
    iCounter : INT;
END_VAR
```

**Result**: The variable `iLocal` is invisible.

Example for hiding a function block:

```
{attribute 'conditionalshow' := 'Library_Developer'}
FUNCTION_BLOCK FB_DataManager
VAR
    iLocal : INT;
    iCounter : INT;
END_VAR
```

**Result**: The identifiers `FB_DataManager`, `iLocal`, and `iCounter` are invisible.

## Command Line Arguments Activating Visibility

You can unhide hidden variables without the source library (*\*.library*) being available in the repository by using the command line argument `conditionalshowsymbols`. To enable visibility, enter the attribute values of the pragma separated by commas.

**Syntax:**

```
LogicBuilder.exe --conditionalshowsymbols=" <some text> ( ,<next text> )* "
```

```
LogicBuilder.exe --conditionalshowsymbols="Library_Developer"
```

```
LogicBuilder.exe --conditionalshowsymbols="Group_A,Group_B"
```

The following table illustrates the behavior when using the compiled library containing `FB_DataManager` of the above examples, whereby FB1 is used as abbreviation for the first and FB2 for the second `FB_DataManager` example. fb1 and fb2 are instances of FB1 and FB2. Additionally, it is assumed a library containing a function `Func1` that is tagged with `{attribute 'conditionalshow' := 'Library_Developer'}`.

|  | Starting Logic Builder without option | Starting Logic Builder `--conditionalshowsymbols="Library_Developer")` |
| --- | --- | --- |
| Input Assistant | * FB1 is displayed. * FB2 is not displayed. * `Func1` is not displayed. * Variables of fb1 and fb2 are not displayed. | * FB1 is displayed. * FB2 is displayed. * `Func1` is displayed. * Variables of fb1 and fb2 are not displayed.(1) |
| SmartCoding | * FB1 is displayed. * FB2 is not displayed. * `Func1` is not displayed. * Variables of fb1 and fb2 are not displayed. | * FB1 is displayed. * FB2 is displayed. * `Func1` is displayed. * Variables of fb1 and fb2 are not displayed.(1) |
| Library Manager | * FB1 is displayed. * FB2 is not displayed. * `Func1` is not displayed. | * FB1 is displayed. * FB2 is displayed. * `Func1` is displayed. |
| Monitoring | * Only `iCounter` of fb1 is displayed. * Both variables of fb2 are displayed.(2) | Both variables of fb1 and fb2 are displayed. |
| Symbol Configuration | * Only `iCounter` of fb1 is displayed. * No variables of fb2 are displayed. | Both variables of fb1 and fb2 are displayed (and can be selected, for example, for data exchange). |
| (1) Variables declared as VAR in the scope of a function block are for internal usage and therefore not displayed in the Input Assistant or SmartCoding.  (2) As FB2 is not displayed in the Input Assistant, SmartCoding or the Library Manager, it is not known how to declare fb2 as FB2. Therefore, monitoring of fb2 is not applicable. | | |
