# Operating Modes

## Presentation

You can use the operating modes of the controller to protect the project against unintended changes.

For example, a programmer can lock the state of the controller to help to avoid changes while programming another controller. After programming has been finished, the controller can be switched to a defined and recognizable state.

You can change the operating mode from Locked or Operational to Debug in offline mode. A login is not required to change the operating mode.

This function is not available for all supported controllers. Consult the *Programming Guide* specific to your controller.

## Operating Modes

The controller can be switched to the following modes by using the commands in the Online > Operating Modes menu or by double-clicking the icon in the status bar:

| Operating mode | Icon | Description | Actions prevented by the runtime system |
| --- | --- | --- | --- |
| Debug |  | No restrictions. | |
| Locked |  | This operating mode can help prevent a download to an incorrect controller when, for example, multiple controllers of a plant are programmed.  The writing of variables is possible. Breakpoints which are already set remain enabled.  NOTE: If the controller is restarted, only the RUN state of an application is preserved. Otherwise the application remains in STOP state. | Application actions prevented:   * Downloading an application * Online change * Forcing variables * Setting breakpoints * Stopping the application * Resetting the application * Starting the application * Deleting the application  File transfer actions prevented:   * Downloading a file to the controller * Deleting a file on the controller * Renaming a file on the controller * Creating a directory on the controller * Deleting a directory on the controller * Renaming a directory on the controller |
| Operational |  | This operating mode helps to ensure that the same application is loaded by a controller after a restart and that the debug features are disabled. Select this operating mode when controller programming is complete and is ready to be released or it has already been released.  Also refer to the [Preconditions for the Activation of the Operational Mode](#D-SE-0084016__D-SE-0084016.4). |

NOTE: A direct switch between the modes Locked and Operational is not possible.

## Preconditions for the Activation of the Operational Mode

* For each application, a boot application must exist on the controller.
* There must not be set active breakpoints.
* The application must be running on the controller.
* No forced values.
* Depending on the device, there may be more restrictions.
