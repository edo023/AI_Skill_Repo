# Commands for Access to Saved Traces

## Load Trace...

The Load Trace... command is used to load a complete trace with configuration, data, and time stamps, which have been stored in a file, back to the editor. By executing this command, the Load Trace dialog box, which is a standard file-browsing dialog box, opens. It allows you to browse a saved trace configuration. Select between the formats trace dump (*\*trace.csv*) or trace file (*trace\*.*) from the list. Select a file from the matching file list and close the dialog box by clicking Open. Then the graph will be displayed with the configuration settings stored in the file. The file may have been created by executing the command Save Trace....

## Save Trace...

The Save Trace... command is used to save traces into a file. It opens the Save Trace dialog box (standard dialog for saving a file) opens. It allows you to define a file name and a location. Select between the formats TRACE (*\*.trace*), TXT (*\*.txt*), or CSV (*\*.csv*) from the list. Enter a name and a path and close the dialog box by clicking Save.

| File format | Description |
| --- | --- |
| TRACE format: *\*.trace* | Traces stored as trace file contain the configuration settings and the captured data with associated time stamp. This data can be reloaded in the trace editor. There, an offline analysis of the trace can be performed. Execute the Load Trace... command for reloading. |
| TXT format: *\*.txt* | Traces stored as text file do not contain configuration data but list the data with associated time stamps. Such a text file can be loaded and processed from tools knowing the CSV format. The trace editor itself cannot read (Load Trace...) this format. |
| CSV format: *\*.trace.csv* (data only) | Traces stored as comma-separated values file (CSV) do not contain configuration data. Address information is provided for each trace variable. The generated file can be read by the runtime system. The data is imported but the trace cannot be started because the addresses of the variables are not saved. |

## Export Symbolic Trace Config

The Export Symbolic Trace Config command is used for creating a configuration file in symbolic trace config format (*\*.traceconfig*) containing the configuration of the edited trace object.

The following preconditions have to be fulfilled:

* A symbol configuration is defined in the application.
* The variables used in the trace configuration also have to be defined in the [symbol configuration editor](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083586).
* The task set in the trace configuration even exists with identical name in the application.
* The name of the application, in which the trace is stored, is identical to the application name in which the trace is loaded.

The variable access is performed symbolically. This has the advantage that the configuration file can be used by any application that has this symbolic configuration. If the preconditions are fulfilled, the trace manager can use the stored configuration on any runtime system.

NOTE: Complex expressions such as pointer dereferencing, array accesses, and device parameters are not supported.

When executing the Export Symbolic Trace Config command, a dialog box opens which allows you to save a symbolic configuration file. Browse for a directory, enter a file name, and close the dialog box by clicking Save.

NOTE: With the [file transfer functionality](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083390), you can load any configuration file into the runtime system.
