# Project Settings - Compile options

## Overview

The Project Settings - Compile options provides information about the compiler version and allows you to configure project-wide compile options.

NOTE: Consult the *Programming Guide* specific to your controller for information on whether the data type LREAL will be treated as 64-bit type or converted to REAL.

## Settings Section

| Element | Description |
| --- | --- |
| Allow unicode characters for identifiers check box | By default, this option is deactivated because the use of unicode characters in identifier names is not matching the IEC standard. However, it can be needed, for example, for Asian languages. |
| UTF8 Encoding for STRING check box | By default, this option is deactivated. Strings with the data type STRING are project-wide encoded in Latin 1 format. Index access to the literal is possible. To encode a single literal in UTF-8 format, add the `UTF8#` type prefix to the literal. For further information, refer to [UTF-8 Encoding: Encoding Single Literals in UTF-8 Format](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=UTF_8Encoding_3F7F7388).  Activate this option to achieve project-wide interpretation of strings of data type STRING as UTF-8. This applies to every STRING literal and is also used for monitoring. Index access to UTF-8-coded literals is not practical. |
| Replace constants check box | By default, this option is activated. In online mode, the constants are indicated by a  icon preceding the value in the declaration or watch view. In this case, constant values cannot be modified. If the option is deactivated, the constant can be accessed.  NOTE: When designing an [interface library](../../../../../api/crossBook?lang=en-US&virtualBookName=SoLibref&topicID=D_SE_0081244), activate the Replace constants option to help avoid an online change. |
| Logging in breakpoints  check box | By default, this option is activated. For breakpoints which are defined as execution points, you can define a message text in the dialog box Breakpoint Properties. This text is displayed in the device Log [view](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083391) when the program stops at the breakpoint. |
| Edit project defines... button | Click the Edit project defines... button to open the Edit Project Defines dialog box that allows you to add project–wide defines. |

## Edit Project Defines Dialog Box

The Edit Project Defines dialog box allows you to add project defines as follows:

| Step | Action |
| --- | --- |
| 1 | Click the Add Project Define button.  **Result**: A new line is added to the list of project defines. |
| 2 | Select the new line and click to edit. |
| 3 | Enter a valid IECidentifier for the new project define. |
| 4 | Click OK to close the dialog box. |

The project defines are valid throughout the entire project and in the referenced libraries. They do not apply to public interfaces. For more information about project defines, refer to [`project_defined (<global define>)`](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=project_defined_1D738A0A).
