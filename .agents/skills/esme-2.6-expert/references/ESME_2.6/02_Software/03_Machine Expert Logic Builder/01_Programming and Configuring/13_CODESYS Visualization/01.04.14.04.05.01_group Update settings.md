# Group: Update settings

Permissions of the new group

|  |  |
| --- | --- |
| **Add New Group** | Requirement: New groups have been added in the user management.  From the list box, select the new group whose permissions configuration should be updated at all occurring locations.  Example: `VISU_GRP1` |
| **Setting for New Group** |  |
| **new group in visualization / hotkey will get the right like group** | Configures which permissions the new group should get  : From the list box, select the group with exactly the permissions to be given to the new group. |
| **new group should get the following right**  **for visualization elements** | : From the list box, select the permission to be assigned to the new group.  Possible permissions are: **Operable**, **Only visible**, **Invisible** |
| **new group should get the following right**  **for hotkeys** | : From the list box, select the permission to be assigned to the new group.  Possible permissions are: **Operable**, **Not operable** |

Group names

|  |  |
| --- | --- |
| **Delete not existing groups** | : Groups which do not or no longer exist in the user management, but are still referenced in the permissions, should be deleted there |
| **Rename groups** | : Groups which have been renamed in the user management are also renamed in the permissions |

Buttons (below)

|  |  |
| --- | --- |
| **Update** | Updates the configured permissions according to the above settings and closes the dialog  The **Messages** view notifies you about the changed locations. A message indicates which visualization object was updated at which location.  Note: Only the configured permissions in visualization objects located in the parent application or in the `POUs` view are updated. |
| **Close Dialog** | Discards the settings and closes the dialog |
