# Dialog: Visualization Tags View

The view displays the visualization tags in relation to the variables called in the visualization elements.

The visualization path is displayed in the view. The number of visualization tags required for the license determination is calculated and displayed for the visualization implemented in the project.

The visualizations, the subordinate visualization elements, and the variables used are listed by name at the top level. In addition, information is provided on the number of visualization tags required there.

Because visualizations (template visualizations) can be referenced and used repeatedly for structured user interfaces, the repeated use is ignored in the calculation of the tags and ultimately only included one time in the calculation. The relevant result is displayed in the **Total tags : <number of found tags>** column.

|  |  |
| --- | --- |
| **Update** | Refreshes the view with the subordinate elements |
| **Collapse All** | Collapses the view. The top level is the selected visualization. |
| **Expand All** | Expands the view and displays all licensed visualization tags per node. |
| **Total tags: <number of found tags>** | Final result of the visualization tag count for all visualizations. This is the software metric in the number of visualization tags for the CODESYS Visualization add-on.  Example: 100 |
| Search field **Filtering by full or partial name** | Input field for the search text to be filtered in the table  Hint: You can, for example, search for all occurrence locations of a template visualization. To do this, specify the name of the visualization. |
