# Status & Staging View for Git Version Control (exclusive to \*.fbs projects)

## Overview

This function is exclusive to projects stored in file-based storage format.

EcoStruxure Machine Expert supports Git version control for \*.fbs projects. When you open an \*.fbs project, the Status & Staging view is open by default and allows you to initialize a repository. To enable source control features, click the Initialize Repository button and browse to the folder you want to configure as local repository. By default, the local repository of an \*.fbs project is saved to the subdirectory .git of the folder where your \*.fbs project is saved. If you use Machine Expert, each \*.fbs project is embedded in a virtual container named Solution. For further information, refer to the [EcoStruxure Machine Expert User Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=esmeug&topicID=). For multi-user deployments, configure an external Git server, such as GitHub.

When a local or remote repository is defined, the Status & Staging view lists the modifications that are performed on the project and provides different Commit commands for uploading them to the database.

## Elements of the View

| Elements | | Description |
| --- | --- | --- |
| Commit changes list | | List of branches  By default, the master branch is selected. Click to open the list and execute the command Create New Branch to create a new branch or select a branch from the list to open this branch and to display the respective content. |
| Publish Branch button |  | Click the button to configure an external Git server.  In the Add Repository Url dialog box, enter the URL of the Git server, for example:  *https://github.<your company>.com/<organization>/<repository name>.git*  NOTE: For the connection to the remote repository, EcoStruxure Machine Expert uses the Windows Credential Manager. Add generic credentials with the development platform address (must start with git, such as. *git:https://github.<your company>.com*), user name and password/personal access token.  NOTE: You can also add and adapt the repository URL with More Actions > Remote.  Click the Add or the Add & Sync button. |
| Search button |  | Click the button to display a search field.  Enter a search string and, if necessary, click the  button to refine your search by activating the options Match case and / or Whole word. |
| More Actions button |  | Click the button to display the list of available commands.  Git commands are provided for SCM (Source Code Management). For detailed information, refer to <https://git-scm.com/>. |
|  | Refresh command | Click to refresh the Status & Staging view. |
|  | View & Sort command with subcommands | Subcommands:   * View As List * View As Tree * Sort by Name * Sort by Path * Sort by Status |
|  | Pull command | – |
|  | Push command | – |
|  | Fetch command | – |
|  | Commit command with subcommands | Subcommands:   * Commit * Stage * Commit All * Undo Last Commit |
|  | Changes command with subcommands | Subcommands:   * Stage All Changes * Unstage All Changes * Discard All Changes |
|  | Pull, Push command with subcommands | Subcommands:   * Sync * Pull * Pull From... * Push * Fetch * Fetch From All Remotes |
|  | Branch command with subcommands | Subcommands:   * Create Branch * Rename Branch * Delete Branch * Publish Branch |
|  | Remote command with subcommands | Subcommands:   * Add Remote... * Remove Remote |
|  | Stash command with subcommands | Subcommands:   * Stash * Stash (Include Untracked) * Stash Staged * Apply Latest Stash * Apply Stash * Pop Latest Stash * Pop Stash * Drop Stash * Drop All Stashes |
| Staged Changes section | | Lists the elements with modifications that are staged.  At the end of each line, the state is indicated (such as U = Untracked or M = Modified). Additionally, the following buttons are available:   * Unstage Changes to unstage the modifications. * Compare Raw Data to open a comparison view in the Logic Builder similar to the [Project Comparison - Differences view](D-SE-0083939.html). * Open File to open the file in an editor view in the Logic Builder. |
| Changes section | | Lists the committed modifications. |
| Commit message field | | Enter a describing text before you click the Commit button. The text is saved with the modifications. |
| Commit button | | The button is enabled after you have entered text in the Commit message field.  Click to commit the modifications listed in the Staged Changes section. |
| Commit & Push button | | Click to save the modifications listed in the Changes section to the database and to update the references. |
| Commit & Sync button | | Click to save the modifications listed in the Changes section to the database and to synchronize the local project with the version of the project stored on the database. |
