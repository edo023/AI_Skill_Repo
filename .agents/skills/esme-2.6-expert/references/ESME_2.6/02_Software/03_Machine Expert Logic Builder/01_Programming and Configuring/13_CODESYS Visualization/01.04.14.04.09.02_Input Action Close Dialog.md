# Input Action: Close Dialog

|  |  |
| --- | --- |
| **Dialog** | The visualization of type **Dialog** which will be closed  : List box with all **Dialog** type visualizations available in the project  Example:  **Default dialogs of the `VisuDialogs` library, which is usually integrated in the project.**   * `FileOpenSave` * `Keypad` * `Login` * `Numpad` * `NumpadExtended` * `TextinputWithLimits`   IMPORTANT: The setting in the object property (**Visualization** tab) of a visualization determines whether or not a visualization can be used as a dialog. |

|  |  |
| --- | --- |
| **Result** | Return value which is passed to the dialog by the input action  NOTE: If more input actions should follow when the dialog is closed, then you can configure these in the input configuration of the element which calls the dialog (**Input configuration → OnDialogClosed)**.  IMPORTANT: The result is an internal variable which cannot be accessed programmatically. The value cannot be read from the IEC code. The value can be used together with the **Open dialog** input action for a conditional update of the transfer parameters.  **Selection of the result value**   * **None**  The result is internally set to "None". * **OK**  The result is internally set to "OK". * **Cancel**  The result is internally set to "Cancel". * **Abort**  The result is internally set to "Abort". * **Retry**  The result is internally set to "Retry". * **Ignore**  The result is internally set to "Ignore". * **Yes**  The result is internally set to "Yes". * **No**  The result is internally set to "No". |
