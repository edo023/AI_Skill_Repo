Alarm Storage Task

|  |  |
| --- | --- |
| **Alarm Storage Task** |  |
| **Enable** | : The alarm evaluation is distributed to two tasks. The faster task (`AlarmManagerTask` object) is responsible for evaluating the alarm conditions. The slower task (alarm storage task) is responsible for saving the alarm information in the alarm database. When you select the option, the alarm storage task is created **automatically**. In the task configuration, this task is not visible below the **Task Configuration** object.  : The alarm evaluation and the alarm storage are processed in one task (`AlarmManagerTask`object). This is the default setting for existing projects which were created before V4.6.0.0. |
| **Task Interval** | Task cycle time of the alarm storage task in ms `(1..1000000`)  Example: `2000`  NOTE: Set a higher task cycle time and a lower priority for the alarm storage task than for the alarm manager task. The alarm storage task requires a slow frequency to execute file access to the alarm database. |
| **Priority (0..31)** | Priority of the alarm storage task  Example: `31` |

For more information, see the following: [Distributing the Alarm Management to Two Tasks](_visu_split_alarm_management_between_two_tasks.html#_visu_split_alarm_management_between_two_tasks)

Alarm History Behavior

|  |  |
| --- | --- |
| **Alarm History Behavior** |  |
| **Store only completed alarms** | : Alarm entries are added to the alarm history only as soon as the alarm has completed its cycle.  IMPORTANT: No active alarms are displayed in the alarm history. |
| **Alarm history updates during runtime start** | |
| **Change state of not normal alarms to normal** | : When the application is started, all alarms in the alarm history which are not in the "Normal" state are automatically assigned the "Normal" state.  : The alarm state is retained when the application is started. |
| **Assign missing inactive timestamps to runtime start** | : When the application is started, missing inactive time stamps are assigned the time of the application start.  : The time stamp is retained when the application is started. |
