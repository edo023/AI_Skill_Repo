# FBD, LD, and IL Editor

## Overview

The Tools > Options > FBD, LD and IL Editor dialog box allows settings concerning the editing in the FBD and LD editor.

## General Tab

|  |  |
| --- | --- |
| View | |
| Show network title | The network title - if defined - is displayed in the upper left corner of a network. |
| Show network comment | The network comment - if defined - is displayed in the upper left corner line of a network. If the network title is visible, the comment will display in the line below the title, too. |
| Show box icon | If a function block or a function available via a library or via the [object properties](D-SE-0083921.html#D-SE-0083921) is provided with an icon (bitmap), this icon will be displayed in the box in the FBD and LD editor. Also the standard operators are provided with icons. |
| Show operand comment | The comment which can be assigned to an operand in the implementation part of the editor is displayed. The operand comment only refers to the usage of the operand in place, in contrast to the symbol comment, which is defined at the declaration of a variable. Operand comments can be edited in the editor.  This comment is truncated automatically depending on available space.  To limit the comment to a defined width, activate the option Fixed size for operand fields and set the parameters of the dialog box Operand Sizes according to your individual requirements. |
| Show symbol comment | For each variable which has got a comment in its declaration, this comment will be displayed above the identifier of the symbol. |
| Show symbol address | For each symbol (variable) the assigned address will be displayed above the identifier of the symbol. |
| Show network separators | A separator line is displayed between the individual networks. |
| Behavior | |
| Placeholder for new operands | Not implemented yet. |
| Empty operands for function block pins | If this option is activated, the operand field for the pins of new function blocks will be left empty (instead of ???). |
| Fixed size for operand fields: The editor of FBD, LD, and IL adjust the size of the operand fields automatically. With this option, you can adjust the length and the height of the operand fields to a specific size. | |
| Operand width, characters | This value defines the width of the operand field (number of characters). |
| Operand height, lines | This value defines the height of the operand field (number of lines). |
| Operand comment height, lines | This value defines the height of the operand comment field (number of lines). |
| Symbol comment height, lines | This value defines the height of the symbol comment field (number of lines). |

## FBD Tab

|  |  |
| --- | --- |
| View | |
| Network with linebreaks | If this option is activated, the line of the connections of a network will be broken in a way where as many elements as possible are visible in the current editor window width. This can affect which networks are enlarged in height. If the editor window is too small, networks will not be broken. |
| Connect boxes with straight lines | If this option is activated, the network components will be arranged in a way where the lines in between are of a fixed, short length, thus reducing the horizontal space required for displaying the networks. This can affect that boxes are enlarged in height, for example, to provide enough space for input and output elements. If the option is not activated, the elements will keep their standard size and the connection lines adapt to the need of space. |
| Behavior | |
| Default network content | This option defines which content is displayed in the editor when you insert a new network. (Empty, Assignment, and Empty box) |
| After insertion select | This option defines which element is selected after a new network has been inserted (Element, Network) |

## LD Tab

|  |  |

This can affect which networks are enlarged in height. If the editor window is too small, networks will not be broken. |
| Behavior | |
