# TripleStorageBackendTypes

class

TripleStorageBackendTypes.

TripleStorageBackendTypes

Bases: enum.Enum

Definition of the strategy used for backend type selection.

OutProc

= 0

Force Out-Proc (use external TripleStorage program, e.g. with In-Memory as backend and CodeAnalysis WCF based protocol as communication)

DebugOutProc

= 1

Force debug Out-Proc (like ForceOutProc but with debug configuration for TripleStorage)

Automatic

= 2

Automatic selection

InMemory

= 3

Force In-Memory RDF backend -- see dotNetRDF

Http

= 4

Force http based RDF backend -- see dotNetRDF
