# Behavior in Case of Error

If an axis error occurs (for example, the axis switches to the state `Errorstop`), the active movement will report an error along with all other accepted movements.

If an FB error occurs in the function block of an active movement, then all movements accepted later also report an error. This is in contrast to PLCopen, Section 2.2.2, in which subsequent commands will continue then execution after an FB error.
