# Placeholders for User Management

Placeholders for the **Login visualization** visualization type

These placeholders are specified in the **Text** element property for the elements of the **Login visualization** visualization type.

| Placeholders | Description | Display when logging in to the visualization |
| --- | --- | --- |
| `$USERNAME$` | Usage: Input of the user name (example: in the **Rectangle** visualization element)  Display as plain text | Initially an empty input field  The input user name is displayed as text. |
| `$USERNAME_PH$` | Usage: Input of the user name (example: in the **Rectangle** visualization element)  The placeholder text can be configured as follows: `$USERNAME_PH$User name$` | A placeholder text is displayed in the input field before typing the user name. |
| `$USERNAME_STAR$` | Usage: Input of the user name (example: in the **Rectangle** visualization element) | Initially an empty input field  The characters of the input user name are displayed as \*. |
| `$USERNAME_STAR_PH$` | Usage: Input of the user name (example: in the **Rectangle** visualization element)  The placeholder text can be configured as follows: `$USERNAME_STAR_PH$User name$` | A placeholder text is displayed in the input field before typing the user name.  The characters of the input user name are displayed as \*. |
| `$PASSWORD$` | Usage: Input of the password (example: in the **Rectangle** visualization element) | Initially an empty input field  The characters of the input password are displayed as \*. |
| `$PASSWORD_PH$` | Usage: Input of the password (example: in the **Rectangle** visualization element)  The placeholder text can be configured as follows: `$PASSWORD_PH$Password$` | A placeholder text is displayed in the input field before typing the password.  The password is not displayed. |
| `$PASSWORD_CLEARTEXT$` | Usage: Input of the password in the **Rectangle** visualization element | Initially an empty input field  The password is displayed as text. |
| `$PASSWORD_CLEARTEXT_PH$` | Usage: Input of the password (example: in the **Rectangle** visualization element)  The placeholder text can be configured as follows: `$PASSWORD_CLEARTEXT_PH$Password$` | A placeholder text is displayed in the input field before typing the password.  The input password is displayed as text. |
| `$LOGIN$` | Usage: In **Button** and **Rectangle** visualization elements to complete the input of credentials | **LOGIN** |
| `$LOGIN_DOT$` | Usage: In **Button** and **Rectangle** visualization elements to complete the input of credentials | **...** |
| `$LOGIN_EMPTY$` | Usage: In **Button** and **Rectangle** visualization elements to complete the input of credentials | Blank field |
| `$LOGIN_OK$` | Usage: In **Button** and **Rectangle** visualization elements to complete the input of credentials | Text: **OK** |
| `$LOGIN_CUSTOMIZED$xxx$` | Usage: In **Button** and **Rectangle** visualization elements to complete the input of credentials | The text specified as `xxx` is displayed. |
| `$CANCEL$` | Usage: In **Button** and **Rectangle** visualization elements to cancel the input of credentials | Text: `CANCEL` |
| `$CANCEL_DOT$` | Usage: In **Button** and **Rectangle** visualization elements to cancel the input of credentials | **...** |
| `$CANCEL_EMPTY$` | Usage: In **Button** and **Rectangle** visualization elements to cancel the input of credentials | Blank field |
| `$CANCEL_CUSTOMIZED$` | Usage: In **Button** and **Rectangle** visualization elements to cancel the input of credentials | The text specified as `xxx` is displayed. |
| `$SYS_ERROR$` | Usage: In **Rectangle** and **Text Field**visualization elements to display an error message in case of incorrect credentials.  If there is no error message, then the element is hidden. | If the credentials are correct, then no message is displayed. |
| `$URL$<text>$<URL>$` | Usage: In a visualization element with an available input configuration  URL assigned to a text | Clicking on the displayed text opens the URL in a new tab |
| `$URL$<text>$<URL>$replace$` | Usage: In a visualization element with an available input configuration  URL assigned to a text | Clicking on the displayed text opens the URL in the current tab |
| `$AUTOSIGNIN` | The "Store credentials in browser" option is available on the default sign-in page (`VisuUserManagement.VUM_LoginVisualization`). The option is displayed only for browsers such as Chrome, Edge, or Opera. | The "**Store credentials in browser**" option is available. When it is selected , the password is saved in the browser. Then it is possible to sign in without entering a password ("Auto-Signin"). |
| `$AUTOSIGNIN_CUSTOMIZED` | Usage: To display a customized text for the option made visible with `$AUTOSIGNIN`.  The default text "**Store credentials in Browser**" is displayed if nothing is specified for `$AUTOSIGNIN_CUSTOMIZED`. | The option is available and the customized text is displayed next to it. |
