# General Information About Device Editors

## Overview

The device editor provides parameters for the configuration of a device, which is managed in the Devices tree.

To open the device editor for a specific device, do the following:

* Double-click the node of the device in the Devices tree or
* Select device in the Devices tree and run the Edit Object command from the contextual menu or from the Project menu.

The Tools > Options > Device editor dialog box allows you to make the generic device [configuration views invisible](D-SE-0083387.html#D-SE-0083387).

This chapter describes the main device editor dialogs. Bus-specific configuration dialogs are described separately.

## Main Device Editor Dialogs

The title of the main dialog consists of the device name, for example MyPlc.

Depending on the device type, the device editor can provide the following tabs:

| --- | --- |
| [Communication Settings in Controller selection mode](D-SE-0083385.html#D-SE-0083385) | Configuration of the connection between programming system and a programmable device (controller). The tab that is displayed depends on the mode selected with the parameter Communication page in the Tools > Options > Device editor [dialog box](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMMenu&topicID=D_SE_0084057). |
| [Communication Settings in Simple mode](D-SE-0083386.html#D-SE-0083386) |

| [Synchronized Files](D-SE-0083389.html#D-SE-0083389) | List of files that are downloaded to the controller with an application download. |
| [Files](D-SE-0083390.html#D-SE-0083390) | Configuration of a file transfer between host and controller. |
| [Log](D-SE-0083391.html#D-SE-0083391) | Display of the controller log file. |
| [PLC Settings](D-SE-0083392.html#D-SE-0083392) | Configuration of:   * Application for I/O handling * I/O behavior in stop status * Bus cycle options |
| Services | Lets you configure the online services of the controller (RTC, device identification). |
| [Users and Groups](D-SE-0083876.html#D-SE-0083876) | User management concerning device access during runtime. |
| [Access Rights](D-SE-0083877.html#D-SE-0083877) | Configuration of the access rights on runtime objects and files for the particular user groups. |
| [Symbol Rights](D-SE-0106022.html#D-SE-0106022) | Access rights of individual user groups to symbols (symbol sets) on the device. |
| [Task Deployment](D-SE-0083394.html#D-SE-0083394) | Display of inputs and outputs assigned to the defined task - used for troubleshooting. |

| [I/O Mapping](D-SE-0083397.html#D-SE-0083397) | Mapping of the input and output channels of an I/O device on project (application) variables. |
| [IEC Objects](D-SE-0106082.html) | Lists objects that allow access to the device from the application. In online mode, it serves as monitoring view. |
| OPC UA server configuration | Provides various functionalities to edit the server configuration file ServerConfig.ini on the Compact Flash (CF) card of the controller.  The OPC UA server configuration allows you to perform the following actions:   * Define default properties for new server certificates. * View and delete existing server certificates. * View and add sampling rates. * Import and export server certificates. * Enable and manage various settings.   This function is not available for all controllers. Consult the *Programming Guide* specific to your controller. |
| [Emulation Settings](EmulationSet-62025FC3.html) | For configuring the emulation with EcoStruxure Machine Expert Twin running in the background and the communication between EcoStruxure Machine Expert Twin and the controller. |
