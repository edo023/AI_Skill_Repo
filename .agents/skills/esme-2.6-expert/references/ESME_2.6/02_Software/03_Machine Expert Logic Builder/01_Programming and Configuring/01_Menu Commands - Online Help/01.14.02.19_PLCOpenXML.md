# PLCOpenXML

## Overview

The Tools > Options > PLCOpenXML dialog box allows some settings concerning the PLCOpenXML [export](D-SE-0083972.html#D-SE-0083972) and [import](D-SE-0083973.html#D-SE-0083973).

The General tab allows you to configure the options for the Project > Export PLCOpenXML... [command](D-SE-0083972.html#D-SE-0083972) and the Project > Import PLCOpenXML... [command](D-SE-0083973.html#D-SE-0083973).

The Fast Export/Import tab allows you to configure the options for the commands PLCOpenXML Export and PLCOpenXML Import that are available in the contextual menu upon right-clicking a program node in the Devices tree.

## PLCOpenXML General Settings

The General tab contains the following parameters:

|  |  |
| --- | --- |
| PLCOpenXML export settings | |
| Additionally export declarations as plain text | By default, declaration parts in PLCOpenXML are split into single variables. To help avoid losing formatting and comment information during this process, keep this setting activated. In that case, the plain text of the exported declaration part is additionally written into the PLCOpenXML file.  NOTE: When this option is deactivated, the code can differ after import. |
| Export folder structure | If this option is activated, the folders are also exported if they contain one of the selected objects. |
| PLCOpenXML import settings | |
| Import folder structure | If the import file contains information about the folder structure of the objects, this structure is also imported. If the option is not activated, the objects are imported without any structure. |

## PLCOpenXML Fast Export/Import Settings

The Fast Export/Import tab contains the following parameters:

|  |  |
| --- | --- |
| PLCOpenXML export settings | |
| Additionally export declarations as plain text | By default, declaration parts in PLCOpenXML are split into single variables. To help avoid losing formatting and comment information during this process, keep this setting activated. In that case, the plain text of the exported declaration part is additionally written into the PLCOpenXML file.  NOTE: When this option is deactivated, the code can differ after import. |
| Export folder structure | If this option is activated, the folders are also exported also if they contain one of the selected objects. |
| Export children | If this option is activated, the subelements of the selected objects are also exported. |
| Export each object to a single file if more than one file is selected | If this option is activated, every selected object will be exported to an individual file named after the object name.  A message is displayed, requesting you to specify a folder where the files are to be saved. |
| PLCOpenXML import settings | |
| Import folder structure | If the import file contains information about the folder structure of the objects, this structure is also imported. If the option is not activated, the objects are imported without any structure. |
| Import content header | If this option is activated, the information stored in the `<contentHeader>` element of the PLCOpenXML file is imported into the Project Information [object](D-SE-0083944.html#D-SE-0083944) of the specified project. |
