# Exception handling at runtime

If an error or exception occurs in a visualization at runtime, then the execution of the visualization is stopped without halting the execution of the application. An error screen opens to notify you. Moreover, the error screen (usually) allows you to restart the visualization.

This exception handling is done as of visualization profile  `CODESYS V3.5 SP7` , compiler version `3.5.7.0`, and runtime system `3.5.7.0`.

Click **Stop Execution on Handled Exceptions** to investigate the cause of exceptions and the error location.
