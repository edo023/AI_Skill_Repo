# Dialog: Static Analysis Settings: Metrics

Symbol: ![](images/uuid-44ea3399-0860-6561-667b-e17736.png)

**Function**: Configures the metrics so that they are calculated and displayed accordingly when the [Command: View Standard Metrics](_san_cmd_view_standard_metrics.html#_san_cmd_view_standard_metrics) command is executed.

**Call**:

* **Project → Project Settings** menu, **Static Analysis** category, **Open configuration dialog** link
* **Build → Static Analysis → Settings** menu

**Requirement**:

* The CODESYS Static Analysis package is installed.
* A project is open.

For more information, see: [Configuring and Running Static Analysis](_san_configuring_and_running.html#_san_configuring_and_running)

|  |  |
| --- | --- |
| **Metric** | All selectable [Metrics](_san_reference_metrics.html#_san_reference_metrics) are displayed in the column. |
| **Active** | : The metric is displayed for each POU in the **Standard Metrics** view following the **Build → Static Analysis → View Standard Metrics** command.  : The metric is not displayed in the **Standard Metrics** view following the **Build → Static Analysis → View Standard Metrics** command. |
| **Lower Limit** | Lower value from which the metric is displayed |
| **Upper Limit** | Upper value to which the metric is displayed |

TIP:

The **Code size**, **Variable size**, **Stack size**, and **Number of calls** metrics are reported only for POUs from libraries which are integrated in the project.

TIP:

Violations of the upper and lower limits of the activated metrics can be reported as build errors by means of static analysis rule [`SA0150`](_san_rule_sa0150.html#_san_rule_sa0150).
