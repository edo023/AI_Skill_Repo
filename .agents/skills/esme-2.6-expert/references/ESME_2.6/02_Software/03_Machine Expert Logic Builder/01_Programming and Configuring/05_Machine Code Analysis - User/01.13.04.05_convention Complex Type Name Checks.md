# Convention: Complex Type Name Checks

## User Description

Coding style is a set of rules or guidelines applied when writing source code. Following a specified coding style helps:

* To read and understand the source code
* To avoid and find programming issues
* To maintain the source code

Based on the Programming Guidelines ([Naming Conventions](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083678), [Prefixes](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083680)) for source code, complex type name convention queries are available to verify the suggested complex type names per type.

## Convention Verification Rule

For convention verification, the complex type is combined with the object name.

Complex type name prefixes based on complex types:

* Function block: FB\_ as prefix
* Program: SR\_ as prefix
* Enumeration: ET\_ as prefix
* Structure: ST\_ as prefix
* etc.

## Example

```
FB_MyAxisWrapper
ST_MyDataStruct
ET_MyEnumeration
SR_Main
etc.
```
