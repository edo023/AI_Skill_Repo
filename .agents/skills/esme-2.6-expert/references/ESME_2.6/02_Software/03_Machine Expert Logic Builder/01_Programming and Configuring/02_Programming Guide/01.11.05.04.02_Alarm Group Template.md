# Alarm Group Template

## Overview

The Alarm group template object is useful for library developers to define alarm conditions with variables of a custom type (function block or structure). The data type can be part of a development library. The variables of the function block can be checked for alarm conditions.

The object can be added to the Devices tree or to the Global node of the Tools tree. A text list object with the same name is automatically created.

## Alarm Group Template Editor

Double click the Alarm group template node to open the Alarm group template editor. It provides the following elements:

| Element | Description |
| --- | --- |
| Text list | Name of the text list object that is created with the alarm group template object.  Default name: AlarmGroupTemplate  By default, the automatically generated text list is selected. You can select another text list from the list. |
| Deactivation | Boolean variable to control the deactivation of the alarms in this group.  NOTE: Alarms that have been deactivated are not reported, even if the alarm condition is fulfilled. |
| Type for alarm definition | Interface variable for alarm definitions. |
| inst : <type> | Type (function block or structure) of the alarm instance.  The interface variable allows access to the variables of the customized type. Alarm conditions can be defined. |

## Table of Alarms

Below the configuration elements, the Alarm group template editor contains the table of alarms. Enter the properties of the individual alarms in the corresponding table columns. Depending on the selected Observation Type, input fields are provided below the table for additionally required parameters. For details, refer to [Additional Observation Type Parameters Below the Alarm Table](#AlarmGTemp-4A63D991__ObservationTypes-4A6649E6).

| Table Column | Description |
| --- | --- |
| ID | Alarm ID that is automatically assigned.  The ID is a combination of the instance name and the ID from the library.  Example: GVL.g\_modul1Fan1  Function block FAN\_ALARM\_PRIO with the resulting alarm ID GVL\_g\_modul1Fan1\_ID\_FAN\_ALARM\_PRIO. |
| Observation Type | Observation type selected from the list. Depending on the selected Observation Type, input fields are provided below the table for additionally required parameters. For details, refer to [Additional Observation Type Parameters Below the Alarm Table](#AlarmGTemp-4A63D991__ObservationTypes-4A6649E6). The fields describe an expression for triggering the alarm. |
| Details | Displays the configuration settings that are defined in the fields below the table for the selected alarm type.  Alternatively, you can enter the condition directly in the field. If you enter an expression that does not match the selected Observation Type, the type is automatically adapted. |
| Deactivation | You can optionally configure a variable for deactivating the alarm. |
| Class | Select an alarm class from the list.  Example: AGL.ACL.Error |
| Message | Enter text for a message that is displayed in the Alarm Table visualization element when the alarm is detected. The text is managed in the text list of the alarm group.  You can extend the message with information from latch variables. To insert line breaks, press Ctrl + Enter.  The following placeholders are supported.   * `<DATE>`: Date when the change to the present state was triggered. * `<TIME>`: Time when the change to the present state was triggered. * `<EXPRESSION>`: Expression (defined within the alarm) that triggered the alarm. * `<PRIORITY>`: Priority of the alarm (configured in the [alarm class editor](AlarmClass-0EB53297.html)). * `<TRIGGERVALUE>`\*: Value that caused the alarm condition to become TRUE. * `<ALARMID>`: Alarm ID as displayed in the first column of the alarm table. * `<CLASS>`: Name of the alarm class (refer to [Configuring an Alarm Class](ConfAlarmClass-0C3A7DD7.html)). * `<ALLDEFAULT>`: Complete information about the alarm. * `<CURRENTVALUE>`\*: Present value of the monitored variable. * `<LATCH1>`\*: Value of the first latch variable at the time when the alarm was triggered. To use more than the two default latch variables, insert additional columns from the contextual menu. * `<LATCH2>`\*: Value of the second latch variable at the time when the alarm was triggered. To use more than the two default latch variables, insert additional columns from the contextual menu. * `<ALARM>`: TRUE, if the alarm state is Active. FALSE for other alarm states. * `<STATE>`: Alarm state:    + 0 = normal   + 1 = active   + 2 = waiting for ACK   + 3 = active, acknowledged   **\*** For `<TRIGGERVALUE>`, `<CURRENTVALUE>` and `<LATCHn>`, you can additionally use formatting specifications such as those that are valid for the C-function `printf`. Example: `The value is <CURRENTVALUE %d>`  By default, one column is available for message text. You can add more columns from the contextual menu that is available in the Data type property for column configuration of the Alarm table and Alarm banner visualization elements if configured in the Tools > Customize dialog box. |
| On-Delay time | Time for switch-on delay. Defines the minimum time that the alarm condition must be fulfilled before the alarm is triggered. With this parameter, you can specify that alarm states that are TRUE for less than the configured time will not trigger an alarm. The parameter must have an IEC 61131-compliant format.  Example: `T#2s` |
| Off-Delay time | Time for switch-off delay. Defines the minimum time that the alarm condition must be inactive before an alarm is treated as inactive. Similar to the On-Delay time, this parameter can be used to ignore short-term value changes.  Example: `T#4s` |
| Latch Var 1 | Reserved for instance name. The instance name is generated as a STRING in LatchVar1 so that the alarms can be assigned to an instance. |
| Latch Var 2 | Another latch variable in order to record additional values when the alarm becomes active.  Data type: Scalar (8 bytes) or of type STRING or WSTRING  The latch variable or the latch expression is recorded when the alarm is triggered and provides additional information about the alarm.  By default, columns are available for two latch variables. You can add more columns from the contextual menu that is available in the Data type property for column configuration of the Alarm table and Alarm banner visualization elements if configured in the Tools > Customize dialog box. |
| Higher Prio. Alarm | When the alarm is triggered, it can be acknowledged by triggering another alarm.  Example: A temperature monitor is configured to issue a warning message at temperatures > 60° C and an error messages at temperatures > 90 °C. The warning message is displayed when the temperature reaches 60° C. When the temperature reaches 90° C, the 60° C warning message is acknowledged and the 90 °C error message is displayed. |

## Additional Observation Type Parameters Below the Alarm Table

Depending on the selected Observation Type, below the table of alarms, input fields are provided for additionally required parameters.

| Observation Type | Additional parameters |
| --- | --- |
| Digital | Expression: Enter the expression to be monitored or select it from the Input Assistant.  On the right-hand side, enter the expression for the comparison.  In between both expressions, select the comparison operator from the list (`=` or `<>`). |
| Upper limit1) | * Expression: As described for Digital, but with the comparison operators `>` or `>=` . * Hysteresis in % (optional) |
| Lower limit1) | * Expression: As described for Digital, but with the comparison operators `<` or `<=` . * Hysteresis in % (optional) |
| Outside range1) | The alarm is triggered when the monitored expression is outside the defined range.   * Expression to be monitored. * Area: On the left-hand side, enter the expression for the lower limit of the range. On the right-hand side, enter the expression for the upper limit. The expression to be monitored is displayed in the middle. Set the comparison operators in-between the three fields accordingly. * Hysteresis in % (optional) |
| Inside the range1) | The alarm is triggered when the monitored expression is inside the defined range.   * Expression to be monitored. * Area: On the left-hand side, enter the expression for the lower limit of the range. On the right-hand side, enter the expression for the upper limit. The expression to be monitored is displayed in the middle. Set the comparison operators in-between the three fields accordingly. * Hysteresis in % (optional) |
| Change | Expression: Enter the expression to be monitored or select it from the Input Assistant. An alarm is triggered when the value changes. |
| Event | An event alarm is triggered by using functions from the AlarmManager library. |
| 1) For these observation types, you can monitor the specified expression as absolute or relative. Absolute means that the limit value is defined by a fixed value or a variable that returns a fixed value. Relative means that the limit value depends on an expression; for example: upper limit: `Variable x >= 0.9 * y`.  Hysteresis in %: When you configure a hysteresis, the alarm condition is fulfilled until a certain deviation from the specified limit value is reached. The size of the deviation is specified as a percentage [%] of the limit value.  Example for Upper limit: `i_temp >= 30 Hysteresis: 10%`. The alarm is active as soon as the variable i\_temp reaches or exceeds the value 30. The alarm is no longer active if the value falls below 27. | |

## Commands of the Contextual Menu

The contextual menu includes the following commands for adding and removing additional columns for messages and latch variables:

* Add message column
* Remove message column
* Add latch var column
* Remove latch var column

Select multiple rows in the table to apply the commands of the contextual menu simultaneously.

## Exporting and Importing Alarms

The contextual menu of the alarm tables provides the following commands for the export and import of alarms to and from a CSV file:

* Export Alarms

  : The command opens a default Save As dialog box. It allows you to export the open alarm list to a file of type Alarm Group Export File (\*.csv).
* Import Alarms: The command opens a default dialog box for opening a file of type Alarm Group Export File (\*.csv) that describes alarm definitions.
