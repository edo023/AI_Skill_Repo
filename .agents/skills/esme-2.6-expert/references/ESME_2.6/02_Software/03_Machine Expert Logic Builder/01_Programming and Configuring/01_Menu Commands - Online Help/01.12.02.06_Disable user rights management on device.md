# Disable user rights management on device / Enable user rights management on device

## Disabling / Enabling with EcoStruxure Machine Expert

The Online > Security > Disable user rights management on device / Enable user rights management on device commands are also available in the contextual menu of the controllers.

The Disable user rights management on device command allows you to deactivate the device user rights management in your controller. Upon executing this command, a dialog box is displayed requesting you to specify a folder on your EcoStruxure Machine Expert PC or selecting the controller to save a backup of the user rights that are active. For saving the file to the PC, you must enter a password. If you confirm the dialog box with OK, you are disconnected from the controller and the user is logged off from the device user management.

In order to help prevent unauthorized access to your controller, keep the device user rights management function activated. If you ensure that your machine or process is not accessible to unauthorized personnel, you can deactivate the function by executing the Disable user rights management on device command. Your individual credentials are required for this procedure.

| WARNING | |
| --- | --- |
|  | UNAUTHENTICATED ACCESS AND MACHINE OPERATION  Do not disable user rights management if your machine or process is accessible to unauthorized personnel either directly or via a network.  Failure to follow these instructions can result in death, serious injury, or equipment damage. |

The Enable user rights management on device command re-enables the device user rights management that was previously disabled. A dialog box is displayed requesting you to select whether to restore the device user rights settings from the controller or from the PC, requiring the password that was assigned when the user rights management was disabled.

## Disabling / Enabling with Webserver

The Modicon M241 Logic Controller, Modicon M251 Logic Controller, and the Modicon M262 Logic/Motion Controller allow you to disable and enable the device user rights management via the embedded webserver: MAINTENANCE > USER MANAGEMENT > USER ACCOUNTS MANAGEMENT > DEACTIVATE

Consult the *Programming Guide* specific to your controller for further information:

## Deactivating User Rights for the Simulation Device in Logic Builder

The simulation device in Logic Builder has its own user rights that can differ from those that are defined in the real controller.

NOTE: To help avoid account lockout (deadlocking), first disconnect Logic Builder from the controller and make sure no other client, for example, an HMI, automatically attempts to connect using the previous user rights configuration.

In order to deactivate user rights in the simulation device, proceed as follows:

| Step | Action |
| --- | --- |
| 1 | Close all instances of Logic Builder. |
| 2 | Close all instances of Vijeo-Designer. |

From the contextual menu of the simulated controller, select [Reset Origin Device > User Management](D-SE-0091259.html).
