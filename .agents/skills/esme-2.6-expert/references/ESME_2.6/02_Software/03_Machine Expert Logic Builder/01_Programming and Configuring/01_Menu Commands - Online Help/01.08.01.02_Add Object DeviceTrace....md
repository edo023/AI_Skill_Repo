# Add Object > DeviceTrace...

## Overview

The Project > Add Object > DeviceTrace... command is available if a controller node is selected in the Devices tree. It allows you to add a device trace node below the selected controller node.

While opening the editor in online mode, right-clicking the root node Application.Trace opens the contextual menu that contains the commands Online list... and Upload Trace to update the traces running on the controller. This is also possible while just being connected to the controller (no login necessary). It is not a prerequisite that the associated application is compilable for the controller.

For further information, refer to the description of the trace editor in the [Programming Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083557).

## Displaying the CPU Load with DeviceTrace Objects in the Project

Displaying the CPU load in the project can be helpful when using the multicore functionality.

As a prerequisite, the controller supports the Trace Manager (CmpTrageMgr component). In this example, the controller provides traces of the individual CPU loads (CpuLoad), as well as traces of the CPU load caused by the runtime (PlcLoad).

| Step | Action |
| --- | --- |
| 1 | In the project, define the Communication Settings for the controller. |
| 2 | Select the controller node in the Devices tree and add a DeviceTrace object. |
| 3 | Rename the DeviceTrace to `Trace_PlcLoad` in the Properties. |
| 4 | Set the focus in the trace editor and execute the command Trace > Upload Trace.  **Result**: The connection to the controller is established and the Online List dialog box opens. |
| 5 | In the Online List dialog box, select the `PlcLoad` entry and click Upload and close the dialog.  **Result**: Multiple trace views open in the trace editor to display the CPU load in the runtime. They represent the traces for the individual CPU logical cores of the controller and a trace for the average value. The text No samples have been recorded. is displayed in each view. |
| 6 | Execute the command Trace > Start Trace  **Result**: The trace record for the four parameters is displayed. |
| 7 | To additionally display the traces for the `CpuLoad` per CPU core with their average value in the project, insert another DeviceTrace object into the Devices tree. Assign a name as, for example, `Trace_CpuLoad`. Load and start the traces for `CpuLoad` in the editor as described above.  **Result**: You can monitor all traces in the project. |
| 8 | To change the appearance of the presentation, click Configuration in the respective trace editor window to access the configuration dialog box. You can use this dialog box in the same way as for an IEC variable trace created in a project, except for variable assignments. |
| 9 | To disconnect from the controller, close the open DeviceTrace editor windows.  If you are logged in to the device, log out to terminate the connection. |
