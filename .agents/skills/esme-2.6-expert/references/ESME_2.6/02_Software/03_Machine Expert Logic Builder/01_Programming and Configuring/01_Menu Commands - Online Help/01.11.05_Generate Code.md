# Generate Code

## Overview

The Build > Generate Code command allows compilation of the application just for test purposes. A code generation run will be performed by default when logging in with the application. However, no code will be downloaded and no compile information file will be created. Thus, you can check for any compilation errors.

During the generation of code, some informal messages regarding memory consumption are displayed in the Messages view.

NOTE: The Total allocated memory size for code and data can be less than the sum of Size of generated code and Size of global data because these two areas can partially share the same memory area (for example, for constants in code). To obtain a more detailed overview of memory consumption, execute the Device Memory Info command.
