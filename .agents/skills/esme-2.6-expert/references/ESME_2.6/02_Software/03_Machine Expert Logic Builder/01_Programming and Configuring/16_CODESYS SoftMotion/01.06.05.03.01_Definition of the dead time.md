# Definition of the dead time

The dead time is the time it takes for the drive to actually reach the set position sent by the PLC. That means it is the time lag between the `fActPosition` and the `fSetPosition`, at the beginning of the current bus task cycle. The dead time can be configured in the general drive editor ([Tab: General](_sm_edt_drive_general.html#_sm_edt_drive_general)) or using the MC\_WriteParameter function block with the parameter number 1070 (`fSetActTimeLagCycles`).

**The dead time is the sum the following times:**

* The time it takes to send the set position to the drive
* The time the drive controller needs to reach the set position
* The time it takes to receive the actual position from the drive
