# Online

## Overview

The Tools > Options > Online dialog box allows you to configure a delay that has to elapse without user interaction after login to a controller before the connection is terminated.

## Terminating the Controller Connection

To terminate the controller connection automatically, configure the following options:

| Option | Description |
| --- | --- |
| Force disconnect from device after time of inactivity | By default, this option is unselected.  Select this option to terminate the connection to the controller after the delay configured for <nnnnn> seconds on the right-hand side has elapsed without user interaction (mouse or keyboard inputs). |
| <nnnnn> seconds | Enter a delay (in seconds) that has to elapse without user interaction (mouse or keyboard inputs) before the connection to the controller is terminated.  Default setting: 1800 seconds (30 minutes)  Value range: 10...10800 seconds (10 seconds...3 hours) |

If the connection is terminated after the configured time of inactivity, a message is displayed in the Messages view. You will then have to login manually to reestablish the connection to the controller (for further information, refer to the [*Login* chapter](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083444)). In case a device user management is activated on the controller, you will have to re-enter the corresponding credentials.
