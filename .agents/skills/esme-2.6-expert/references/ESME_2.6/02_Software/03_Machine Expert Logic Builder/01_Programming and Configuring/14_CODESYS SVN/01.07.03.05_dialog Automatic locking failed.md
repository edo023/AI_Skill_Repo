# Dialog: Automatic locking failed

The dialog shows a list of all objects for which an automatic lock was not possible. In the options, you define how CODESYS SVN should resolve the conflict.

Automatic locking and merging

|  |  |
| --- | --- |
| * **Try to steal the lock for the affected objects** * **Activate the "Offline Mode" to temporarily suppress locking** | These options are displayed when another user has locked the object. |
| * **Update the affected objects to the newest revision** * **Update the whole project to the newest revision** * **Activate the "Offline Mode" to temporarily suppress locking** | These options are displayed when a more current version of the object is available on the server. |
| * **Activate the "Offline Mode" to temporarily suppress locking** | These options are displayed when a connection to the server cannot be established. |
| **SVN Project Settings** | Opens the dialog for the [Dialog: Options – SVN Settings](_svn_dlg_options.html#_svn_dlg_options)  There you can change the settings for automatic locking. |
| **SVN Settings** | Opens the dialog for the general [SVN Settings](_svn_dlg_options.html#_svn_dlg_options) |
