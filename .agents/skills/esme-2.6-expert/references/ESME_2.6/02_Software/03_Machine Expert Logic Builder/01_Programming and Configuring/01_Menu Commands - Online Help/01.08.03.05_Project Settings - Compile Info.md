# Project Settings - Compile Info Synchronization Options

## Overview

The Project Settings > Compile Info Synchronization Options dialog box allows you to store the compile information to be able to connect to a controller without download to a dedicated folder.

NOTE: Machine Expert allows you to define a folder where to store the compile information in a file .compile-info-sync.json for each Solution. The settings in this file have priority over the settings configured in this Compile Info Synchronization Options dialog box. If a .compile-info-sync.json file exists, the settings in this dialog box are not considered.

When different users are working on the same project and the option is selected, the compile information is synchronized between the users. Whenever one of the users attempts to connect to a controller and the application running on the controller is exactly the application that the user has opened in EcoStruxure Machine Expert, the connection can be established without requiring a [download or online change](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083446) as the compile information is provided on the common database.

The following options are available:

| Option | Description |
| --- | --- |
| Full Sync | The compile information is automatically stored in the project directory to a file named *<projectname>.<devicename>.<application ID>.compileinfo* when a download to a controller is performed.  The compile information is automatically searched in a local folder. |
| Conditional Sync (ask before Sync) | If a download to a controller is performed, a message is displayed requesting you to save the compile information.  With this option selected, enter the path in the Synchronization Folder >  Location text field or click the browse button (...) to navigate to the location of the synchronization folder. |
| Inactive Sync | By default, the compile information is not saved. |
