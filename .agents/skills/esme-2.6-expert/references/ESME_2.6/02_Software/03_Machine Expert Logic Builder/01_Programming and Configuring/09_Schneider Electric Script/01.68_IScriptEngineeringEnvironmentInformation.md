build

Gets the build number of the executive environment with which the project was last edited. The build number changes with each development step of the application and can be used to distinguish, for example, between a field-test version and an official release. The version number consist of four components: major, minor, build and revision. The format of the [build](#iscriptengineeringenvironmentinformation_module_IScriptEngineeringEnvironmentInformation__iscriptengineeringenvironmentinformation_IScriptEngineeringEnvironmentInformation_IScriptEngineeringEnvironmentInformation_build) number is as follows: major.minor.build.0 The components are used as follows: - Major: Indicates the main version of a release. Each full-installation has its own major version number. - Minor: Indicates an in-place update for an existing release (typically a patch). - Build: Indicates an (internal) release cycle (a development step).

**Note**

See also interface IScriptEngineeringEnvironment .

informational\_product\_version

Gets the version name of the application (for instance 'Version 1.0') with which the project was last edited.

**Note**

This returns the version name of the application in form of a free text. Be careful when trying to parse a version number from the value. If the version is to be evaluated, [build](#iscriptengineeringenvironmentinformation_module_IScriptEngineeringEnvironmentInformation__iscriptengineeringenvironmentinformation_IScriptEngineeringEnvironmentInformation_IScriptEngineeringEnvironmentInformation_build) should be used instead.

Valid examples: - '1.0' - 'V1.0' - 'Version 1.0' - 'Version 2104' - 'Version 1.0.1' - 'Version 1.0 Patch 1' - 'Version 1.0 Update 1' - '...'

**Note**

See also interface IScriptEngineeringEnvironment .
