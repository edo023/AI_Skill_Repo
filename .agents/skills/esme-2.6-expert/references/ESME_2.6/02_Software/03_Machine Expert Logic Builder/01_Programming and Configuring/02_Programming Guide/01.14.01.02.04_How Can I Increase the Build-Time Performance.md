# How Can I Increase the Build-Time Performance?

## Good Practices

The list of good practices may help you to avoid a slow performance:

* Verify that the hardware of the PC meets the [system requirements](../../../../../api/crossBook?lang=en-US&virtualBookName=SoGloRef&topicID=D_SE_0080412).
* Use a Solid State Drive (SSD) and verify that sufficient memory space is available. Contact your IT administration for further details.
* Consider uninstalling components that you do not need.
* If you are using Vijeo-Designer, consider to disable the automatic symbol export function. To achieve this, activate the option Disable automatic symbol export in the Logic Builder Options > Vijeo-Designer [dialog box](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMMenu&topicID=D_SE_0084062).
* Consider activating the option Disable undo after deleting a DTM (performance optimization) in the Logic Builder Options > FDT Options [dialog box](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMMenu&topicID=D_SE_0084054) if this function is not frequently used.
* Avoid large POUs. The size of POUs programmed in graphical languages like LD is not limited. Nevertheless, it is a good practice to call POUs in sequences instead of creating one large POU.
