# Task Groups Tab

## Overview

The Task Groups tab of the Task Configuration allows you to define task groups that can be distributed over the individual processor cores in [Multicore](Multicore-CE144766.html) controllers. Use the Core parameter to define the strategy for binding the task of a task group to processor cores.

NOTE: This tab is exclusive to Multicore controllers.

## Elements of the Tab

The Task Groups tab provides the following elements:

| Element | Description |
| --- | --- |
| Add Group button | Click to create a new task group with the default name `NewGroup_<no>`. |
| Remove Group button | Click to delete the selected task group. |
| Group Name field | Double-click the field to edit the name of the group. |
| Core field | Select one option from the list:   * Free floating  As a prerequisite for this option, use multicore operators, such as [TEST\_AND\_SET](D-SE-0107189.html) or [\_\_COMPARE\_AND\_SWAP](D-SE-0107121.html).  The IEC tasks of the task group are dynamically distributed to the individual processor cores by the load balancing function of the operating system. This has the effect that the load is automatically distributed and manual intervention is not possible. * Sequentially pinned(1)  As a prerequisite for this option, use multicore operators, such as [TEST\_AND\_SET](D-SE-0107189.html) or [\_\_COMPARE\_AND\_SWAP](D-SE-0107121.html).  The IEC tasks of the task group are permanently bound to different processor cores. If the number of tasks exceeds the number of processor cores, the tasks are bound to the individual processor cores, starting with Core1 and continuing with Core2, Core3, etc. In the event of an overflow, the process is continued with Core0, Core1, etc. Manual intervention is not possible. * Fixed pinned(1)  The IEC tasks of the task group are permanently bound to a processor core that is assigned by the runtime (by default, Core1). If several task groups with Fixed pinned setting are available, each task group is bound to a different processor core starting with Core1. * <fixed core number>(1)  Enter a processor core number. The IEC tasks of the task group are permanently bound to this processor core. If the processor core does not exist on the controller, an error is detected during the download. |
| Priority field | Displays the priority of the task as configured with the [Priority parameter](D-SE-0083541.html). |
| **(1)**: The Sequentially pinned, Fixed pinned, and <fixed core number> settings can help to obtain more deterministic jitter times because these settings are not influenced by the dynamic distribution of tasks to the individual cores at runtime. However, the settings prevent the operating system from distributing the load across the individual processor cores. NOTE: Test the Sequentially pinned, Fixed pinned, and <fixed core number> settings before usage to verify that they lead to a significantly improved real-time behavior. | |

NOTE: For data consistency without using multicore operators, assign a task group/tasks for one machine (for example, motion, inputs/outputs, HMI) to a single core.
