# Project Localization > Manage Localizations...

## Overview

The Project > Project Localization > Manage Localizations... command opens the Manage Localizations dialog box. It allows you to select the localization language or the original version of the project. You can add or remove localization files (\*.<language>.po) to or from the project.

To convert localization template (\*.pot) files into localization (\*. po) files, use a third-party translation software application. Create one specific \*. po file for each target language.

## Manage Localizations Dialog Box

The Manage Localizations dialog box contains the following elements:

| Element | Description |
| --- | --- |
| Available Localizations | List of the localization files available in the project.  Example:  proj1-de.poproj1-en.po<Original Version>  The entry <Original Version> represents the localization template (\*.pot file) that is the source file. You can edit only this original version of the project. |
| Add | Click the Add button to open a dialog box for selecting an additional localization file (\*. po) from the file system. |
| Remove | Select a localization file in the Available Localizations list and click the Remove button to remove the file from the project. |
| Default Localization | Activate this option to define the localization file selected in the Available Localizations list as default localization. The entry is displayed in bold.  You can then use the Toggle Localization command to switch between the <Original Version> and this Default Localization. |
| Switch Localization | Click the Switch Localization button to switch to the localization selected in the Available Localizations list. |
| OK | Click the OK button to display the project in the language that is provided by the localization file selected in the Available Localizations list. If you select the entry <Original Version>, the project is displayed in the editable non-localized version. |
