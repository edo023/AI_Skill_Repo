# Project Settings - Implicit Checks

## Overview

The Project Settings > Implicit checks dialog box allows you to select which [POUs for implicit checks](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083416) are verified for their availability during compilation for each application.

You can select the following types of POUs for implicit checks:

* check\_bounds (selected by default)
* check\_pointer
* check\_range\_unsigned
* check\_range\_signed
* check\_div\_real64
* check\_div\_real32
* check\_div\_int64
* check\_div\_int32

The availability of the selected POUs for implicit checks is automatically verified during compilation of the project. If any of the selected POUs is missing, a message is displayed in the Messages [view](D-SE-0083922.html#D-SE-0083922) if the option Show compiler messages, if POUs for implicit checks are missing. is activated in the Tools > Options > POUs for implicit checks dialog box.

The generated [implicit check function](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083669) CheckPointer does by default not detect invalid [pointers](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083669).
