# Implicit Enumerations

## Overview

The implicit enumeration declares an enumeration variable that is valid within the local POU. A variable of this type can take a limited number of integer values, which are declared in the component list.

The implicit enumeration does not have a data type name or identifier.

## Declaration

```
<variable name> : ( <list of component names> ) := <initialization>;
```

|  |  |
| --- | --- |
| `<list of component names>` | Comma-delimited list of component names containing a number of components of your choice  Each component represents an integer value, starting at 0. The value of each following component is incremented by 1.  Example: `Alfa, Bravo, Charlie, Delta, Echo`  A fixed value can also be assigned to each component.  Example: `Zero, Alfa:=10, Bravo:=20, Charlie:=30, Delta:=40, Echo` |
| `<initialization>` | An optional initialization value, usually a component |

## Call Example

```
PROGRAM PLC_PRG
VAR
    iAlphabet : (Alfa, Bravo, Charlie, Delta, Echo) := Echo;
    strText : STRING;
    iNumber : INT;
END_VAR
```

```
CASE iAlphabet OF
    Alfa:
    strText := 'A';
	
    Bravo: 
    strText := 'B';
	
    Charlie:
    strText := 'C';
	
    Delta:
    strText := 'D';
	
    Echo:
    strText := 'E';
END_CASE
iNumber := Alphabet;
```

## Internal Data Type

An implicit enumeration creates an internal data type assigning a name according to the following scheme:

```
IMPLICIT_ENUM_ <POU name> _ <variable name>
```

Do not use this internal identifier because it is possible that the internal identifier is different.
