# References

## Overview

This data type is available in extension to the IEC 61131-3 standard.

A reference stores the address of an object (a variable) that is located elsewhere in memory; in this respect, the behavior is identical to a pointer.

In contrast to a pointer, a variable behaves like an object when the syntax is concerned. Additionally, variables declared as REFERENCE provide the following advantages compared to POINTERS:

* A reference does not have to be dereferenced explicitly (with `^`) to access the contents of the referenced object.
* When passing values to input parameters of functions/function blocks/methods, the following applies: If an input is declared as `REFERENCE TO` `<data type>`, a variable of the corresponding `<data type>` can be passed (`refInput :=` variable instead of `pInput :=ADR`(variable)).
* The compiler verifies that references of the same data type are assigned to each other.

For further information, refer to the *Assignment operator* REF [description](D-SE-0083515.html#D-SE-0083515__D-SE-0083515.3).

You can also use an implicit pointer monitoring function as described in the paragraph CheckPointer [function](D-SE-0083669.html#D-SE-0083669__D-SE-0083669.7).

**Examples with `:=`**

When references are assigned with `:=`, a value is always copied, regardless of whether the reference is on the left-hand side, on the right-hand side, or on both sides:

| Reference at different positions | Copies the value | Pointer notation |
| --- | --- | --- |
| ``` ref_int := iValue; ``` | The value `iValue` is written to the location to which the reference points to. | ``` p_int^ := iValue; ``` |
| ``` iValue := ref_int; ``` | The value to which the reference points to is written after `iValue`. | ``` iValue := p_int^; ``` |
| ``` ref_intA := ref_intB; ``` | The value to which `ref_intB` points to is written to the location to which `ref_intA` points to. | ``` p_intA^ := p_intB^; ``` |

**Examples with `REF=`**

When references are assigned with `REF=`, the address is applied and a reference is required on the left-hand side:

| Reference with `REF=` | Applies the address | Pointer notation |
| --- | --- | --- |
| ``` ref_int REF= iValue; ``` | The reference points to `value`. | ``` p_int := ADR(iValue); ``` |
| ``` ref_intA REF= ref_intB; ``` | `ref_intA` points to the same value as `ref_intB`. | ``` p_intB := p_intB; ``` |
| ``` iValue REF= ref_int; ``` | A compile error is detected. | - |

NOTE: In the declaration, `REFERENCE TO INT REF= iValue` behaves like `REFERENCE TO INT := iValue`.

## Syntax

<identifier> : REFERENCE TO <data type>

## Example Declaration

```
VAR
    p_int : POINTER TO INT;
    ref_int : REFERENCE TO INT;
    iVarA : INT;
    iVarB : INT;
END_VAR

ref_int REF= iVarA; // corresponds to p_int := ADR(iVarA)
ref_int := iVarB; // corresponds to p_int^ := iVarB)
```

NOTE: It is not possible to declare references like REFERENCE TO REFERENCE or ARRAY OF REFERENCE or POINTER TO REFERENCE.

## Check for Valid References

You can use the operator `__ISVALIDREF` to check whether a reference points to a valid value that is a value unequal to 0.

**Syntax**

```
<Boolean variable name> := __ISVALIDREF( <reference name> );
```

`<reference name>`: Identifier declared with type `REFERENCE TO` `<data type>`.

The boolean variable is TRUE, if the reference points to a valid value. Otherwise it is FALSE.

**Example**

Declaration

```
ivar : INT;
ref_int : REFERENCE TO INT;
ref_int0: REFERENCE TO INT;
testref: BOOL := FALSE;
```

Implementation

```
ref_int REF= ivar;
ref_int0 REF= 0;
testref := __ISVALIDREF(ref_int);   (* will be TRUE, because ref_int points to ivar, which is unequal 0 *)
testref := __ISVALIDREF(ref_int0); (* will be FALSE, because ref_int is set to 0 *)
```
