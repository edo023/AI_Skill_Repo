# Starting and Stopping a Recording

You can configure a condition for a trend recording during the recording. When the Boolean variable becomes `TRUE`, the recording is started. When the Boolean variable becomes `FALSE`, the recording is stopped.

1. In the device tree, double-click a **Trend Recording** object.
2. Below **Common Settings**, in the **Record condition** input field, specify a Boolean variable, bit access, or property.

   * At application runtime, data is recorded only when the value is `TRUE`.

NOTE:

The configuration is not possible depending on a trigger variable. For this you need a [Object Trace](../../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D-SE-0083558).
