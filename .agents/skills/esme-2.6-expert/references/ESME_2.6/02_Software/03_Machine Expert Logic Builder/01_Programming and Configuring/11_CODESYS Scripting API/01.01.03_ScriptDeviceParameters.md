ScriptDataElement

Bases: object

A data element of an online parameter.

Notice that instances of this interface usually also implement at least one of the related interfaces [ScriptCompoundDataElement](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptCompoundDataElement), [ScriptValueDataElement](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptValueDataElement), [ScriptDeviceParameter](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptDeviceParameter), [ScriptCompoundDataElement](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptCompoundDataElement), [ScriptEnumerationDataElement](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptEnumerationDataElement), [ScriptRangeDataElement](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptRangeDataElement), which contain additional useful members, e. G. [ScriptValueDataElement.value](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptValueDataElement_value). Which interface applies can be determined via the properties [has\_sub\_elements](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptDataElement_has_sub_elements), [is\_range\_type](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptDataElement_is_range_type), [is\_enumeration](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptDataElement_is_enumeration), [is\_union](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptDataElement_is_union) and [parameter](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptDataElement_parameter).

ScriptIoMapping

Bases: [ScriptIoVariableMapping](#scriptdeviceparameters_module_ScriptDeviceParameters__scriptdeviceparameters_ScriptDeviceParameters_ScriptIoVariableMapping)

Describing the I/O Mapping options. This object also represents the first actual mapping for the variable.

Currently the device IO mapping editor UI allows only a single mapping to be configured, so we’ll implicitly handle this mapping similar to what the device IO mapping editor does. When an output variable (or a member thereof) is a struct, either the struct itself or its components may be mapped, but not both, as the semantics of the colliding mappings is not clear.
