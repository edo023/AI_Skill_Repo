# Stack Usage Tab

## Overview

The Stack Usage tab of the Task Configuration displays the stack usage per application, per task and for the POUs in the maximum call stack column.

As a prerequisite for data to be displayed in this tab, the application code must have been generated. A code generation run is performed automatically when logging in with the application to the controller. To start code generation explicitly, execute the Build > Generate Code command.

## Elements of the Tab

The Stack Usage tab provides the following elements:

| Element | Description |
| --- | --- |
| Override default stack size from the target settings option | By default, this check box is cleared and the Available stack size value cannot be edited.  Select the option to make the Available stack size field editable. If you enter a value that is lower than the calculated value, consider the hazard message below this table. |
| Reset to default button | When the Override default stack size from the target settings option is selected, click this button to reset the value of the Available stack size field to the default value that is read from the device description. |
| Available stack size field | By default, this field is read-only and indicates the available stack size (in bytes) that is read from the device description.  Example: 4,194,304  To edit this field manually, select the Override default stack size from the target settings check box. Consider the hazard message below the table.  During compilation, the value entered is verified and an error is detected when the calculated stack size exceeds the value entered. |
| Assumed stack size for external calls field | This field is read-only and indicates the general estimated value of stack size consumption by an external call (in bytes) that is read from the device description.  Example: The value 24 indicates that it is estimated that a call to an external function consumes 24 bytes of the stack.  To use another value than the estimated stack size provided by the device description, assign a value of your choice to an external call by using the attribute [`'estimated-stack-usage'`](D-SE-0083629.html). Consider the hazard message above the table. |
| Task/Maximum Call Stack column | This column indicates call hierarchy, that is, the call stack that was determined during compile.  To populate this view with values, start a code generation by executing the Build > Generate Code command. |
| Stack Size column | The cells of the Stack Size column indicate the calculated stack size (in bytes) per called POU or the stack size added up by the call hierarchy for a task.  NOTE: The figures displayed are calculated depending on estimations. For further information, refer to [Calculation of the Stack Usage](#StackUsageTab-CDBACAE1__CalculationOfTheStackUsage-CE0F3FFF).  The figures are updated during each compile operation.  For POUs displayed in red, a stack overflow has been detected. The POUs that are in the hierarchy below are also displayed in red due to memory overflow. |

Manually decreasing the Available stack size value may lead to stack overflow exceptions.

## Calculation of the Stack Usage

The stack usage is calculated statically using the static call stack. The calculation is based on the following principles:

* The calculation starts with the hidden/implicit callee for the entry of the task.
* For each IEC program call (for example, `PLC_PRG`), the stack usage of each IEC POU that is called is calculated recursively. The largest possible stack usage of a POU is added to its own local stack usage in order to determine the maximum stack usage of the POU.
* For dynamic calls, the stack usage is calculated as follows:

  The stack usage of possible implementations is determined and the largest stack usage is assumed. For interface methods, possible implementations and for function block methods, possible overrides are taken into account.
* For external calls, the value specified by the target device is assumed. If no value is specified by the device, then 0 is assumed.
* In the case of recursion, the calculation is aborted. The value of the type pragma attribute[`'estimated-stack-usage'`](D-SE-0083629.html) is applied for the recursively called code. If no value is specified, then 0 is assumed.

NOTE: The displayed values are estimations. They may be too high because the assumed deepest call path is never run through at runtime. They may also be too low because the estimations of recursions or external calls were not adequate.
