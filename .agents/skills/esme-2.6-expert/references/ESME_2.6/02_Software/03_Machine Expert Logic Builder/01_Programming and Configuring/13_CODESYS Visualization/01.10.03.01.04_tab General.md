# Tab: General

The tab contains the general metadata of the open style file and allows it to be edited.

Identification

|  |  |
| --- | --- |
| **Company** | Example: `Xy-z GmbH`  TIP: In the installed styles, CODESYS can filter by the company names specified here. |
| **Name** | Example: `Style_A` |
| **Version** | User-defined version number  Example: `1.1.1.1` |

Common Settings

|  |  |
| --- | --- |
| **Base style** | Name and version of the style that the open style is based on.  Hint: The derived style properties from the base style are highlighted in yellow on the **Style Properties** tab. |
| **Partial style (usable only as base for other visualization styles)** | : The style is identified as partial. Therefore, it can be used for other styles as a base style only.  Example: A style which has been defined only with color definitions and which is passed to many other styles.  Note: A partial style on its own is not checked for consistency, but only in the hierarchy together with the styles which it inherits.  IMPORTANT: A style with this setting enabled will not be displayed in the Visualization Manager (under **Style Settings)** in the **Selected style** list box). The partial style is not selectable.  : Complete style which is checked and then displayed in the Visualization Manager (under **Style Settings** in the **Selected style** list box).  For more information, see the following: [Tab: Settings](_visu_obj_manager_settings.html#_visu_obj_manager_settings) |
| **From** | The **Select base style** dialog opens. The dialog prompts you to select a style file that is saved to the file system (and not does not have to be installed). The file is used as a base style. |

Informational

|  |  |
| --- | --- |
| **Visualization Profile** | The profile is intended for informational purposes. For example, you find elements that are not preconfigured with special style entries, and information from the profile. In addition, CODESYS checks in the profile whether a required style is missing. |
