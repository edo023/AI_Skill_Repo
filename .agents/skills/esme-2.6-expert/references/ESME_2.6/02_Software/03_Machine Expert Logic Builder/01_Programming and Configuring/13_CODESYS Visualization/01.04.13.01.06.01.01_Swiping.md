# Swiping

With this feature, the visualizations referenced in the element (**Frame** or **Tabs**) can be toggled by means of a swipe gesture (pan or flick gesture).

|  |  |
| --- | --- |
| **Swiping behavior** | Requirement: Swiping is possible only when the **Multitouch handling** option is selected.  **Options for how a visualization user needs to perform the swipe (pan or flick gesture) to trigger an image change to the next referenced visualization in the element:**   * **Horizontal swiping**: Swiping from right to left, and vice versa * **Vertical swiping**: Swiping from top to bottom, and vice versa * **Not swipable**: Swiping is disabled (default)   Hint: Set the scaling type to **Anisotropic**. |
| **Swiping preview** | Requirement: The option is available only in overlay mode (**Support client animations and overlay of native elements** option) to display the preview images on multitouch devices.  : During the swipe gesture, a preview image of the next visualization is displayed in the frame. |

For more information, see: [Referencing of Visualizations](_visu_display_visualizations_in_a_visualization.html#_visu_display_visualizations_in_a_visualization) or [Tab: Settings](_visu_obj_manager_settings.html#_visu_obj_manager_settings)
