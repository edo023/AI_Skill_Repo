# Adding Communication Managers

## Overview

Communication managers are mandatory to activate and configure any hardware bus interface, for example CANopen or serial line.

Two types of communication managers exist:

* Fieldbus managers which allow to configure fieldbus devices (for example CANopen or Modbus devices)
* General communication managers

Available communication managers are listed below:

| Name | Interface type | Description |
| --- | --- | --- |
| ASCII Manager | Serial line | Used to transmit and/or receive data with a simple device. |
| Machine Expert-Network Manager | * Serial line (maximum 1) * Ethernet (maximum 3) | For exchanging data with HMIs. |
| Modbus IOScanner | Serial line | Modbus RTU or ASCII protocol manager used to define implicit exchanges (I/O scanning) with Modbus devices. |
| Modbus Manager | Serial line | Used for Modbus RTU or ASCII protocol in client or server mode. |
| CANopen Performance | CAN | CANopen manager for performance controllers (M241, M251). |
| CANopen\_Manager | CAN | CANopen manager for PacDrive controllers. |
| Modbus TCP Slave Device | Ethernet | Modbus TCP manager for controllers with Ethernet port. |
| EthernetIP | Ethernet | EtherNet/IP manager for controllers with Ethernet port (M251). |
| Industrial Ethernet Network | Ethernet | Used to configure EtherNet/IP and Modbus TCP scanner services on M241 / M251 controllers with Ethernet port. |
| EtherNet/IP Scanner | Ethernet | Used to configure EtherNet/IP Scanner on M262 controllers. |
| ModbusTCP IOScanner | Ethernet | Used to configure Modbus TCP scanner services on M262 controllers. |
| Sercos Master | Ethernet | Used to configure a Sercos Master on M262 and PacDrive controllers. |
| PROFINETIO-Controller | - | Used to configure a PROFINETIO-Controller based on Hilscher netX on PacDrive controllers. |
| EtherNet-IP-Scanner | - | Used to configure an EtherNet-IP-Scanner based on Hilscher netX on PacDrive controllers. |
| TCP/UDP Manager | - | Used to create a customized protocol based on TCP or UDP for exchanging data with a third party device on M262 controllers. |
| PROFIBUS-DPV1-Master | - | Used to configure a ProfiBus master based on Hilscher netX on PacDrive controllers. |
| EtherCAT-Master | - | For configuring an EtherCAT master based on Hilscher netX on PacDrive controllers. |

## Adding the Communication Manager

Communication managers are automatically added with the respective device.

To add a communication manager separately, proceed as follows:

| Step | Action |
| --- | --- |
| 1 | In the Devices Tree, select the bus interface (Serial Line, CANopen bus / CANbus, Ethernet) and click the green plus button of the node or right-click the bus interface node and execute the Add Device... command from the contextual menu.  **Result**: The Add Device dialog box opens. |
| 2 | In the Add Device dialog box, select Schneider Electric from the list Vendor.  **Note**: You can filter the devices by brand by clicking the list Vendor. |
| 3 | Select the Communication manager from the list. |
| 4 | Rename your device by typing a name in the Name box.  **Note**: Do not use spaces within the name. If you do not rename the device, a name is given by default.  Naming the device meaningfully may ease the organization of your project. |
| 5 | Click the Add Device button. |
| 6 | Click the Close button to close the Add Device dialog box. |
| 7 | Configure the Communication manager. |
