# Add Exit Action

## Overview

The Add Exit Action command opens the Add Exit Action dialog box that allows you to define a new exit action. Depending on the settings made for Default insertion method in the Tools > Options > SFC Editor [dialog box](D-SE-0084059.html#D-SE-0084059__D-SE-0084059.5), a dialog box is displayed prompting you to select the duplication mode for the new step.

## Adding an Exit Action

To add a new exit action, proceed as follows:

| Step | Action | Result |
| --- | --- | --- |
| 1 | Execute the command SFC > Add Exit Action. | By default, a dialog box is displayed prompting you to select the duplication mode for the new action. |
| 2 | Select a duplication mode, and click OK. | The Add Exit Action dialog box opens.  For further information, refer to the [*Selecting the Duplication Mode for Step Actions*](D-SE-0084153.html#D-SE-0084153__D-SE-0084153.4) . |
| 3 | Enter the Name for the new step, for example Step0\_exit, select Structured Text (ST) from the Implementation language list, and click Add. | * A new node Step0\_exit is automatically inserted in the Applications tree below the SFC POU node. * An ST editor opens allowing you to program statements that are executed once after the Step0 has become inactive. |
| 4 | Close the Step0\_exit editor. | * The `Step0` block in the SFC editor is now marked with an `X` in the lower right corner. Double-clicking this marker opens the ST editor. * `Step0_exit` is available in the Properties of the step as Specific > Actions > Exit action. |

If you copy and paste the `Step0` block, the same exit actions are available in the copied step, and the new step calls the same action.
