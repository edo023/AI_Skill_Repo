# Compiler Error C0415

## Message

Out of persistent memory. Variable '<variable name>', <count> bytes (Largest contiguous memory gap `<size>`). Editing persistent variable lists may produce fragmented memory. Perform Declarations, Reorder List and Clear Gaps command to compact persistent variable lists.

## Message Cause

There is not sufficient persistent memory available on your controller for the declared persistent variables.

The persistent memory is intended for variables that are intended to retain their values after the download. As a prerequisite, the type of the variables must not be modified. Therefore, map as few variables as possible with complex data types (for example, structures or arrays of structures) in the persistent area. Modifications of these variables can lead to large gaps in the persistent data that result in this error being detected.

NOTE: Persistent memory on your controller is limited. Therefore, verify whether the variables declared as persistent are truly necessary to be made persistent or, if possible, move variable declarations to a Global Variable List.

## Solution

The following action can correct the detected error. Execute the Reorder List and Clear Gaps command to reorder the persistent variables and reallocate the memory. This leads to reinitialization of the persistent variables. Alternatively, execute the Save Current Values to Recipe command for the present values. After a download, run the Restore Values from Recipe command to restore the values.

For further information, refer to the commands Reorder List and Clear Gaps and Restore Values from Recipe in [Commands of the Persistent Variable Editor](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMMenu&topicID=D_SE_0084168).
