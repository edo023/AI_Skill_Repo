# Add Device User

## Overview

The Online > Security > Add device user... command is only available if you are logged in on a target system (controller) supporting an online user management. It serves to define a new online user, who is added to the administrators group automatically.

NOTE: You can handle the online user management via the Users and Groups view of the device editor ([**Users and Groups**](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083876)). The commands of the menu Online > Security provide an additional simplified opportunity to protect the access on the target device.

A dialog box opens where you have to specify a Name and Password for the new user.

NOTE: Use a secure password. A secure password is one that has not been shared or distributed to any unauthorized personnel and does not contain any personal or otherwise obvious information. Further, a mix of upper and lower case letters and numbers offer greater security. You should choose a password length of at least seven characters. The password must comply with the password policy configured in the [Change Runtime Password Policy dialog box](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=ChangeRuntimePasswordPolicyDialogBo_9C3C10B5).

NOTE: After having added a new user in this way, you will not be able to log in anymore with an empty username and password.

If you want to deactivate the device user rights management, use the Disable user rights management on device [command](D-SE-0104289.html#D-SE-0104289).
