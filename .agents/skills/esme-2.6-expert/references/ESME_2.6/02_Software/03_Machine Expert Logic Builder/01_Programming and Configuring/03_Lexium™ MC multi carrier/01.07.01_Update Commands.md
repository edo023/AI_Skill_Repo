# Update Commands

## Overview

An Update button is provided according to the editor mode:

* In single-track mode, the Update button is provided in the [Track Tab](TPC_MLS-Config_Tab_Track-D99C4272.html).
* In multi-track mode, the Update button is provided in the [Overview tab](OverviewTab-E4C6B98A.html).

The Update button provides further commands to manually adapt the devices in the Devices tree or the multi carrier objects in the Applications tree to the content of the Multicarrier Configuration editor and vice versa.

## Automatic Updates (Project Settings)

If you activate the option Automatic updates in the Project > Project Settings > Multicarrier Configuration Editor > General tab, the content of the multi carrier configuration is automatically synchronized with the Devices tree and the generated code.

* If you make modifications in the Multicarrier Configuration / track editor, these modifications are automatically updated (Update To All) when another editor within EcoStruxure Machine Expert becomes the active window.
* If you make modifications to a multi carrier device or to the GVL\_MulticarrierConfiguration, these changes are automatically updated (update From Last Modified) to the Multicarrier Configuration editor / track editor when the Multicarrier Configuration editor / track editor becomes the active window.

NOTE: If you activate the Automatic Updates option, you must complete and verify any modifications you make with the Multicarrier Configuration editor / track editor. If modifications are left incomplete and you leave the Multicarrier Configuration editor / track editor, the automatic update is executed on incomplete or inconsistent data which may corrupt part of the multi carrier configuration.

## Update > To Devices Command

Running the Update > To Devices command creates or updates the following nodes in the Devices tree:

* For each track you created in the Multicarrier Configuration editor, a Lexium MC Track device is inserted under the Sercos Master node.
* For the segments of your track, Lexium MC Segment devices are inserted under the Lexium MC Track device.
* For the carriers, a Lexium MC Carrier Pool node is inserted under the PacDrive LMC device.

  The Lexium MC Carrier Pool node contains the Lexium MC Carrier devices.

If the aforementioned devices already exist in the Devices tree, they are adapted to the content of the Multicarrier Configuration editor.

To configure the names that are assigned to the newly created devices in the Devices tree, open the Device Generation tab of the Project > Project Settings > Multicarrier Configuration Editor dialog box (refer to [Project Settings - Multicarrier Configuration Editor](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMMenu&topicID=TPC_MLS_Config_Tools_Options_D995A499)).

The naming of track devices depends on the track mode:

* In single track mode, the name of the track device is defined with the parameter Track Base Name in the Project > Project Settings > Multicarrier Configuration Editor dialog box (refer to [Project Settings - Multicarrier Configuration Editor](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMMenu&topicID=TPC_MLS_Config_Tools_Options_D995A499)).
* In multi-track mode, the name of the track device is set to the name of the corresponding track object.

Assigning addresses with the [Topological Start Address command](TPC_MLS-Config_Tab_Track-D99C4272.html#TPC_MLS-Config_Tab_Track-D99C4272__TopologicalStartAddress-F1A964ED):

If the Topological Start Address of a track contains a valid Sercos address, the segments of the track are assigned a unique topological address, starting with the configured value.

If the Topological Start Address is set to -1, no addresses are assigned to segments of the track. In this case, set the appropriate segment addresses manually.

NOTE: In multi-track mode, select Topological Start Address values so that the address areas of the tracks do not overlap. Otherwise the command cannot be executed.

## Update > To Code Command

Running the Update > To Code command generates or updates IEC code and data needed for running the Lexium™ MC multi carrier.

| NOTICE | |
| --- | --- |
|  | LOSS OF DATA  Do not modify the generated code as it is overwritten, without further notice, when running the Update > To Code command.  Failure to follow these instructions can result in equipment damage. |

**For tracks:**

For the track you created in the Track tab, a MulticarrierConfiguration folder node is inserted under the Application node in the Applications tree.

This folder contains the GVL\_MulticarrierConfiguration and the associated structures (for example, ST\_Carrier, ST\_Segment, ST\_Station, ST\_ToolType).

If code already exists in the Applications tree, it is adapted to the content of the Multicarrier Configuration editor.

**For emulation data:**

If you activate the option Create Emulation Data in the Project > Project Settings > Multicarrier Configuration Editor > Code Generation tab, the Update > To Code operation also creates the emulation data that is needed for emulating the track with the EcoStruxure Machine Expert Twin.

Emulation data comprises:

* A MulticarrierEmulation folder node which is inserted under the Application node in the Applications tree. This folder contains the GVL\_MulticarrierEmulation and the SR\_EmulationInterface program.
* The SR\_EmulationInterface program which synchronizes the OPC UA structures with the GVL\_MulticarrierEmulation. Do not change the content of this program.

  The GVL\_MulticarrierEmulation must be set by your application so that the EcoStruxure Machine Expert Twin can emulate the track. For further information, refer to the [DigitalTwinCommunication Library Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=DiTwCoLB&topicID=DigitalTwinCommunication_66B162AC).

  NOTE: As a prerequisite for setting the emulation variables as published in the Symbol Configuration, ensure that the build process of the application has been performed successfully.

**For stations:**

If you activate the option Create Code for Stations in the Project > Project Settings > Multicarrier Configuration Editor > Code Generation tab, the Update > To Code operation also creates code for stations.

The station data comprises a MulticarrierStations folder node that is inserted under the Application node in the Applications tree. This folder contains a subfolder Autogenerated Stations that, in turn, contains the following elements:

* A subfolder User Stations that contains a subfolder for each User Station that is available in the [Stations tab](TPC_MLS-Config_Tab_Stations-DA9DBC46.html). User Stations allow you to enter your own code because the content of User Stations is not overwritten with the Update > To Code command.
* A subfolder Visualizations that contains one visualization for each station. The visualizations can be used to control the generated station(s).
* One program is created per generated track starting with `SR_CallStationGenerated (PRG)` for the first track. The node name is extended by consecutive numbers `SR_CallStationGenerated_<n> (PRG)` per track. For further information, refer to the [Lexium™ MC12 multi carrier Example Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=exMulCar&topicID=AutoGenMode_3F724595).

## Update > To All Command

The Update > To All command synchronizes both the code and the devices in your project with the Multicarrier Configuration editor.

## Update > From Devices Command

You can modify your multi carrier configuration by adding/changing/deleting segments or carriers. To apply the modifications to the Multicarrier Configuration editor, run the Update > From Devices command. Your configuration in the Multicarrier Configuration editor is then updated from the configuration in the Devices tree.

## Update > From Code Command

You can add/change/remove your multi carrier definition in the code, like adding stations, carriers or segments by editing the GVL\_MulticarrierEmulation. To apply the modifications to the Multicarrier Configuration editor, you can read from code by executing the Update > From Code command. The configuration in the Multicarrier Configuration editor is updated according to your definition in the code.

## Update > From Last Modified Command

Run the Update > From Last Modified command to synchronize the track in the Multicarrier Configuration editor with the Devices tree or with the GVL depending on what was last modified.
