# Convert to File-Based Project... (exclusive to \*.project and \*.library files)

## Overview

This function is exclusive to \*.project and \*.library files in classic storage format.

The File > Convert to File-Based Project... command converts a classic \*.project or \*.library file in classic storage format to a file-based storage \*.fbs project.

## Converting a Classic Project to a File-Based Project

To convert a classic project to a file-based project, proceed as follows:

| Step | Action |
| --- | --- |
| 1 | In the Logic Builder, open the File menu and select the Convert to File-Based Project... command.  **Result**: The Convert to File-Based Project dialog box is displayed. |
| 2 | Define the Name of the file-based project file and browse to the Location where you want your \*.fbs file to be saved.  The folder path and the name of the folder that will be created for the file-based format is displayed in the Project Folder text box. |
| 3 | Click OK to start the conversion.  As this process can take some time, a message is displayed in the lower right corner of the Logic Builder screen that allows you to cancel the operation.  Before the conversion is performed, the project is verified to detect conversion issues. Detected issues are displayed in the Convert project to file-based storage dialog box. |
| 4 | Verify the messages that are displayed in the table of the Convert project to file-based storage dialog box.  Messages of type error must be fixed before the conversion process can be continued. To achieve this, refer to the column Fix of the table.  Messages of type warning provide useful information but do not stop the conversion process. |
| 5 | To start the conversion process, click the OK button.  **Result**: The classic \*.project or \*.library file has been converted to a file-based storage \*.fbs project. In the Logic Builder, the [Status & Staging view](StatusStaging-2CE9FB60.html) opens that allows you to initialize a repository to enable source control features. |
