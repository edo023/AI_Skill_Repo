# Multicore - Tools

## Overview

Multicore is a term associated with parallel processing and refers to the use of several cores that are simultaneously engaged in the execution of a program. Parallel processing divides computational-intensive operations into smaller parts and distributes these tasks among several cores. The goal is to speed up processing of computationally-intensive operations.

Characteristics of computationally-intensive applications:

* Large amount of data to process.
* Complex algorithms requiring heavy computations.

We can understand the control program for a system, that was previously calculated on a single processor core, as a complex algorithm. The division of this complex program flow into several subprograms (task partitioning) that can be executed as independently as possible enables the reduction of the cycle times. This results in a better reaction time of the system because the sampling rates can be increased.

Goals of task partitioning:

* Evenly dividing the effort among the available cores.
* Minimization of contention for shared resources.

Up to now, the processing of very large amounts of data (for example, image processing in real time) has often been outsourced to specialized control components because the computing capacity of controllers with one processor core has not been sufficient to handle both parts of the data processing in the required time window.

The ability to use multiple processor cores out of the controller program brings additional possibilities. The parallel processing of data, distributed over several processor cores, makes it possible to execute complex processing sequences (for example, for image processing in real time) in a required time window. In many cases, it is no longer necessary to implement parallel file processing outside the controller with special tools and programming languages. Parallel data processing becomes an integral part of a controller program.

## Example

The new possibilities are illustrated with the example of industrial image processing, that is based on the evaluation of taken images, that then influences the sequence of further production steps.

In this example, the image processing consists of three steps:

1. Image capture due to an external event.
2. Image analysis.
3. Decision on rejection as scrap or forwarding to subsequent process steps.

With single core controllers, the image analysis step is outsourced to an external system that is triggered by an external event. The result of the image analysis is provided through a physical input of the controller.

With multi-core controllers, the following application design is feasible:

Image analysis is performed in a library block that is triggered by a local input variable. This reduces the processing time for image evaluation to a minimum based on the number of available processor cores. The calculations are thus dynamically distributed to an optimal number of processor cores. Subsequently, the results are summarized in order to be able to make a decision about whether the present product meets the requirements. In addition to the computational effort for image processing, there is also the effort for distributing the data and the effort for summarizing the results. Therefore, the image processing efficiency does not increase linearly with the number of processor cores. The different efforts are to be evaluated to lead to shorter processing times per product.

## Atomic Operations

Atomic operations are mainly used to implement locking mechanisms (locking) used in the synchronization between tasks when accessing shared data.

An operation acting on shared memory is atomic if it completes in a single step relative to other task. When an atomic store is performed on a shared variable, no other task can observe the modification half-complete. When an atomic load is performed on a shared variable, it reads the entire value as it appeared at a single moment in time.

Typical processor instructions for implementing atomic operations are:

* Atomic load and store of integral IEC data types as, for example, LINT, \_\_XWORD, POINTER, INTERFACE, …
* Multicore operators
