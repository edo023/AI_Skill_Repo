# Project Settings - Library development

## Overview

The Project Settings - Library development is used to configure parameters for the [Check All Pool Objects command](D-SE-0083983.html).

During the compile process of an application, only the function blocks/programs/functions that are actually used directly or indirectly by the application code are compiled. The Check All Pool Objects command additionally verifies the function blocks/programs/functions that are not used. All execution paths in the code are taken into account.

## Compiler Defines to Use Field

Click the Scan button to start scanning the function blocks/programs/functions of the library. The compiler defines that are found are provided as a comma-separated list in the Compiler Defines to Use field.

When cascading statements `{IF defined(...)}-{ELSIF defined(...)}` are detected, only the define at `{IF}` is taken into account.

## Check All Pool Objects Command

If you save a POINTER variable in a DWORD variable in the library, the library can exclusively be compiled for 32-bit devices.

To detect and avoid this issue, specify the pointer size to be used for the Check All Pool Objects command. The Pointer size to use list provides the following values:

* 32 Bit
* 64 Bit
* 32 and 64 Bit

## Example

To use the Check All Pool Objects command to detect invalid assignments for the 64-bit pointer size, proceed as follows:

```
FUNCTION F : INT
VAR
               
ptr : POINTER TO BYTE;
               
ptrSave : DWORD;
END_VAR
ptrSave := ptr;
```

**Result**: Detected error C0032: Type 'POINTER TO BYTE' cannot be converted to type 'DWORD'.

Use the following declaration for `ptrSave` to avoid this detected error:

```
ptrSave : __XWORD;
```
