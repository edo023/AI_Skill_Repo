# `Attribute no_init`

## Overview

Variables provided with the pragma {attribute no\_init} will not be initialized implicitly. The pragma belongs to the variable declared subsequently.

## Syntax

```
{attribute 'no_init'}
```

also possible

```
{attribute 'no-init'}
{attribute 'noinit'}
```

## Example

```
PROGRAM PLC_PRG
VAR
A : INT;
{attribute 'no_init'}
B : INT;
END_VAR
```

If a reset is performed on the associated application, the integer variable `A` will be again initialized implicitly with 0, whereas variable `B` maintains the value it is currently assigned to.
