# Reset Origin Device

## Overview

The Reset origin device command is available in the contextual menu of a controller in the Devices tree if the controller is in online mode with the application. It resets the controller to the default settings.

The command opens a dialog box with the following options:

* User Management: User management is removed from the controller.

  NOTE: In comparison to the [command Reset user rights management to default](../../../../../api/crossBook?lang=en-US&virtualBookName=D-SE-0104288.html), it does not change the settings of the communication policy automatically. If Enforced user management is required, you can change this manually in the [Change Communication Policy dialog box](../../SoMProg&topicID=D_SE_0083386) or call the command Reset user rights management to default.
* Plc Logic: Applications, boot applications, remnant variables, visualization files, alarm files and recipe files are removed from the controller.
* Certificates: Certificates are removed from the controller.

NOTE: The results of this function may vary according to the particular controllers configured, as not all controllers support the different objects affected. Consult the *Programming Guide* specific to your controller.
