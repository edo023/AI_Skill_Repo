# Changed working copy format in CODESYS SVN V4.1.0.0 and higher

For projects in version CODESYS SVN `V4.1.0.0` and higher, the working directory (working copy) has a new format.

If you open a project that was created with `V4.0.4.0` or lower, then the project is updated automatically to the new format when it is opened.

If you open a project that was created with `V4.0.4.0` or lower and the project is based on an older SVN version of `1.7.x` or lower, then you are prompted whether or not CODESYS should update the format. If you reject the update, then the SVN link of the project will be deactivated. You can still load and edit the project.

The update does not have an effect on saving to the SVN server. You can also check out projects with earlier versions of the client. The new format affects only the local working directory.

For more information, see: <http://svnbook.red-bean.com/en/1.8/svn.ref.svn.c.upgrade.html>
