# Alias

## Overview

A user-defined data type alias allows you to declare an alternative name for a basic type, data type, or function block.

Select the Type = Alias in the [Add DUT dialog box](D-SE-0083427.html).

## Declaration of an Alias in a DUT

```
TYPE A_MESSAGE : STRING[50];
END_TYPE
```

## Syntax

```
TYPE <DUT name> : <type name>;
END_TYPE
```

The following types are permitted for `<type name>`:

* `<basic type name>`
* `<data type name>`
* `<function block name>`

## Example for Variables of Type STRING or ARRAY with Specific Length

Declaration part:

```
TYPE FRAME : ARRAY[0..1499] OF BYTE; END_TYPE
TYPE SYMBOL : STRING(512); END_TYPE
```

Call:

```
PROGRAM PLC_PRG
VAR
    frameF : FRAME;
    symbolS : SYMBOL;
END_VAR
```

## Example for Variables Requiring a Different Initial Value

This example is valid for an alias data type for variables that require a different initial value than the one provided by the compiler.

```
TYPE INDEX : DINT := -1; END_TYPE
```

## Example for Variables Applying a Subset of Values

This example is valid for an alias data type for variables of a specific type where a specific subset of values of the original type is to be applied.

```
{attribute 'qualified_only'}
VAR_GLOBAL CONSTANT
    c_diMaxRune : DINT := DINT#16#0010FFFF;
END_VAR
```

```
TYPE RUNE : DINT(0..GVL.c_diMaxRune); END_TYPE
```

## Example for Types From Another Namespace

This example is valid, for instance, for types from a subordinate library to make them available in the present namespace.

```
TYPE ENCODING: SBB.ENCODING; END_TYPE (*ENUM*)
TYPE RUNE : UTF8.RUNE;  END_TYPE (*SUBTYPE*)
TYPE INFO : STR.INFO;  END_TYPE (*STRUCT*)
TYPE IBuilder : SBB.IBuilder; END_TYPE (*INTERFACE*)
TYPE Range: SBB.Range; END_TYPE (*FUNCTION_BLOCK*)
```

Since the types of subordinate libraries become part of your own library, a [container library](../../../../../api/crossBook?lang=en-US&virtualBookName=SoLibref&topicID=D_SE_0081247) becomes superfluous in most cases.
