# 8512 Sercos wrong device type

## Description

[Refer to Diagnostic class (standard): 3](D-SE-0063169.html#D-SE-0063169)

The device type of the Sercos slave found on position (x) does not match the expected type (type).

| Cause | Solution |
| --- | --- |
| Detected error when assigning the identification features. | Verify and correct if necessary the settings of the identification features (for example: ConfiguredTopologyAddress). |
| * Ext. Diagnosis NoPS connected  Unable to assign a power supply to the drive.  (see parameter ConfiguredPowerSupply and PowerSupply) * The parameter ConfiguredPowerSupply is empty, but no PowerSupply (for example, LXM62PSM) is contained in the project. | Add an appropriate PowerSupply to the project (LXM62PSM). |
| The parameter ConfiguredPowerSupply contains a name which does not comply with a PowerSupply (for example, LXM62PSM) in the project. | Add another PowerSupply to the project with the correct name or enter the name of a projected PowerSupply into the parameter ConfiguredPowerSupply. |
| Inconsistent feature configuration in a Lexium 62 Double Drive, for example:   * Drive\_A = Standard / 1 * Drive\_B = Standard Plus / 2   The selected values in the parameter DeviceVariantSet must be identical. | Verify and correct if necessary the settings of the feature configuration for both drive instances. |

When starting up the Sercos bus in Phase 2, the type of the detected Sercos slave is validated with the slave that is entered in the PLC configuration. The interruption of the Sercos bus has not been received in at least twice the cycle time. The type detected is entered in the extended diagnostic.
