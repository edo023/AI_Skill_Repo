# 8112 Sercos telegram invalid

## Description

[Refer to Diagnostic class (standard): 3](D-SE-0063169.html#D-SE-0063169)

| [Device reactions](D-SE-0063393.html#D-SE-0063393__D-SE-0063393.2) | Default reaction | [Minimum reaction](D-SE-0063393.html#D-SE-0063393__D-SE-0063393.4) |
| --- | --- | --- |
| [Drive](D-SE-0063389.html#D-SE-0063389) | BD2 | BD2 |
| [Power supply](D-SE-0063391.html#D-SE-0063391) | BP2 | BP2 |

The device (Sercos Slave) did not receive a valid telegram or valid data within the telegram over the real-time bus (Sercos) during the Sercos phases 3 or 4. The diagnostic message is triggered after two consecutive telegrams containing invalid data or 10 consecutive invalid telegrams.

NOTE: The diagnostic message 8112 Sercos telegram invalid is not sent for invalid data or telegrams to the system since the Sercos slave can no longer report the diagnostic code to the controller via the Sercos bus. In this case, diagnostic code [8506 Sercos Master communication not possible](D-SE-0064000.html#D-SE-0064000) is reported in the controller (Sercos master).

| Cause | Solution |
| --- | --- |
| Detected hardware error: Cable is disconnected or inoperable. | Verify and replace if necessary the cable. |
| Detected hardware error: Device is inoperable. | Replace the device. |
