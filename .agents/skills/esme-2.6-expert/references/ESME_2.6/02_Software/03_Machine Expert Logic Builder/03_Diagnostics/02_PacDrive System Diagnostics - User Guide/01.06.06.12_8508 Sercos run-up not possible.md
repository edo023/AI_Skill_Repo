# 8508 Sercos run-up not possible

## Description

[Refer to Diagnostic class (standard): 3](D-SE-0063169.html#D-SE-0063169)

| Cause | Solution |
| --- | --- |
| At least one Sercos slave does not react on time after the phase change to phase 3. | Reset the Sercos slave and start Sercos phase-up again. |

No telegrams were received in phase 0.

| Cause | Solution |
| --- | --- |
| Wiring error detected (cable separated). | Verify and correct if necessary the wiring. |
| Non-Operating Sercos slave after the controller. | Verify the state and condition of the device and reset or replace if necessary. |
| ExtDiagMsg `C_Time=20000`  An error is detected during the parameter transfer for the configuration of the Sercos timing.  For example: One set ProducerCycleTime of a Sercos I/O device is not supported by the device. | Verify and correct if necessary the Sercos timing for the device. |
| ExtDiagMsg `C_SP0=2000F`  No valid Sercos topology detected. Phasing-up was not possible. | Verify correct connection at both ports. |
| ExtDiagMsg `C_SP3-2000B`  At least one Sercos slave does not react on time after the phase change to phase 3. | Reset the Sercos slave and start Sercos phase-up again. |
| ExtDiagMsg `C_SP3-20037`  For at least one Sercos slave, an error is detected during synchronization command execution. | * Reset the Sercos slave and start Sercos phase-up again. * Verify and correct if necessary the Sercos device. |
| ExtDiagMsg `C_SP4 =20028`  The number of configured additional real-time parameters is exceeded. | Verify and correct if necessary the parameters AdditionalIDNListMDT and AdditionalIDNListAT. |
| ExtDiagMsg `ReconfSercAddr`  Automatic distribution of unique Sercos addresses to the devices not using `IdentificationMode SercosAddress` was not successful. The Sercos phase-up is aborted. | Refer to [8042 Sercos phase switched](D-SE-0063826.html#D-SE-0063826). |
| ExtDiagMsg `Wrong Topology`  Using `IdentificationMode TopologyAddress /2` the Sercos bus can only phase-up in bus-topology 'Line' on port 1 or in bus-topology 'Ring'. | Verify and modify if necessary the Identification Mode if using an unsupported topology. |
| ExtDiagMsg `C_TGet=1 - Sercos run-up not possible`  Timeout while reading all parameters necessary for timing and telegram calculation from all Sercos slaves. | * Reset the Sercos slave and start Sercos phase-up again. * Verify and correct if necessary the Sercos devices. |
| ExtDiagMsg `Err —1`  Sercos phasing-up not possible.  Indeterminable root cause. | Verify the Sercos communication, Sercos configuration, and firmware compatibility.  If the condition persists, contact your local Schneider Electric service representative. |
