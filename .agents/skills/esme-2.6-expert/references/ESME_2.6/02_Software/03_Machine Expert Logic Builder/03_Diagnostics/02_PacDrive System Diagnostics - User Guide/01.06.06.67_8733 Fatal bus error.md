# 8733 Fatal bus error

## Description

[Refer to Diagnostic class (standard): 3](D-SE-0063169.html#D-SE-0063169)

Further bus communication is not possible due to a detected bus error.

NOTE: No data exchange with bus terminal **BT-4/ENC1**. Position loss of the master encoder connected to bus terminal **BT-4/ENC1**.

| CAUTION | |
| --- | --- |
|  | POSITION LOSS DUE TO SERCOS BUS ERROR  Only acknowledge the diagnostic message after reinitialization or homing.  Failure to follow these instructions can result in injury or equipment damage. |

If necessary, use the function FC\_SysReset() or FC\_PrgResetAndStart().

| Cause | Solution |
| --- | --- |
| Detected wiring error: Incorrect bus topology. | Verify and correct or replace if necessary the terminals, transmission lines, and so on. |
| EMC errors found. | Verify and correct or replace if necessary the wiring and EMC protection. |

| Ext. diagnosis | Meaning |
| --- | --- |
| `asynchronous` | A synchronization error is detected. The touchprobes on the BT-4/DIO1 are not working.  Verify the jumper cable that connects the controller to the optional module PN-4 (in the controller). |
| `clock miss` | A communication error is detected in PacNet.  This may be due to various causes:   * PacNet loop not closed * PacNet terminating plug is missing * A module connected to PacNet is inoperable (for example, no power)   Verify the PacNet loop and the modules in the loop. |

| Cause | Solution |
| --- | --- |
| Sercos cycle time 4 ms and PacDrive LMC-FPGA version < V0x02 (x = Hardware-ID, for example Id=2 -> FPGA version = V0202) | Update PacDrive LMC-FPGA version to at least version V0x02 |
