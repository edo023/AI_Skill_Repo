# 8826 PIC update not possible

## Description

[Refer to Diagnostic class (standard): 4](D-SE-0063169.html#D-SE-0063169)

An error is detected during a firmware exchange for the PIC controller. The PacDrive controller is in bootloader-mode and is trying to exchange the firmware again. The firmware (such as file c6p\_0511.bin) is on the flash disk. When the firmware has been exchanged, the controller is reset (Reset).

| Ext. diagnosis | Meaning |
| --- | --- |
| `bootl.active` | This message appears after an unsuccessful PIC firmware update.   * The PacDrive controller attempts to exchange the firmware again. * If you do not succeed, contact your local Schneider Electric service representative. |
| `File not found` | This message appears with message bootl. active. This means that it appears when an update is forced in the PIC bootloader but the necessary file is not on the flash disk.   * Transmit the firmware file to the flash disk. * If this attempt also does not succeed, contact your local Schneider Electric service representative. |
| `File not OK` | This message appears when the PIC update file is incorrect.   * Transmit a correct firmware file to the flash disk. * If you do not succeed, contact your local Schneider Electric service representative. |
| `Header not OK` | This message appears if the header in the file is not correct.   * Transmit a correct firmware file to the flash disk. * If you do not succeed, contact your local Schneider Electric service representative. |
| `HWIndex <> <Index>`  (for example: `HWIndex <> 5`) | The message appears if PIC update files are found that have an incorrect hardware code. The index provides the required HW index.  Use the PIC update file that matches the device (hardware). |
