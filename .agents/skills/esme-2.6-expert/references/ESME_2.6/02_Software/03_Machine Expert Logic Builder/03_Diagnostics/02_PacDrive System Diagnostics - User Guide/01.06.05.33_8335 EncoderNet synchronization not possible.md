# 8335 EncoderNet synchronization not possible

## Description

[Refer to Diagnostic class (standard): 3](D-SE-0063169.html#D-SE-0063169)

Filter type 2 diagnostic messages

An error is detected in the encoder network when synchronizing the controller system clocks.

## Ext. diagnosis: Sync Init (Object SYN\_M or SYN\_S)

| Cause | Solution |
| --- | --- |
| The initialization of synchronization of master or slave not successful. The IP address (SlaveIPAddress) of the sync. Module is assigned twice. | Verify and correct if necessary the parameter SlaveIPAddress of the synchronization modules in the synchronization master PLC configuration. |

## Ext. diagnosis: SercosPhase xxx

| Cause | Solution |
| --- | --- |
| The Sercos loop is not in phase 4. | Sercos bus in phase 4 restarted. |

## Ext. diagnostic: Dupl. MasterID

| Cause | Solution |
| --- | --- |
| The set MasterID is already being used by another synchronization master which is in the same network. | Change the MasterID. |

## Ext. diagnosis: ProtVer xxx

| Cause | Solution |
| --- | --- |
| The synchronization master protocol version no longer matches the synchronization slave protocol version. | * Use compatible software versions for the controllers. * Contact your local Schneider Electric service representative. |

## Ext. diagnosis: Ping Master

| Cause | Solution |
| --- | --- |
| The synchronization master does not respond to a ping. | Verify and correct if necessary the Ethernet connection to the synchronization master. |

## Ext. diagnosis: Dupl. Slave IP (Object SYN\_MOD)

| Cause | Solution |
| --- | --- |
| The IP address (SlaveIPAddress) of the sync. Module is assigned twice. | Verify and correct if necessary the parameter SlaveIPAddress of the synchronization modules in the synchronization master PLC configuration. |

## Ext. diagnosis: SlaveCycleTime

| Cause | Solution |
| --- | --- |
| The SlaveCycleTime is not a multiple of the Sercos cycle time. | Verify and correct if necessary the parameter SlaveCycleTime or adjust the Sercos cycle time. |

## Ext. diagnosis: Wrong config (object SYN\_MOD)

| Cause | Solution |
| --- | --- |
| The activated synchronization slave is not configured properly or is not responding. | * Verify and correct if necessary the configuration of the synchronization slave (for example, same CycleTime). * Verify and correct if necessary the TCP/IP connection between synchronization master and slave. |

## Ext. diagnosis: Wrong config (object SYN\_S)

| Cause | Solution |
| --- | --- |
| The Sercos cycle times differ between synchronization master and slave or the synchronization slave-configuration cannot be transmitted. | Adjust the CycleTime parameter of the synchronization master and/or slaves. |

## Ext. diagnosis: Wrong State

| Cause | Solution |
| --- | --- |
| The activated synchronization slave is in an error state. | Acknowledge the detected error in the synchronization slave. |

## Ext. diagnosis: No DataOut

| Cause | Solution |
| --- | --- |
| No Sync. Encoder output has been configured or activated. | * Verify and correct if necessary the PLC configuration. * Verify and correct if necessary the TCP/IP connection synchronization master to slave. Activate encoder output (parameter Enable). |

## Ext. diagnosis: No Module

| Cause | Solution |
| --- | --- |
| No Sync. module and no Sync. encoder input is configured or enabled. | * Verify and correct if necessary the PLC configuration. * Verify and correct if necessary the TCP/IP connection synchronization master to slave. Module or Sync. Activate encoder output (sync. module parameter Enable or sync. encoder input parameter Enable). |

## Ext. diagnosis: Sync Err

| Cause | Solution |
| --- | --- |
| The synchronization slaves are unable to synchronize with the synchronization master.  The Ethernet (see parameter SyncQuality) and/or the CPU load (see parameter AvailableLoad) is too high. | * Verify and correct if necessary the Ethernet connection. * Optimize the IEC project to reduce the CPU load. |

## Ext. diagnosis: No Sync: xxx

| Cause | Solution |
| --- | --- |
| The number of sequential synchronization telegrams (xxx) which were not received, exceeded the number specified in SlaveCycleErrorLimit. The Ethernet (see parameter SyncQuality) and/or the CPU load (see parameter AvailableLoad) is too high. | * Verify and correct if necessary the Ethernet connection. * Optimize the IEC project to reduce the CPU load. * Increase SlaveCycleErrorLimit. |

## Ext. diagnostic: DataReady Err

| Cause | Solution |
| --- | --- |
| The Sync. Encoder outputs cannot configure sync. encoder inputs or a data error is detected during normal operation. | Verify the parameter DiagCode of sync. modules for more detailed information on the error source. |

## Ext. diagnostic: DataEnable Err

| Cause | Solution |
| --- | --- |
| Either the sync. encoder outputs are unable to configure the sync. encoder inputs or a data error is detected while the system is running. Encoder outputs and sync. encoder inputs are ready for the data exchange. | Verify the parameter DiagCode of sync. modules for more detailed information on the error source. |

## Ext. diagnostic: Wrong IP Add

| Cause | Solution |
| --- | --- |
| The parameter SlaveIPAddress is not set. | Contact your local Schneider Electric service representative. |

## Ext. diagnostic: Wrong IP Add

| Cause | Solution |
| --- | --- |
| Internal error code | Contact your local Schneider Electric service representative. |

## Ext. diagnostic: Sync Err Wnd

The synchronization has been interrupted due to excessive telegram delays.

| Cause | Solution |
| --- | --- |
| Ethernet data communication saturation. | This can appear for a number of external reasons related to the network such as wiring, network traffic, performance or other related equipment. Verify and correct if necessary any external equipment or wiring disturbing communications. |

## Ext. diagnostic: Sync Err LCyc

The synchronization has been interrupted due to excessive unsuccessful telegrams.

## Ext. diagnostic: Sync Err CtrlM

The synchronization master has received a diagnostic message from a slave reporting a detected communication error.

## Ext. diagnostic: Sync Err CtrlS

The synchronization slave has received a diagnostic message from the master reporting a detected communication error.

## Ext. diagnostic: Sync Err CtrlN

The synchronization module has received a diagnostic message from the corresponding synchronization slave reporting a detected communication error and was unable to read more detailed diagnostic information.

## Ext. diagnostic: Sync TimeoutM

The synchronization master has determined a timeout for establishing synchronization (> 5 min).

## Ext. diagnostic: Sync TimeoutS

The synchronization slave has determined a timeout for establishing synchronization (> 5 min).
