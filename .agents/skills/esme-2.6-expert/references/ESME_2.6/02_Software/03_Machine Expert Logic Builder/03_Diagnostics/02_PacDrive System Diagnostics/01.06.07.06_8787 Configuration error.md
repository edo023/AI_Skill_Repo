# 8787 Configuration error

## Description

[Refer to Diagnostic class (standard): 3](D-SE-0063169.html#D-SE-0063169)

A configuration error is detected.

| Ext. diagnosis | Meaning |
| --- | --- |
| `Mtype=xx wrong` (xx = detected module) | A non-configured module was detected on PacNet. The module cannot be used.  Verify and correct if necessary the configuration of the module in the PLC configuration. |
| `Mtype=0 wrong` | No module was found on PacNet.  Switch on the control voltage on the module. |
| `M=xx n.res` | The module with bus address xx repeatedly gives no response. This may be caused by a power outage or an inoperable module. The module cannot be used.   * Verify and correct if necessary the module address on the module. * Verify and correct if necessary the module address in the PLC configuration. * Verify and correct if necessary the power supply of the module. * Replace the module if necessary. |
| `Powerfail 0-7` | The power supply for outputs 0-7 of the BT-4/DIO1 or DIO8 module (optional module of Lexium 62 ILM Drive Module) is not available or is too low. These outputs cannot be used. The DIO8 module triggers the diagnostic message only if PowerSupply = Extern / TRUE.  Verify and correct or replace if necessary the power supply of the outputs. |

NOTE: The DiagPF0\_7Enable parameter allows you to disable the message.

| Ext. diagnosis | Meaning |
| --- | --- |
| `Powerfail 8-15` | The supply voltage for outputs 8-15 of the BT-4/DIO1 is not available or too low. These outputs cannot be used.  Verify and correct or replace if necessary the power supply of the outputs. |

NOTE: The DiagPF8\_15Enable parameter allows you to disable the message.

| Ext. diagnosis | Meaning |
| --- | --- |
| `FPGA not supp.` | The firmware on the controller does not match the controller hardware.  Install a matching firmware version on the controller with the Firmware Assistant. |
| `FPGA Version` | An FPGA version was found on the controller that is specified as a beta test version. The pinout of the FPGA firmware is the number represented by p in the zpxx of the FPGA firmware version.  The pinout of the controller can be read in the ControllerType1 parameter at position p. The highest bit indicated by z is a beta test version.  Install a released FPGA firmware file on the controller. |
| `too many IM` | With DIO8 module (optional module of Lexium 62 ILM Drive Module), the sum of the measuring inputs and the Impulse Counters is more than two.  Verify and correct if necessary the configuration (parameter IO0\_Mode ... IO7\_Mode). |

NOTE: In certain cases, another diagnostic code or diagnostic class is entered in the message logger of the controller for DIO8 module and Lexium 62 ILM Drive/Lexium 52 Drive onboard I/O modules in the message logger of the drive.

| Ext. diagnosis | Meaning |
| --- | --- |
| `SetSizePDIN/SetSizePDOUT` | Detected internal error. The data size for the drive safety module could not be read. The default values are used.  Contact your local Schneider Electric service representative. |
