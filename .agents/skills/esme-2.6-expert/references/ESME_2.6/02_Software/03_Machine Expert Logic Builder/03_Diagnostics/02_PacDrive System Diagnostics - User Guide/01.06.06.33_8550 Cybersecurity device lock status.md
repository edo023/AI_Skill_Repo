# 8550 Cybersecurity device lock status

This diagnostic message relates to the multi carrier transport system.

A logger entry in the segment is triggered indicating that a cybersecurity device unlock/lock operation was performed.

For in-depth diagnosis, a cybersecurity device unlock/lock operation can be performed by authorized and qualified personnel (Schneider Electric field application engineer).

Segments are shipped by Schneider Electric in the cybersecurity device state “locked”.

If the cybersecurity device state of a segment is set to "unlocked", the segment can potentially be accessed without authorization.

| WARNING | |
| --- | --- |
|  | UNAUTHENTICATED ACCESS AND SUBSEQUENT UNAUTHORIZED MACHINE OPERATION  Ensure that the cybersecurity device state of the segment is set back to “locked” before you put your system back into operation.  Failure to follow these instructions can result in death, serious injury, or equipment damage. |

Information on the cybersecurity device state is provided by three parameters of the segment logger entry:

* Parameter 1 (device lock/unlock attempt):

  + 1 = device lock/unlock attempt was successful
  + 0 = device lock/unlock attempt was unsuccessful
* Parameter 2 (new lock state):

  + 0 = device locked
  + 255 = device unlocked
  + Other values are for test purposes only.
* Parameter 3 (previous lock state):

  + 0 = device locked
  + 255 = device unlocked
  + Other values are for test purposes only.

Examples:

* Successful unlock:

  + Parameter 1 = 1
  + Parameter 2 = 255
  + Parameter 3 = is not relevant
* Successful lock:

  + Parameter 1 = 1
  + Parameter 2 = 0
  + Parameter 3 = is not relevant
