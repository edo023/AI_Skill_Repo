# 8123 Safe Torque Off incorrect

## Description

[Refer to Diagnostic class (standard): 3](D-SE-0063169.html#D-SE-0063169)

| [Device reactions](D-SE-0063393.html#D-SE-0063393__D-SE-0063393.2) | Default reaction | [Minimum reaction](D-SE-0063393.html#D-SE-0063393__D-SE-0063393.4) |
| --- | --- | --- |
| [Drive](D-SE-0063389.html#D-SE-0063389) | AD | E |
| [Power supply](D-SE-0063391.html#D-SE-0063391) | - | - |
| [Multi carrier transport system](DeviceReactions-MultiCarrier-36D130D7.html#DeviceReactions-MultiCarrier-36D130D7) | AD | - |

* The Inverter Enable input is set to low, while the drive is enabled (`AxisState` >1) and parameter InverterEnableFunction is set to `Safe Stop 1 / 1`.

  NOTE: If the drive is shut down with `ControllerEnable`, then the controller is active until the drive stops and the brake is engaged (BrakeCouplingTime). The maximum time for this stop is defined with the parameter StopTimeLim.

| For the motor/drive system: | |
| --- | --- |
| Cause | Solution |
| Detected wiring error. | Verify and correct if necessary the wiring of the Inverter Enable input. |
| Detected circuit error: The Inverter Enable input is set to low. | Verify and correct if necessary the control of the Inverter Enable input. |
| The safety function `Safe Torque Off` is used. | Verify and correct if necessary the parameter InverterEnableFunction. |

* The `Safe Force Off (SFO)` input is set to low, while the multi carrier segment is enabled.

| For the Multi carrier transport system: | |
| --- | --- |
| Cause | Solution |
| Wiring error at the `Safe Force Off (SFO)` input. | Verify the wiring and the control of the `Safe Force Off (SFO)` input at the multi carrier segment, and correct if necessary. |
| The communication interconnects are not inserted correctly. | Verify the mounting and fastening of the communication interconnects of the multi carrier segments. |
