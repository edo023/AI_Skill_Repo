# 8342 On/OffPos inverted (On/OffDelay ok?)

## Description

[Refer to Diagnostic class (standard): 3](D-SE-0063169.html#D-SE-0063169)

The values of the parameters OnPosition and OffPosition of a Cam Switch object lead to invalid positions in conjunction with the values of the parameters OnDelay and OffDelay and the velocity of the associated Cam Track.

For example, the value of the parameter OnDelay in conjunction with the velocity of a Cam Track can result in a calculated OnPosition that is greater than the calculated OffPosition. In such a case, the OnPosition value is invalid.

| Cause | Solution |
| --- | --- |
| The calculated OffPosition value of a Cam Switch object is greater than the calculated OnPosition Value.  To calculate the OnPosition and OffPosition values of a Cam Switch object, the velocity and the OnDelay and OffDelay values of the corresponding Cam Track are used. | * Reduce velocity of the associated Cam Track. * Reduce the OnDelay or OffDelay values. * Increase the difference between the OnPosition and OffPosition values of the Cam Switch object. |
