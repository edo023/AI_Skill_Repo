# Connecting to a Controller

## Encrypted TLS Communication

Some controllers, starting with firmware versions released with EcoStruxure Machine Expert V1.1, provide encrypted TLS communication.

Where supported, the tool connects using this secured communication to encrypt data transfer.

The communication status is displayed on the left-hand side at the bottom of the tool:

* Connected with secured encrypted communication
* Connected without encrypted communication

On the first connection to a new controller, or if the node name of the controller is modified, you are prompted to accept the certificate of the controller.

If accepted, the certificate is automatically saved to the Windows *User certificates* inside the *Controllers* folder. You are not prompted again to accept the certificate until or unless you remove the certificate.

NOTE: You can use [command-line arguments](D-SE-0041408.html#D-SE-0041408__D-SE-0041408.2) to alter the first connection behavior.

## Device User Rights Credentials

If device user rights management is activated on the controller, you are prompted for a user name and the password. The user name and password must have already been defined on the controller.

In some situations after you entered valid credentials, you may be requested to renew the existing password.

At initial startup of a controller, or if the user rights management has been reset, the default  User Name /  Password combination is  Administrator /  Administrator.

NOTE: You can use [command-line arguments](D-SE-0041408.html#D-SE-0041408__D-SE-0041408.2) to alter the first connection behavior.
