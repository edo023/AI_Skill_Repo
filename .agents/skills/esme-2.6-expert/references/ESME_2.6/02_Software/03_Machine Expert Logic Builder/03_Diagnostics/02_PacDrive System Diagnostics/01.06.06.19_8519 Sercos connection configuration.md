# 8519 Sercos connection configuration

## Description

[Refer to Diagnostic class (standard): 3](D-SE-0063169.html#D-SE-0063169)

This diagnostic message is used to display detected Sercos configuration issues. If possible, the Sercos master tries to continue the Sercos phase up and only disables the real-time data for the slaves that have the configuration issue. If this is possible, these slaves provide the ExtDiagMsg **not configured** diagnostic which cannot be acknowledged.

This message is only resettable if the Sercos phase is reset to CP0 again.

The table indicates the meaning of Ext. Diagnosis

| Ext. diagnosis | Meaning | Devices |
| --- | --- | --- |
| not configured | Sercos communication is active (CP2-CP4) but there is no cyclic real-time data configured for this slave. | The Sercos slaves (Lexium 62 Power Supply, Lexium 62 Power Supply, Lexium 52 Drive, TM5NS31, TM5CSLCx00FS, and so on) |
| len. mismatch | There was a misconfiguration with the length of the cyclic real-time data transmitted. Producer and consumer deliver different length information for the configured data. Producer and consumer can be identified by additional diagnostic messages. | Sercos master |
| P aaa Len=bbb | The producer of the connection aaa delivers the length information bbb. The producer is the source of this diagnostic message. | * All Sercos slaves (Lexium 62 Power Supply, Lexium LXM62 Drive, Lexium LXM52 Drive, TM5NS31, TM5CSLCx00FS, and so on) * Sercos master |
| C aaa Len=bbb | The consumer of the connection aaa delivers the length information bbb. The consumer is the source of this diagnostic message. | * The Sercos slaves (Lexium 62 Power Supply, Lexium LXM62 Drive, Lexium LXM52 Drive, TM5NS31, TM5CSLCx00FS, and so on) * Sercos master |
| SSDO aaa/bbb | Configuration error bbb during the exchange of safety data for SafeNode with ID aaa. bbb can be an error code described in the online help chapter Error codes of the Sercos parameter channel (service channel) or FC\_SercosWriteServiceData.  This diagnostic message is triggered by the TM5CSLCx00FS in case of an inappropriate, respectively empty application on the Safe-Key.   * Open EcoStruxure Machine Expert - Safety with the current configuration from EcoStruxure Machine Expert. * Download the EcoStruxure Machine Expert - Safety application to the SLC. * Restart the SLC. | TM5NS31, TM5CSLCx00FS |
| IO data length | Invalid I/O data length of the SLC configuration has been detected. (The I/O configuration in the EcoStruxure Machine Expert project does not match the EcoStruxure Machine Expert - Safety project that is running on the SLC).   * Open EcoStruxure Machine Expert - Safety with the current configuration from EcoStruxure Machine Expert. * Download the EcoStruxure Machine Expert - Safety application to the SLC. * Restart the SLC. | TM5CSLCx00FS |
| FW version/SRx | The firmware version of the SLC does not match the selected safety release.  (Safety release V1.2 requires the SLC version V1.xx, safety release V2.0 requires the SLC version V2.xx)   * Verify and correct if necessary the parameter SLC.SafetyRelease. * Update the SLC firmware to the correct version. | TM5CSLCx00FS |
