## Task

Set the position of the axis based on a digital signal.

## Description

The function block [FB\_Home](FB_Home-GeneralInformation-459EBABF.html#FB_Home-GeneralInformation-459EBABF) internally uses FB\_HomingDigitalInput from the PLCopen Part 1 library. For detailed information on the configuration and the axis behavior during the homing process, refer to the .

The method HomingToDigitalInput sets the position of the axis based on a digital signal that is linked to a physical sensor providing a reference point of the system.

* The homing movement is started at a constant velocity (i\_lrHighVelocity) in the configured direction.
* When the rising edge of the reference switch is detected, the velocity decreases (i\_lrLowVelocity). The direction of this second movement depends on the configured homing mode.
* When the falling edge of the reference switch is detected, the corresponding position value is set as the home position and the movement stops. The standstill position differs from the home position in terms of the distance covered during deceleration.
* Depending on the value of the input i\_xReturnToZero, the movement either stops or it continues until it reaches zero.

If the edge of the reference switch is not detected within this configured maximum travel distance, the execution of the function block is aborted with an error. If a hardware limit switch is triggered during the homing movement, the execution of the function block is aborted with an error.

NOTE: To perform this homing command the axis must be in state Standstill.

NOTE: If an online change is detected while the homing command is active the homing procedure is immediately aborted. In this case, the axis is brought to a stop using the ErrorStop parameters defined on the axis.

## Inputs

| Input | Data type | Description |
| --- | --- | --- |
| i\_lrHomePosition | LREAL | Position that is set as axis position when the falling edge of the reference switch signal is detected at low velocity. |
| i\_lrMaxTravelDistance | LREAL | Maximum distance of the movement for searching for the rising edge of the reference switch signal.  NOTE: Values less than 0 disable the monitoring of the maximum movement distance. |
| i\_lrHighVelocity | LREAL | Value of the velocity for the homing movement towards the reference switch until the signal is detected (= high velocity). |
| i\_lrLowVelocity | LREAL | Value of the velocity for the movement away from the reference switch after the rising edge is detected (= low velocity). |
| i\_lrAcceleration | LREAL | Value of the acceleration for the homing movement towards the reference switch (high velocity) and for the movement away from the reference switch (low velocity). |
| i\_lrDeceleration | LREAL | Value of the deceleration for the homing movement after the signal of the reference switch has been detected (high velocity) and for the movement away from the reference switch (low velocity). |
| i\_lrJerk | LREAL | Value for the maximum jerk with which the acceleration is modified.  NOTE: Value 0 disables the jerk limit and results in a jump from zero to maximum acceleration. |
| i\_etDirection | ET\_Direction | Direction of the homing movement. |
| i\_etHomingMode | ET\_HomingMode | Specifies whether or not the slow movement with low velocity after the fast movement with high velocity is performed with reversed direction. |
| i\_xReturnToZero | BOOL | Specifies if after the home position has been set, the movement is continued to the zero position 0 at high velocity.  NOTE: The PLCopen operating state remains at the state Homing as long as the return to zero movement lasts. |
| i\_refxReferenceSwitch | REFERENCE TO BOOL | Reference to the digital signal that represents the reference point of the system. |

## Troubleshooting

This table describes the possible issues and their solutions:

| Issue  (error information q\_xError = TRUE) | Cause | Solution |
| --- | --- | --- |
| q\_etResult = HomingIsAlreadyActive | Another homing process is already ongoing. | Retry after the active homing process has been completed |
| q\_etResult = NotInitialized | The FB\_Home function block instance has not been initialized yet. | Initialize the function block by calling the method Init. |
| q\_etResult = AxisNotReady | The axis is not in standstill. | Stop the axis before triggering the homing command. |
| q\_etResult = InvalidInput | The input i\_refxReferenceSwitch is not assigned. | Assign a variable of type BOOL to the input i\_refxReferenceSwitch. |
| q\_etResult = HomingFailed | The underlying motion function block has detected an error. Details are provided in the q\_sResultMsg. | Correct the cause of the error and retry the operation. |
| q\_etResult = HomingInterrupted | Due to an external command or an abort caused by an online change, the control of the axis has been lost. | Trigger the homing command again. |
