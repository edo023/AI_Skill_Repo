AutoTune Reports: Axis Not Ready (lResult = -10050)

The axis cannot be optimized, even though no error has occurred on the drive.

| Cause | Removal |
| --- | --- |
| The DC bus voltage of the drive is too low. Since the drive is not being controlled, no diagnostic message is generated via the drive diagnostic. | Activate the mains contactor or check that the mains is switched on.  Check the IEC program whether a program section to connect the mains when the optimization mode is activated was switched off. |
| PowerSupplyCheck is not set for the corresponding PowerSupply. | Check the PowerSupplyCheck parameter. |
| The drive cannot be switched on for some other reason. | Check the parameter InternalDeviceState. The drive status is displayed here. This way the reason why the drive is not ready becomes obvious. |

AutoTune Reports: Axis Velocity Out of Range (lResult = -10051)

The permitted maximum velocity of the drive has been exceeded.

NOTE: It might not be possible to optimize drives with a highly limited velocity with AutoTune!

| Cause | Removal |
| --- | --- |
| The start parameters were selected unfavorable for the optimization. Oscillations occurred on the drive which increased the velocity excessively. | Start the optimization with default parameters.  Click on Start with default control loop parameters in the visualization Advanced Configuration.  Then restart the optimization. |
| The velocity limit max. Velocity selected in the visualization is too low. To optimize the axis, higher speeds are required. | Increase the velocity limit max. Velocity selected in the Main Configuration visualization window. Ensure that the maximum velocity permitted at the drive is not exceeded. |

AutoTune Reports: Axis Acceleration Out of Range (lResult = -10052)

The acceleration of the drive exceeded the permitted value.

NOTE: It might not be possible to optimize drives with a highly limited velocity with AutoTune!

| Cause | Removal |
| --- | --- |
| Unfavorable start parameters were selected for optimization. Oscillations occurred on the drive which increased the (actual) acceleration excessively. | Start the optimization with default parameters.  Click on Start with default control loop parameters in the visualization Advanced Configuration.  Then restart the optimization. |
| The acceleration limit max. Acceleration selected in the visualization is too low. Higher acceleration values are required to optimize the axis. | Increase the acceleration limit max. Acceleration selected in the visualization. Ensure that the maximum (reference) acceleration permitted at the drive is not exceeded. |

AutoTune Reports: Axis Position Out of Range (lResult = -10053)

The position of the drive is not within the permitted range.

NOTE: It might not be possible to optimize drives with a highly limited position range with AutoTune!

| Cause | Removal |
| --- | --- |
| The start parameters were selected unfavorable for the optimization. Oscillations occurred on the drive that only allow an insufficient control over the compliance of the position range. | Start the optimization with default parameters.  Click on Default parameter in the visualization Results.  Then restart the optimization. |
| The controller parameters were not optimized or not optimized successfully, with the result that oscillation occurs at the drive which does not allow sufficient control over the observation of the position range. | Optimize the controller parameters and ensure that the optimization is completed successfully. |
| The position range defined in the visualization by min. Position and max. Position is too small. To optimize the axis a greater position range is required. | Increase the position range with min. Position and max. Position in the visualization. Ensure that the maximum position range permitted on the drive is not exceeded. |

AutoTune Reports: Axis Stopped at Lower Position Limit (lResult = -10054)

The axis was brought to a standstill because an exceeding of the position range by Min. Position was expected. But the drive was stopped while it was still within the permitted position range.

NOTE: It might not be possible to optimize drives with a highly limited position range with AutoTune!

See diagnostic message for causes and troubleshooting: Axis Position out of range (lResult = -10053)

AutoTune Reports: Axis Stopped at Upper Position Limit (lResult = -10055)

The axis was brought to a standstill because an exceeding of the position range by Max. Position was expected. But the drive was stopped while it was still within the permitted position range.

NOTE: It might not be possible to optimize drives with a highly limited position range with AutoTune!

See diagnostic message for causes and removals: Axis Position out of range (lResult = -10053)

AutoTune Reports: No Enable While Optimizing (lResult = -10057)

The axis was stopped because the input AUT.G\_xAutoTuneEnable was switched off during the optimization.

NOTE: The Enable status is displayed as Enable in the top right of the visualization.

| Cause | Removal |
| --- | --- |
| The variable AUT.G\_xAutoTuneEnable was switched over from optimization mode to normal mode. | An optimization can only be performed if the input AUT.G\_xAutoTuneEnable remains TRUE from the beginning to the end of the optimization. |

AutoTune reports: Drive changed while optimizing (lResult = -10058)

The axis was stopped because a different drive was selected during the optimization.

| Cause | Removal |
| --- | --- |
| A different axis was selected in the Axis Select visualization window. | Optimization can be performed only if the selected drive remains unchanged from the start to the end of optimization. |

AutoTune reports: CutOffFrequency of velocity control too low (lResult = -10080)

The value calculated for the cut-off frequency of the velocity control loop is too small.

| Cause | Removal |
| --- | --- |
| No suitable controller parameters were found for the required dynamic. | Change the required dynamic in the visualization window Advanced Configuration under Dynamic response of controller. |

AutoTune Reports: CutOffFrequency of Position Control Too Low (lResult = -10081)

A too small value was calculated for the cut-off frequency of the position control loop.

| Cause | Removal |
| --- | --- |
| No suitable controller parameters were found for the required dynamic. | Change the required dynamic in the visualization window Advanced Configuration under Dynamic response of controller. |

AutoTune reports: Wrong drive firmware version (lResult = -10200)

The optimization cannot be performed with the current firmware version of the drive.

| Cause | Removal |
| --- | --- |
| To be able to use the AutoTune\_V0020xx library, a software version according to chapter [System Requirements](../System_Requirements/System_Requirements.htm#XREF_D_SE_0091176_4) is required for the drive. | Install a software version according to chapter [System Requirements](../System_Requirements/System_Requirements.htm#XREF_D_SE_0091176_4) on the drive (LXM62, LXM52 or ILM62).  Install a version of the AutoTune library via the library repository that matches the firmware version in your drive. |
