# Providing Data for Visualization

## General Information

For displaying unit-specific data in a visualization through the [FB\_VisApplicationManager](FB_VisAppManager-Gen-DA85FB15.html), you must create a function block together with an appropriate visualization frame object. Each unit instances this function block and collects the data of the unit. The FB\_VisApplicationManager then displays the data in the visualization frame according to the selected unit.

A visualization usually contains data from various sources such as diagnostics, operating modes or axes. This means that you must create the visualization frames and the corresponding function blocks for the various data or you can use pre-defined frames and function blocks from a library.

## Code Example

In the following, you find a code example for the example function block FB\_VisConnectorUnitData from step 1 in the description above:

Instancing:

```
fbVisConnectorUnitData : FB_VisConnectorUnitData;
```

Call:

```
fbVisConnectorUnitData(
  i_uiUnitId              := 100,
  i_sUnitName             := 'SR_SubUnitA',
  i_xActive               := xActive,
  i_xReady                := xReady,
  i_etPackMlState         := etPackMlState,
  i_etActiveOperationMode := etActiveOperationMode);
```

  

```
fbVisManager.UpdateVisConnectorReference(
  i_uiVisConnectorIndex   := ET_VisConnectors.UnitData,
  i_pbyVisConnectorAddress  := ADR(fbVisConnectorUnitData));
```
