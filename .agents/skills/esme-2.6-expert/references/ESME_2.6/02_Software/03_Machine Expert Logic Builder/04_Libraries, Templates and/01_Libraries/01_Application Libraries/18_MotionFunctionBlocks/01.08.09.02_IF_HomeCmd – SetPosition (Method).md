# IF\_HomeCmd – SetPosition (Method)

## Overview

|  |  |
| --- | --- |
| Type: | Method |
| Available as of: | V1.3.1.0 |
| Versions: | Installed version |

## Task

Set the position of the axis.

## Description

Use this method to define the present position of an axis without requiring physical movement.

NOTE: To perform this homing command the axis must be in state Disabled or Standstill.

## Inputs

| Input | Data type | Description |
| --- | --- | --- |
| i\_lrPosition | LREAL | Position to be set in user-defined units. |

## Outputs

| Output | Data type | Description |
| --- | --- | --- |
| q\_xError | BOOL | The output is set to TRUE if an error has been detected during the execution. |
| q\_etResult | ET\_Result | POU-specific output on the diagnostic; q\_xError = FALSE -> Status message; q\_xError = TRUE -> Diagnostic message. |
| q\_sResultMsg | STRING[255] | Event-triggered message that gives additional information on the diagnostic state. |

## Troubleshooting

This table describes the possible issues and their solutions:

| Issue  (error information q\_xError = TRUE) | Cause | Solution |
| --- | --- | --- |
| q\_etResult = HomingIsAlreadyActive | Another homing process is already ongoing. | Retry after the active homing process has been completed |
| q\_etResult = NotInitialized | The FB\_Home function block instance has not been initialized yet. | Initialize the function block by calling the method Init. |
| q\_etResult = AxisNotReady | The axis state is invalid. | Only the states Disabled and Standstill are accepted. |
| q\_etResult = HomingFailed | The underlying motion function block has detected an error. Details are provided in the q\_sResultMsg. | Correct the cause of the error and retry the operation. |
