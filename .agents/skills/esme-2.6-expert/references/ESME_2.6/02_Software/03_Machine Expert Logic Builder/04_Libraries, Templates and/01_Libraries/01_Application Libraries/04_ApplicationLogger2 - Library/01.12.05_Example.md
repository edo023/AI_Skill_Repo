# Example

## General

The following example is a short source code example of using the Application Logger via function blocks.

It is also possible to use the corresponding interfaces instead, for example IF\_LoggerPoint2 or G\_ifApplicationLogger.

**Declaration part**

```
 VAR
   udiState : UDINT;
   xSomethingHappened : BOOL;
   fbLoggerPoint : APL2.FB_LoggerPoint;
   etResult : APL2.ET_Result;
 END_VAR
```

**Program part**

```
 CASE udiState OF
   0: // Machine initialization
      // Register ApplicationLogger service
      APL2.G_ifApplicationLogger.RegisterCommunicationService();
      // Register first LoggerPoint
      fbLoggerPoint.RegisterLoggerPoint(
         i_ifParent := APL2.G_ifApplicationLogger,
         i_sName := 'AplTest',
         i_sType := 'MyProject',
         i_sSource := 'Project',
         q_stResult => etResult);
      IF (etResult = APL2.ET_Result.OK) OR (etResult = APL2.ET_Result.LoggerPointAlreadyRegistered) THEN
         udiState := 10;
      ELSE
         // not able to register logger point -> verify output q_etResult and output q_sResultMsg
         udiState := 100;
      END_IF
      
   10: // cyclic program
      IF xSomethingHappened THEN
         xSomethingHappened := FALSE;
         // write message wia LoggerPoint
         fbLoggerPoint.AddLogEntry(
            i_etLogLevel := APL2.ET_LogLevel.Exception,
            i_udiDiagCode := 4242,
            i_sMessage := 'An exception has occurred',
            q_etResult => etResult);
         IF (etResult <> APL2.ET_Result.OK) THEN
            // not able to add a log entry -> verify output q_etResult and output q_sResultMsg
            udiState := 100;
         END_IF
      END_IF
 END_CASE
```

By running this code on a controller and setting xSomethingHappened to TRUE, there is no message entry in the Application Logger by default. This is because the internal LogLevel of the LoggerPoints is set to Nothing per default. The LoggerPoint only logs messages which are of interest at the moment. This means that the input i\_etLogLevel of the Method fbLoggerPoint.AddLogEntry is set to an equal or higher level than the internal LogLevel of the LoggerPoint.

A detailed explanation how this internal LogLevel is set with the Logger Points tab of the ApplicationLogger is described in the section [Modifying the Log Level of a Logger Point](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMMenu&topicID=D_SE_0091294_8). The order of the LogLevels is defined in the [ET\_LogLevel](D-SE-0077662.html#D-SE-0077662) enumeration.

If the internal LogLevel is set to StatusMessage, for example, the messages which are sent from this point on are added to the LogMessage list, previous send messages are still ignored.
