# FB\_TorqueFeedForward2 - GetConfiguration (Method)

## Overview

|  |  |
| --- | --- |
| Type: | Method |
| Available as of: | V1.5.1.0 |
| Versions: | Installed version |

## Description

This method returns the axis for which the torque feed forward function is configured, after the Configure() method has completed successfully. The method can be used to retrieve and verify the axis that was assigned during configuration. If the configuration did not execute or was interrupted, the output q\_xError is set to TRUE and diagnostic messages are displayed.

The following prerequisites must be fulfilled to use the method:

* The method Configure() must be executed. Also refer to the property xIsConfigured.
* The Sercos ActualState must be phase 4 (operational).

## Interface

| Output | Data type | Description |
| --- | --- | --- |
| q\_ifAxis | CMI.If\_AxisIdentification | Axis for which the torque feed forward is configured. |
| q\_xError | BOOL | If this output is set to TRUE, an error is detected.  For details, refer to q\_etResult and q\_etResultMsg. |
| q\_etResult | ET\_Result | Provides diagnostic and status information as a numeric value. |
| q\_etResultMsg | STRING[80] | Provides additional diagnostic and status information as a text message. |

## Troubleshooting

Possible diagnostic information (q\_xError = TRUE)

| Outputs of the method indicate the values | Cause | Solution |
| --- | --- | --- |
| q\_etResult = ConfigurationNotDone | The function block is not configured. | Configure the function block with the Configure() method. |
| q\_etResult = InvalidSercosState | The Sercos state is not as required. | Set the Sercos state to 4. |
