# Commands from the Visualization to the First Unit

## Command Flow

The TopLoader application contains the structure ST\_ApplicationInputData (refer to the [figure of the Applications tree](ProjectStructure-7AD3B398.html#ProjectStructure-7AD3B398__SoftwareUnitsRepresentingMechanicUn-7B34E3C0)). It holds the command signals that can be sent to the machine (for example, start, stop, selected unit control mode, machine velocity).

Inside the [SR\_Application program](ProjectStructure-7AD3B398.html#ProjectStructure-7AD3B398__SR_ApplicationProgram-7B32F451), multiple instances of these input data are maintained, one for each source the input can come from (such as visualization, HMI, Scada). The [control buttons of the visualization](Controls-7B3FD551.html#Controls-7B3FD551__ControlButtons-7B48B32C) are connected to the stVisuInputs structure directly in the visualization.

![](images/DD01018014.png)

When SR\_Application is in the working state, the method InputsAndCommands is called to retrieve the inputs.

![](images/DD01018013.png)

In order to superimpose multiple command channels (like, for example, visualization and HMI), SR\_Application contains one function block per command signal. It combines the signals from the different sources into one command signal, as, for example xResetButtonPressed.

![](images/DD01018012.png)

The InputsAndCommands method uses these function blocks to combine the sources and to provide diagnostics and logging information.

![](images/DD01018011.png)

Afterwards, the combined information of the command buttons are provided to an instance of the [FB\_StacklightsAndButtons function block of the ApplicationFrameworkUtility library](../../../../../api/crossBook?lang=en-US&virtualBookName=AFULib&topicID=FB_StackLightsAndButtons_GeneralInf_3AE70A15). It generates PackML commands for the unit master. The commands are transferred to the unit together with the other input data for the unit master.
