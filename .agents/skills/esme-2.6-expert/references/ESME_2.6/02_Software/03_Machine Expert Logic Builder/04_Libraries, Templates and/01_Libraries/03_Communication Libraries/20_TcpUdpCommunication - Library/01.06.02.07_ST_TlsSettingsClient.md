# ST\_TlsSettingsClient

## Overview

|  |  |
| --- | --- |
| Type: | Structure |
| Available as of: | V1.0.4.0 |
| Inherits from: | - |

## Description

The structure ST\_TlsSettingsClient is used to provide the TLS (Transport Layer Security) settings for the secured connection to be established by the TCP client.

## Structure Elements

| Name | Data type | Default value | Description |
| --- | --- | --- | --- |
| xSendClientCert | BOOL | FALSE | Set to TRUE if a client certificate is required. |
| sCertName | STRING(255) | - | Specifies the client certificate file. If the value is a null string, the default certificate of the controller is used.  The name corresponds to the value “Issued for” from the controller certificate. The name can be obtained from the certificate using the Security Screen in Logic Builder. |
| etProtocol | ET\_TlsProtocol | TLSv12 | Protocol used for secured connection. |
| etCertVerifyMode | ET\_CertVerifyMode | TrustedOnly | Mode for the certificate verification. |
| sHostnameForSNI | STRING(255) | - | Specifies the name of the host to connect at the start of the handshaking process as part of the Server Name Indication (SNI) extension. |
| ifVerifyCertCallback | IF\_VerifyCertCallback | - | Interface to retrieve detailed diagnostic information of the certificate verification. It allows you to manipulate the certificate verification result. |
