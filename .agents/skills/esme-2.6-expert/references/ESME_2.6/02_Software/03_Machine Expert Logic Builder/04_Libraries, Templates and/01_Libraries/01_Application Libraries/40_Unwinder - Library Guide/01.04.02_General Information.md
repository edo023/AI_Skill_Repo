# General Information

General Information

Description

The library "Unwinder" has been designed mainly for packaging applications. It is suited for mechanics with dancer systems as well as mechanics with foil tension measurement systems.

It supports the two tasks of an unwinder:

o Supplying packaging material in the form of foils to a packaging process.

o Winding up (garbage) foil at the end of the packaging process.

The direction of movement can also be reversed for specific applications.

Use Cases

The Unwinder Library supports the following use cases:

oDancer mechanics

oWinding

oRadius measurement

oDancer shifting

oSplicing from standstill

oSplicing during movement

oMasterless

oDancer/bobbin movement calm down

oDancerless

oTest without mechanic

oRadius determination

oFeedback controller

oPreControl

oFoil crack detection

oJ-load compensation

Working Principle of an Unwinder Process

The graphic shows a schematic representation of an unwinder process. The functions provided by the Unwinder Library are divided into six parts:

![G-SE-0064646.1.gif-high.gif](images/G-SE-0064646.1.gif-high.gif)

1 Feedback handler

2 Radius determination

3 SMG (SoMotionGenerator)

4 PreControl (PreControl with master connection or PreControl by given velocity)

5 Control (Control by a three point controller or by a PI controller)

6 Start movements (start by moving to reference position or start by splicing from standstill)

Namespace of the Library

The default namespace of this library is UW.

The POUs, data structures, enumerations and constants have to be addressed via this namespace.
