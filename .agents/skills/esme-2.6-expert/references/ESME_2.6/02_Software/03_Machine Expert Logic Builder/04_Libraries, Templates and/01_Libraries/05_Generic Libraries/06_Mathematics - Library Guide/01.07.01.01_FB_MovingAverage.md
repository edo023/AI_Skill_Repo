## Description

This function block calculates the moving average of an adjustable number of values. Numerical values of the type LREAL are entered into a FIFO buffer. The size of the FIFO buffer is specified with the input ic\_uiMaxOrder.

First, the buffer has to be initialized with the method Init. This specifies the number of values to be averaged (you can read this setting via the property uiOrder). If the buffer contains fewer values than the setting, the average value is calculated for the values entered.

The number of values to be averaged can be modified with the method SetOrder while the system is running. The values already entered in the buffer are kept and remain considered.

Use the property lrAverage to get the average value according to the settings.

Use the property uiLevel to read the number of valid values entered in the buffer. If uiLevel is equal to the value ic\_uiMaxOrder, the oldest value is overwritten when a new value is entered into the buffer, following the FIFO principle.

The method Init reinitializes the buffer, which means that the number of valid values in the buffer and the calculated average value are set to 0.

## Interface

The input of this function block needs to be assigned during build time. The value cannot be modified during runtime.

The initial value must be defined in the declaration of the instance of this function block.

For example:

```
VAR
	fbMoveAvg :SE_MATH.FB_MovingAverage := (ic_uiMaxOrder := 20);
END_VAR
```

The function block must not be called in the application.

| Input | Data type | Description |
| --- | --- | --- |
| ic\_uiMaxOrder | UINT | Number of values that can be stored in the internal buffer.  Default value: 16 |
