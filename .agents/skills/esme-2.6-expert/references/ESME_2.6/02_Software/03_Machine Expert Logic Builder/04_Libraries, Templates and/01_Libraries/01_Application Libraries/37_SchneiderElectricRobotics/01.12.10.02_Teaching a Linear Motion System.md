# Teaching a Linear Motion System

## Overview

If the target coordinate system is a linear moving system, the samples required to estimate the orientation can be acquired using reference objects placed on the system to teach.

## Procedure for Using two Reference Targets on a Conveyor Belt

| Step | Action |
| --- | --- |
| 1 | Start by printing the provided teaching grid. In this example the [**Reference Grid**](QualityOfTheEstimatedOrientationOrP-E8FA8E39.html#QualityOfTheEstimatedOrientationOrP-E8FA8E39__ExampleOfReferenceGrid-E8FAF666). Only Set 1 of the grid is used during the procedure. |
| 2 | Place the printed teaching grid on the physical coordinate system to teach. Align the X and Y axis of the teaching grid to the equivalent axes of the coordinate system to teach. Compared to the [fixed coordinate system](EstimateTheOrientationOfATargetCoor-E4668C26.html#EstimateTheOrientationOfATargetCoor-E4668C26__FixedCoordinateSystem-E466E8E9), the presence of misalignments do not influence the estimated orientation.  NOTE: It is essential that the teaching grid does not move with respect to the surface of the other system. |
| 3 | Define an instance of the teaching function block:  ``` fbTeachingProcedure : FB_TeachingLinearMotionSystem; ``` |
| 4 | Perform the base configuration of the teaching procedure:   ``` fbTeachingProcedure.Configuration(     i_stReferenceSystemBasePose := stBasePose,     i_xIsRobotReferenceSystem := TRUE, //the robot is the reference "known" system for this teaching     i_ifRobotFeedback := stRoboticModule.iq_ifFeedback,     i_ifRobotConfigurationReadOnly := := stRoboticModule.iq_ifRobotConfigurationReadOnly,     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ```   stBasePose represents the global pose of the robot or the other system involved in the teaching, depending on the value of i\_xIsRobotReferenceSystem. If a SERT.FB\_EntitiesHandler is used to define the layout of the robot cell, then this pose can be retrieved from the function block and it is equal to the base pose of the entity (for example: a robot, a conveyor, and so on). The Feedback and Configuration Read Only interfaces can be retrieved from a robot module interface of from a robot function block. |
| 5 | Create a variable of type [ST\_TeachingLinearMotionSystemData](ST_TeachingLinearMotionSystemData-DC995353.html) and fill it with the required information.  For example:  stProcedureData.ifVelocitySource:= DRV\_Conveyor;  stProcedureData.stLEncLogicalAddresslrY := LE\_Teaching;  stProcedureData.astReferencePoint[1].lrX := 100.0;  stProcedureData.astReferencePoint[1].lrY := 83.0;  stProcedureData.alrReferencePointEncoderPosition[1]:= LE\_Teaching.Position;  stProcedureData.astReferencePoint[2].lrX := 100.0;  stProcedureData.astReferencePoint[2].lrY := 183.0;  stProcedureData.alrReferencePointEncoderPosition[2]:= LE\_Teaching.Position;  stProcedureData.udiNumberOfReferencePoints:= 2;  The positions astReferencePoint are referred to the coordinate system of the linear motion system. The elements of alrReferencePointEncoderPosition are the teaching encoder positions at the time the positions stored inside astReferencePoint were measured.  Then provide a structure to the procedure:   ``` fbTeachingProcedure.SetProcedureData(     i_stData := stProcedureData,     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ``` |
| 6  (Optional) | The teaching requires touching points with the robot on the surface of the other system to teach. This procedure is performed with a tool mounted on the robot. The cartesian pose transformation introduced by a tool can be described with ROB.IF\_RobotConfiguration.ConfigureTool. Then the tool can be selected with the method ROB.IF\_RobotMotion.SetTool. Once a tool is set, the feedback ROB.IF\_RobotFeedback.rstRefPositionTCP accounts for that.  If a rotation of the tool could influence the TCP position, the orientation must be considered, and it is then required to configure the robot so that the orientation is properly handled (see also ROB.IF\_RobotConfiguration.MapComponentToDrive). |
| 7 | Move the robot to a position where it is aligned to the circle marked with the number 1 inside Set 1, as displayed in the graphic.  NOTE: The angle α marked in the image represents a possible misalignment between the orientation of the coordinate system to teach and the teaching grid printout. |
| 8 | Call the method AddSample to store a new set of data:   ``` fbTeachingProcedure.AddSample(     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ```   NOTE: After calling AddSample a first time for the selected set, the value of udiNumberOfSamplesInActiveSet increases by one. |
| 9 | Move the robot to a position where it is aligned to the circle marked with the number 2 inside Set 1, as displayed in the graphic. |
| 10 | Call the method AddSample to store a new set of data:   ``` fbTeachingProcedure.AddSample(     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ```   NOTE: After calling AddSample a second time for the selected set, the set is completed and the value of udiNumberOfCompleteSets increases by one. |
| 11 | Perform a forward motion of the conveyor belt. The length of the motion must be greater than or equal to the distance between the points 1 and 2 of the set.  NOTE: The distance ∆X marked in the image represents the distance of the relative positioning. |
| 12 | Move the robot to a position where it is aligned to the circle marked with the number 1 inside Set 2, as displayed in the graphic. In this case the sampling is performed on the set marked as Set 1 on the teaching grid that was moved to a different position once a set was completed. |
| 13 | Call the method AddSample to store a new set of data:   ``` fbTeachingProcedure.AddSample(     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ``` |
| 14 | Move the robot to a position where it is aligned to the circle marked with the number 2 inside Set 1, as displayed in the graphic. |
| 15 | Call the method AddSample to store a new set of data:   ``` fbTeachingProcedure.AddSample(     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ``` |
| 16 | Perform a forward motion of the conveyor belt. The length of the motion must be greater than or equal to the distance between the points 1 and 2 of the set.  NOTE: The distance ∆X marked in the image represents the length of the relative positioning. The length of the relative positioning between the acquisition of two sets may differ. |
| 17 | Move the robot to a position where it is aligned to the circle marked with the number 1 inside Set 1, as displayed in the graphic. |
| 18 | Call the method AddSample to store a new set of data:   ``` fbTeachingProcedure.AddSample(     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ``` |
| 19 | Move the robot to a position where it is aligned to the circle marked with the number 2 inside Set 1, as displayed in the graphic. |
| 20 | Call the method AddSample to store a new set of data:   ``` fbTeachingProcedure.AddSample(     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ``` |
| 21 | Call the method EstimateOrientation to estimate the orientation of the target coordinate system based on the acquired sets of samples:   ``` stEstimatedPose := fbTeachingProcedure.EstimateCartesianPose (     i_etOrientationConvention := ROB.ET_OrientationConvention.ZYX,     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ```   NOTE:  * stEstimatedOrientation is a structure of type GEM.ST\_CartesianPose that is used to represent a position and an orientation in the cartesian space. * If fbTeachingProcedure.xIsRobotReferenceSystem is TRUE, the returned value is the global pose of the other system involved in the teaching. If it is FALSE, the returned value is the global pose of the robot. |
