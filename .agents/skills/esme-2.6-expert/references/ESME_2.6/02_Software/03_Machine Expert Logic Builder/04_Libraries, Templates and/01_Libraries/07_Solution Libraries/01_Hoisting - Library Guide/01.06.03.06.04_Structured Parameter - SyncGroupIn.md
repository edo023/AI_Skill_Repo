xSync

Synchronization command for the axis group. All axes fulfilling the conditions to enter synchroni­zation are added to the synchronization group when i\_astSyncGroup[i].xSync is TRUE. Axes can be added to synchronization groups only while they are ready and no alarm state is detected. All axes must be in standstill in order to enter or leave a synchronization group, unless they have the parameter i\_astAxisIntf[i].xInstSync set to TRUE.

There must be at least one axis within a synchronization group to allow the group to enter and maintain the synchronous state.

xDrvFwdSync

Forward direction command for all axes within the synchronization group. This input supersedes individual command inputs of each axis interface while the axes are within the synchronization group.

xDrvRevSync

Reverse direction command for all axes within the synchronization group. This input supersedes individual command inputs of each axis interface while the axes are within the synchronization group.

wDrvSpdRefSync

Speed reference for the axis group [RPM]. The value is related to the nominal speed of the synchronization group wDrvSpdNomSync. The output speed reference for axes belonging to the synchronization group is limited by their respective high and low speed values.

rKpPosSync

Proportional gain of the position controller. The gain is used for all axes within the synchronization group and is scaled according to number of pulses per revolution of each motor axis. This allows synchronization of axes with different encoders.

The gain is normalized to a value corresponding to 1024 encoder pulses per revolution. Therefore, with proportional gain of 0.05, the function block applies a correction of 51 RPM when the axis deviates by one motor revolution from the synchronized position.

Start the commissioning with a low value of proportional gain and increase it slowly. Trace the outputs q\_astAxisIntf.rEncPosDev of all axes within the axis group. If the deviations remain too high, increase the proportional gain. If they oscillate, decrease the proportional gain.

rMaxPosDevSync

Maximum allowed position deviation for all axes inside the synchronization group. The parameter is in meters if the axes have defined linear and nominal speeds, or in pulses if they are not so defined. If the deviation of any of the axes within the synchronization group exceeds this value, all axes within the synchronization groups are stopped and an alarm message is signaled.

wDrvSpdNomSync

Nominal speed of the synchronization group [RPM]. The nominal speed serves as a reference for the speed reference defined in parameter wDrvSpdRefSync and for the acceleration and deceleration ramp times. The output speed references for the axes that are part of the synchroni­zation group still respect the configured high and low speeds.

wDrvAccSync

Acceleration ramp time for the synchronization group. Time of acceleration of the synchronization group from standstill to the nominal speed. Acceleration values for the axes get modified dynamically based on their calculated speed ratios.

wDrvDecSync

Deceleration ramp time for the synchronization group. Time of deceleration of the synchronization group from the nominal speed to standstill. Deceleration values for the axes get modified dynamically based on their calculated speed ratios.

xFlexDir

Setting this input to TRUE authorizes axes synchronized within given synchronization group to move towards their respective preset offset positions in an arbitrary direction as long as any direction command is present. i\_astAxisIntf.wDrvLsp parameter of all axes within the synchronization group must be 0 in this case.

Setting this input to FALSE allows movement of axes towards their respective preset offset positions only in the direction specified by the direction command.
