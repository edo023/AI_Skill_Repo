# Operating the Application

![](images/logo.png)

Operating the Application

Overview

The example applications implement a web visualization for monitoring and control purposes.

To open the web visualization, enter the following address in the address bar of a web browser:

http://<IP Address>:8080\webvisu.htm

The visualization is self-explanatory. However, this section provides a brief description of each function. In addition, refer to the [visualization screens](Project_Information-12.htm#XREF_D_SE_0079252_1).

TCP Client

Enter the [IP address](../glossary/glossary.htm#XREF_D_SE_0055638_11) and the [port](../glossary/glossary.htm#XREF_D_SE_0055638_16) of the target [TCP](../glossary/glossary.htm#XREF_D_SE_0055638_20) [server](../glossary/glossary.htm#XREF_D_SE_0055638_18), which is usually the second of the two controllers. Enter the text message that shall be sent to the [TCP](../glossary/glossary.htm#XREF_D_SE_0055638_20) [server](../glossary/glossary.htm#XREF_D_SE_0055638_18). Click the Connect & Send button from the [TCP](../glossary/glossary.htm#XREF_D_SE_0055638_20) [client](../glossary/glossary.htm#XREF_D_SE_0055638_5) visualization to establish a connection to the server and to send data. The status of the [TCP](../glossary/glossary.htm#XREF_D_SE_0055638_20) [client](../glossary/glossary.htm#XREF_D_SE_0055638_5) and the exchanged data are displayed on the visualization.

In case of TLS encryption accepting only trusted certificates the server certificate has to be known and rated as trusted. For more information about certificate management on the controller, refer to [How to Manage Certificates on the Controller - User Guide](../../../../../../api/crossBook?lang=en-US&virtualBookName=HowMgCer&topicID=D_SE_0095876_3).

TCP Server

Open a configurable TCP [port](../glossary/glossary.htm#XREF_D_SE_0055638_16) to allow incoming client connections to be accepted. Data can be received from the client through the established TCP connection. Depending on the content of a received message, a response is sent to the TCP [client](../glossary/glossary.htm#XREF_D_SE_0055638_5). The status of the [TCP](../glossary/glossary.htm#XREF_D_SE_0055638_20) [server](../glossary/glossary.htm#XREF_D_SE_0055638_18) and the exchanged data are displayed in the visualization.

UDP Peer

Open and bind a configurable UDP [port](../glossary/glossary.htm#XREF_D_SE_0055638_16) so that data can be received from another [UDP](../glossary/glossary.htm#XREF_D_SE_0055638_21) [peer](../glossary/glossary.htm#XREF_D_SE_0055638_15). The other [UDP](../glossary/glossary.htm#XREF_D_SE_0055638_21) [peer](../glossary/glossary.htm#XREF_D_SE_0055638_15) is usually the second controller. Enter the [IP address](../glossary/glossary.htm#XREF_D_SE_0055638_11) and the [port](../glossary/glossary.htm#XREF_D_SE_0055638_16) of the target [UDP](../glossary/glossary.htm#XREF_D_SE_0055638_21) [peer](../glossary/glossary.htm#XREF_D_SE_0055638_15). Enter the text message that shall be sent to the other [UDP](../glossary/glossary.htm#XREF_D_SE_0055638_21) [peer](../glossary/glossary.htm#XREF_D_SE_0055638_15). Click the Send button in the visualization to send the data. The status of the [UDP](../glossary/glossary.htm#XREF_D_SE_0055638_21) [peer](../glossary/glossary.htm#XREF_D_SE_0055638_15) and the exchanged data are displayed in the visualization.
