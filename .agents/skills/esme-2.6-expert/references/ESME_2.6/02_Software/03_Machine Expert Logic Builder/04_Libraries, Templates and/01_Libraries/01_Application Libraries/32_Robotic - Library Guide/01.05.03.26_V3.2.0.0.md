## New Features

* **IF\_RobotFeedbackConnectedPath.etActiveMotionLimit**

  The robot feedback etAcitiveMotionLimit indicates the robot component which limits the active robot path movement most. The feedback is available when robot path movement is performed and can be used for example to verify if a synchronous auxiliary axis movement issued by IF\_RobotMotion.MoveSync() influences the robot path movement.
* **IF\_RobotConfigurationAdvanced.SetFeedbackState() and GetFeedbackState()**

  With the method SetFeedbackState() the state of a robot feedback can be defined. In order to save CPU calculation time, the possibility to disable robot feedback by calling the method SetFeedbackState() is introduced. The same method can be used to enable the robot feedback again. By calling the method GetFeedbackSet() the actual configuration can be retrieved.

## Enhancements

* **IF\_RobotFeedback.etLastTargetCoordinateSystem improved**

  The robot feedback etLastTargetCoordinateSystem is updated properly now when calling the method IF\_RobotMotion.SetCoordinateSystem().

* **Using IF\_RobotMotion.MoveSync() for auxiliary axes, when AdditionalTransformationAxes is configured, improved**

  In rare cases an invalid reference position was generated for an auxiliary axis when axis positions of main axes have influenced the position of auxiliary axes in AdditionaltransformationAxes. This has been improved now.
* **Calculating a Motion Profile**

  + Under certain conditions, the diagnostic message ET\_Diag.UnexpectedProgramBehavior was returned by the robot when setting IF\_RobotMotion.lrVelOverride to a very low value. This was related to calculating the motion profile of the connected path. Calculating the motion profile is improved.
  + Under certain conditions, the diagnostic message ET\_Diag.UnexpectedProgramBehavior was returned by the robot due to calculate the time to synchronize for tracking triggered by IF\_RobotMotion.ChangeCoordinateSystem2(). This was related to calculating the motion profile of the connected path. Calculating the motion profile is improved.

* **IF\_RobotFeedbakTracking.xInCoordinateSystem and IF\_RobotFeedbackTracking.etPhase improved**

  The robot feedback xInCoordinateSystem stays FALSE now after tracking is stopped during synchronization phase is active, for example due to reaching the boarder of the work envelope of the robot. The robot feedback etPhase returns the value Inactive in this situation.
* **Keeping maximum motion parameters of synchronous auxiliary axis movement improved**

  In rare cases the defined maximum motion parameters were not kept for synchronous auxiliary axis movements. This was related to calculating the motion profile of the connected path. Calculating the motion profile is improved.
* **Watchdog exception**

  In case IF\_RobotConfigurationAdvanced.ExtendConnectedPath() was configured with i\_udiMinAvailableSegements equal to 1 in rare cases a watchdog exception was caused by the robot. This has been improved.
* **Robot feedback TrackingDeviation fully supported**

  In library Robotic V3.1.2.0 the robot feedback TrackingDeviation was not rendering the correct value.

  The robot feedback TrackingDeviation on PacDrive Lexium Motion Controllers is fully supported again.

  A proper value for TrackingDeviation is reported again. Affected robot feedback:

  + IF\_RobotFeedback.ifSpace.lrTrackingDeviation
  + IF\_RobotFeedback.ifSpace.ifX.lrTrackingDeviation
  + IF\_RobotFeedback.ifSpace.ifY.lrTrackingDeviation

  + IF\_RobotFeedback.ifTrajectoryStorage.ifSpace.lrTrackingDeviation
  + IF\_RobotFeedback.ifTrajectoryStorage.ifSpace.ifX.lrTrackingDeviation
  + IF\_RobotFeedback.ifTrajectoryStorage.ifSpace.ifY.lrTrackingDeviation
  + IF\_RobotFeedback.ifTrajectoryStorage.ifSpace.ifZ.lrTrackingDeviation

  + IF\_RobotFeedback.raifOrientation[ROB.ET\_RobotComponent.OrientationY].lrTrackingDeviation
  + IF\_RobotFeedback.raifOrientation[ROB.ET\_RobotComponent.OrientationZ].lrTrackingDeviation

  Therefore, the default monitoring of the tracking deviation and the monitoring of the maximum tracking deviation configured is also fully supported again. The maximum tracking deviation can be configured by calling the method IF\_RobotConfigurationAdvanced.SetTrackingDeviationLimit().

  A tracking deviation alert and a ControllerStop of robot axes is triggered again in case the TrackingDeviation would exceed the configured maximum value (input i\_lrValue) multiplied by the configured factor (input i\_lrFactor).
