# Error Description

![](images/logo.png)

Error Description

Error Description

Description

The following errors can occur with AutoTune (Error states of the ControllerOptimization function block):

| Detected error | Error text and meaning |
| --- | --- |
| -10000 | rMaxVelocity out of range: The rMaxVelocity parameter is too large or negative. It must be between zero and MaxVel. |
| -10001 | rMaxAcceleration out of range: The rMaxAcceleration is too large or negative. It must be between zero and the value of MaxAcc at J\_Load equal to zero and J\_Gear equal to zero. |
| -10006 | lrMinPosition too great: The value of lrMinPosition is too large. It must be less than lrMaxPosition and less than the position. |
| -10007 | lrMaxPosition too less: The value of lrMaxPosition is too small. It must be greater than lrMinPosition and greater than the position. |
| -10020 | rMaxVelocity changed while optimizing: The rMaxVelocity parameter has been changed during optimization (bOptimizing = TRUE). |
| -10021 | rMaxAcceleration changed while optimizing: The rMaxAcceleration parameter has been changed during optimization (bOptimizing = TRUE). |
| -10026 | lrMinPosition changed while optimizing: The lrMinPosition parameter has been changed during optimization (bOptimizing = TRUE). |
| -10027 | lrMaxPosition changed while optimizing: The lrMaxPosition parameter has been changed during optimization (bOptimizing = TRUE). |
| -10030 | iResponse changed while optimizing: The iResponse parameter has been changed during optimization (bOptimizing = TRUE). |
| -10033 | iOptimizationMode changed while optimizing: The iOptimizationMode parameter has been changed during optimization (bOptimizing = TRUE). |
| -10035 | Controller parameter(s) changed while optimizing: The controller parameter has been manually changed during optimization. |
| -10036 | Mechanical parameter(s) changed while optimizing: The mechanical parameters have been manually changed during optimization. |
| -10040 | rMaxVelocity out of range at AutoTuneCheck: The rMaxVelocity parameter has an invalid value at the AutoTuneCheck function block. |
| -10050 | Axis not ready: The axis is not ready to be switched on. The diagnostic message is in the message log or in the DiagCode (PLC configuration). |
| -10051 | Axis Velocity out of range: The motor velocity has exceeded the permitted maximum velocity rMaxVelocity by 20%. |
| -10052 | Axis Acceleration out of range: The motor acceleration has exceeded the permitted maximum acceleration MotorPeakAcc, which is calculated by multiplying rMaxAcceleration and rMotorAccFactor. |
| -10053 | Axis Position out of range: The actual position of the drive is outside the permitted range, which is defined by lrMinPosition and lrMaxPosition. |
| -10054 | Axis stopped at lower position limit: The axis has been stopped as the value was expected to fall below position limit lrMinPosition. The axis is, however, still within the permitted range of positions. |
| -10055 | Axis stopped at upper position limit: The axis has been stopped as the value was expected to exceed position limit lrMaxPosition. The axis is, however, still within the permitted range of positions. |
| -10057 | No enable while optimizing: At the function block, bEnable was set to FALSE during optimization (bOptimizing = TRUE). |
| -10058 | Axis ID changed while optimizing: The r\_DRIVE parameter was changed during optimization (bOptimizing = TRUE). |
| -10060 | Invalid axis ID: The r\_DRIVE parameter has an invalid value. |
| -10080 | rCutOffFrequencyVelContr is zero: The value zero was calculated for the cut-off frequency of the velocity controller. Further optimization is not possible. |
| -10081 | rCutOffFrequencyPosContr is zero: The value zero was calculated for the cut-off frequency of the position controller. Further optimization is not possible. |
| -10200 | Wrong drive firmware version: The function block only works with the software versions specified in chapter [System Requirements](../System_Requirements/System_Requirements.htm#XREF_D_SE_0091176_4) in the LXM62, LXM52 or ILM62. |
| -10201 | Function block is called twice: The function block can only be called up once. If there is more than one instance of the function block you always have to ensure that only one instance is enabled (bActive = TRUE). |
| -10202 | Sercos not in phase 4: The sercos bus is not in phase 4. |
