# Stop Modes

## General

The RoboticModule provides three different stop modes which are either triggered by command or by exceptions.

These stop modes are:

* Synchronous stop with the configured motion parameters
* Synchronous stop with emergency stop parameters
* Asynchronous stop of the drives

## Synchronous Stop with Motion Parameters

**Stop behavior**

This stop mode stops the robot movement with the configured motion parameters (for example, set by the method IF\_Motion.SetMotionParameters). The robot synchronously ramps down the TCP movement along the given path.

Components which are performing an asynchronous movement are stopped using the motion parameters that are configured for the individual component. An active tracking is not stopped in this case.

**Commands and triggers for the stop**

This stop is triggered by the module command ET\_Cmd.Hold or when the bit i\_xRobotStart in the module interface is set to FALSE. An exception with the reaction StopEndOfCycle also triggers this behavior.

## Synchronous Stop with Emergency Parameters

**Stop behavior**

The robot ramps down the TCP movement with the configured emergency parameters, set by the method IF\_Configuration.SetEmergencyParameter2.

Asynchronous movements are stopped using the emergency parameters configured for the individual component.

The tracking movement uses the configured emergency parameters to stop its motion independent from the TCP movement.

A synchronous stop with emergency parameters, that is not finished within the specified stop time is escalated to an asynchronous stop. The stop time is defined by the input i\_timRobotStopTimeLimit of the method IF\_Configuration.SetEmergencyParameter2.

**Commands and triggers for the stop**

This stop is triggered by the module commands that forces the RoboticModule to exit the automatic mode. These commands are the OpMode commands Manual, Homing or BreakRelease and the commands Stop, Abort, DriveEnableOn and DriveEnableOff .

An exception with the reaction SyncStopEH or SyncStopEL also triggers this stop behavior.

## Asynchronous Stop

**Stop behavior**

In case of an asynchronous stop, the configured drives are stopped individually, depending on the ControllerEnableStopMode selected in the drive.

There are no robotic parameters that are used in this stop mode.

**Commands and triggers for the stop**

This stop mode cannot be triggered by a command. Only a disable of the RoboticModule (iq\_stStandardModuleItf.i\_xEnable = FALSE) or exceptions with a reaction AsyncStop can trigger it.
