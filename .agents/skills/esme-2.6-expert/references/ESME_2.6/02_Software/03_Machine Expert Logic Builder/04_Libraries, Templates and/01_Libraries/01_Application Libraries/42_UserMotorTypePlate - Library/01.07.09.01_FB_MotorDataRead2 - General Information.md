# FB\_MotorDataRead2 - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Function block |
| Available as of: | V1.7.4.0 |

## Task

This function block reads the motor data from the selected axis and writes it to a binary file.

## Description

This function block reads the motor data of the selected axis and writes it to a binary file. You can define the file name. Use FB\_MotorDataFileRead2 to convert the binary file into an ST\_UserMotorData data structure.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_xExecute | BOOL | Upon a rising edge the reading process is started. |
| i\_ifDrive | CMI.IF\_AxisIdentification | Drive from which the motor data is read. |
| i\_sFilename | STRING[255] | File path to the binary motor data file located on the file system of the controller. |
| i\_etStorageLocation | [ET\_StorageLocation](D-SE-0090526.html#D-SE-0090526) | Storage location from which the motor data is read. |

| Output | Data type | Description |
| --- | --- | --- |
| q\_xBusy | BOOL | Indicates that the process of reading the motor data file is in progress. |
| q\_xDone | BOOL | Indicates that the motor data file was read successfully. |
| q\_xError | BOOL | Indicates that an error is detected during the process of reading the motor data file. |
| q\_etDiag | [GD.ET\_Diag](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.GlobalDiagnostic&topicID=D_SE_0076228) | General library-independent statement on the diagnostic.  A value unequal to GD.ET\_Diag.Ok corresponds to a diagnostic message. |
| q\_etDiagExt | [ET\_DiagExt](D-SE-0090520.html#D-SE-0090520) | POU-specific output on the diagnostic.  q\_etDiag = GD.ET\_Diag.Ok -> status message  q\_etDiag <> GD.ET\_Diag.Ok -> diagnostic message |
| q\_sMsg | STRING[80] | Event-triggered message which gives more detailed information on the diagnostic state. |

## CouldNotCreateFile

| Cause | Solution |
| --- | --- |
| There is already a file with the file name i\_sFilename. | Use a different file name. |

## DriveInvalid

| Cause | Solution |
| --- | --- |
| The drive does not support the required interfaces. | Use the function block only with supported drives.  Refer to [Presentation of the library – General Information](D-SE-0090554.html). |

## FunctionNotSupportedByThisDevice

| Cause | Solution |
| --- | --- |
| The WorkingMode of the drive is not physical. | Verify that the WorkingMode of the drive is set to physical. |

## InternalError

| Cause | Solution |
| --- | --- |
| An internal error was detected. | Contact your Schneider Electric service representative. |

## InvalidFilename

| Cause | Solution |
| --- | --- |
| The parameter i\_sFilename is a null string. | Assign the correct filename to i\_sFilename. |

## InvalidStorageLocation

| Cause | Solution |
| --- | --- |
| An invalid value is assigned at the input i\_etStorageLocation. | Assign a valid value to i\_etStorageLocation. |

## NoMotorDataFoundInDrive

| Cause | Solution |
| --- | --- |
| No motor data found in drive. | Verify that the drive has a readable motor typeplate stored. |

## SercosNotInPhaseTwo

| Cause | Solution |
| --- | --- |
| Sercos is not in phase 2. | Verify that the Sercos is in state 2. |

## SercosCommunicationNotPossible

| Cause | Solution |
| --- | --- |
| The function block detects an error during communication with the drive over the Sercos service channel. | Verify the Sercos connection to the drive and that the drive is operational. |

## UnknownState

| Cause | Solution |
| --- | --- |
| An error occurred during internal processing. | Contact your Schneider Electric service representative. |
