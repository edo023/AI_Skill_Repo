# Operating the Application Example

## General

The application example implements a web visualization for monitoring and control purposes.

This section provides a brief description of each function. In addition, refer to [Visualization screens](D-SE-0085455.html#D-SE-0085455).

## Open the Web Visualization

To open the web visualization, enter the following address in the address bar of your web browser:

*http://<IP Address from the controller>:8080\webvisu.htm*

NOTE: By default, the web visualization protocol is blocked by the firewall of the controller. Consult the *Programming Guide* specific to your controller for information about the configuration of the firewall.

## Read CSV File

To read data from a file, you need to specify the parameters for the read operation.

* Enter the file path and name to the file to read.
* Enter the character code used as delimiter for the single values.
* Select the read mode to specify which values to read from the file.
* Set the values for the parameters Row and Column, depending on the read mode.
* Adjust the timeout parameter, if required.

To start the read operation, click Read File. The progress of the operation is displayed in the lower area of the visualization screen.

After a successful read operation, the values are displayed in the Read table in the visualization screen. The table displays 50 values (10 rows by 5 columns) of the buffer where the read values are stored. Use the scroll function to select the rows to be displayed.

In addition, the following information is provided on the visualization screen:

| Display element | Description |
| --- | --- |
| Table structure | Provides information about the dimensions of the available buffer for the read data. |
| File information | Provides information about the file which has been processed recently. |
| Status from last operation | Provides information about the status and details of the last Read File operation. |
| Performance from last operation | Provides information about execution time of the last read operation. Values can be used to find the appropriate value for the timeout parameter and to adjust the task configuration. |

## Write CSV File

To create a file or to append data to an existing file, you need to specify the parameters for the write operation.

* Enter the file path and file name to which you intend to create or append data. If the file path is omitted, the default root directory is assumed.
* Enter the character code used as delimiter for the values.
* Select the mode for opening the file to determine whether a file is to be created or whether data is to be appended.
* Set the values for the parameters Row and Column, depending on the amount of data to be written.
* Adjust the timeout parameter, if required.
* Type/Modify the values in the table that represents the data to write into the file. Beforehand, you can clear the table (button Clear the table) or you can create a copy of the content of the read buffer (button Sync with the read table).

The table displays 50 values (10 rows and 5 columns) of the buffer where the values are stored. Use the scroll function to select the rows to be displayed.

To start the write operation, click Write File. The progress of the operation is displayed in the lower area of the visualization screen.

After a successful write operation, the new size of the file is provided in File information.

In addition, the following information is provided on the visualization screen:

| Display element | Description |
| --- | --- |
| Table structure | Provides information about the dimensions of the available buffer for data to write. |
| Status from last operation | Provides information about the status and details of the last Write File operation. |
| Performance from last operation | Provides information about execution time of the last write operation. Values can be used to find the appropriate value for the timeout parameter and to adjust the task configuration. |
