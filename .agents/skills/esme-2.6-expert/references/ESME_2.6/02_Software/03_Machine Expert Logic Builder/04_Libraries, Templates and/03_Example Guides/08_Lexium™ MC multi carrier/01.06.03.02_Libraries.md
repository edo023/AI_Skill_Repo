# Libraries

## Overview

The following libraries have been added to the Library Manager in order to operate with the Multicarrier example project in the template:

| Library | Company | Function |
| --- | --- | --- |
| Multicarrier | Schneider Electric | Provides methods and interfaces for controlling the Lexium™ MC multi carrier.  For more information, refer to the [Multicarrier library](../../../../../api/crossBook?lang=en-US&virtualBookName=MLSLib&topicID=) |
| MulticarrierStation | Schneider Electric | Provides methods and interfaces for controlling the handling of carriers within and between stations.  For more information, refer to the [MulticarrierStation library](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRSLib&topicID=) |
| ApplicationLogger | Schneider Electric | Offers the functionality to establish a logging for the application.  For more information on the Application Logger, refer to the [ApplicationLogger library](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.ApplicationLogger&topicID=). |

The Multicarrier library provides methods and interfaces to configure and use the multi carrier transport system. The entry point is the interface IF\_Multicarrier.

In the MulticarrierStation library, the function block FB\_CoreStation allows you to create your own stations. Furthermore, ready-made stations are available that can be parameterized.

Both libraries use the Application Logger for writing messages and events into the corresponding library.
