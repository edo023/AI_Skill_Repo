# FB\_MotorDayOfManufactureWrite2 - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Function block |
| Available as of: | V1.7.4.0 |

## Task

This function block writes the day of manufacture of a motor to the electronic motor typeplate.

## Description

The day of manufacture is stored as part of the electronic motor typeplate.

## Requirements

* Sercos must be in phase 2.
* The *[MotorIdentification](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM62Drive&topicID=D_SE_0071763)* parameter of the axis must be set to the correct value.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_xExecute | BOOL | Upon a rising edge the write process is started. |
| i\_ifDrive | CMI.IF\_AxisIdentification | Drive to which the motor data is written. |
| i\_etStorageLocation | [ET\_StorageLocation](D-SE-0090526.html#D-SE-0090526) | Storage location to which the motor data is written.  The [MotorIdentification](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM62Drive&topicID=D_SE_0071763) parameter of the axis has to be set according to the selected storage location.   * ET\_StorageLocation.Drive => Motor without typeplate / 2 * ET\_StorageLocation.Encoder => Motor with typeplate / 0 |
| i\_dtDayOfManufacture | DATE | Date of manufacture to be written to the typeplate. |
| i\_xReinit | BOOL | The drive is reinitialized after the date of manufacture is written successfully. |

| Output | Data type | Description |
| --- | --- | --- |
| q\_xBusy | BOOL | Indicates that the process to write the date of manufacture is in progress. |
| q\_xDone | BOOL | Indicates that the date of manufacture was written successfully. |
| q\_xError | BOOL | Indicates that an error is detected during the process to write the date of manufacture. |
| q\_etDiag | [GD.ET\_Diag](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.GlobalDiagnostic&topicID=D_SE_0076228) | General library-independent statement on the diagnostic.  A value unequal to GD.ET\_Diag.Ok corresponds to a diagnostic message. |
| q\_etDiagExt | [ET\_DiagExt](D-SE-0090520.html#D-SE-0090520) | POU-specific output on the diagnostic.  q\_etDiag = GD.ET\_Diag.Ok -> status message  q\_etDiag <> GD.ET\_Diag.Ok -> diagnostic message |
| q\_sMsg | STRING[80] | Event-triggered message which gives more detailed information on the diagnostic state. |
