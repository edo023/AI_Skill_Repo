Task

Function for changing cam points.

Description

Using this function, the xInstantNewCam bit can be set within the AxisModuleInterface structure. The structure assigned to a specific axis is transferred to the function via the input/output iq\_stAxisModuleItf. The xInstantNewCam signal will cause the axis to reload the cam points specified by the currently selected MultiCam table and run the changes in the current cycle as shown below.

![G-SE-0064627.1.gif-high.gif](images/G-SE-0064627.1.gif-high.gif)

The new cam data takes effect at the InstantXLimMax blend point. The blend point can be specified with the [FC\_SetInstantNewCamPosition](Functions-28.htm#XREF_D_SE_0077203_1) function.

The xInstantNewCam signal provides feedback. The axis will reset the signal to false at the blend point if the new cam points are accepted. The [FC\_GetInstantNewCamStatus](Functions-8.htm#XREF_D_SE_0077163_1) function can be used to retrieve the status of the xInstantNewCam signal. The xInstantNewCam signal has to be set TRUE before the axis arrives at the blend point for the new cam points to be used in the current cycle.

The default MultiCam table is specified with the [FC\_InitCamParameter](Functions-10.htm#XREF_D_SE_0077167_1) function.

This function can’t be continually scanned because it will continually set the signal.
