# Operation with Tuning

## Regular Operation

* When tuning is enabled and the tracking deviation is configured, the FastMove functionality tries to tune the acceleration and deceleration of each individual movement to the maximum while keeping the tracking deviation at the TCP within the given limits.

  In this case, the values AccelerationFactor and DecelerationFactor that can be set by IF\_FastMove.SetParameter are ignored.
* When an individual movement is fully tuned, the feedback bit SER.IF\_FastMove.xPreviousMovementTuned is set to TRUE.
* It is possible to keep the tuning active all the time. If tuning is not necessary anymore, the values are not modified.
* When the tuning is done for all movements, the tuning process can be put on hold with the property IF\_FastMove.xReadOnly. When this property is TRUE, the present values are used and not adapted any more. The property can be reset at any time to continue with the tuning process.

## Restart the Tuning Process

* The tuning process can be restarted using the method IF\_FastMove.RestartTuning. It can be restarted from scratch with the input i\_xCompleteTuning set to TRUE. In this case, all tuning values are discarded, and the tuning starts with its initial values.
* Restart a complete tuning when the new payload or a new gripper have a significant difference in weight compared to the present weights.
* When only a small change was made, the tuning can be restarted with i\_xCompleteTuning set to FALSE. Then the present tuning values are kept and used as base for the new tuning process.

## Save and Restore the Tuning Values

The methods SetTuningData and GetTuningData can be used to read and write the tuning values.

As the methods only provide pointers to the data, you must implement the save and load process yourself. There is no consistency check for the data when the data is loaded.
