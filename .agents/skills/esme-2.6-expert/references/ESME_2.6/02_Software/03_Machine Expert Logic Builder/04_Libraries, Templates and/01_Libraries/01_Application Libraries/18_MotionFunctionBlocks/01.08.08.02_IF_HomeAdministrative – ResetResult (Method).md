# IF\_HomeAdministrative – ResetResult (Method)

## Overview

|  |  |
| --- | --- |
| Type: | Method |
| Available as of: | V1.3.1.0 |
| Versions: | Installed version |

## Task

Resets the diagnostic outputs and command state of the function block [FB\_Home](FB_Home-459E662D.html#FB_Home-459E662D).

## Description

The method is used to reset the diagnostic outputs of the method [FB\_Home.Cycle](FB_Home-CycleMethod-45AB3661.html#FB_Home-CycleMethod-45AB3661) and the command state structure. Execution of the method is only possible if no homing process is running.

NOTE: An existing ErrorStatus of the axis is not reset. Use the method [ResetError](../../../../../api/crossBook?lang=en-US&virtualBookName=IF_AdministrativeResetErrorMethod-43166D4C.html#IF_AdministrativeResetErrorMethod-43166D4C) of FB\_AxisMotion or the PLCopen function block MC\_Reset to clear it. Refer to the [M262 Synchronized Motion Control Library Guide](../../PLCO&topicID=D_SE_0086568).

## Outputs

| Output | Data type | Description |
| --- | --- | --- |
| q\_xError | BOOL | The output is set to TRUE if an error has been detected during the execution. |
| q\_etResult | ET\_Result | POU-specific output on the diagnostic; q\_xError = FALSE -> Status message; q\_xError = TRUE -> Diagnostic message. |
| q\_sResultMsg | STRING[80] | Event-triggered message that gives additional information on the diagnostic state. |

## Troubleshooting

This table describes the possible issues and their solutions:

| Issue  (error information q\_xError = TRUE) | Cause | Solution |
| --- | --- | --- |
| q\_etResult = HomingIsAlreadyActive | FB\_Home is processing a homing command. | Retry after running the homing command has been completed. |
