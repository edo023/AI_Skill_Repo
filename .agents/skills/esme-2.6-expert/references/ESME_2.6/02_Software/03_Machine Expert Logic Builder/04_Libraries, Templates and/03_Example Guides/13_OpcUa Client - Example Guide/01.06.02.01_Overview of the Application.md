# Overview of the Application

## Overview

The application includes the logic for controlling the OPC UA client provided on the controller.

The following features are supported:

* Establishing the connection to an OPC UA server and opening an OPC UA session.
* Cyclic monitoring of connection status.
* Querying the namespace indexes and their node handles of the defined namespace URIs.
* Reading (one-shot or cyclic) a list of variables from the OPC UA server.
* Writing (one-shot or cyclic) a list of variables on the OPC UA server.

  + Creating a subscription.
  + Adding monitored items to a subscription.
  + Updating values of monitored items in controller synchronization mode.
  + Verifying if values are updated in firmware synchronization mode.
  + Removing monitored items from a subscription.
  + Deleting a subscription.
* Stopping the OPC UA session and closing the connection to the OPC UA server.
* Dynamic configuration of the controller firewall from the application program during runtime.

Furthermore, an OPC UA server is configured on the same controller. The application implements simulation logic to modify the values of the variables provided via the OPC UA server.

In addition, the application implements a visualization to monitor and control the OPC UA client and the data exchange.
