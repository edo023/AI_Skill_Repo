# Secured Connection to SysLog Server

## Overview

This section shows how to retrieve a certificate needed for a secured connection between SysLog agent and server.

## Certificate Retrieval

For auto-signed certificates, you have to transfer the certificate received from server (pki/untrusted directory) into the pki/trusted/certs directory.

This operation is possible from FTP server or from the File Tab of the controller in EcoStruxure Machine Expert.

For other kind of certificates, like a certificate signed by a Certificate Authority, copy the certificate into the pki/trusted/certs directory.

The following procedure describes how to retrieve the certificate for secured server connection:

| Step | Action |
| --- | --- |
| 1 | In EcoStruxure Machine Expert, double-click on your controller in the Devices tree. |
| 2 | Select the Files tab, then click the Refresh icon. |
| 3 | A hazard message window opens. Read the text carefully and press Alt+F to confirm. |
| 4 | In the right window, open the pki/untrusted directory and browse to the certificate (.crt file). |
| 5 | In the left window, select a temporary location to store the certificate and transfer it by clicking on the central arrow button. |
| 6 | In the right window, open the pki/trusted/certs directory. |
| 7 | Select the certificate in the left window and transfer it from its temporary location to the pki/trusted/certs directory by clicking on the central arrow button. |
