# FB\_TorqueFeedForward - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Function block |
| Available as of: | V1.0.0.0 |
| Versions: | Installed version |

## Task

Converts the feed forward value and transfer it to the drive.

## Description

This POU receives the target values for the torque feed forward, and transfers the converted values to the drive.

Four operating modes are available for the torque feed forward. They can be selected with the i\_iMode input:

| Mode | Description |
| --- | --- |
| 0 | Feed forward disabled  The feed forward is disabled. Neither the drive nor the POU perform any calculations. The torque feed forward value set by the POUs in this library has no effect. |
| 1 | Default feed forward in the drive.  The feed forward is activated. The feed forward values are automatically calculated in the drive. The feed forward is influenced by the drive parameters   * J\_Gear * GearIn * GearOut * J\_Load * StaticFriction * ViscousFriction   The POU does not perform any calculations. The torque feed forward value set by the POUs in this library has no effect. |
| 2 | Feed forward by the user.  The feed forward is activated. You must calculate the feed forward values and transfer them to the POU with the corresponding parameter ([FB\_TorqueFeedForward.i\_lrTorque](D-SE-0090563.html)). In doing so, torques must be calculated that occur due to the load (mechanics without gear box and product). The torque for acceleration of the motor (including gear box) is automatically calculated in the drive and added to the feed forward value. This calculation is influenced by the drive parameter J\_Gear. |
| 3 | Feed forward by the user and default feed forward.  The feed forward is activated. You must calculate the feed forward values that deviate from the default feed forward and transfer them to the POU with the corresponding parameter ([FB\_TorqueFeedForward.i\_lrTorque](D-SE-0090563.html)). The default feed forward is automatically calculated in the drive (refer to ET\_ControlMode.Default).  The user-specific feed forward value is added to the automatically calculated feed forward value. |
| 4 | Feed forward by the user only.  The feed forward is activated. You must calculate the feed forward values and transfer them to the POU with the corresponding parameter ([FB\_TorqueFeedForward.i\_lrTorque](D-SE-0090563.html)). In doing so, torques must be calculated that occur due to the load, gearbox and motor.  No calculations are performed within the drive. |

The following prerequisites must be fulfilled to use the POU:

* Sercos must be in phase 4.
* The feed forward must be configured (the q\_xDone output of FB\_TorqueFeedForwardConfig provides the value TRUE).

NOTE: This POU must be executed by a task that is triggered by an external event. "MDT\_WRITE\_ACCESS" must be selected as external event. These settings enable processing synchronously to the Sercos cycle.

NOTE: This function block is only supported on LMC PacDrive controller.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_xEnable | BOOL | A rising edge FALSE -> TRUE activates the POU, a falling edge TRUE -> FALSE deactivates the POU.  A deactivated POU does not execute any actions. |
| i\_xExecute | BOOL | FALSE -> TRUE: POU performs calculations and transfers feed forward values to the drive. This is only possible after the configuration has been completed (q\_xDone output of iq\_fbConfig provides the value TRUE). |
| i\_lrTorque | LREAL | Gear box output side feed forward torque in Nm. |
| i\_iMode | INT | Operation mode of the feed forward (see upper table "Operating modes") |
| i\_xReset | BOOL | FALSE -> TRUE: Resets the POU to its initial state (q\_xActive = FALSE, q\_xReady = FALSE) |

| Output | Data type | Description |
| --- | --- | --- |
| q\_xActive | BOOL | TRUE: The POU is active and has to be executed further.  FALSE: The POU is inactive. |
| q\_xReady | BOOL | TRUE: The POU is ready to operate and can accept user commands.  FALSE: The function block is not ready to accept user commands. |
| q\_etDiag | [GD.ET\_Diag](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.GlobalDiagnostic&topicID=D_SE_0076228) | General library-independent statement on the diagnostic.  A value unequal to GD.ET\_Diag.Ok corresponds to a diagnostic message. |
| q\_etDiagExt | [ET\_DiagExt](D-SE-0090561.html#D-SE-0090561) | POU-specific output on the diagnostic.  q\_etDiag = GD.ET\_Diag.Ok -> status message  q\_etDiag <> GD.ET\_Diag.Ok -> diagnostic message |
| q\_sMsg | STRING[80] | Event-triggered message which gives more detailed information on the diagnostic state. |

| Input/Output | Data type | Description |
| --- | --- | --- |
| iq\_fbConfig | [FB\_TorqueFeedForwardConfig](D-SE-0090565.html#D-SE-0090565) | Configuration POU  The q\_xDone output is TRUE if the configuration has been completed. Only then the FB\_TorqueFeedForward POU can be executed (i\_xExecute: FALSE -> TRUE). |

## Diagnostic Messages

| q\_etDiag | q\_etDiagExt | Enumeration value | Description |
| --- | --- | --- | --- |
| OK | Disabled | 0 | The POU is disabled. |
| OK | Executing | 5 | Version |
| OK | WaitForConfiguration | 14 | POU waits for the configuration. |
| OK | WaitForExecute | 3 | POU waits for the execution. |
| UnexpectedProgramBehavior | ConfigurationNotDone | 13 | The configuration has not been performed. |

## ConfigurationNotDone

| Issue | Cause | Solution |
| --- | --- | --- |
| - | The i\_xExecute input has been set to TRUE before the configuration was completed (q\_xDone output of FB\_TorqueFeedForwardConfig is FALSE). | Set i\_xExecute = TRUE only after the q\_xDone output of FB\_TorqueFeedForwardConfig is TRUE. |

## Disabled

The function block is disabled and executes no actions. i\_xEnable and q\_xActive are set to FALSE

## Executing

The function block is active and transfers the feed forward values to the selected axis.

## WaitForConfiguration

The POU has completed its initialization and waits until the configuration has been completed (q\_xDone output of FB\_TorqueFeedForwardConfig is TRUE).

## WaitForExecute

The POU has completed its initialization and is waiting for a positive edge at the i\_xExecute input before continuing processing.
