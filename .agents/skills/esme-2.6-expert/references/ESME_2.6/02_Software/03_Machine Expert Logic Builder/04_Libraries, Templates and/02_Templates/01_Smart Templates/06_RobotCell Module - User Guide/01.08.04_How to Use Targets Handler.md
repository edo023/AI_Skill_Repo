# How to Use Targets Handler

## Overview

If a TargetsGenerator (for example, Camera/Sensor) has created some targets you must handle them. The camera/sensor indicates if new target data are available (for example, SR\_<MyCamera>.xVisionDataReceived or SR\_<MySensor>.xNewTargetsAvailable).

To handle the targets in the application, you can add them to an assigned targets handler (for example, SR\_<RobotCell>.q\_aifTargetsHandler[ROB.ET\_CoordinateSystem.Tracking1].AddTarget()).

## Example IEC-Code for Camera

Example for the declaration part.

```
//TargetHandler
aifTargetsHandler	: ARRAY[ROB.ET_CoordinateSystem.Tracking1..ROB.ET_CoordinateSystem.Tracking30] OF SERT.IF_TargetsHandler;
etDiag 		: GD.ET_Diag;
etDiagExtSert 	: SERT.ET_DiagExt;
sMsg		: STRING[80];
udiListIndex 	: UDINT;
udiUnivocalId 	: UDINT;

//Targets SmartVisu
stVisionDataProducts : SERT.ST_VisionDataProducts;
udiUnivocalId 	: UDINT;
udiTargetsCounter 	:UDINT;
xIsTargetSlipping 	: BOOL;
stCartesianPose 	: SERT.ST_CartesianPose;
```

Example for the implementation part.

```
// Set aifTargetsHandler 
aifTargetsHandler  := SR_<RobotCell>.q_aifTargetsHandle;  

IF SR_<MyCamera>.xVisionDataReceived THEN 
         //New target data has been detected
         //read out the data
         stVisionDataProducts := SR_<MyCamera>.GetProducts();
         FOR udiTargetsCounter := 1 TO stVisionDataProducts.uiNumberOfProducts DO  

          stCartesianPose.stPosition := stVisionDataProducts.astProductData[udiTargetsCounter].stPosition;
          stCartesianPose.stOrientation.lrZ := stVisionDataProducts.astProductData[udiTargetsCounter].lrRotation;
          stCartesianPose.etOrientationConvention := ROB.ET_OrientationConvention.XYZ;
          //AddTargets to targetsHandler
          //User must select the TargetsHandler the targets should be added to, by select the right value between ROB.ET_CoordinateSystem.Tracking1.. ROB.ET_CoordinateSystem.Tracking30 
          IF (aifTargetsHandler[ROB.ET_CoordinateSystem.Tracking1] <> 0) THEN  

                    aifTargetsHandler[ROB.ET_CoordinateSystem.Tracking1].AddTarget(i_stPose := stCartesianPose,  
                    i_lrEncoderPosition:= LE_TargetHandler1.Position, 
                    // User must select the Encoder connected to the TargetsHandler.
                    // User must set a Type. In case the camera already has an ObjectType information this can be used. The same TargetType should be used for TargetsHandler.
                    i_uiType:= stVisionDataProducts.astProductData[udiTargetsCounter].uiObjectType, // SERT.ET_TargetType.Type1,  
                    q_etDiag=> etDiag, 
                    q_etDiagExt=> etDiagExtSert, 
                    q_sMsg=> sMsg, 
                    q_udiListIndex=> udiListIndex, 
                    q_udiUnivocalId=> udiUnivocalId);
    END_IF
		
  END_FOR

// Call ClearVisionData to be ready for new data.
  SR_Camera.ClearVisionData();
END_IF
```
