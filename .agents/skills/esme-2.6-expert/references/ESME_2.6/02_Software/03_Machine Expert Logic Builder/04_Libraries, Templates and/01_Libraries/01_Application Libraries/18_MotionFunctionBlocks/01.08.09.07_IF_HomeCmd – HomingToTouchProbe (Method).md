# IF\_HomeCmd – HomingToTouchProbe (Method)

## Overview

|  |  |
| --- | --- |
| Type: | Method |
| Available as of: | V1.3.1.0 |
| Versions: | Installed version |

## Task

Set the position of the axis based on the position capture value of a touchprobe.

## Description

The function block FB\_Home internally uses FB\_HomingTouchProbe from the PLCopen MC Part 1 library. For detailed information on the configuration and the axis behavior during the homing process, refer to the functional description for FB\_HomingTouchProbe in the *M262 Synchronized Motion Control Library Guide* online help.

The method sets the position of the axis based on the position capture value of a touch probe. The input i\_ifTriggerRef specifies the edge of the capture signal to be detected.

* The homing movement is started to a constant velocity in the configured direction.
* When the specified edge of the capture signal is detected, the corresponding position value is set as the home position and the movement decelerates.
* Depending on the value of the input i\_xReturnToZero, the movement either stops or is continued to the zero point.

NOTE: The edge of the position capture signal is detected by IDNs in the library. This implies a delay which depends on how many times the method [Cycle](FB_Home-CycleMethod-45AB3661.html#FB_Home-CycleMethod-45AB3661)  has to be called to be completed. If the input i\_xReturnToZero is set to FALSE, the delay corresponds to the distance to the touchprobe after the state xDone has been set to TRUE.

If the edge of the capture signal is not detected within the configured maximum travel distance, the execution of the function block is aborted with an error. If a hardware limit switch is triggered during the homing movement, the execution of the function block is aborted with an error.

NOTE: To perform this homing command the axis must be in state Standstill.

## Inputs

| Input | Data type | Description |
| --- | --- | --- |
| i\_ifTriggerRef | PLCO. MC\_TriggerRef | Reference to the position capture source of the axis for which the function block is to be executed. The function block detects the specified edge of the capture signal and homes the axis to this position value. |
| i\_lrHomePosition | LREAL | Position in user-defined units that is set as home position when the specified edge of the capture signal is detected. |
| i\_lrMaxTravelDistance | LREAL | Maximum distance in user-defined units of the movement for searching for the edge of the capture signal.  NOTE: Values less than 0 disable the monitoring of the maximum movement distance. |
| i\_lrVelocity | LREAL | Value of the velocity for the homing movement at constant velocity. |
| i\_lrAcceleration | LREAL | Value of the acceleration for the homing movement at constant velocity. |
| i\_lrDeceleration | LREAL | Value of the deceleration for the homing movement after the edge of the capture signal has been detected. |
| i\_lrJerk | LREAL | Value for the maximum jerk with which the acceleration is modified.  NOTE: Value 0 disables the jerk limit and results in a jump from zero to maximum acceleration. |
| i\_etDirection | ET\_Direction | Direction of the homing movement. |
| i\_xReturnToZero | BOOL | Specifies whether the movement is continued to the zero position 0 after the home position has been set. |

## Outputs

| Output | Data type | Description |

| q\_xError | BOOL | The output is set to TRUE if an error has been detected during the execution. |
| q\_etResult | ET\_Result | POU-specific output on the diagnostic; q\_xError = FALSE -> Status message; q\_xError = TRUE -> Diagnostic message. |

## Troubleshooting

This table describes the possible issues and their solutions:

| Issue  (error information q\_xError = TRUE) | Cause | Solution |
| --- | --- | --- |
| q\_etResult = HomingIsAlreadyActive | Another homing process is already ongoing. | Retry after the active homing process has been completed |
| q\_etResult = NotInitialized | The FB\_Home function block instance has not been initialized yet. | Initialize the function block by calling the method Init. |
| q\_etResult = AxisNotReady | The axis is not in standstill. | Stop the axis before triggering the homing command. |
| q\_etResult = InvalidInput | The interface assigned at i\_ifTriggerRef is invalid. | Assign a valid interface to the position capture source. |
| q\_etResult = HomingFailed | FB\_HomingTouchProbe could not execute the homing command. | Refer to q\_sResultMsg for more details. |
| q\_etResult = HomingInterrupted | Due to an external command or an abort caused by an online change, the control of the axis has been lost. | Trigger the homing command again. |
