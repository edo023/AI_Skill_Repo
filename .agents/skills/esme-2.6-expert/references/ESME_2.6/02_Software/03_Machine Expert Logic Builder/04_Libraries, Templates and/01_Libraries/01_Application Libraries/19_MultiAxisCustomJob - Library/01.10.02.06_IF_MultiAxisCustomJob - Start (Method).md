## Description

The method Start of the interface [IF\_MultiAxisCustomJob](IF_MultiAxisCustomJob-4D7A4899.html) initiates the multi-axis custom job. Calling this method triggers the transition of the interface from the Idle state to the Linked state, and subsequently to the Executing state in the next RTP cycle.

The method requires a reference to a user-defined function block that implements the IF\_MultiAxisCustomJobCallback interface. This link allows the multi-axis custom job to execute the custom motion logic via the Prepare and Calculate methods.

Upon activation, the interface verifies the availability of the specified configuration. The method returns a result indicating whether the start sequence was successfully initiated. It does not wait for all subordinate channels to reach their target operating state. If the start sequence is successful, the subordinate axes abort their current motion jobs to transfer control to the multi-axis custom job.

For details on the interaction of function blocks, interfaces and methods of a multi-axis custom job, refer to [Multi-Axis Custom Jobs](Multi-AxisCustomJobs-572EC46B.html).
