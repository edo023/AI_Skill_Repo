## Overview

## Description

The method Prepare is called once by the function block [FB\_MultiAxisCustomJob](FB_MultiAxisCustomJob-4CFA57F9.html) during the initialization phase in the first RTP cycle of the job execution. This method is intended for the initialization of your custom motion algorithm for the multi-axis custom job.

At the time of the call, the master channel values have already been updated for the current RTP cycle, while the subordinate channels reflect the movement values of the axes from the RTP cycle immediately preceding the job activation. This allows you to synchronize your algorithm with the current movement state of the axes if they were already moving.

Implementing code within the Prepare method is optional; the multi-axis group will function even if the method remains empty. The system proceeds to call the Calculate method in the same RTP cycle immediately after Prepare returns.

For details on the interaction of function blocks, interfaces and methods of a multi-axis custom job, refer to [Multi-Axis Custom Jobs](Multi-AxisCustomJobs-572EC46B.html).
