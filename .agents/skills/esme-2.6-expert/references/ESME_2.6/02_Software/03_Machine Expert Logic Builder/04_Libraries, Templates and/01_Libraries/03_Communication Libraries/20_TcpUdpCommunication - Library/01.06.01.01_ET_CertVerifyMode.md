# ET\_CertVerifyMode

## Overview

|  |  |
| --- | --- |
| Type: | Enumeration |
| Available as of: | V1.0.4.0 |

## Description

The enumeration ET\_CertVerifyMode is used to define the mode for the certificate verification for TCP/IP connections using TLS (Transport Layer Security).

In the context of TLS, certificates can be used to verify the identity of the communication partners. Certificates are sent while establishing the connection, it is called the TLS handshake. The sending of certificates during TLS handshake is optional, but if the communication partner requires a certificate, it is obligatory. Only if the result of the verification of the certificate is positive, a connection with the communication partner can be established. If the client or server is configured to verify the server certificate in mode TrustedOnly, it is required to manage the certificates on your controller manually. This can be done using the editor Security Screen in the Logic Builder.

NOTE: TrustedOnly is the only way to verify the identity of the communication partner.

## Enumeration Elements

| Name | Value | Description |
| --- | --- | --- |
| NotVerified | 1 | No verification of the certificate. Also, peers without sending a certificate are accepted. This setting is used for the server by default. |
| AllCertificates | 4 | A certificate by peer is obligatory but can be undefined. Each certificate is considered as trusted. |
| TrustedOnly | 6 | A certificate by peer is obligatory and must be known and rated as trusted. |

For more information about certificate management on the controller, refer to [*How To Manage Certificates on the Controller User Guide*](../../../../../api/crossBook?lang=en-US&virtualBookName=HowMgCer&topicID=D_SE_0095876).
