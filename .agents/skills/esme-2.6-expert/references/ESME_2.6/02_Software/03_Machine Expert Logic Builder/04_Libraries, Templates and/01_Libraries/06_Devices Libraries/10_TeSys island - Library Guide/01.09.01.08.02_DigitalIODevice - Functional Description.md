# DigitalIODevice - Functional Description

## Overview

|  |  |
| --- | --- |
| Type: | Function block |
| Available as of: | V4.0.1.0 |

## Functional Description

The function block DigitalIODevice provides information about the digital I/O device with four inputs and two outputs.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| iq\_TeSysIslandRef | FB\_TeSys\_island | Reference to the TeSys island device. |
| i\_sDeviceName | STRING(GPL.Gc\_uiLengthOfDeviceName) | Reference to the device defined in the TeSys island configuration. |
| i\_xEnable | BOOL | A rising edge at the i\_xEnable input starts the cyclic data exchange between the function block and the selected device. As long as i\_xEnable is TRUE, the cyclic data is exchanged at every controller cycle. A falling edge at the i\_xEnable input stops the data exchange. Refer to [*Behavior of the Function Blocks with the Input i\_xEnable*](D-SE-0095075.html#D-SE-0095075). |
| i\_xDQ0 | BOOL | If this input is set to TRUE, the digital output 0 is set to TRUE.  The input is only active if the DigitalIODevice is used as a subdevice of another avatar, for example, a Motor One Direction avatar and is not assigned to an I/O port in the IO PORT ASSIGNMENT table. The input i\_xDQ0 is inactive if the DigitalIODevice is used as an individual Digital I/O avatar or is assigned to an I/O port in the IO PORT ASSIGNMENT table. |
| i\_xDQ1 | BOOL | If this input is set to TRUE, the digital output 1 is set to TRUE.  The input is only active if the DigitalIODevice is used as a subdevice of another avatar, for example, a Motor One Direction avatar and is not assigned to an I/O port in the IO PORT ASSIGNMENT table. The input i\_xDQ1 is inactive if the DigitalIODevice is used as an individual Digital I/O avatar or is assigned to an I/O port in the IO PORT ASSIGNMENT table. |

| Output | Data type | Description |
| --- | --- | --- |
| q\_xActive | BOOL | If the function block is active, this output is set to TRUE. |
| q\_xError | BOOL | If this output is set to TRUE, an error has been detected. For details, refer to q\_etResult and q\_etResultMsg. |
| q\_etResult | ET\_Result | Provides diagnostic and status information as a numeric value. |
| q\_sResultMsg | STRING[30] | Provides additional diagnostic and status information as a text message. |
| q\_xReady | BOOL | Indicates TRUE if the device is ready to receive commands. |
| q\_xDI0 | BOOL | If this output is set to TRUE, the digital input 0 of the Digital I/O device is set to TRUE. |
| q\_xDI1 | BOOL | If this output is set to TRUE, the digital input 1 of the Digital I/O device is set to TRUE. |
| q\_xDI2 | BOOL | If this output is set to TRUE, the digital input 2 of the Digital I/O device is set to TRUE. |
| q\_xDI3 | BOOL | If this output is set to TRUE, the digital input 3 of the Digital I/O device is set to TRUE. |
