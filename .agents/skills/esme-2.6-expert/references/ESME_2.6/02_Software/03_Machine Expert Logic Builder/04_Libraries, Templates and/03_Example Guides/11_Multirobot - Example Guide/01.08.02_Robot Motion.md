# Robot Motion

## Default Actions

Each instance of the function block FB\_RobotMotion provides a set of default actions that the interface Supervisor Interface can require from the robot. In the demo project, each Smart Template robot module runs an instance of FB\_RobotMotion in their Logic method.

Four default actions are implemented in the project and a command can be sent using stTaskInterface.stCommand by the supervisor:

| Parameter | Description |
| --- | --- |
| MoveToSafe | A vertical motion, performed at the beginning of an Auto mode to bring the robot to a position free from obstacles. |
| MoveToTarget | A generic motion to a target pose, this is used, for example, when the robot must reach a wait position, if no targets are available. |
| PickTarget | The robot executes a motion to reach a target to pick and it handles the activation of the gripper. |
| PlaceOnTarget | The robot executes a motion to reach a target place pose and it handles the deactivation of the gripper. |

## Path Planning and Trajectory Generation

The path planning of the default actions is generated with a method of FB\_RobotMotion called MoveSTarget. This method generates a path from the present robot pose to the target pose depending on a set of constraints related to the geometry of the system. The following types of paths are possible:

| Parameter | Description |
| --- | --- |
| LinearMoveInSafeArea | Both the start and target poses are above the collision-free height. A linear movement is performed. |
| LinearMoveVertical | The start and target poses have the same X and Y coordinates, but a different Z coordinate. A vertical linear movement is performed. |
| HalfSplineExt | Either the start or the target pose is above the collision-free height. A half spline is performed, taking into account the obstacle restrictions described by stPickPlaceConstraints. |
| FullSplineExt | Start and target are below the collision-free height. A complete spline is performed taking into account the obstacle restrictions described by stPickPlaceConstraints. |

Once a path from start to target pose is generated, FB\_RobotMotion generates a trajectory using the parameters defined by stMotionParameters for the selected action.
