# Index File - Example and Structure Description for an ATV340S

![](images/logo.png)

Index File - Example and Structure Description for an ATV340S

Example

Example of an Index File that is intended to read parameters from an ATV340S:

ATV340S  
Name;LogicAddress;IDN;LenBytes;DataType  
CMI;8504;P-0-2055.0.4;2;WORD (BitString16)  
ACC;9001;P-0-2060.0.1;2;UINT (Unsigned16)  
DEC;9002;P-0-2060.0.2;2;UINT (Unsigned16)  
…  
CD1;8423;P-0-2054.0.23;2;WORD (Enumeration)  
FR1;8413;P-0-2054.0.13;2;WORD (Enumeration)

Each line of the Index File is a data record that corresponds to one parameter.

Each parameter is thus defined by five fields that are separated by semicolons.

First Row - VendorDeviceID

The device name of the targeted drive. This entry must contain the VendorDeviceID (part of the drive identification parameters). For example, for an ATV340S the first row must contain ATV340S.

Second Row - Header

The second line of the Index File names these fields:

| Name of field | Description |
| --- | --- |
| Name | Parameter name or code, for example ACC or HSP for ATV340S |
| LogicAddress | Logical Address of the parameter |
| IDN | Identification number (IDN) based on the [Sercos III notification](../../../../../../api/crossBook?lang=en-US&virtualBookName=SerComLi&topicID=D_SE_0055484_1). |
| LenBytes | Length of the parameter in bytes. |
| DataType | Data type of the parameter, for example UINT16, INT32 |

Third Row - Save to Nonvolatile Memory

The third line of the Index File must contain the request that stores the parameters to the nonvolatile memory.

Example:

CMI;8504;P-0-2055.0.4;2;WORD (BitString16)

NOTE: Parameters which are changed via fieldbus are not automatically saved to the nonvolatile memory.

Fourth Row and Following

The fourth and following lines of the Index File contain the parameters of the drive which shall be stored in the Parameter File in the controller.
