# ST\_AccessRule

## Overview

|  |  |
| --- | --- |
| Type: | Structure |
| Available as of: | V1.1.0.0 |
| Extends: | - |

## Description

The structure ST\_AccessRule specifies the access rule which is used by the Modbus TCP server to control the access from external Modbus TCP clients.

## Structure Elements

| Name | Data type | Default value | Description |
| --- | --- | --- | --- |
| sIpAddressFirst | STRING[15] | NULL | Specifies the first client IP address the access rule is applied to. |
| sIpAddressLast | STRING[15] | NULL | If an IP address range is defined, this element specifies the last client IP address the access rule is applied to.  To apply the rule to a single IP address, leave this parameter empty or set it equal to the parameter sIpAddressFirst. |
| xAllowAccessToAllBlocks | BOOL | FALSE | If this element is TRUE, the clients defined by the IP address filter have access to all addresses of the memory blocks (discrete inputs, coils, input registers, holding registers).  If this element is TRUE, the following parameters for starting address, length, and offset are ignored. |
| uiDiscreteInputsStartAddress | UINT | 1 | This element determines the first address of the discrete inputs which the client is allowed to access. |
| uiDiscreteInputsAddressOffset | UINT | 0 | This element determines an offset to the address requested by the client.  Example: The client requests data from address 1 and the offset is set to 10, then the request is processed at address 11. |
| uiDiscreteInputsLength | UINT | 0 | This element determines the number of discrete inputs starting from the start address plus the offset that the client is allowed to access. |
| uiCoilsStartAddress | UINT | 1 | This element determines the first address of the coils which the client is allowed to access. |
| uiCoilsAddressOffset | UINT | 0 | This element determines an offset to the address requested by the client.  Example: The client requests data from address 1 and the offset is set to 10 then the request is processed at address 11. |
| uiCoilsLength | UINT | 0 | This element determines the number of coils starting from the start address plus the offset that the client is allowed to access. |
| uiInputRegistersStartAddress | UINT | 1 | This element determines the first address of the input registers which the client is allowed to access. |
| uiInputRegistersAddressOffset | UINT | 0 | This element determines an offset to the address requested by the client.  Example: The client requests data from address 1 and the offset is set to 10 then the request is processed at address 11. |
| uiInputRegistersLength | UINT | 0 | This element determines the number of input registers starting from the start address plus the offset that the client is allowed to access. |
| uiHoldingRegistersStartAddress | UINT | 1 | This element determines the first address of the holding registers which the client is allowed to access. |
| uiHoldingRegistersAddressOffset | UINT | 0 | This element determines an offset to the address requested by the client.  Example: The client requests data from address 1 and the offset is set to 10 then the request is processed at address 11. |
| uiHoldingRegistersLength | UINT | 0 | This element determines the number of holding registers starting from the start address plus the offset that the client is allowed to access. |
