# FB\_MotorDataWrite2 - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Function block |
| Available as of: | V1.7.4.0 |

## Task

This function block writes the motor data in the drive or to the motor encoder, depending on the selected i\_etStorageLocation. Also refer to the chapter [Asynchronous Motor](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.3rdPartyMotors&topicID=D_SE_0088217).

NOTE: Execute this POU only when required for application needs. The encoder memory area has a limited number of write cycles.

NOTE: Incorrect machine encoder typeplate data has a direct influence on the behavior of the controller.

| DANGER | |
| --- | --- |
|  | UNINTENDED OPERATION OF THE MOTOR  * Verify assignment of machine encoder data to the motor before writing. * Ensure that no one is in the operating zone during the startup.  Failure to follow these instructions will result in death or serious injury. |

NOTE: A deactivated monitoring function can lead to a possible undetected overheating of the motor.

| DANGER | |
| --- | --- |
|  | OVERHEATING OF THE MOTOR  * Verify assignment of motor data to the motor before writing. * Use appropriate methods to verify thermal overload of the motor.  Failure to follow these instructions will result in death or serious injury. |

## Requirements

* Sercos must be in phase 2.
* No motor data must be stored in the selected drive (use FB\_MotorDataDelete2, if necessary).
* The *[MotorIdentification](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM62Drive&topicID=D_SE_0071763)* parameter of the axis must be set to the correct value.
* WorkingState of drive must be physical.

## Description

This POU receives a file with motor data and writes it into the selected axis. The file must be created beforehand with FC\_MotorDataFileCreate2.

The motor data is stored to the selected storage location.

* In the memory of the encoder, if sufficient memory is available in the encoder.
* In the servo amplifier, if the encoder has no or insufficient memory or if no encoder is used.

The motor data is permanently stored. Thus, the motor data must be written only once.

NOTE: When replacing drives or motors, make sure that the correct motor data is stored in the drive or motor. In all likelihood, the motor data must be rewritten.

| DANGER | |
| --- | --- |
|  | UNINTENDED OPERATION OF THE MOTOR  Ensure that the typeplate of the new drive or motor is consistent with the typeplate of replaced drive or motor.  Failure to follow these instructions will result in death or serious injury. |

If motor is to be modified (for example, because the motor was replaced), you must first delete the existing motor data with FB\_MotorDataDelete2.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_xExecute | BOOL | Upon a rising edge the write process is started. |
| i\_ifDrive | CMI.IF\_AxisIdentification | Drive to which the motor data is written. |
| i\_sFilename | STRING[255] | File path to the binary motor data file located on the file system of the controller. |
| i\_etStorageLocation | [ET\_StorageLocation](D-SE-0090526.html#D-SE-0090526) | Storage location to which the motor data is to be written. Default value: drive.  The *[MotorIdentification](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM62Drive&topicID=D_SE_0071763)* parameter of the axis has to be set according to the selected storage location.   * ET\_StorageLocation.Drive => Motor without typeplate / 2 * ET\_StorageLocation.Encoder => Motor with typeplate / 0 |

| Output | Data type | Description |
| --- | --- | --- |
| q\_xBusy | BOOL | Indicates that the process to write the motor data is in progress. |
| q\_xDone | BOOL | Indicates that the motor data was written successfully. |
| q\_xError | BOOL | Indicates that an error is detected during the process to write the motor data. |
| q\_etDiag | [GD.ET\_Diag](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.GlobalDiagnostic&topicID=D_SE_0076228) | General library-independent statement on the diagnostic.  A value unequal to GD.ET\_Diag.Ok corresponds to a diagnostic message. |
| q\_etDiagExt | [ET\_DiagExt](D-SE-0090520.html#D-SE-0090520) | POU-specific output on the diagnostic.  q\_etDiag = GD.ET\_Diag.Ok -> status message  q\_etDiag <> GD.ET\_Diag.Ok -> diagnostic message |
| q\_sMsg | STRING[80] | Event-triggered message which gives more detailed information on the diagnostic state. |

## DriveInvalid

| Cause | Solution |
| --- | --- |
| The drive does not support the required interfaces. | Use the function block only with supported drives.  Refer to [Presentation of the library – General Information](D-SE-0090554.html). |

## FileAccessNotPossible

| Cause | Solution |
| --- | --- |
| The specified file does not exist. | Verify if the file exists on the controller. |
| The content of the file is not readable. | Only files created by POUs out of this library are supported. |

## FunctionNotSupportedByThisDevice

| Cause | Solution |
| --- | --- |
| The WorkingMode of the drive is not physical. | Verify that the WorkingMode of the drive is set to physical. |

## InvalidDatafield

| Cause | Solution |
| --- | --- |
| The entry for the data field size is greater than 128. | Only files created by POUs out of this library are supported. |

## InvalidFilename

| Cause | Solution |
| --- | --- |
| The parameter i\_sFilename is a null string. | Assign the correct filename to i\_sFilename. |

## InvalidStorageLocation

| Cause | Solution |
| --- | --- |
| An invalid value is assigned at the input i\_etStorageLocation. | Assign a valid value to i\_etStorageLocation. |

## MotorDataAlreadyStoredInDrive

| Cause | Solution |
| --- | --- |
| There are already motor data stored in drive. | Before writing new motor data, delete the existing motor data using FB\_MotorDataDelete2. |

## SercosNotInPhaseTwo

| Cause | Solution |
| --- | --- |
| Sercos is not in phase 2. | Verify that the Sercos is in state 2. |

## SercosCommunicationNotPossible

| Cause | Solution |
| --- | --- |
| The function block detects an error during communication with the drive over the Sercos service channel. | Verify the Sercos connection to the drive and that the drive is operational. |

## UnknownState

| Cause | Solution |
| --- | --- |
| An error occurred during internal processing. | Contact your Schneider Electric service representative. |
