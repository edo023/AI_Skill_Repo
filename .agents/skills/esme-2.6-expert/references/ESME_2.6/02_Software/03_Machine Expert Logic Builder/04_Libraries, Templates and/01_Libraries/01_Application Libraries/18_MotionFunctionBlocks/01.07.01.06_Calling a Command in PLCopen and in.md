# Calling a Command in PLCopen and in FB\_AxisMotion

## Description

| PLCopen | FB\_AxisMotion |
| --- | --- |
| * Only with a rising edge on the input from FALSE to TRUE, a command is executed. * Before executing a new command, the input Execute must be set to FALSE. | * The methods provided from the function block FB\_AxisMotion have no input Execute. * Setting and resetting the input Execute is not necessary. The move command is executed by calling the method. |
| Example MC\_MoveAbsolute: | Example MoveAbsolute: |
| Call example for two MC\_MoveAbsolute commands:   ``` MC_MoveAbsolute(Execute := true,…); MC_MoveAbsolute(Execute := false,…); MC_MoveAbsolute(Execute := true,…); MC_MoveAbsolute(Execute := false,…); ``` | Call example for two MoveAbsolute commands:   ``` fbAxisMotion.ifCmd.MoveAbsolute(..); fbAxisMotion.ifCmd.MoveAbsolute(..); ``` |

## Example for Calling a Command from FB\_AxisMotion

Call the command only once in your state machine when going into the state:

```
20: // start master
  IF xFirstCycleStateEntry THEN
     xFirstCycleStateEntry :=  FALSE;

     fbAxisMotionMaster.ifCmd.MoveVelocity(
        i_etBufferMode  :=  PLCO.MC_Buffer_Mode.Aborting,
        i_lrVelocity    :=  25,
        i_lrAcceleration:=  1000,
        i_lrDeceleration:=  1000,
        i_lrJerk        :=  10000,
        i_etDirection   :=   PLCO.MC_Direction.PositiveDirection,
 );
END_IF
```

As soon as the condition to jump into the next state is fulfilled, set the Boolean for next state and jump into it.

```
//Check if Master has reached its velocity
IF fbAxisMotionMaster.ifCmd.stGetRecentCmdState.xContinuous THEN
   xFirstCycleStateEntry :=  TRUE;
   uiState :=  50;
END_IF
```
