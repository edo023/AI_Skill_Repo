Task

Get the complete path description for further analysis via [FB\_PathForJerkPositioning](../Function_Blocks_I_to_Q/Function_Blocks_I_to_Q-19.htm#XREF_D_SE_0087320_1) and its methods.

Description

A motion positioning operation can be defined by means of [FB\_RgPosStartOptimised](Function_Blocks_R_to_Z-8.htm#XREF_D_SE_0087349_1) (motion operation defined and calculated based on velocity, acceleration and travel distance) or [FB\_PathForJerkPositioning](../Function_Blocks_I_to_Q/Function_Blocks_I_to_Q-19.htm#XREF_D_SE_0087320_1) (motion operation defined and calculated based on jerk phases and their duration times).

Both POUs deliver different results concerning point of view:

o[FB\_RgPosStartOptimised](Function_Blocks_R_to_Z-8.htm#XREF_D_SE_0087349_1) delivers jerk phases and their duration times.

o[FB\_PathForJerkPositioning](../Function_Blocks_I_to_Q/Function_Blocks_I_to_Q-19.htm#XREF_D_SE_0087320_1) delivers position, velocity and acceleration at a specific point in time of the motion positioning movement.

With the method [GetPathDescription](#XREF_D_SE_0087354_1) it is possible to convert the motion instruction defined by FB\_RgPosStartOptimised into a function block which can be handled by analysis methods of FB\_PathForJerkPositioning.

This way the same motion positioning movement can be analyzed from different points of view.

Before GetPathDescription can be executed, the motion operation has to be defined with the method [InitialiseFB](Function_Blocks_R_to_Z-15.htm#XREF_D_SE_0087356_1) and calculated with the method [CalculateRequiredMotion](Function_Blocks_R_to_Z-9.htm#XREF_D_SE_0087350_1) (see Usage section steps 1, 2 and 5 in description of [FB\_RgPosStartOptimised](Function_Blocks_R_to_Z-8.htm#XREF_D_SE_0087349_1)).
