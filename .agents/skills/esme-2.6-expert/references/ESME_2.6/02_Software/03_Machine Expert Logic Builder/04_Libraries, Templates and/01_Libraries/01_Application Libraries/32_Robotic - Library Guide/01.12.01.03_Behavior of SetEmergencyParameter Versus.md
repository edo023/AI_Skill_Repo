# Behavior of SetEmergencyParameter Versus SetEmergencyParameter2

## General

Setting the emergency parameters with the method SetEmergencyParameter is equivalent to SetEmergencyParameter2 with i\_etComponent = ET\_RobotComponent.All.

## Emergency Parameters Are Set by the Method SetEmergencyParameter

In case FB\_Robot.xEnable changes from TRUE to FALSE and one of the following conditions is active

* a path movement (with synchronous auxiliary axis movement), or
* an asynchronous auxiliary axis movement, or
* a path jogging movement, or
* a cartesian jogging movement, or
* an auxiliary axis jogging movement

then the movements are stopped with the **same** parameterized emergency stop parameters for the robot components (ET\_RobotComponent.All).

## Emergency Parameters Are Set by the Method SetEmergencyParameter2

In case FB\_Robot.xEnable changes from TRUE to FALSE and one of the following conditions is active

* a path movement (with synchronous auxiliary axis movement), or
* an asynchronous auxiliary axis movement, or
* a path jogging movement, or
* a cartesian jogging movement, or
* an auxiliary axis jogging movement

then the movements are stopped with the **specific** parameterized emergency stop parameters of each robot component.

## Impact on Behavior of Configuration Method ConfigDone

If the configuration method SetEmergencyParameter2 is used to configure the emergency parameters of the robot components, the configuration method ConfigDone (to verify the necessary robot configurations) can only be called successfully if the emergency parameters are configured for all robot components (i\_etComponent = ET\_RobotComponent.All).
