## Task

Reading the physical IDs of Smart Carriers.

## Description

With the method StartReadingCarriersPhysicalIds, you can start the process of reading the physical IDs of the Smart Carriers on a Lexium™ MC multi carrier track.

**Preconditions for the process:**

* The outputs q\_xActive and q\_xReady of the FB\_SmartCarrierUtility are TRUE.
* The instance of the function block does not execute another process.
* The multi carrier objects for the segments and the carriers (Lexium MC Segment and Lexium MC Carrier) must be in WorkingMode real / 1.
* The parameter State of the multi carrier segment objects (Lexium MC Segment) must have the value Enabled / 3.
* The multi carrier carrier objects (Lexium MC Carrier) must not be enabled.

  

For more information on multi carrier objects and on the parameters WorkingMode and State, refer to the [Lexium™ MC multi carrier Device Objects and Parameters Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=).

**Reading process:**

By calling the method StartReadingCarriersPhysicalIds, you start the reading process that runs without further user action. You can verify the status of the process through the property etState (see [FB\_SmartCarrierUtility](SmartCarrierUtility-B8A36EDB.html#SmartCarrierUtility-B8A36EDB)).

The reading process asynchronously calls the function FC\_ReadCarriersPhysicalIds from the SystemInterface library. For more information on the function FC\_ReadCarriersPhysicalIds, refer to the [SystemInterface library](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.SystemInterface&topicID=ReadCarriersPhysicalId_Gen_420EB61F).

  

NOTE: Do not modify the state of the Lexium MC Carrier objects during the reading process.

NOTE: If at the end of the reading process, the status message ReadCarriersPhysicalIdsSucessful is not displayed, you must repeat the process.

NOTE: After the reading process, the physical IDs of the Smart Carriers are available in the user function CarrierIdentification.  

For more information on the user function CarrierIdentification, refer to the [Lexium™ MC multi carrier Device Objects and Parameters Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=CarrIdent_9D6C0CE9).
