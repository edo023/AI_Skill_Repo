# IF\_LoggerPointConfiguration – Register (Method)

## Overview

|  |  |
| --- | --- |
| Type: | Method |
| Available as of: | V1.3.1.0 |
| Versions: | Installed version |

## Task

Registers a logger point to the Application Logger.

## Description

The method registers a logger point representing the function block to the Application Logger. The logger point must be configured with a name and is added to a specified parent. Unless modified, the logger point inherits the log level of the parent. If required, the logger level can be modified after registration of the logger point using the property etLogLevel.

NOTE: As a prerequisite for using the Register() method, the Application Logger plug-in must be added to the project and the Application Logger communication service must be registered.

For more information refer to the [ApplicationLogger Library Guide – Using the Library](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.ApplicationLogger&topicID=D_SE_0075331).

## Inputs

| Input | Data type | Description |
| --- | --- | --- |
| i\_ifParent | APL2. IF\_LoggerPoint | The value of the input specifies the parent logger under which the logger point of the function block must be registered. |
| i\_sName | STRING(80) | The name of the logger point shown in the Application Logger must be declared here. |

## Outputs

| Output | Data type | Description |
| --- | --- | --- |
| q\_xError | BOOL | The output is set to TRUE if an error has been detected during the execution. |
| q\_etResult | APL2.ET\_Result | POU-specific output on the diagnostic; q\_xError = FALSE -> Status message; q\_xError = TRUE -> Diagnostic message. |
| q\_sResultMsg | STRING[255] | Event-triggered message that gives additional information on the diagnostic state. |

## Troubleshooting

This table describes the possible issues and their solutions:

| Issue  (error information q\_xError = TRUE) | Cause | Solution |
| --- | --- | --- |
| q\_etResult = LoggerPointAlreadyRegistered | Logger point is already registered | Do not call this method again after the logger point is successfully registered. |
| q\_etResult = LoggerPointInvalid | The value at i\_ifParent is invalid. | Assign an interface of type APL\_Base.IF\_LoggerPoint to the input i\_ifParent. |
| An internal error was detected. | Execute the function block again. If the issue persists, contact your Schneider Electric support representative. |
| q\_etResult = ParentNotRegistered | The logger point defined as parent is not registered. | Register the logger point assigned to i\_ifParent at the global ApplicationLogger. |
| q\_etResult = ServiceNotRegistered | The communication service is not registered. | Register the communication service for the global ApplicationLogger using the APL2.G\_ifApplicationLogger.RegisterCommunicationService() method. |
| q\_etResult = MaxNumberOfLoggerPointsExeeded | The maximum number of supported logger points is already registered. | Increase the library parameter APL\_Base.GPL.Gc\_udiMaxNumberOfLoggerPoints for maximum number of supported logger points. |
