# Teaching a Fixed System

## Overview

The FB\_TeachingFixedSystem library function block handles the teaching a system based on a grid of reference points.

NOTE:

* The grid must have a minimum of six points; each set must contain a minimum of two points.
* It is important that the X direction of the grid is aligned to the direction of the X axis of the orientation to teach.
* If the reference grid is printed on a sheet of paper, the alignment of the grid to the X and Y axes of the target system to teach influences the result of the estimation.

## Procedure for Using the Provided Teaching Grid

| Step | Action |
| --- | --- |
| 1 | Start by printing the provided teaching grid. In this example the [**Reference Grid**](QualityOfTheEstimatedOrientationOrP-E8FA8E39.html#QualityOfTheEstimatedOrientationOrP-E8FA8E39__ExampleOfReferenceGrid-E8FAF666). |
| 2 | Place the printed teaching grid on the physical coordinate system to teach, ensuring that the X and Y axes drawn on the grid are aligned as much as possible with the equivalent axes of the system to teach. Any misalignment between the teaching grid orientation and the coordinate system to teach influences the estimated orientation.  NOTE: Ensure that the teaching grid cannot move with respect to the surface of the system on which it is placed. |
| 3 | Define an instance of the teaching function block:  ``` fbTeachingProcedure : FB_TeachingFixedSystem; ``` |
| 4 | Perform the base configuration of the teaching procedure:   ``` fbTeachingProcedure.Configuration(     i_stReferenceSystemBasePose := stBasePose,     i_xIsRobotReferenceSystem := TRUE, //the robot is the reference "known" system for this teaching     i_ifRobotFeedback := stRoboticModule.iq_ifFeedback,     i_ifRobotConfigurationReadOnly := := stRoboticModule.iq_ifRobotConfigurationReadOnly,     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ```   stBasePose represents the global pose of the robot or the other system involved in the teaching, depending on the value of i\_xIsRobotReferenceSystem. If a SERT.FB\_EntitiesHandler is used to define the layout of the robot cell, then this pose can be retrieved from the function block and it is equal to the base pose of the entity (for example: a robot, a conveyor, and so on). The Feedback and Configuration Read Only interfaces can be retrieved from a robot module interface of from a robot function block. |
| 5 | Create a variable of type ST\_TeachingFixedSystemData and fill it with the required information.  For example:  stProcedureData.stOriginPosition.lrX := 50.0;  stProcedureData.stOriginPosition.lrY := 0.0;  stProcedureData.stOriginPosition.lrZ := 0.0;  stProcedureData.lrXSpacing := 100.0;  stProcedureData.lrYSpacing := 100.0;  stProcedureData.udiNumberOfPointsAlongY := 2;  You can measure a proper value for lrXSpacing and lrYSpacing directly on the printout of the reference grid. lrXSpacing is the distance between the first point of two adjacent sets. lrYSpacing is the distance between two adjacent points within the same set.  Then provide a structure to the procedure:   ``` fbTeachingProcedure.SetProcedureData(     i_stData := stProcedureData,     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ``` |
| 6  (Optional) | The teaching requires touching points with the robot on the surface of the other system to teach. This procedure is performed with a tool mounted on the robot. The cartesian pose transformation introduced by a tool can be described with ROB.IF\_RobotConfiguration.ConfigureTool. Then the tool can be selected with the method ROB.IF\_RobotMotion.SetTool. Once a tool is set, the feedback ROB.IF\_RobotFeedback.rstRefPositionTCP accounts for that.  If a rotation of the tool could influence the TCP position, the orientation must be considered, and it is then required to configure the robot so that the orientation is properly handled (see also ROB.IF\_RobotConfiguration.MapComponentToDrive). |
| 7 | Move the robot to a position where it is aligned to the circle marked with the number 1 inside Set 1, as displayed in the graphic. |
| 8 | Call the method AddSample to store a new set of data:   ``` fbTeachingProcedure.AddSample(     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ```   NOTE: After calling AddSample a first time for the selected set, the value of udiNumberOfSamplesInActiveSet increases by one. |
| 9 | Move the robot to a position where it is aligned to the circle marked with the number 2 inside Set 1, as displayed in the graphic. |
| 10 | Call the method AddSample to store a new set of data:   ``` fbTeachingProcedure.AddSample(     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ```   NOTE: After calling AddSample a second time for the selected set, the set is completed and the value of udiNumberOfCompleteSets increases by one. |
| 11 | Move the robot to a position where it is aligned to the circle marked with the number 1 inside Set 2, as displayed in the graphic. |
| 12 | Call the method AddSample to store a new set of data:   ``` fbTeachingProcedure.AddSample(     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ``` |
| 13 | Move the robot to a position where it is aligned to the circle marked with the number 2 inside Set 2, as displayed in the graphic. |
| 14 | Call the method AddSample to store a new set of data:   ``` fbTeachingProcedure.AddSample(     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ``` |
| 15 | Move the robot to a position where it is aligned to the circle marked with the number 1 inside Set 3, as displayed in the graphic. |
| 16 | Call the method AddSample to store a new set of data:   ``` fbTeachingProcedure.AddSample(     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ``` |
| 17 | Move the robot to a position where it is aligned to the circle marked with the number 2 inside Set 3, as displayed in the graphic. |
| 18 | Call the method AddSample to store a new set of data:   ``` fbTeachingProcedure.AddSample(     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ``` |
| 19 | Call the method EstimateOrientation to estimate the orientation of the target coordinate system based on the acquired sets of samples:   ``` stEstimatedPose := fbTeachingProcedure.EstimateCartesianPose (     i_etOrientationConvention := ROB.ET_OrientationConvention.ZYX,     q_xError => xError,     q_etResult => etResult,     q_sResultMsg => sResultMsg ); ```   NOTE:  * stEstimatedOrientation is a structure of type GEM.ST\_CartesianPose that is used to represent a position and an orientation in the cartesian space. * If fbTeachingProcedure.xIsRobotReferenceSystem is TRUE, the returned value is the global pose of the other system involved in the teaching. If it is FALSE, the returned value is the global pose of the robot. |
