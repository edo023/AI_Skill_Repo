# Adding a Lexium™ MC12 multi carrier Track

## Overview

As indicated in the figure provided in the [Overview of the Application](OverviewApp-2C7B2314.html#OverviewApp-2C7B2314__EquipmentModuleLayer-2C7CE567), the SR\_MainMachine controls equipment modules that control the Actuator Layer.

In this example, each Lexium™ MC12 multi carrier track is controlled by an instance of FB\_Multicarrier contained in an SR\_MulticarrierModuleX commanded by the SR\_MainMachine.

## Procedure

To add a Lexium™ MC12 multi carrier track, proceed as follows:

| Step | Action |
| --- | --- |
| 1 | As described in [State 0](MCConfig-2CD81190.html#MCConfig-2CD81190__State0-31145B91), add track, segment and carrier device objects, update the global variable list GVL\_MulticarrierConfiguration and the global parameters of the Multicarrier library.  For more information, refer to the [Lexium™ MC multi carrier Device Objects and Parameters Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=CarrCtrlLoop_8EE5F492) and the [EcoStruxure Machine Expert Lexium™ MC multi carrier Configuration Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=MLSConfG&topicID=TPC_MLS_Config_Tab_Track_D99C4272). |
| 2 | In the folder TemplateFullProgrammingFramework > EquipmentModules > MulticarrierModules, copy and paste SR\_MulticarrierModule1 and rename it with the number of MulticarrierModule. |
| 3 | In the action ControlModule of the new SR\_MulticarrierModuleX program, comment the call to external program for the operation modes Auto, TestMulticarrier or AutoGenerated mode in the states 120, 350 and 750. |
| 4 | In SR\_MainMachine, increase the constant c\_udiNumberOfSubModules by 1 and add a constant c\_udiMultiCarrierX. |
| 5 | In SR\_MainMachine, declare a new structure stMulticarrierModuleX of the type ST\_MulticarrierModule. |
| 6 | In SR\_MainMachine, update the action Init\_CmdTables for the relevant commands `StopPrepare`, `Auto`, `StopAuto`, `TestMulticarrier`, `Auto Generated`, `TrackCalibration`. |
| 7 | In SR\_MainMachine and the folder EquipmentModules, perform the following operations:   1. Copy Init\_Multicarrier1, insert and rename it. 2. Replace inside the new action the constant variable c\_udiMultiCarrier1 by the constant declared in step 4. 3. Replace SR\_Multicarrier1 by the MulticarrierModule created in step 2. 4. Replace the string `Multicarrier 1` accordingly. |
| 8 | In SR\_MainMachine and the folder ExceptionHandling, update the action MulticarrierModulesFaults with the MulticarrierModule created in step 2. |
| 9 | In SR\_MainMachine and the folder Multicarrier, perform the following operations:   1. Copy the folder Multicarrier1 and paste it. 2. Rename the new folder and its methods accordingly. 3. Inside each action AssignTrackX / AssignSegmentsX / AssignCarriersX, assign the objects belonging to the track created in step 1 to the ST\_MulticarrierModule structure variable created in step 5. |
| 10 | In SR\_MainMachine and the folder Template\_Struct, perform the following operations in the Start\_Action:   * Modify the state 0 by copying and pasting the code for the first track. * Replace stMulticarrierModule1 by the structure variable created in step 5. * Assign the number of carriers on the track to the variable i\_udiNumberOfCarrierOnTrack of the instance of ST\_MulticarrierModule. * Replace the actions AssignTrack1 / AssignSegments1 / AssignCarriers1 by the ones created in step 9. * Replace MC\_Track\_1 by the track created in step 1. * Replace 1 by the corresponding index of your track in GVL\_MulticarrierConfiguration. |
| 11 | This step is optional.  In SR\_MainMachine and the folder Template\_Struct, perform the following operations:   1. Update the action Start\_Action. 2. Modify the state 30, and add extra states for reading and writing the control loop parameters based on the states 60, 70, 80. |
| 12 | In SR\_MainMachine and the folder Template\_Struct, perform the following operations:   1. Update the action Start\_Action. 2. Modify the state `200` by copying and pasting the line for the first MulticarrierModule and adapting it for the new one. 3. Update the state `210` by copying and pasting the line for the first MulticarrierModule and adapting it for the new one. |
| 13 | In SR\_MainMachine and the folder Template\_Struct, add a call to the new MulticarrierModule. Use the corresponding variables as inputs for the module. |
| 14 | In the folder MulticarrierStations > AutomaticMode, perform the following operations:   1. Create a new program to control the stations of the new track. 2. Call it in the MulticarrierModule created in step 2 in the action ControlModule state `120`. |
| 15 | You can create a visualization in the Vizualization folder of the AutomaticMode. Add it to the frame of the Vis\_Auto as a reference. |
| 16 | Go to TemplateFullProgrammingFramework > Visualization > Multicarrier.    Copy and paste VIS\_MulticarrierModule1 and rename it. Open it and replace all `SR_MulticarrierModule1` by the name of the MulticarrierModule created in step 2. |
| 17 | Open Vis\_Jogging, select the frame and add a new reference to FR\_Jogging, configure it with the variables of the MulticarrierModule created in step 2.    Open the folder MulticarrierTrackCalibration > Vis\_TrackCalibration. Select the frame and add a new reference to FR\_TrackCalibration, configure it with the variables of the MulticarrierModule created in step 2. |
