# General Information

## Description

The library PD\_MultiBelt, compatible with the PacDrive controllers, is a collection of technology functions for grouping products using parallel belts. The table below depicts an example for each of the most common use cases.

NOTE: For this library, qualified-access-only is set. Therefore, the POUs, data structures, enumerations, and constants have to be accessed using the namespace of the library.

|  |  |
| --- | --- |
| **Three belt grouping** | **Four belt infeed** |
|  |  |

You will find information about the application and parameterization of the library [here](D-SE-0077877.html#D-SE-0077877).

The key features of this library include:

* Controlling several parallel belts - tested for up to four belts
* Defining several stations for loading and unloading products - tested for up to three stations
* Defining up to 127 compartments per train
* Mounting several trains per belt - tested for up to two trains
* Filtering of sensor signals
* Performing warm start routines

This library has been designed to cover a general range of use cases. If you need further features for your application, contact your Schneider Electric representative to find an individual solution.

## General Considerations and Restrictions

Besides these general considerations and restrictions, some features are only available for certain station types. They are described at their respective position within this library guide.

* The servo axis type **SercDrive** is not supported.
* The program calling the MultiBelt functionalities must be executed in a cyclical task.
* In case two stations are located very close compared with the train length, the system may not move at all.
* The feature xStationLock is not available.
* In certain, rare scenarios, trains might move backwards and collide into one another. To avoid this, use the same value for ST\_Station.lrTrainsDistances in all stations.
