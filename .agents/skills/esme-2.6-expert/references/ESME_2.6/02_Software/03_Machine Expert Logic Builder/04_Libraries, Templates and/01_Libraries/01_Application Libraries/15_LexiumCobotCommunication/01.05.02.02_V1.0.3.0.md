## Enhancements

The following improvements are implemented to the function block [FB\_CobotControllerFeedback](FB_CobotControllerFeedback-GeneralI-27E4BC7A.html):

**Verification of the task cycle time**

The feedback data is provided by the Lexium Cobot controller at a frequency of 20 ms.

To avoid unintended behavior when retrieving the feedback data from the Lexium Cobot controller, the function block [FB\_CobotControllerFeedback](FB_CobotControllerFeedback-GeneralI-27E4BC7A.html) must be called in a task with a cycle time of less than or equal to 20 ms.

A verification of the task cycle time is implemented. If the configured task cycle time does not match the specification, a diagnostic message is issued.

**Improvement on required CPU load**

The feedback data is provided by the Lexium Cobot controller at a frequency of 20 ms.

An improvement of the required CPU load when processing the received feedback data from Lexium Cobot controller is implemented. Processing the feedback data received from the Lexium Cobot controller using the [FB\_CobotControllerFeedback](FB_CobotControllerFeedback-GeneralI-27E4BC7A.html) function block requires on average 40% less CPU load than in the previous version.

The following improvements are implemented to the methods of the function block FB\_ CobotController:

* The input parameter i\_stData.lrMass of the method FB\_CobotController - SetPayload can be set to the value 0.0.
* For the method [FB\_CobotController - JogStart](FB_CobotController-JogStartMethod-2651ADED.html) the diagnostic message ET\_Result.PositionRange is returned for i\_etMode = ET\_JogMode.Relative if at the input parameter i\_stPosition the value 0.0 is provided.
