# Instancing a Station

## Overview

To add stations to the operation mode Auto, execute the following steps:

| Step | Action |
| --- | --- |
| 1 | Create the station function block and the associated structure in the AutomaticMode folder. For an easier implementation, you can inherit the FB\_BaseStation from the Base Station folder. This step is optional as you can reuse the station function block and structure. |
| 2 | Declare the station function block and structure in the SR\_AutomaticTrackX program depending on the track you want to add your station. |
| 3 | Modify the following actions:   * CallAllStations  With this action, you can add the cyclic call of the additional stations and the parametrization action SetParameterStationX for the corresponding station, for example SetParameterStation1. * ResetAllTriggersAndStates  When leaving the operation mode Auto or if an error is detected, a cold start is necessary. This requires that the carriers must move to the first station and must be added to the first station. As a carrier can only be assigned to one station, all stations have to release the carriers assigned to them. * ErrorQuitStations  With this action, you call the error quit of the stations. * ReportError  With this action, you detect station errors and activate the exception handling. For more information, refer to [Exception Handling](ExceptHandl-2DA91519.html). * RegisterLoggerPoint  In order to read messages from your station in the Application Logger, the station must be registered to the Application Logger. In the example project, the registration mechanism is inherited from the FB\_CoreStation so that you only need to activate the registration. * SetParameterStation1  Copy and rename this action. Adapt the parameters of the action to your specific station. * AutoTrigger  This action sets time-dependent triggers for the different stations to move the carriers out of the station. For each station, an individual timer can be set.  NOTE: If the triggering of the station between the tracks must be synchronized, then the AutoTrigger must be handled by SR\_MainMachine in the AutoStep action. |
