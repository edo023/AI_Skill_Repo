# Behavior in the Operating Modes

![](images/logo.png)

Behavior in the Operating Modes

Command Processing

A successful executed command is displayed as follows:

q\_xCMDActive = FALSE  
q\_diCMDActive = last command  
q\_xCMDDone = TRUE  
iq\_diCMD = 0

Operating mode commands are not possible if an error reaction [TPL.ET\_Reaction](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.Template&topicID=D_SE_0078354_1).AsyncStop, [TPL.ET\_Reaction](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.Template&topicID=D_SE_0078354_1).SyncStopEL or [TPL.ET\_Reaction](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.Template&topicID=D_SE_0078354_1).SyncStopEH is pending.

You cannot enable the position control via the [DriveEnableOn](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) command if an error reaction [TPL.ET\_Reaction](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.Template&topicID=D_SE_0078354_1).AsyncStop, [TPL.ET\_Reaction](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.Template&topicID=D_SE_0078354_1).SyncStopEL is pending.

NoOp

No operation mode is selected. The module is in wait state.

The technology POU [PDL.FB\_DriveEnable](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.PacDriveLib&topicID=D_SE_0087260_1) is being activated.

Basic monitoring systems are enabled:

oMonitoring the diagnostic messages of the axis

oMonitoring the hardware limit switches

oMonitoring the reduced velocity

Possible state commands

o[DriveEnableOn](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) -> The position control of the axis is activated.

o[DriveEnableOff](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) -> The position control of the axis is disabled.

Possible operation mode commands

o[Homing](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) -> Change to operation mode [Homing](#XREF_D_SE_0088804_4) takes place.

o[Manual](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) -> Change to operation mode [Manual](#XREF_D_SE_0088804_5) takes place.

o[CamCs](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) -> Change to operation mode [MultiCam](#XREF_D_SE_0088804_6) takes place and a coldstart is preselected.

o[CamWs](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) -> Change to operation mode [MultiCam](#XREF_D_SE_0088804_6) takes place and a warmstart is preselected.

o[Endless](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) -> Change to operation mode [Endless](#XREF_D_SE_0088804_7) takes place.

o[Positioning](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) -> Change to operation mode [Positioning](#XREF_D_SE_0088804_8) takes place.

oMonitoring the reduced velocity

MultiCam

The operation mode MultiCam is selected. The axis is attached as slave to a master via cams.

The operation mode is parameterized in the structures [AXM.ST\_MultiCam](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077229_1) and [AXM.ST\_Main](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077223_1).

The technology POUs [PDL.FB\_DriveEnable](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.PacDriveLib&topicID=D_SE_0087260_1) and [PDL.FB\_MultiCam](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.PacDriveLib&topicID=D_SE_0087313_1) are being activated.

If the axis shall be activated in an ILS network, the pstIlsData parameter of the ST\_MultiCam structure must be linked to the (global) stIlsData structure, to which also the pstIlsData parameter of the ST\_EndlessFeed structure of the master module was linked. Subsequently, the technology POU [PDL.FB\_PreCalcMultiCam](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.PacDriveLib&topicID=D_SE_0087330_1) is called up first.

 If the operation mode was entered via the [CamWs](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) command, a warmstart is preselected. Subsequently, a WsStart command is waited for.

Positioning

The operation mode Positioning is selected. The axis is moved relatively with a certain distance or absolutely to a position.

The operation mode is parameterized in the structures [AXM.ST\_Positioning](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077231_1) and [AXM.ST\_Main](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077223_1).

The technology POUs [PDL.FB\_DriveEnable](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.PacDriveLib&topicID=D_SE_0087260_1) and [PDL.FB\_VarioPosJerk](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.PacDriveLib&topicID=D_SE_0087376_1) are being activated.

In the operation mode Positioning the [Start](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) command may be overwritten with the [Hold](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) command. The internal start bit may also be reset by the i\_xPosStop input of the [AXM.ST\_Main](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077223_1) structure, so that the axis starts to move again with the next start signal without having received a [Hold](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.AxisModule&topicID=D_SE_0077124_1) command.
