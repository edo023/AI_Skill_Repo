# ST\_RegisterDiagSource - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Data structure |
| Available as of: | 1.1.2.0 |
| Inherits from: | - |

## Description

Substructure for ST\_DiagnosticsManagerToDiagSource This is used to coordinate the communication of several diagnostic sources.

## Structure Elements

| Variable | Data type | Description |
| --- | --- | --- |
| xUsed | BOOL | TRUE as soon as the program cycle has been written to the register structure. A diagnostic source first must verify this register and may only write if it still returns FALSE. |
| udiNextId | UDINT | This ID must use a diagnostic source to register itself. The ID is required for all following communication. After the registration, a diagnostic source determines in each cycle whether it was sent a command by checking ST\_DiagnosticsManagerToDiagSource.i\_udiComWithId for the presence of the own ID. |
| sName | STRING(255) | A unique string ID for each connected diagnostic source for communication |
| sTypeName | STRING(255) | A string for describing and allocating the diagnostic source to a diagnostic type |
| xUnsubscribe | BOOL | Communication only permitted as long as the value is FALSE After completion of all communication processes, a diagnostic source must set this value to TRUE. |
| udiSourceId | UDINT | ID of the source |
