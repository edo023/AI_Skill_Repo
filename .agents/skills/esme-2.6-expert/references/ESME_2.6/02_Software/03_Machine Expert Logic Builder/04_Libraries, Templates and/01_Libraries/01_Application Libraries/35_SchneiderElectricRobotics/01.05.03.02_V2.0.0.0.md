# V2.0.0.0

## Overview

This library contains function blocks to parametrize a Schneider Electric robot.

## System Requirements

Using the library with other versions of software or firmware may have results other than those described in the present documentation.

| WARNING | |
| --- | --- |
|  | UNINTENDED EQUIPMENT OPERATION  * Ensure that the software and firmware are of the versions supported by this library. * Contact your Schneider Electric service representative for compatibility information.  Failure to follow these instructions can result in death, serious injury, or equipment damage. |

## Supported Hardware

* PacDrive LMC Eco
* PacDrive LMC Pro
* PacDrive LMC Pro2

## Software Requirements

* SoMachine Motion V4.3

## Firmware Requirements

PacDrive 3 V4.3

* PacDrive LMC Eco V1.54.2.1 or greater
* PacDrive LMC Pro V1.54.2.1 or greater
* PacDrive LMC Pro2 V1.54.2.1 or greater

## New Functions

* The Lexium T Robot has been added to the library.

## Modifications

* Drive parameter DriveStopCurrentLimit

  The drive parameter DriveStopCurrentLimit is watched by FB\_RobotPSeries and FB\_RobotTSeries.

  The value of the parameter must be always set to 0.0.
* Drive parameter UserDefinedStopMode

  The drive parameter UserDefinedStopMode is watched by FB\_RobotPSeries and FB\_RobotTSeries.

  The value of the parameter must be always set to 'Start with actual values / 0'.

## Enhancements

* FB\_RobotPSeries.SetKinematicParameter

  In case of i\_lrValue is less than 0.0, the output q\_etDiag reported GD.ET\_Diag.Ok.

  This behavior has been corrected to GD.ET\_Diag.InputParameterInvalid and is fully supported in the present version.
* FB\_RobotPSeries.GetKinematicParameter

  In case of i\_etName = ET\_KinematicParameter.GripperMass, the outputs q\_lrValue and q\_stCenterOfMass returned the incorrect values.

  This behavior has been corrected and is fully supported in the present version.
* IF\_RobotPSeriesCalibration.WriteEncoderRotationalAxis

  The calibration mode WriteEncoderRotationalAxis is fully supported in the present version.

  In case of a virtual drive, the diagnostic message q\_etDiag = GD.ET\_Diag.ExecutionAborted / q\_etDiagExt = ET\_DiagExt.VirtualDrives is returned. Writing the encoder position is not possible in case of virtual drives.
