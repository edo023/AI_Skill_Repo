# ET\_MultiAxisCustomJobState

## Overview

|  |  |
| --- | --- |
| Type: | Enumeration type |
| Data type: | UDINT |
| Available as of: | V1.0.0.0 |

## Description

The enumeration ET\_MultiAxisCustomJobState contains the values that indicate the current state of the function block FB\_MultiAxisCustomJob.

## Enumeration Elements

| Name | Value | Description |
| --- | --- | --- |
| Idle | 0 | The multi-axis custom job is not active. This is the initial state or the state reached after successfully calling the methods CleanGroup or ForceStop. |
| Linked | 1 | The Start method has been called and the system is preparing the synchronization of the axes. |
| Executing | 2 | The multi-axis custom job is active and in control of the axes. The Calculate method is being executed in each RTP cycle. |
| Steady | 3 | The multi-axis custom job has completed transient movements, such as acceleration, and is operating according to the synchronized custom algorithm. |
| Done | 4 | Execution completed successfully; the output q\_xDone  of the method Calculate  was set to TRUE. |
| Aborted | 5 | The execution of the multi-axis custom job has been interrupted and aborted by another motion command or function block. |
| Error | 6 | An error was detected during the start sequence or during the execution of the job. |
