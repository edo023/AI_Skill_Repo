# FB\_MotorDataDelete2 - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Function block |
| Available as of: | V1.7.4.0 |

## Task

This function block deletes the motor data in the selected storage location.

NOTE: Execute this POU only when required for application needs. The encoder memory area has a limited number of write cycles.

## Requirements

* Sercos must be in phase 2.
* The *[MotorIdentification](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM62Drive&topicID=D_SE_0071763)* parameter of the axis must be set to the correct value.
* WorkingState of drive must be physical.

## Description

After the deletion of the previous motor data, the new motor data can be written to the axis using FB\_MotorDataWrite2.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_xExecute | BOOL | Upon a rising edge the write process is started. |
| i\_ifDrive | CMI.IF\_AxisIdentification | Drive from which the motor data is deleted. |
| i\_etStorageLocation | [ET\_StorageLocation](D-SE-0090526.html#D-SE-0090526) | Storage location to which the motor data is to be written. Default value: drive.  The *[MotorIdentification](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM62Drive&topicID=D_SE_0071763)* parameter of the axis has to be set according to the selected storage location.   * ET\_StorageLocation.Drive => Motor without typeplate / 2 * ET\_StorageLocation.Encoder => Motor with typeplate / 0 |

| Output | Data type | Description |
| --- | --- | --- |
| q\_xBusy | BOOL | Indicates that the process to delete the motor data is in progress. |
| q\_xDone | BOOL | Indicates that the motor data was deleted successfully. |
| q\_xError | BOOL | Indicates that an error is detected during the process to delete the motor data. |
| q\_etDiag | [GD.ET\_Diag](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.GlobalDiagnostic&topicID=D_SE_0076228) | General library-independent statement on the diagnostic.  A value unequal to GD.ET\_Diag.Ok corresponds to a diagnostic message. |
| q\_etDiagExt | [ET\_DiagExt](D-SE-0090520.html#D-SE-0090520) | POU-specific output on the diagnostic.  q\_etDiag = GD.ET\_Diag.Ok -> status message  q\_etDiag <> GD.ET\_Diag.Ok -> diagnostic message |
| q\_sMsg | STRING[80] | Event-triggered message which gives more detailed information on the diagnostic state. |

## DriveInvalid

| Cause | Solution |
| --- | --- |
| The drive does not support the required interfaces. | Use the function block only with supported drives.  Refer to [Presentation of the library – General Information](D-SE-0090554.html). |

## FunctionNotSupportedByThisDevice

| Cause | Solution |
| --- | --- |
| The WorkingMode of the drive is not physical. | Verify that the WorkingMode of the drive is set to physical. |

## InvalidStorageLocation

| Cause | Solution |
| --- | --- |
| An invalid value is assigned at the input i\_etStorageLocation. | Assign a valid value to i\_etStorageLocation. |

## NoMotorDataFoundInDrive

| Cause | Solution |
| --- | --- |
| No motor data found in drive. | Verify that the drive has a readable motor typeplate stored. |

## SercosCommunicationNotPossible

| Cause | Solution |
| --- | --- |
| The function block detects an error during communication with the drive over the Sercos service channel. | Verify the Sercos connection to the drive and that the drive is operational. |

## SercosNotInPhaseTwo

| Cause | Solution |
| --- | --- |
| Sercos is not in phase 2. | Verify that the Sercos is in state 2. |

## UnknownState

| Cause | Solution |
| --- | --- |
| An error occurred during internal processing. | Contact your Schneider Electric service representative. |
