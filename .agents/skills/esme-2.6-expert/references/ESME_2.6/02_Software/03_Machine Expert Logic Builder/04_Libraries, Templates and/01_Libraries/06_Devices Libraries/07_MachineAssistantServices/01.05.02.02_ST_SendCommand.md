## Description

The structure ST\_SendCommand provides the single items, which are merged in the required command string.

## Structure Elements

| Name | Data type | Description |
| --- | --- | --- |
| i\_etCommandType | ET\_SendCommandType | Command to be executed. Refer to ET\_SendCommandType. |
| i\_sMacAddr | STRING[17] | MAC address of the device |
| i\_sExtension1 | STRING[30] | Additional command part, depending on the command used. See matrix below. |
| i\_sExtension2 | STRING[30] |
| i\_sExtension3 | STRING[30] |
| i\_sExtension4 | STRING[30] |

## Command Matrix for FB\_SendCommand

i\_stSendCommand (also refer to ET\_SendCommandType)

| i\_etCommandType | i\_sMacAddress | i\_sExtension1 | i\_sExtension2 | i\_sExtension3 | I\_sExtension4 |
| --- | --- | --- | --- | --- | --- |
| Locate | Mandatory | Not used | Not used | Not used | Not used |
| SetIPv4Config | Mandatory | IPv4 address | IPv4 mask | IPv4 gateway | Save |
| SetDhcp | Mandatory | DHCP name | Save | Not used | Not used |
| SetBootp | Mandatory | Save | Not used | Not used | Not used |
| SetDeviceName | Mandatory | Device name | Save | Not used | Not used |
| Backup | Mandatory | FTP username | FTP password | Not used | Not used |
| Restore | Mandatory | Backup name | FTP username | FTP password | Not used |

NOTE: The parameter Save can be either TRUE or FALSE. Other commands are not accepted (a diagnostic message is shown).

* TRUE = The new settings are saved in nonvolatile memory.
* FALSE = The new settings are only valid as long the device is not power cycled.
