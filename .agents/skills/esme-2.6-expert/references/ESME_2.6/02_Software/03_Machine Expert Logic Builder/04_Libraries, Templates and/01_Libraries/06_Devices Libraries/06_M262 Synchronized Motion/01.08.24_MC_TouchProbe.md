# MC\_TouchProbe

## Functional Description

This function block configures and starts position capture.

The function block returns the axis position when a trigger event occurs. The trigger parameters of the drive are provided by the device implementation. The [timestamp touch probe functionality](#D-SE-0086564__TimestampTouchProbeWithEdgeIOModule-F68740C0) allows you to use Modicon Edge I/O NTS modules as touch probe inputs.

Executing the function block MC\_AbortTrigger while MC\_TouchProbe is busy aborts the function block for the referenced trigger input.

The function block allows for one-time position capture; that is, only the first event after the rising edge at the input Execute is valid for recording. Any subsequent events are ignored.

The input PositionType which is an alias of the enumeration ET\_PositionType of the MotionInterface library allows you to specify the type of position to be captured. With the position type ActualPosition, the values of the outputs of the function blocks MC\_ReadActualPosition and MC\_ReadActualVelocity are used in the calculation of the captured position. If these function blocks do not provide values, for example, because the specified axis is a virtual axis or its working mode is Simulated, the values of RefPosition and RefVelocity are used.

With the position type ReferencePosition, the axis properties RefPosition and RefVelocity are used in the calculation of the captured position.

## Timestamp Touch Probe Functionality with Modicon Edge I/O NTS Modules

Before you can use the timestamp touch probe functionality with Modicon Edge I/O NTS modules, you need to configure the module in EcoStruxure Machine Expert.

Procedure:

| Step | Action |
| --- | --- |
| 1 | In the Devices tree, below the SercosMaster node, add a Modicon Edge I/O NTS bus coupler. |
| 2 | Under the Modicon Edge I/O NTS bus coupler, add a Modicon Edge I/O NTS module supporting the timestamp touch probe functionality (NTSEDM0822 or NTSEDT0822). |
| 3 | Double-click the NTSEDM0822 or NTSEDT0822 icon to display the Edge I/O NTSE editor. |
| 4 | Configure InputChannel\_01 and select the expert function Timestamp\_Input\_1. |
| 5 | Under the NTSEDM0822 or NTSEDT0822 module, add a touch probe Input device (TP\_01). |
| 6 | Double-click the touch probe input device icon TP\_01 to display the parameter editor of the touch probe input device.  In the parameter group ExpertFunctionInformation, the parameter FunctionName is automatically set to MotionTimestampInput and the parameter FunctionName to Timestamp\_Input\_1. |
| 7 | In the parameter group Cap, configure the touch probe input device TP\_01 as required for your application. For details, refer to the M660 Programming Guide and the DeviceIOFunctionsInterface Library Guide. |

After you have configured the touch probe input device, set ifTrigger to TP\_01.triggerCap to connect it to the function block.

The window mode for the timestamp touch probe functionality with Modicon Edge I/O NTS modules allows you to limit position capture to a position window between the inputs FirstPosition and last LastPosition. The value of FirstPosition must be less than the value of LastPosition for a linear axis. Otherwise, an error is detected (WindowPositionsInvalid). To enable the window mode, set WindowOnly to TRUE.

The window mode is only available for the timestamp touch probe functionality. If the window mode is enabled for an input other than the touch probe input object TP\_01 (for example, for LXM32S, LXM28S or M262 encoder input), and if the inputs FirstPosition and LastPosition are not 0, an error is detected (WindowModeNotSupported)

The parameters for the timestamp touch probe capture are only latched on a rising edge of the input Execute if this rising edge starts a new position capture. Modifications of the parameters are not taken into account after the rising edge on the Execute input.

If position capture is started with RisingEdge as the configured capture edge, this configuration can only be changed after aborting the running function block MC\_TouchProbe with MC\_AbortTrigger or after a position has been successfully captured (Done is TRUE).

Retriggering the input Execute if the function block is busy does not start a new position capture. The ongoing position capture remains active until it terminates successfully or is aborted.

If you start a new function block MC\_TouchProbe with a device assigned to ifTrigger which is being used for a different position capture, an error is detected (CaptureSourceAlreadyInUse).

If more than one signal edge of the same type is detected in the same cycle (for example, two rising edges), the timestamp of the first detected edge is used to calculate the captured axis position. This also applies if BothEdges is configured and both rising and falling edge are detected in the same cycle.

## Inputs

| Input | Data type | Description |
| --- | --- | --- |
| Axis | Axis\_Ref | Reference to the axis for which the function block is to be executed. |
| Execute | BOOL | Value range: FALSE, TRUE.  Default value: FALSE.  A rising edge of the input Execute starts the function block. The function block continues execution and the output Busy is set to TRUE.  The function block allows for one-time position capture; that is, only the first event after the rising edge at the input Execute is valid for recording. Any subsequent events are ignored. |
| ifTrigger | MC\_Trigger\_Ref | Edge to trigger position capture.  Refer to [MC\_Trigger\_Ref](D-SE-0094936.html#D-SE-0094936__D-SE-0094936.5) for a description. |
| WindowOnly | BOOL | Limits position capture to a position window.  Possible values:   * Value FALSE: Position capture not limited to position window * Value TRUE: Position capture limited to position window |
| FirstPosition | LREAL | Specifies the first position of the position window. |
| LastPosition | LREAL | Specifies the last position of the position window. |
| PositionType | MC\_PositionType | Type of position to be captured.  Refer to [MC\_PositionType](D-SE-0094936.html#D-SE-0094936__MC_PositionType-F6491CE8) for a description. |

| Output | Data type | Description |
| --- | --- | --- |

## Notes

Performance of the touch probe functionality for LXM32S drives:

* Twelve Sercos cycles are required for the first execution or reconfiguration of position capture, provided that there is no other use of the service channel.
* Nine Sercos cycles are required for the first, subsequent execution of position capture.

Example: With an LXM32S drive and a Sercos cycle time of 2 ms, the minimum time between two capture edges at a capture input (CAP1, CAP2 or CAP3) after the first execution of MC\_TouchProbe is 18 ms (9 \* 2 ms).
