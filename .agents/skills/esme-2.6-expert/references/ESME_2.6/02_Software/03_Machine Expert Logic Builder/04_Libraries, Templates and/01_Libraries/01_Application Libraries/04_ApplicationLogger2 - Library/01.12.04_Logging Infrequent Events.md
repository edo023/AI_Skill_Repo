# Logging Infrequent Events

## General

Sometimes it is necessary to search for an interruption that occurs only once a week or even less. For a better understanding of the interruption, it would be good to have a logging that shows what the application did right before the interruption occurred.

Because the Application Logger stores the logging in a ring buffer and many logger messages may be sent, only a small time period after the event remains. After this time period the logging messages in the ring buffer are overwritten.

In order to help prevent the overwriting of the ring buffer, the logging can be disabled with the method DisableLogger of the function block FB\_ApplicationLoggerHandler.

In order to use this trigger mechanism, the event that is of interest has to be detected in the project. This can be achieved by adding a POU of the type Program to the project that verifies the status of the project cyclically. The program works in parallel with the monitored project. The project sends the logger messages to the Application Logger as normal. Inside the additional program, an instance of the FB\_ApplicationLoggerHandler function block is implemented. As soon as the program detects the event, the method DisableLogger of the FB\_ApplicationLoggerHandler function block is called once.

If a time period after the event shall be logged additionally, the call of the method has to be delayed, for example by using a TON function block.
