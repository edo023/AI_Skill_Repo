# Executing Multiple Asynchronous Methods

![](images/logo.png)

Executing Multiple Asynchronous Methods

Overview

The parallel execution of multiple asynchronous methods can be achieved as a single task or multi task.

Single Task

The advantage of creating a single task is the simple implementation. The disadvantage of using a single task is that the jobs executing this method are put off into a waiting list. This means that if there are n call-ups, e.g. reading/writing the SERCOS service channel parameters (time for reading/writing the SERCOS service channel parameters = 10 ms), the time for writing the parameter is n \* 10 ms.

Multi Task

The advantage of creating a multiple tasks is that the asynchronous executions of the method cannot influence one another. The reading/writing process of a SERCOS service channel parameter can already be finished after 10 ms. The disadvantage of the multi task is that n tasks are not easy to handle. Especially creating n tasks each having an unambiguous combination of task name and task priority.

The async mechanism implemented in PacDrive supports up to 10 parallel tasks. This is specified in the global GC\_udiNumOfAsnycTask (0-10) parameter and activated by GC\_xEnableAsnycTasks.
