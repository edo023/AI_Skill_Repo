Best Practices

General

The following chapter may help you to avoid errors in your application.

To run the application, you must have administrative rights to copy CSV files from an external location to the controller file system and conversely.

Preconditions

oMake sure to have the correct syntax and the correct extension (some Windows tools add the extension \*.txt)

oThe values within a parameter file are integers (no enumeration, no floating point values). During download, the drive adapts them to the correct format. Consider this if you change values within a parameter file manually.

There is no consistency or limit verification of the read data. Each value is inserted into the Parameter File and thereafter written to the drive without a verification.

NOTE: A new parameter upload may be required.

oMake sure that the IDNs in the Index File have read and write access. Furthermore, each IDN must be accessible via the Sercos fieldbus.

oUp- or Downloading of parameters is only possible if the Sercos bus is in phase 4 and the drive is not enabled (AxisState = 1).

oSome drive parameters can only be written if one (or more) other parameter was previously restored. Consider this when creating the Index File.

oExample for ATV340S:

Parameter CD1: This parameter can only be accessed if parameter CHCF is set to Separate or I/O profile.

oDepending on the number of IDNs in the Index File, you must adapt GPL.Gc\_uiMaxNumber­OfRowsRead and/or GPL.Gc\_uiMaxExecutionTime.

Function Block Instances

Even if it is possible to create several instances of FB\_ParameterUpload and/or FB\_Parame­terDownload, it is suggested working with only one function block instance for each function.

You can execute several instanced function blocks in parallel (to handle diffrent drives at the same time). However, due to the asynchronous communication restrictions of Sercos, there is no significant performance improvement.

Executing several function blocks in parallel also requires independent CSV files due to the restricted file access. Refer to File Handling below.

File Handling

oConsider that an Index File and/or a Parameter File stored in the controller can only be access from one function block at the same time.

oIt is not possible to execute several instances of FB\_ParameterUpload and/or FB\_Parame­terDownload with the same file names as inputs in parallel.

oSuccessful execution of FB\_ParamterUpload creates a new Parameter File.

oInput string for an Index File and/or a Parameter File contains the file name (mandatory) and the file path (optional), for example CSV\ATV340S\_IndexFile.csv.

If the Index File and/or the Parameter File name contain the path, the path must already exist in the controller.

oOnly files with the extension .csv are supported. If the file extension is missing, .csv is added automatically.

oDepending on the size of the Index File (number of lines), you have to adapt the global parameter GPL.Gc\_uiMaxNumberOfRowsRead to provide enough memory for reading one or more CSV files.

oMake sure that there is enough memory in the controller to process the required (intended) files.

oEnsure that for FB\_ParameterUpload an Index File is located in the controller and in the appropriate folder.

oCreate the Index File with an external tool (conforming to the specification, for example Notepad++) and then copy the file in the controller file system.

oStore the relevant Index Files and Parameter Files to an external storage device.

NOTE: During an update of the controller firmware, the required Index Files and Parameter Files are deleted
