# FB\_TorqueFeedForwardConfig - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Function Block |
| Available as of: | V1.0.0.0 |
| Versions: | Installed version |

## Task

Configures the torque feed forward for an axis.

## Description

This POU configures the flexible torque feed forward for an axis. If the configuration is successfully completed, transfer this POU as an input parameter to the FB\_TorqueFeedForward POU.

The following prerequisites must be fulfilled to use the POU:

* Sercos must be in phase 4.
* The Sercos IDN P-0-1069.0.0 must be added to the AdditionalIDNListMDT parameter of the axis. If this parameter is changed, a new Sercos phase start-up (phase 0 => phase 4) has to be performed.

  **Example:** DRV\_Lexium62.AdditionalIDNListMDT := 'P-0-1069.0.0'

NOTE: This function block is only supported on LMC PacDrive controller.

## Disabled

The function block is disabled and executes no actions. i\_xEnable and q\_xActive are set to FALSE

## DriveDoesNotSupportThisFunction

| Issue | Cause | Solution |
| --- | --- | --- |
| - | The selected axis does not support this function. | Select an axis that supports this function (for example, Lexium LXM62 Drive, Lexium LXM52 Drive or Lexium ILM62 Drive Module). |

## Executing

The function block is performing the configuration. Waiting until q\_xDone has the value TRUE.

## ParameterNotConfigured

| Issue | Cause | Solution |
| --- | --- | --- |
| - | The Sercos IDN P-0-1069.0.0 has not been added to the AdditionalIDNListMDT parameter of the axis. | Add IDN P-0-1069.0.0 to the AdditionalIDNListMDT parameter of the axis. |
| - | After adding the parameter, no new Sercos phase start-up has been performed. | Perform new Sercos phase start-up (phase 0 => phase 4). |

## SercosCommunicationNotPossible

| Issue | Cause | Solution |
| --- | --- | --- |
| - | A Sercos communication error has been detected during the configuration. | Set the Sercos communication to phase 4. |

## TorqueFeedForwardConfigured

The function block has completed the configuration. It now can be transferred to the FB\_TorqueFeedForward POU.

## WaitForExecute

The POU has completed its initialization and is waiting for a positive edge at the i\_xExecute input before continuing processing.

## WaitForSercos

The function block is waiting for Sercos to be in phase 4.
