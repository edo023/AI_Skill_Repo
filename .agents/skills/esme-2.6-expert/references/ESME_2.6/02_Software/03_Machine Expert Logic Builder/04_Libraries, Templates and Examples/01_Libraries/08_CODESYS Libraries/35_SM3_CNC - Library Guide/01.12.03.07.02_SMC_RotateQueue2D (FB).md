# SMC\_RotateQueue2D (FB)

FUNCTION\_BLOCK SMC\_RotateQueue2D

This function block will rotate the path stored in poqDataIn around the z-axis by the angle given by dPhi [°]. A positive angle effects a positive rotation in mathematical sense (counterclockwise).

The input variable poqDataIn is a pointer to the structure [SMC\_OUTQUEUE](cAH0ifE6NckjZp5eF3Qpy-filyk_smc-outqueue.html#cah0ife6nckjzp5ef3qpy_filyk_smc_outqueue_smc_outqueue_fb) , which describes the path to get rotated. The initialization value FALSE of input variable bEnable will bar the rotation of the path. As soon as bEnable is set to TRUE all [SMC\_GEOINFO](cAH0ifE6NckjZp5eF3Qpy-filyk_smc-geoinfo.html#cah0ife6nckjzp5ef3qpy_filyk_smc_geoinfo_smc_geoinfo_struct) objects found in poqDataIn will be processed. When bEnable is reset to FALSE, the module will not execute any further modifications.

The initialization of input variable bReset with FALSE effects, that the [SMC\_GEOINFO](cAH0ifE6NckjZp5eF3Qpy-filyk_smc-geoinfo.html#cah0ife6nckjzp5ef3qpy_filyk_smc_geoinfo_smc_geoinfo_struct) objects currently found in poqDataIn will not get rotated, but only those that will be added.

As, during the processing, a sudden change of the characteristic parameter dPhi of the rotation might cause the path to be broken, modifications to the corresponding input will not be taken into account, until [SMC\_OUTQUEUE](cAH0ifE6NckjZp5eF3Qpy-filyk_smc-outqueue.html#cah0ife6nckjzp5ef3qpy_filyk_smc_outqueue_smc_outqueue_fb) will be empty at poqDataIn or until a Reset (bReset) will be done.

In order to effect a rotation within the (x,y)-plane around a center differing from (0,0), e.g. around the point (xP,yP), a translation by the vector (-xP,-yP,0) has to be carried out first , followed by the rotation by the desired angle dPhi and followed again by a retranslation with vector (xP,yP,0).
