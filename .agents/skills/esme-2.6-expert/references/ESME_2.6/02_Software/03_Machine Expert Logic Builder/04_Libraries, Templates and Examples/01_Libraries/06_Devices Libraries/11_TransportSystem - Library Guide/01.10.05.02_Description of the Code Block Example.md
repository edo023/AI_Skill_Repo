# Description of the Code Block Example

Description of the Code Block Example

oThis code block would affect the first detected vehicle to prepare a “MoveVehicleToPosition” command, using forward movement, a maximum acceleration and deceleration of 1.0 m/s2, maximum velocity of 1.0 m/s, to destination 0.5 m from the start of path ID 1.

oAt each program cycle, an internal parser verifies the elements for new commands which require a positive edge of the i\_xExecute input.

New commands are reframed to meet the requirements of the HLC for executing commands.

oAfter having received the response from the HLC, the outputs q\_etDiag and q\_etDiagExt of the affected elements (astPath, astVehicle) are updated to indicate whether the command has been accepted or rejected by the HLC

For each command that takes longer to complete, e.g. “MoveVehicleToPosition”, the HLC also sends an asynchronous response after an accepted command has been successfully completed.

The outputs q\_etDiag and q\_etDiagExt in the related elements are updated respectively.

oBefore executing a follow-up command the edge, i\_xExecute has to be FALSE for at least one program cycle so that it can be detected internally.

By sending a follow-up command, previous values in q\_etDiag and q\_etDiagExt will be overwritten respectively.
