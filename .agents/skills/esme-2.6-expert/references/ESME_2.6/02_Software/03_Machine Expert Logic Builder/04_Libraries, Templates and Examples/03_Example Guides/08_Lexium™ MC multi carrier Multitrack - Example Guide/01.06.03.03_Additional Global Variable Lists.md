# Additional Global Variable Lists

| Global Variable List | Description |
| --- | --- |
| GVL\_Project | Contains variable declarations for variables used to configure the Multicarrier. |
| GVL\_MulticarrierConfiguration | Contains variable declarations for variables that are generated by the Multicarrier Configuration editor.  For more information on the Multicarrier Configuration editor, refer to the [Lexium™ MC multi carrier Configuration Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=MLSConfG&topicID=TPC_MLS_Config_Add_Config_Tool_D92EE736). |
| GVL\_MulticarrierEmulation | Contains variable declarations for the interface with EcoStruxure Machine Expert Twin. |
| OPCUA\_Server\_Objects | Contains variable declarations for the data provided by the OPC UA server using the Symbol Configuration. |

| GVL\_Project Constant Variables | Description |
| --- | --- |
| Gc\_xVirtualMulticarrier | For running the Multicarrier as a virtual device, set the parameter Gc\_xVirtualMulticarrier to TRUE. |
| Gc\_xEnableCrashPrevention | For enabling the function block FB\_CrashPrevention, set the parameter Gc\_xEnableCrashPrevention to TRUE. |
| Gc\_xEnableCarrierControlLoop | For using the CarrierControlLoop parameters, set the parameter Gc\_xEnableCarrierControlLoop to TRUE.  The configuration of the control loop for the motion control of the carriers is usually performed per segment. If you need to configure the control loop settings per carrier (instead of per segment), you can enable the necessary parameters in the user function CarrierControlLoop. In this case, the control per segment through Sercos IDNs (IDentification Numbers) is deactivated.  The user function CarrierControlLoop is disabled by default. It must be enabled in the Feature Configuration tab of the corresponding Lexium MC Carrier objects.  For more information on the CarrierControlLoop user function, refer to the [Lexium™ MC multi carrier Device Objects and Parameters Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=CarrCtrlLoop_8EE5F492). |
| Gc\_xSmartCarrier | Set the parameter Gc\_xSmartCarrier to TRUE to indicate that you use a Smart Carrier on the track. |
| Gc\_udiSmartCarrierIndex | Use the parameter Gc\_udiSmartCarrierIndex to define the ConfiguredLogicalId of the Smart Carrier on the track. For more information on the ConfiguredLogicalId, refer to the [Lexium™ MC multi carrier Device Objects and Parameters Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=ConfigLogicId_9C6C7FF5).  The carriers are reordered during the operation mode Prepare. |
