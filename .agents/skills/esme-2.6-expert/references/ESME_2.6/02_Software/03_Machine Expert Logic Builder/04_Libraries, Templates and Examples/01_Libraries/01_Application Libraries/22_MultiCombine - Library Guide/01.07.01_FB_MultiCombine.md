# FB\_MultiCombine

## Overview

|  |  |
| --- | --- |
| Type: | Function block |
| Available as of: | 1.0.0.0 |

## Introduction

The function block FB\_MultiCombine is used to combine movements of several axes (channels) into a single subordinate axis. The function block calculates the sum of the incremental movements of the connected channels and applies the result to the subordinate axis.

## Setup and Initialization

The number of channels is defined during the declaration of the function block using a generic constant in angle brackets. This value determines the number of axes whose movements are to be combined. The maximum number of channels is 10. Example with three channels:

```
fbMultiComb : MLTCMB.FB_MultiCombine<3>;
```

The channels are provided as an array using the output Channels.

Each channel acts as a coordinate axis. You can execute motion jobs on these channels independently. However, these movements only affect the subordinate axis when the function block is active.

Configuration constraints:

* The subordinate axis must not be a feedback axis.
* The function block cannot start if a circular dependency (loop)  is detected between the subordinate axis and the channels, or within the channel configuration itself.

## Starting the Function Block

The combination process is initiated by a rising edge at the Execute input. Conditions for a start of the function block:

* The subordinate axis must be in the operating state Standstill.
* The connected channels must be in the operating state Standstill or Disabled.
* The function block must be started with the buffer mode set to Aborting.

Once the function block is active, the operating state of the subordinate axis transitions to SynchronizedMotion. From this point, any incremental movement on a channel is summed and applied to the subordinate axis.

If a channel is disabled while the function block is active, its target velocity is set to 0.

| WARNING | |
| --- | --- |
|  | UNINTENDED EQUIPMENT OPERATION  Verify that setting the target velocity to 0 cannot cause velocity jumps of the subordinate axis.  Failure to follow these instructions can result in death, serious injury, or equipment damage. |

To transition the operating state of a channel from Disabled to Standstill, set the Enable input of the MC\_Power instance for that channel to TRUE.

If the function block is re-executed while already active, the subordinate axis transitions to the operating state ErrorStop and an error is detected (AxisInErrorStop).

## Aborting and Restarting the Function Block

The function block can be aborted by any motion job that can interrupt a synchronized movement. When the function block is aborted, the subordinate axis is released from the combined movement.

## Channel Bundling

With channel bundling, you can synchronize a specific channel with the state of the subordinate axis. When bundling starts, the velocity and acceleration values of the subordinate axis are passed to a new motion job that is initiated by the method [StartChannelBundling](FB_MultiCombine-StartChannelBundlin-5BC0855A.html).

Bundling channels has the following impacts:

* A new motion job is initiated using the dynamics from the subordinate axis.
* The motion jobs running on other channels in the group are aborted. The operating state of the corresponding axes transitions to Standstill.
* If MC\_MoveSuperimposed is active on the subordinate axis or any channel when bundling is triggered, it is aborted.

A new motion job can be initiated on any channel in the Real-Time Process (RTP) cycle immediately following the one in which the bundling operation was triggered.

Supported PLCO function blocks for channel bundling:

* MC\_MoveRelative
* MC\_MoveAbsolute
* MC\_Halt
* MC\_MoveVelocity
* MC\_CamIn (RampIn)
* MC\_Stop
