# Migration From Other Libraries

## General

The MotionFunctionBlocks library contains functions that are similar to functions of the PD\_PacDriveLib library. An overview of the functions that can be replaced by functions from the MotionFunctionBlocks library can be found in the [EcoStruxure Machine Expert Compatibility and Migration - User Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=CompMigr&topicID=T001639217).

## Migrating Functions from PD\_PacDriveLib

Almost all motion functions of the PD\_PacDriveLib library are available in the MotionFunctionBlocks library. The functions have an updated interface. The new, corresponding names can be found in the [EcoStruxure Machine Expert Compatibility and Migration - User Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=CompMigr&topicID=T001639217).

Procedure for migrating from the PD\_PacDriveLib library to the MotionFunctionBlocks library:

1. Add the MotionFunctionBlocks library to the Library Manager.
2. Determine which function of the MotionFunctionBlocks library replaces the corresponding function from the PD\_PacDriveLib.
3. Replace the previous function name **PDL.FC\_xyz** by the corresponding function name **MFB.FC\_zyx**.
4. Adapt the function call (input names changed, diagnostic outputs changed).

   * If the input names of the function are different, identify the target input names according to the present document.
5. Adapt exception handling

   * The outputs etDiag and etDiagExt have been removed. Now, there are three outputs for error handling:

     + q\_xError indicates the status of the function.
     + q\_etResult provides status information when the function is operational and diagnostic information in the case of a detected error.
     + q\_sResultMsg provides additional information.
6. Remove the PD\_PacDriveLib library from the Library Manager.
