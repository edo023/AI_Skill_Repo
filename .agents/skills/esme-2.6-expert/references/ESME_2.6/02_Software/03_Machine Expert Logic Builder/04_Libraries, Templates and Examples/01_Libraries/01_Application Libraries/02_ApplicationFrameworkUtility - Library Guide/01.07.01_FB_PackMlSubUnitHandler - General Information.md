# FB\_PackMlSubUnitHandler - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Function block |
| Available as of: | V1.0.0.0 |

## Task

The function block FB\_PackMlSubUnitHandler creates PackML commands for multiple subunits, enabling the subunits to follow the state of the leading unit in which the function block is called.

For more information on PackML commands, refer to the [PackML Library Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=PackMLli&topicID=D_SE_0077921).

## Description

The function block FB\_PackMlSubUnitHandler creates PackML commands for multiple subunits so that the subunits follow the state of the leading unit in which the function block is called. In addition, the function block monitors the PackML state of the subunits and verifies whether the subunits are aligned to the state of the leading unit.

While monitoring the state of the subunits, the function block can manage different error reactions between the leading unit and the subunits; for example, if a subunit transitions from the Execute state to Aborted, but the leading unit transitions from Execute to Held.

In addition, the function block is able to coordinate the restart of subunits that are in different states than the leading unit. For example, if the leading unit is in the state Starting, Un-Holding or Un-Suspending to start the machine, the function block FB\_PackMlSubUnitHandler brings the subunits to the state Execute even if they are in Aborted, Stopped, Held or Completing states.

## Generic constants

The function block must be adapted to the number of subunits to be controlled. The number of subunits is specified in the definition of the function block instance with the `VAR_GENERIC CONSTANT` functionality. If the function block is to handle four subunits, the declaration of the function block must appear as follows:  
  
`fbInstanceName : SE_AFU.FB_PackMlSubUnitHandler<4>;`

| Name | Data type | Description |
| --- | --- | --- |
| c\_uiSubUnitCount | UINT | The number of subunits controlled by the instance of the function block FB\_PackMlSubUnitHandler. |

## Inputs

| Input | Data type | Description |
| --- | --- | --- |
| i\_xEnable | BOOL | TRUE: The function block monitors the state of the subunits and creates commands for the subunits according to the state of the leading unit. FALSE: The function block shows default values on its outputs. |
| i\_etState | [PackML.ET\_States](../../../../../api/crossBook?lang=en-US&virtualBookName=PackMLli&topicID=D_SE_0077937) | The state of the leading unit in which the function block is called. |
| i\_xMute | BOOL | TRUE: The function block does not create commands for the subunits. FALSE: FB\_PackMlSubUnitHandler starts its command chain again. This can be used to program command sequences differing from the standard operation of FB\_PackMlSubUnitHandler. For example, if in state Starting only some of the subunits shall receive a Start command and only if these subunits are in Executing state, the Start command is given to the other subunits. |
| i\_astSubUnitUpstreamData | ARRAY [1..c\_uiSubUnitCount] OF [ST\_SubUnitUpStream](ST_SubUnitUpStream-3AF20E49.html#ST_SubUnitUpStream-3AF20E49) | The parameters and data of the subunits. |

## Outputs

| Output | Data type | Description |
| --- | --- | --- |
| q\_xActive | BOOL | Indicates TRUE if the function block is active. As long as the output is TRUE, the function block must be executed cyclically. |
| q\_xReady | BOOL | Indicates that the function block generates commands and monitors the state of the subunits. |
| q\_xSubUnitsAligned | BOOL | TRUE: The state of the subunits is congruent with the state of the leading unit. FALSE: At least one subunit shows a state that is incongruent with the state of the leading unit. This signal can be used to signal an error in the leading unit. |
| q\_xActingStateConcluded | BOOL | If the leading unit is in a PackML acting state (for example, Starting or Stopping), this output indicates whether all subunits have reached their target state. If the leading unit is in a waiting state, the output is always FALSE. This signal can be used to trigger the PackML state Completed. |
| q\_astSubUnitDownstreamData | ARRAY [1..c\_uiSubUnitCount] OF [ST\_SubUnitDownStream](ST_SubUnitDownStream-3AF07B85.html#ST_SubUnitDownStream-3AF07B85) | Output data to transfer to each subunit and status data of each subunit. |
