# FB\_TorqueFeedForward2 - Configure (Method)

## Overview

|  |  |
| --- | --- |
| Type: | Method |
| Available as of: | V1.5.1.0 |
| Versions: | Installed version |

## Description

This method configures the function block for the torque feed forward control.

NOTE: After the method executes successfully, the function block does not re-read the following axis-specific parameters:Gear In, Gear Out, Movement Direction, Motor Torque Constant, Motor Inertia, J Load. To apply updated values for these parameters, reconfigure the function block.

**The following prerequisites must be fulfilled prior to configuration:**

* The Sercos parameter P‑0‑1069.0.0 must be included in the IDN mapping of the drive object.
* The ActualWorkingMode of the drive must be physical.
* The Sercos ActualState must be phase 4 (operational).

**Configuration Process**

* The Configure() method must be called until q\_xBusy = FALSE.
* The result of the configuration is indicated by q\_xDone = TRUE or q\_xError = TRUE.
* If q\_xDone = TRUE, the function block is configured and xIsConfigured = TRUE.
* Calling Configure() again restarts the configuration.

NOTE:

* The IDN mapping is configured either directly in the objects Device Editor (LMC / PacDrive) of the drive, or it must be set programmatically using the function FC\_ConfigureIdnMapping on the M660 Motion controller.
* After every Sercos phase ramp‑up (transition back to CP0 → CP4), the configuration must be repeated using Configure().
* Simultaneous execution of this function block with the legacy function blocks FB\_TorqueFeedForward and FB\_TorqueFeedForwardConfig is not permitted.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_ifAxis | CMI.If\_AxisIdentification | Axis for which the torque feed forward control is to be configured. |

| Output | Data type | Description |
| --- | --- | --- |
| q\_xBusy | BOOL | If this output is set to TRUE, the configuration is in progress. The method must be called as long as the output is set to FALSE. |
| q\_xDone | BOOL | If this output is set to TRUE, the configuration is completed successfully. |
| q\_xError | BOOL | If this output is set to TRUE, an error is detected.  For details, refer to q\_etResult and q\_etResultMsg. |
| q\_etResult | ET\_Result | Provides diagnostic and status information as a numeric value. |
| q\_etResultMsg | STRING[80] | Provides additional diagnostic and status information as a text message. |

## Troubleshooting

Possible diagnostic information (q\_xError = TRUE)

| Outputs of the method indicate the values | Cause | Solution |
| --- | --- | --- |
| q\_etResult = InvalidInterface | The given interface is invalid. | Assign a valid interface to the input i\_ifAxis. |
| q\_etResult = InvalidSercosState | The Sercos ActualState is not as required. | Set the Sercos ActualState to phase 4. |
| q\_etResult = DriveDoesNotSupportThisFunction | The selected axis does not support this function. | Select an axis that supports this function (for example, Lexium LXM62 Drive, Lexium LXM52 Drive or Lexium ILM62 Drive module). |
| q\_etResult = InvalidWorkingMode | The working mode of the assigned drive is invalid. | Set the working mode of the drive to Physical. |
| q\_etResult = RequiredIdnMappingNotDone | The required parameter IDN (P-0-1069.0.0) is not configured as additional IDN for cyclic data exchange. | Add the IDN P-0-1069.0.0 to the cyclic data exchange configuration. |
| q\_etResult = CreatingAsyncTaskFailed | An internal error was detected during initialization of the AsyncManager. | Execute the method again. If the issue persists, contact your Schneider Electric support representative. |
| q\_etResult = AsyncTaskTimedOut | Asynchronous job not completed within the timeout. | Execute the method again. If the issue persists, contact your Schneider Electric support representative. |
| q\_etResult = UnexpectedFeedback | An internal error has been detected. | Execute the method again. If the issue persists, contact your Schneider Electric support representative. |
