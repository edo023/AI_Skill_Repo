## New Features

* The method SetOrientationConvention was added to the interface IF\_RobotConfigurationAdvanced. With this method, the orientation convention in which move commands and robot feedback are interpreted can be set.
* The method SetReducedVelocity was added, to the interface IF\_RobotMotion. With this method, you can set a lower value for a part of the connected path.

## Modifications

* The default orientation convention was changed from XYZ to ZYX.
* The method IF\_RobotConfigurationAdvanced.SetOrientationConvention() can be used to set the orientation convention to the preferred value.
* Configured tools take robot orientation into account. A change of the robot orientation is compensated by a robot movement to keep the TCP at the given coordinate.

## Enhancements

* The usage of IF\_RobotConfigurationAdvanced.AdditionalTransformationAxes is improved when AuxAx with a period is used.
* In manual mode, the tool change by using IF\_RobotMotion.SetTool is improved. The verification of whether a MoveAsync is active or triggered is now correctly performed.
* The behavior of the motion profile calculation when a segment cannot be attached to the connected path due to acceleration limitation is improved.
