# IF\_HomeCmd – WriteAxisEncoder (Method)

## Overview

|  |  |
| --- | --- |
| Type: | Method |
| Available as of: | V1.3.1.0 |
| Versions: | Installed version |

## Task

Write the axis encoder in connection with setting the axis position.

## Description

The WriteAxisEncoder homing mode is used to write a defined value to the encoder of a real axis in order to establish an explicit mechanical reference. This operation establishes a fixed relationship between the position of the physical axis and the internal encoder value (Encoder.Position), enabling the system to reliably reconstruct this reference at a later point using the homing mode [RestorePosFromAxisEncoder](IF_HomeCmdRestorePosFromAxisEncoder-452D28DC.html).

This homing mode is useful during commissioning, machine setup, or service operations where a precise mechanical reference must be established once, and restored later without repeating a mechanical homing cycle.

NOTE: To perform this homing command the axis must be in state Disabled.

## Functional Description

The following sequence is performed when WriteAxisEncoder is executed:

1. Writing the Encoder Value

   Once the axis is confirmed to be stationary, the function block writes the target value to the encoder of the servo axis. This action defines the mechanical reference point.
2. Setting Axis.RefPosition

   Following the encoder update, the internal axis reference position (Axis.RefPosition) is set according to the configuration parameters of the method, primarily i\_lrHomePosition, i\_lrUserPeriod, and i\_lrAbsEncOffset.

   After completion, the resulting axis position corresponds to the target home position and is adjusted according to the selected application mode (periodic or non-periodic). Refer to [Application Modes for WriteAxisEncoder](WriteAxisEncoderMethodApplicationMo-4523D2FC.html) .

   NOTE: This homing mode does not set the axis state to Homing. Therefore, a possible interruption by other motion function blocks cannot be prevented.

## Inputs

| Input | Data type | Description |
| --- | --- | --- |
| i\_lrHomePosition | LREAL | Position value of the axis (calculated back to either range i\_lrUserPeriod or  EncoderPeriod). |
| i\_lrAbsEncOffset | LREAL | Absolute value by which the axis position can be reconstructed in the negative range in the case of non-periodic applications.  NOTE:  * For i\_lrUserPeriod > 0, this value is not taken into account. * For i\_lrUserPeriod = 0, this value must be greater than 0. |
| i\_lrUserPeriod | LREAL | Specifies the machine period.  NOTE:  * i\_lrUserPeriod > 0: periodic application.  After homing the axis position is in the range i\_lrHomePosition ...i\_lrUserPeriod. * i\_lrUserPeriod = 0: non-periodic application.  After homing the axis position is in the range -i\_lrAbsEncOffset ... (EncoderPeriod - i\_lrAbsEncOffset). |

## Troubleshooting

This table describes the possible issues and their solutions:

| Issue  (error information q\_xError = TRUE) | Cause | Solution |
| --- | --- | --- |
| q\_etResult = HomingIsAlreadyActive | Another homing process is already ongoing. | Retry after the active homing process has been completed |
| q\_etResult = NotInitialized | The [FB\_Home](FB_Home-GeneralInformation-459EBABF.html#FB_Home-GeneralInformation-459EBABF) function block instance has not been initialized yet. | Initialize the function block by calling the method [Init](FB_Home-InitMethod-46178154.html#FB_Home-InitMethod-46178154). |
| q\_etResult = AxisNotReady | The axis state is invalid. | Only the states Disabled and Standstill are accepted. |
| q\_etResult = HomingFailed | FB\_HomeAbs could not set the position. | Refer to q\_sResultMsg for more details. |
