# IF\_MultiAxisCustomJob

|  |  |
| --- | --- |
| Type: | Interface |
| Available as of: | 1.0.0.0 |

## Description

The interface IF\_MultiAxisCustomJob is the central interface for the coordination and management of the axes in a multi-axis custom job. It orchestrates the execution of a user-defined motion algorithm across multiple axes within a single RTP cycle.

The interface manages the instances of IF\_SlaveChannel for subordinate axes and IF\_MasterChannel for master axes connected to the group.

The interface uses the interface IF\_MultiAxisCustomJobCallback to access the user-defined logic by orchestrating the calls to the Prepare and Calculate methods provided by the interface.

Upon activation via the Start method, the interface triggers the initialization phase by calling the Prepare method of the interface. Subsequently, it cyclically calls the Calculate method of the interface. This provides the environment in which the user-defined algorithm calculates motion profiles and applies target values to the subordinate axes using the method WriteMovementValues of the respective instances of the interface IF\_SlaveChannel.

Throughout execution, the interface monitors the state of the multi-axis group and indicates the current operating state via the etState property.

For details on the interaction of function blocks, interfaces and methods of a multi-axis custom job, refer to [Multi-Axis Custom Jobs](Multi-AxisCustomJobs-572EC46B.html).
