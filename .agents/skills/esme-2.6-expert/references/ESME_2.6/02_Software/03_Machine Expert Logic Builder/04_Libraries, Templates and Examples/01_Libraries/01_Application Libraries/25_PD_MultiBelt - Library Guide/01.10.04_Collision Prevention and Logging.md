# Collision Prevention and Logging

## Overview

This section deals with the topic of collision prevention and the internal logging functionality.

## Collision Prevention

A collision prevention has been implemented to help prevent mechanical damage to the system. It stops the trains before they come too close. The functionality looks ahead; that is to say, the minimum distance entered in the lrCrashDistance will definitely not be surpassed, even by an upcoming collision the trains will still maintain this distance after stopping.

## Logging

A logging functionality is integrated in the MultiBelt that records the events but does not indicate them in the program.

When calling up the diagnostic data, the logging data is transferred with the PacDrive Diagnostics and can be displayed in this program.

The diagnostic data contains the information required to reproduce the situation. If an error is detected, this data should therefore always be recorded.

The logging data also remains, even after an error has been detected and subsequent Emergency Stop is received and can then be called up using the PacDrive diagnostics. If an error is detected during operation, e.g., excessive acceleration, a copy of the data can also be made during the normal program sequence.
