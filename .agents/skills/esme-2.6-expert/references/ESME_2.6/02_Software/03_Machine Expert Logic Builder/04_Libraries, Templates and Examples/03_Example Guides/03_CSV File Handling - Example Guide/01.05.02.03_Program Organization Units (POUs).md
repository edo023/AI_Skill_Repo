# Program Organization Units (POUs)

## SR\_CsvFileHandling

The program SR\_CsvFileHandling implements the program code to control the read and write operations.

The program code has been split into unique logical functions. The program parts are processed in subroutines, referred to as actions.

The actions called from the program are listed in the following table:

| Name of the subroutine (action) | Description |
| --- | --- |
| ACT\_EndPerformanceMeasurement | The measured values of the present operation are updated. |
| ACT\_InputMapping | The input variables, mainly coming from the visualization, are verified and assigned to local variables for further processing. |
| ACT\_OutputMapping | The local variables are converted into the required format and then assigned to the associated output variables. The output variables are linked to the visualization. |
| ACT\_StartPerformanceMeasurement | The initialization of the measurement of the performance of the operation is implemented. |
| ACT\_StateMachine | This action implements the state machine for the file handling.  The general states are:   * Initialization of the function blocks * Monitoring the control commands * Triggering the commanded function for the corresponding function blocks * Monitoring the execution of the function blocks |

## SR\_VisControl

The program SR\_VisControl implements additional code to control the displayed data tables on the visualization in the Logic Builder within the EcoStruxure Machine Expert software.
