## Description

This function block is used to connect to a remote server via an HTTP proxy server. The function block sends an HTTP CONNECT request to the proxy server, requesting the proxy to establish a tunnel to the remote server.

The function block extends the FB\_ProxyHandlerBase. The ConnectToRemoteServer method is overwritten and implements the processing of the HTTP CONNECT request.

The sequence for establishing the connection is as follows:

1. Call the method ConnectToProxy to establish a connection to the specified proxy server.
2. Call the method ConnectToRemoteServer. The following steps are processed:

   1. The HTTP request CONNECT is created to establish a connection to the remote server, which is specified by its qualified hostname (iq\_sHostname) and the TCP port (i\_uiServerPort).
   2. If a callback is registered, the callback method [MessageToSend](MessageToSendMethod-01D40E4C.html#MessageToSendMethod-01D40E4C) from the registered interface is executed.
   3. The HTTP request is sent to the proxy server.
   4. The response on the HTTP request is received from the proxy server.
   5. The HTTP status code is verified, a value between 200 and 299 is considered as success, for example: HTTP/1.1 200 Connection Established.
   6. If a callback is registered, the callback method [VerifyResponse](VerifyResponseMethod-0240B9C4.html#VerifyResponseMethod-0240B9C4) from the registered interface is executed.
   7. If the option i\_xUpgradeToTls is set, the method UpgradeToTls of the referenced instance of the FB\_TcpClient2 is executed and monitored.

If both methods are completed successfully one after the other, q\_xDone = TRUE, the connection to the remote server is established and ready for the data exchange. The instance of the FB\_ProxyHandlerHttpConnect is no longer required for this connection.
