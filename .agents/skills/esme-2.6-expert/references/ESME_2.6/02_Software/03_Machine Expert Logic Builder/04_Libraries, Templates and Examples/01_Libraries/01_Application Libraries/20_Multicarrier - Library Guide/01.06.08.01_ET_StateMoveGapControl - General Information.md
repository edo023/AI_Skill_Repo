# ET\_StateMoveGapControl - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Enumeration |
| Available as of: | V1.0.0.0 |

## Description

The enumeration ET\_StateMoveGapControl specifies the status of the MoveGapControl move command of a carrier (refer to [IF\_MoveGapControl](IF_MoveGapControl-5B81ACFA.html#IF_MoveGapControl-5B81ACFA)).

## Enumeration Elements

| Name | Value (UINT) | Description |
| --- | --- | --- |
| NotActive | 0 | The move command MoveGapControl is not active. |
| MoveIntoEndTarget | 1 | The carrier is moving to the target position specified by the parameter [i\_IrTarget](IF_MoveGapControl-StartMethod-53C5DF88.html#IF_MoveGapControl-StartMethod-53C5DF88__Inputs-53C5FB73). |
| MoveIntoTemporaryTarget | 2 | The carrier cannot move to the end target position. The carrier moves to the next possible temporary target position. |
| InEndTarget | 3 | The carrier is in the specified end target position. |
| InTemporaryTarget | 4 | The carrier is in the next possible temporary target position. |
| StopCarrier | 5 | The carrier is being stopped. The stop is requested by the user or internally by the method [IF\_MoveGapControl - Stop](MoveGapCtrlStop-03D5662C.html#MoveGapCtrlStop-03D5662C). |
| CarrierStopped | 6 | The carrier is in standstill after being stopped. |
| NewStartIsBlockedStartAsSoonAsPossible | 7 | The carrier receives the command to move to a new target. The movement cannot be executed for one of the following reasons:  * The next carrier in the moving direction is inside the minimum gap defined by the parameter [i\_lrRefMinGap](IF_Motion-SetRefMinGapToCarrierInFr-6E20C338.html#IF_Motion-SetRefMinGapToCarrierInFr-6E20C338__Inputs-6CFA455F). * The next carrier in the moving direction maintains the minimum gap but is not moving. * The next carrier in the moving direction uses the move command [MovePureSmg](IF_MovePureSmg-5B83BBA5.html#IF_MovePureSmg-5B83BBA5). The selected carrier waits until the next carrier in the moving direction is in standstill or is outside the target. * The next carrier in the moving direction uses the move command [StartSyncToExternalMaster](MoveSyncExtMaster-080435F3.html#MoveSyncExtMaster-080435F3). The selected carrier waits until the next carrier in the moving direction is in standstill or is outside the target. |
