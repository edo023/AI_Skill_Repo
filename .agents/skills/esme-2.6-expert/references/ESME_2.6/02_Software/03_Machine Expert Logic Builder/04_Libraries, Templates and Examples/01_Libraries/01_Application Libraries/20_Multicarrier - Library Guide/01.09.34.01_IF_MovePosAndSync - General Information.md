## Task

Interface for controlling synchronized movements of carriers through cam and positioning modes.

## Description

The interface provides several methods for defining synchronized movements of carriers with the move command MovePosAndSync that offers the possibility to combine cam and positioning jobs.

The move command provides the following features:

* Possibility to combine positioning and cam jobs.
* Cam and positioning methods can be started on channel A, B or C (see [Move Commands and Channels](Move_Channels-36D35D8B.html)).
* Positioning methods for all channels with optional channel bundling when started on channel A (see [channel bundling](Move_Channels-36D35D8B.html#Move_Channels-36D35D8B__ChannelBundl-36D389A6)).
* Cam methods are only applicable to movements of a master and a connected carrier if their movement direction is identical. The master velocity must be greater than or equal to 0. The master can be an external master or a master carrier in front or behind the selected carrier.
* The move command MovePosAndSync cannot be used for superimposing other move command like, for example, MoveDirectly.

NOTE: For the behavior of carriers using the move command MovePosAndSync in combination with carriers using the move command MoveGapControl, refer to the appropriate [behavior principles](IF_MoveGapControl-StartMethod-53C5DF88.html#IF_MoveGapControl-StartMethod-53C5DF88__Principles_NotMoveGap-495858BC).

NOTE: The reference values (position, velocity, acceleration) of the different channels are summed up.
