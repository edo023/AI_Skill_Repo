# FB\_TorqueFeedForward2 - SetRefTorque (Method)

## Overview

|  |  |
| --- | --- |
| Type: | Method |
| Available as of: | V1.5.1.0 |
| Versions: | Installed version |

## Description

The method is used to set the requested torque feed forward value. The method converts it into an equivalent q-axis current value (based on motor data) and writes it to the feed forward parameter of the drive.

The following conditions must be fulfilled:

* The function block must be successfully configured.
* The Sercos ActualState must be phase 4 (operational).

NOTE: For optimal drive performance, the torque feed forward value must be calculated every Sercos cycle and written to the real‑time parameter with SetRefTorque().

To execute the method synchronously with the Sercos cycle, use one of the following implementation options:

| M660 Motion Controller | LMC PacDrive / M660 Motion Controller |
| --- | --- |
| Use a callback function associated with the system event AfterMotionCycleSercos. This callback can be configured in the Task Configuration editor and executes directly after the event. | Configure an EventTriggered task that is activated by the corresponding event of the RealTimeProcess (RTP). This executes the method within the deterministic timing of the Real‑Time Process and maintains synchronization with the Sercos cycle. |

## Troubleshooting

Possible diagnostic information (q\_xError = TRUE)

| Outputs of the method indicate the values | Cause | Solution |
| --- | --- | --- |
| q\_etResult = ConfigurationNotDone | The function block is not configured. | Ensure that the method Configure is completed.  Use the property xIsConfigured. |
| q\_etResult = InvalidSercosState | The Sercos ActualState is not as required. | Set the Sercos ActualState is phase 4. |
| q\_etResult = InvalidWorkingMode | The working mode of the assigned drive is invalid. | Set the working mode of the drive to Physical. |
| q\_etResult = DriveDoesNotSupportThisFunction | The selected axis does not support this function. | Select an axis that supports this function (for example, Lexium LXM62 Drive, Lexium LXM52 Drive or Lexium ILM62 Drive module). |
| q\_etResult = DivisonByZero | At least one parameter used by the internal conversion functions is equal to 0. | Set GearOut and TorqueConstant to values greater than 0. If the issue persists, contact your Schneider Electric support representative. |
