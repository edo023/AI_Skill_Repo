## VAR\_OUTPUT

After the caller has executed the callee, the data in this area is provided by the callee (writer) to the caller (reader). The caller has read access. The callee should only write this data. See rule SA0038 of the CODESYS Static Analysis “Read access to output variables”. The transfer method used is **by value**. The CODESYS compiler will initialize the output variables of a PROGRAM or FUNCTION\_BLOCK with default values. So the initialisation of the output variables by the callee is optional and it is possible to use “unconnected” output variables in this context.
