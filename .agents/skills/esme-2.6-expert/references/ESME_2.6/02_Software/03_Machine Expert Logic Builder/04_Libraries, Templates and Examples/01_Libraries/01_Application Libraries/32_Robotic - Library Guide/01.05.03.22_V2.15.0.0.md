## New Features

* **ET\_ArmConfiguration.InnerRight and ET\_ArmConfiguration.InnerLeft**

  By using the enumeration elements ET\_ArmConfiguration.InnerRight or ET\_ArmConfiguration.InnerLeft on the input i\_etArmconfiguration of the method IF\_RobotMotion.MoveJ(), the inner solution is applied in case of a necessary modification of the arm configuration of a SCARA configured by IF\_RobotConfiguration.Scara4Ax().
* **IF\_RobotConfigurationAdvanced.udiNumberOfPeriodsSetPosB**

  If a robot is configured using the Scara4Ax transformation, AxisB acts as a periodic axis. A SetPos is executed to realize the period modification of AxisB and therefore manage overflow of the axis position if AxisB rotates without limit in any one direction by using the possibility to modify the arm configuration with the newly introduced enumeration elements ET\_ArmConfiguration.InnerRight or ET\_ArmConfiguration.InnerLeft.

## Mitigated Anomalies

* **IF\_RobotMotion.MoveSync() and IF\_RobotMotion.MoveAsyc()**

  Sending an auxiliary axis or an orientation to the same target in consecutive connected paths is now handled in a proper way and reported by the diagnostics message ET\_DiagExt.IdenticalTarget.
* **Motion parameters for warm start of auxiliary axes and orientations**

  If no motion parameters for warm start (ET\_WarmStartMode.Default) of auxiliary axes or orientations are defined by a previous movement of the corresponding component, the configured motion parameters are read. If no motion parameters are configured, a diagnostics message ET\_DiagExt.WarmStartNotPossible is reported.
* **Tracking in combination of using IF\_AdditionalTransformationTCP.Change()**

  Using IF\_AdditionalTransformationTCP.Change() during an active tracking is fully supported now.
* **Algorithm of resulting acceleration limitation improved**

  The algorithm of resulting acceleration limitation was improved.

  In some rare cases, a path velocity which was too slow has been used.
