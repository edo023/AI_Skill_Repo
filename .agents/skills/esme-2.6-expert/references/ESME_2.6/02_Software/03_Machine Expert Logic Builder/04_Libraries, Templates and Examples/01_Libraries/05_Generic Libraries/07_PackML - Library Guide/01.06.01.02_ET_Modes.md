# ET\_Modes

## Overview

|  |  |
| --- | --- |
| Type: | Enumeration |
| Available as of: | V1.0.1.0 |

## Description

The enumeration includes predefined operation modes (for example, production). An operation mode is an ordered subset of states, commands, and transitions which can be initialized via the structure ST\_UnitModeDefinition.

## Enumeration Elements

| Name | Data type | Value | Description |
| --- | --- | --- | --- |
| Undefined | DINT | 0 | The operation mode is undefined or invalid. |
| Producing (1) | DINT | 1 | Operation mode for production. |
| Maintenance | DINT | 2 | Operation mode for maintenance. |
| Manual | DINT | 3 | Manual operation mode |
| **(1)** With the 2015 revision of the ANSI/ISA TR88.00.02-2015, the control mode Producing was changed to Production. For reasons of compatibility with the previous version of the PackML library, the name Producing has been maintained. | | | |

## Used By

The values of the enumeration can be used to fill values of the variables UnitModeCurrent of ST\_Status and UnitMode of ST\_Command.
