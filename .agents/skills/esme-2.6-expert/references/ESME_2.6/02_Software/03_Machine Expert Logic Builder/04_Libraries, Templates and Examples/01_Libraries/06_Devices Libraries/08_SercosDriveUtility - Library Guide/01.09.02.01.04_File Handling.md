# File Handling

File Handling

oConsider that an Index File and/or a Parameter File stored in the controller can only be access from one function block at the same time.

oIt is not possible to execute several instances of FB\_ParameterUpload and/or FB\_Parame­terDownload with the same file names as inputs in parallel.

oSuccessful execution of FB\_ParamterUpload creates a new Parameter File.

oInput string for an Index File and/or a Parameter File contains the file name (mandatory) and the file path (optional), for example CSV\ATV340S\_IndexFile.csv.

If the Index File and/or the Parameter File name contain the path, the path must already exist in the controller.

oOnly files with the extension .csv are supported. If the file extension is missing, .csv is added automatically.

oDepending on the size of the Index File (number of lines), you have to adapt the global parameter GPL.Gc\_uiMaxNumberOfRowsRead to provide enough memory for reading one or more CSV files.

oMake sure that there is enough memory in the controller to process the required (intended) files.

oEnsure that for FB\_ParameterUpload an Index File is located in the controller and in the appropriate folder.

oCreate the Index File with an external tool (conforming to the specification, for example Notepad++) and then copy the file in the controller file system.

oStore the relevant Index Files and Parameter Files to an external storage device.

NOTE: During an update of the controller firmware, the required Index Files and Parameter Files are deleted
