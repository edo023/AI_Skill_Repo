# ET\_ReasonCode

## Overview

|  |  |
| --- | --- |
| Type: | Enumeration |
| Available as of: | V2.4.4.0 |

## Description

The enumeration ET\_ReasonCode contains the possible values that indicate the reason code inside messages exchanged between the MQTT client and the MQTT broker according to the MQTT V5 specification.

## Enumeration Elements

| Name | Value (UDINT) | Description |
| --- | --- | --- |
| Success\_NormalDisconnection\_GrantedQos0 | 0 | The reason code 0 is interpreted according to the message where it is contained as follows:   * CONNACK, PUBACK, UNSUBACK, AUTH: Success * DISCONNECT: Normal disconnection * SUBACK: Topic is subscribed with QoS level 0. |
| GrantedQoS1 | 1 | Topic is subscribed with QoS level 1. |
| GrantedQoS2 | 2 | Topic is subscribed with QoS level 2. |
| DisconnectWithWillMessage | 4 | The connection is terminated and the MQTT broker receives the information that the MQTT client expects the server to publish the Will message. |
| NoMatchingSubscribers | 16 | The MQTT broker provides the information that no subscriber is available for the topic. |
| NoSubscriptionsExisted | 17 | A matching subscription is not found during unsubscribing. |

| ReAuthenticate | 25 | The client is requested to initiate a re-authentication. |
