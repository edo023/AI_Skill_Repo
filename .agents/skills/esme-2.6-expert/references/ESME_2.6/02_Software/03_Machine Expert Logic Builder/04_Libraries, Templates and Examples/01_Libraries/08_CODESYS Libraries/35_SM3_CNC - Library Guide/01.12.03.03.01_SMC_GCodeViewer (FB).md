# SMC\_GCodeViewer (FB)

FUNCTION\_BLOCK SMC\_GCodeViewer

This function block produces a textual representation of the G-Code that has been processed by [SMC\_NCInterpreter](VVFvsggjXKxZ6qmLw10MvZPswnU_smc-ncinterprete.html#vvfvsggjxkxz6qmlw10mvzpswnu_smc_ncinterprete_smc_ncinterpreter_fb) or [SMC\_NCDecoder](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-ncdecoder.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_ncdecoder_smc_ncdecoder_fb). This textual representation can be displayed in a visualization. With the input interpolatedObjectId, typically connected with the [SMC\_Interpolator](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-interpolator.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_interpolator_smc_interpolator_fb) output actObjectId, the function block gets the information which line has already been processed. It will remove the text of already processed lines.

**Note**

The output of the function block does not exactly match the G-code stored in the editor or in the text file. For example, no blank lines or comments will be output and variables will be replaced with their value.

The function block is called in the same task as [SMC\_NCInterpreter](VVFvsggjXKxZ6qmLw10MvZPswnU_smc-ncinterprete.html#vvfvsggjxkxz6qmlw10mvzpswnu_smc_ncinterprete_smc_ncinterpreter_fb).

The number of [SMC\_GCODEVIEWER\_DATA](cAH0ifE6NckjZp5eF3Qpy-filyk_smc-gcodeviewer.html#cah0ife6nckjzp5ef3qpy_filyk_smc_gcodeviewer_smc_gcodeviewer_data_struct) objects in the buffer pBuffer must be at least as large as the sum of all buffer sizes of the [SMC\_NCInterpreter](VVFvsggjXKxZ6qmLw10MvZPswnU_smc-ncinterprete.html#vvfvsggjxkxz6qmlw10mvzpswnu_smc_ncinterprete_smc_ncinterpreter_fb) and the path-preprocessing function blocks.
