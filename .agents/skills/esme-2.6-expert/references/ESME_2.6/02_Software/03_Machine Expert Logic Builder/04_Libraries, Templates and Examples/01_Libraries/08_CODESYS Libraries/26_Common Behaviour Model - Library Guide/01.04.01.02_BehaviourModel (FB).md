# BehaviourModel (FB)

FUNCTION\_BLOCK BehaviourModel IMPLEMENTS [IBehaviourModel](HxFoZ2ysCUMXYKT8P2lbmhX-bw0_ibehaviourmodel.html#hxfoz2yscumxykt8p2lbmhx_bw0_ibehaviourmodel_ibehaviourmodel_itf), [IConfigurationProvider](HxFoZ2ysCUMXYKT8P2lbmhX-bw0_iconfigurationpr.html#hxfoz2yscumxykt8p2lbmhx_bw0_iconfigurationpr_iconfigurationprovider_itf), [IConfigurationProvider2](HxFoZ2ysCUMXYKT8P2lbmhX-bw0_iconfigurationpr1.html#hxfoz2yscumxykt8p2lbmhx_bw0_iconfigurationpr1_iconfigurationprovider2_itf)

The BehaviourModel function block will serve as a core for other more specialized function blocks like [ETrigATl](d5Ek35equq6Geb4iMYOV1LabLwI_etrigatl.html#d5ek35equq6geb4imyov1lablwi_etrigatl_etrigatl_fb). Because *PLCopen* defines two different variables (xEnable/xExecute) for starting the function block depending on its type, a neutral name xControl has been chosen. The xControl input variable can act as a xExecute or an xEnable input variable. This depends on how the related action provider is decorated. If the action provider is decorated with [IEdgeTriggered](7Dy_vkh4hU084C7tLiDR95ZL0SQ_iedgetriggered.html#7dy_vkh4hu084c7tlidr95zl0sq_iedgetriggered_iedgetriggered_itf) then the xControl input variable acts as an xExecute. If the action provider is decorated with [ILevelControlled](7Dy_vkh4hU084C7tLiDR95ZL0SQ_ilevelcontrolled.html#7dy_vkh4hu084c7tlidr95zl0sq_ilevelcontrolled_ilevelcontrolled_itf) then the xControl input variable acts as an xEnable.
