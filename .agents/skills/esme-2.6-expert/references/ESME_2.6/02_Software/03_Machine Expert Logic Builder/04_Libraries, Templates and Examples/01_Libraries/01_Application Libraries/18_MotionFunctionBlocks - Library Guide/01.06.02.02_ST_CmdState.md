# ST\_CmdState

## Overview

|  |  |
| --- | --- |
| Type: | Structure |
| Available as of: | V1.0.1.0 |
| Inherits from: | — |

## Description

The structure ST\_CmdState is used to get and show the feedback of the recent and prior move command.

## Structure Elements

| Name | Data Type | Description |
| --- | --- | --- |
| etCmdType | ET\_AxisMotionCommand | Enumeration of the command |
| xBusy | BOOL | Default value: FALSE   * FALSE: The command is not being executed. * TRUE: The command is being executed. |
| xActive | BOOL | Default value: FALSE   * FALSE: The command does not control the movement of the axis. * TRUE: The command controls the movement of the axis. |
| xDone | BOOL | For discrete motion commands.  Default value: FALSE   * FALSE: Execution is not finished, or an error has been detected. * TRUE: Execution terminated without an error detected. |
| xContinuous | BOOL | On PLCopen, there are non-discrete Movecommands, they are called In.... feedback signals (InSync, InVelocity, InSteadyState, InGear).  xContinuous is the relevant feedback for the In.... feedback signals. |
| xAborted | BOOL | Default value: FALSE   * FALSE: Execution has not been aborted. * TRUE: Execution has been aborted by another command. |
| xError | BOOL | Default value: FALSE   * FALSE: The command is being executed, no error has been detected during execution. * TRUE: An error has been detected during the execution of the command. |
| etResult | ET\_Result | Provides diagnostic and status information as a numeric value.   * If q\_xError = FALSE, q\_etResult provides status information. * If q\_xError = TRUE, q\_etResult provides diagnostic or error information. |
| diJobId | DINT | User specific job number |
