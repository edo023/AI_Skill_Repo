# Overview

Overview

| Step | Action | Comment |
| --- | --- | --- |
| 1 | Set up the hardware according to the [network layout](Project_Information-2.htm#XREF_D_SE_0088480_1). | Both controllers and the PC must be connected to the same [Ethernet](../glossary/glossary.htm#XREF_D_SE_0055638_7) network. |
| 2 | Open the TcpUdpCommunicationExample project in EcoStruxure Machine Expert and open the Logic Builder. | – |
| 3 | Double-click the MyController node in the Devices tree. | A network scan is performed automatically. Available controllers are provided in the Controller selection view of the device editor. |
| 4 | Select the target controller from the list. | – |
| 5 | Right-click the controller entry and execute the command Process communication settings... from the context menu. | – |
| 6 | Enter the Communication parameter. | – |
| 7 | Activate the option Save settings permanently and click the OK button. | Result: The communication parameters are stored on the controller. They are retained even if it is restarted. These parameters override the application settings. |
| 8 | Repeat steps 3...7 for the other controller. | – |
| 9 | Execute the Multiple Download... command from the Online menu. | – |
| 10 | In the Multiple Download dialog box, select the applications, and click the OK button. | Result: The selected applications are downloaded to the previously selected controllers. |

For more information on downloading an application and the communication settings, refer to the EcoStruxure Machine Expert [Programming Guide](../../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SG_0026478_8) .
