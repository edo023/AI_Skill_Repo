# SMC\_Interpolator2Dir (FB)

FUNCTION\_BLOCK FINAL SMC\_Interpolator2Dir IMPLEMENTS [SMC\_IInterpolator](pSc41CfRieQozA2o_fxRBRpweGA_smc-iinterpolato.html#psc41cfrieqoza2o_fxrbrpwega_smc_iinterpolato_smc_iinterpolator_itf)

This function block is similar to function block [SMC\_Interpolator](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-interpolator.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_interpolator_smc_interpolator_fb), regarding function and allocation of inputs and outputs, but it is able to interpolate a path also in the opposite direction.

Reverse interpolation:

If dOverride has a negative value, SMC\_Interpolator2Dir interpolates in the opposite direction. For example, if this input is assigned to an analog velocity input of a hand wheel, you can move forward and backward with the desired velocity.

The use of the function block depends on the following preconditions:

> * If input bStartAtEnd is TRUE, the path has to be stored completely in structure poqDataIn.
> * If the path does not fit completely, you can define how many elements of the queue are reserved for moving backward. ([SMC\_QueueSetReservedEntries](JQrmasFMYZh4yi3HZW-UzcjGrwU_smc-queuesetrese.html#jqrmasfmyzh4yi3hzw_uzcjgrwu_smc_queuesetrese_smc_queuesetreservedentries_fb)) For example, if you want to move 10 elements backwards, you must reserve 13 (10 + 3) elements (three elements are reserved as a safety buffer). The queue needs to be large enough, because the reserved elements are no longer available for the look ahead.
> * The use of SMC\_Interpolator2Dir requires the additional call of [SMC\_Interpolator2Dir\_SlowTask](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-interpolator2.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_interpolator2_smc_interpolator2dir_slowtask_fb). This call must be done in the same task that reads the g-code, performs the path preprocessing, and fills the [SMC\_OutQueue](cAH0ifE6NckjZp5eF3Qpy-filyk_smc-outqueue.html#cah0ife6nckjzp5ef3qpy_filyk_smc_outqueue_smc_outqueue_fb) of the interpolator.
> * Reversing of probe movements / G31 is not supported. Going backwards is limited to the movement after the last probe movement / G31.

See [SMC\_Interpolator](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-interpolator.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_interpolator_smc_interpolator_fb) for a description of all inputs and outputs.
