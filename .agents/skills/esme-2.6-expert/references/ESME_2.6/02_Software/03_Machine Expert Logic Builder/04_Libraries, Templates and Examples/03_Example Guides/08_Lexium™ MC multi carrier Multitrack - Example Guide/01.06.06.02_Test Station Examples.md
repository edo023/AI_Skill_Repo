# Test Station Examples

## Overview

| Name | Description |  |
| --- | --- | --- |
| Sync | The Sync example includes two stations on each track. The first station of each track synchronizes the carriers in the station to a master axis. The second station stops the synchronization when the synchronous carriers reach its process position.  When Autotrigger is active, the stations are triggered automatically, making the new synchronous carriers on each track move face to face.  For more information on the move command for synchronization, refer to the [IF\_MoveSyncFromStandstill - StartSyncToExternalMaster (Method) of the Multicarrier Library Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=MLSLib&topicID=MoveSyncExtMaster_080435F3). | For a visual illustration of this example, refer to the [Sync](../../../../../api/video?lang=en-US&bookKey=decf20c3030487a6fadf59293c33f31751472c1c3a21f1b21e2b6c7ee997eae7&videoName=exMultrck_sync.mp4) video sequence. |
| PairTransfer | The PairTransfer example illustrates the transfer of a product from a track to another by pairs of carriers.  The transfer of the product from the first track to the second track is performed in movement. To help ensure a smooth transfer, the carriers of the second track are synchronized with the carriers of the first track and then superimposed.  For more information on the move command for synchronization and superimposed movements, refer to:   * [IF\_MoveSyncFromStandstill - StartSyncToExternalMaster (Method) of the Multicarrier Library Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=MLSLib&topicID=MoveSyncExtMaster_080435F3) * [IF\_MoveSyncFromStandstillSuperimposedChannelC (Method) of the Multicarrier Library Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=MLSLib&topicID=MoveSyncFrStandstSuperimpC_2F9D684A) | For a visual illustration of this example, refer to the [PairTransfer](../../../../../api/video?lang=en-US&bookKey=decf20c3030487a6fadf59293c33f31751472c1c3a21f1b21e2b6c7ee997eae7&videoName=exMultrck_pairTransfer.mp4) video sequence. |
