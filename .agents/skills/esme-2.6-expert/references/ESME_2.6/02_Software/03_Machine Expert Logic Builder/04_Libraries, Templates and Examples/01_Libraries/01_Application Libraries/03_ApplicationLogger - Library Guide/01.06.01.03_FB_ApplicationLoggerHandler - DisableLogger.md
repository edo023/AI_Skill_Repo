# FB\_ApplicationLoggerHandler - DisableLogger (Method)

## Overview

|  |  |
| --- | --- |
| Type: | Method |
| Available as of: | V1.0.4.0 |

## Description

This method can be called to disable further logging and to prevent the overwriting of logger messages inside the ring buffer. A logger message is created to indicate the disabling. By default the logger is enabled.

Using this function, you can define a trigger in the project in order to search for a root cause. The logging is always enabled until the trigger occurs. Then this method is called, which deactivates further logging. This way the behavior of the machine is stored inside the buffer of the Application Logger and can be read out with EcoStruxure Machine Expert any time.
