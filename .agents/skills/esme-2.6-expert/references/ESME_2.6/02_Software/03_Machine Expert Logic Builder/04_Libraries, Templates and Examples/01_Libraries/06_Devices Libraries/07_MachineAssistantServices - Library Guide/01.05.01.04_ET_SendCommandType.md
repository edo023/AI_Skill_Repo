# ET\_SendCommandType

## Overview

|  |  |
| --- | --- |
| Type: | Enumeration |
| Available as of: | V1.0.0.0 |

## Description

The enumeration ET\_SendCommandType specifies the supported command types.

## Enumeration Elements

Except Locate and Backup, the commands have an impact on the device configuration.

| NOTICE | |
| --- | --- |
|  | LOSS OF DATA  Do not use the SetIPv4Config, SetDhcp, SetBootp, SetDeviceName or Restore command on a running machine.  Failure to follow these instructions can result in equipment damage. |

| Name | Value (UDINT) | Description |
| --- | --- | --- |
| Locate | 0 | Request to a detected device to send an optical indication that it can be located. For example, the status LEDs of a Modicon M251 Logic Controller are flashing for about 10 s. |
| SetIPv4Config | 1 | Request to set the network configuration of the device to a fixed IPv4 address (including subnet mask and gateway).  NOTE:  * The time to update an IP configuration depends on several factors like network traffic or device load. * There is no confirmation if a command was executed successfully. Due to this, wait 10 s and then verify (new scan) if the settings are modified as expected. * Depending on the parameter Save, the new IP settings are stored permanently or not. |
| SetDhcp | 2 | Request to set network configuration of the device to DHCP.  NOTE:  * The time to update a DHCP configuration depends on several factors like network traffic or device load. * There is no confirmation if a command was executed successfully. Due to this, wait 10 s and then verify (new scan) if the settings are modified as expected. * Depending on the parameter Save, the new DHCP settings are stored permanently or not. |
| SetBootp | 3 | Request to set network configuration of the device to BOOTP.  NOTE:  * The time to update a BOOTP configuration depends on several factors like network traffic or device load. * There is no confirmation if a command was executed successfully. Due to this, wait 10 s and then verify (new scan) if the settings are modified as expected. * Depending on the parameter Save, the new BOOTP settings are stored permanently or not. |
| SetDeviceName | 4 | Request to set the name of an ATV9••, ATV6•• or ATV340.  NOTE:  * The time to update a device configuration depends on several factors like network traffic or device load. * There is no confirmation if a command was executed successfully. Due to this, wait 10 s and then verify (new scan) if the settings are modified as expected. * Depending on the parameter Save, the new device settings are stored permanently or not. |
| Backup | 5 | Request to copy the folders *sys* and *usr* from the addressed controller (via MAC address) to an SD card which must be inserted in the controller where the application is running before executing the command.  If the transfer is executed successfully, the backup files are located on the SD card.  For example:  */sd0/Dev\_backups/TM251MESE/TM251MESE@0080F40A61AD*   * */sd0/Dev\_backups* = default path for the backup files * *TM251MESE* = device type * *TM251MESE@0080F40A61AD* = controller referenced by the device name   NOTE:  * This command is only available for M241 and M251 Logic Controllers. * There is no dedicated diagnostic message to indicate that no SD card is inserted to save the data. The function block terminates with a general diagnostic message. |
| Restore | 6 | Request to restore (copy) the files from a previous backup to a dedicated controller.  The [backup name](D-SE-0093285.html#D-SE-0093285__D-SE-0093285.6) defines the location on the SD card where the files were stored before a successfully executed backup command, for example, TM251MESE@0080F40A61AD.  NOTE:  * This command is only available for M241 and M251 Logic Controllers. * There is no dedicated diagnostic message to indicate that no SD card with a backup image is available. The function block terminates with a general diagnostic message. * This command is for device replacement. * This command only transfers the new image files. To activate the new image, a power cycle (switch off/on) is required.  NOTE: Data in the folders ***sys*** and ***usr*** of the addressed controller are overwritten. See important hazard message after the table. |

| NOTICE | |
| --- | --- |
|  | LOSS OF DATA  Be sure to backup important information in the *sys* and *usr* directories before executing the Restore command.  Failure to follow these instructions can result in equipment damage. |
