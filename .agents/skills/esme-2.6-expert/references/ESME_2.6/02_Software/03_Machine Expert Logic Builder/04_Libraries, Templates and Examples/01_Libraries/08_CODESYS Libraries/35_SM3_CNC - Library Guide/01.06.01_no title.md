# 

The function blocks of this library allow to read, interpret, and interpolate g-code files (according to DIN 66025). The interpreter ([SMC\_NCInterpreter](VVFvsggjXKxZ6qmLw10MvZPswnU_smc-ncinterprete.html#vvfvsggjxkxz6qmlw10mvzpswnu_smc_ncinterprete_smc_ncinterpreter_fb)) transforms the path into a structure that can be modified, transformed, and interpolated in order to create cyclic set positions for the drives. (e.g. [SMC\_ReadNCFile2](TW1SQNUTI52-XXX1HJ6us52paR8_smc-readncfile2.html#tw1sqnuti52_xxx1hj6us52par8_smc_readncfile2_smc_readncfile2_fb), [SMC\_NCInterpreter](VVFvsggjXKxZ6qmLw10MvZPswnU_smc-ncinterprete.html#vvfvsggjxkxz6qmlw10mvzpswnu_smc_ncinterprete_smc_ncinterpreter_fb), [SMC\_ToolRadiusCorr](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-toolradiusco.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_toolradiusco_smc_toolradiuscorr_fb), [SMC\_AvoidLoop](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-avoidloop.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_avoidloop_smc_avoidloop_fb), [SMC\_SmoothPath](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-smoothpath.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_smoothpath_smc_smoothpath_fb), [SMC\_RoundPath](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-roundpath.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_roundpath_smc_roundpath_fb), [SMC\_Interpolator](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-interpolator.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_interpolator_smc_interpolator_fb)).

The functional breakdown to function blocks allows particular components to be replaced by function blocks designed for special requirements. Moreover, further function blocks – like an individual path pre-processor – can be combinded with the existing components without any problems.

The data structures of the library (e.g. [SMC\_POSINFO](cAH0ifE6NckjZp5eF3Qpy-filyk_smc-posinfo.html#cah0ife6nckjzp5ef3qpy_filyk_smc_posinfo_smc_posinfo_struct), [SMC\_GEOINFO](cAH0ifE6NckjZp5eF3Qpy-filyk_smc-geoinfo.html#cah0ife6nckjzp5ef3qpy_filyk_smc_geoinfo_smc_geoinfo_struct) and SMC\_VECTOR3D) describe positions, path segments and vectors. The [SMC\_OUTQUEUE](cAH0ifE6NckjZp5eF3Qpy-filyk_smc-outqueue.html#cah0ife6nckjzp5ef3qpy_filyk_smc_outqueue_smc_outqueue_fb) structure represents the interpreted g-code path..

The following compiler defines are supported:

> * SOFTMOTION\_NC\_INTERPRETER\_LOG\_GCODE: logs all gcode that is processed by [SMC\_NCInterpreter](VVFvsggjXKxZ6qmLw10MvZPswnU_smc-ncinterprete.html#vvfvsggjxkxz6qmlw10mvzpswnu_smc_ncinterprete_smc_ncinterpreter_fb)
> * SOFTMOTION\_NC\_NAVIGATOR\_LOG: logs information about all executed gcode jumps and subprogram calls/returns
> * SOFTMOTION\_NC\_SYMBOLTABLE\_LOG: logs information about the symbol table
> * SOFTMOTION\_NC\_JUMPBUFFER\_LOG: logs information about jumps in jump buffer
> * SM3\_CNC\_LOG\_IPO2DIR: logs information about [SMC\_Interpolator2Dir](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-interpolator1.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_interpolator1_smc_interpolator2dir_fb).
