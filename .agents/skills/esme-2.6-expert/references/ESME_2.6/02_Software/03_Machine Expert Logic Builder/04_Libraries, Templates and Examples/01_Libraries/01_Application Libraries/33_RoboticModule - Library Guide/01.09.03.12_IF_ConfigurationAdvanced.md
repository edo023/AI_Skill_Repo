## Task

Retrieve a set of parameters from a specific tracking system.

## Description

With the method GetTrackingParameters(…), a set of parameters can be retrieved from a tracking system.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_etTrackingSystem | [ROB.ET\_CoordinateSystem](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.Robotic&topicID=D_SE_0075477) | Tracking system from which the parameters should be retrieved.  Valid value: Identifier of a configured tracking system. |

| Output | Data type | Description |
| --- | --- | --- |
| q\_etDiag | [GD.ET\_Diag](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.GlobalDiagnostic&topicID=D_SE_0076228) | General library-independent statement on the diagnostic.  A value not equal to GD.ET\_Diag.Ok corresponds to a diagnostic message. |
| q\_etDiagExt | [ET\_DiagExt](ET_DiagExt-GeneralInformation-CBB54036.html#ET_DiagExt-GeneralInformation-CBB54036) | POU-specific output on the diagnostic.  q\_etDiag = ET\_Diag.Ok -> Status message  q\_etDiag <> ET\_Diag.Ok -> Diagnostic message |
| q\_sMsg | STRING[80] | Event-triggered message that gives additional information on the diagnostic state. |
| q\_stParameters | [ROB.ST\_TrackingParameters](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.Robotic&topicID=D_SE_0100785) | Parameters that should be retrieved from the tracking system transferred at input i\_etTrackingSystem. |
