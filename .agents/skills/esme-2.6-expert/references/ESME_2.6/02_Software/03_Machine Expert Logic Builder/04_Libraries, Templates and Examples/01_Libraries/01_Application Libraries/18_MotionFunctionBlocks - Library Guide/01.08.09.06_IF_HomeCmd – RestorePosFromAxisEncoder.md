# IF\_HomeCmd – RestorePosFromAxisEncoder (Method)

## Overview

|  |  |
| --- | --- |
| Type: | Method |
| Available as of: | V1.3.1.0 |
| Versions: | Installed version |

## Task

Restore the axis position from the axis encoder value.

## Description

The homing mode RestorePosFromAxisEncoder is used in combination with the homing mode WriteAxisEncoder that must be executed before. Refer to [IF\_HomeCmd – WriteAxisEncoder (Method)](IF_HomeCmdWriteAxisEncoderMethod-451E54DB.html) and to [Application Modes for WriteAxisEncoder (Method)](WriteAxisEncoderMethodApplicationMo-4523D2FC.html).

The method WriteAxisEncoder stores the mechanical reference once, and RestorePosFromAxisEncoder brings the axis back to this reference later – either with an offset (linear axis) or wrapped into a period (rotary axis). Both modes must be configured with the same parameters (i\_lrUserPeriod, i\_lrAbsEncOffset, i\_lrHomePosition), or the resulting position reference will not match.

The homing mode works in two modes, in periodic and non-periodic applications. The use case is selected with the value at the input i\_IrUserPeriod:

* Periodic applications (i\_lrUserPeriod > 0):

  + In periodic mode, the parameter i\_lrAbsEncOffset is not considered.   
    The RefPosition is set directly based on the encoder value.
  + If the encoder value is outside the defined period, the function automatically wraps the position into the valid period range.
  + This helps to ensure that the axis reference position remains within the configured periodic interval, for example, within 0...360° for rotary axes.
* Non-periodic applications (i\_lrUserPeriod = 0):

  + In non-periodic mode, the parameter i\_lrAbsEncOffset is taken into account when calculating the RefPosition.   
    This offset introduces a defined shift between the encoder position and the reference position.
  + If the axis is moved into the negative position range and the encoder position is read again, the function restores the corresponding RefPosition correctly including negative values.
  + This helps to ensure that the mechanical reference is accurately reconstructed outside the positive range.

When using RestorePosFromAxisEncoder, the configured Scaling.InvertDirection is taken into account. When the direction is inverted, the encoder counts in the opposite direction compared to the reference position. The method automatically compensates for this, but both WriteAxisEncoder and RestorePosFromAxisEncoder must use the same direction setting to ensure a correct and consistent position restoration.

NOTE: In order to perform this homing command, the axis must be in state Disabled or Standstill.

NOTE: This homing mode does not set the axis state to Homing. Therefore, a possible interruption by other motion function blocks cannot be prevented.

## Inputs

| Input | Data type | Description |
| --- | --- | --- |
| i\_lrAbsEncOffset | LREAL | Absolute value by which the axis position can be reconstructed in the negative range in the case of non-periodic applications.  NOTE: The value must be identical to the one used for writing WriteAxisEncoder. |
| i\_lrUserPeriod | LREAL | Specifies the machine period.  NOTE: The value must be identical to the one used for writing WriteAxisEncoder. |

## Troubleshooting

This table describes the possible issues and their solutions:

| Issue  (error information q\_xError = TRUE) | Cause | Solution |
| --- | --- | --- |
| q\_etResult = HomingIsAlreadyActive | Another homing process is already ongoing. | Retry after the active homing process has been completed |
| q\_etResult = NotInitialized | The FB\_Home function block instance has not been initialized yet. | Initialize the function block by calling the method Init. |
| q\_etResult = AxisNotReady | The axis is not in state Disabled. | Only the states Disabled and Standstill are accepted. |
| q\_etResult = HomingFailed | The function block FB\_HomeAbs could not set the position. | Refer to q\_sResultMsg for more details. |
