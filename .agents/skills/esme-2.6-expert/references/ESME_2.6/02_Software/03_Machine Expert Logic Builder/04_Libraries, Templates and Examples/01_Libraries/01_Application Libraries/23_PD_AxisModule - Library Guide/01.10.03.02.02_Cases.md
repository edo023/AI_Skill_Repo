# Cases

Cases

When starting the reference travel, the following cases are distinguished:

Case 1

The input i\_ifTouchProbe displays the condition "Sensor not assigned" when the reference travel is started.

The axis moves in the parameterized direction until Touchprobe detect the required edge. The direction and edge is defined by the input i\_etMode. Afterwards the axis stops at the following position: "Position when the reference signal is given" + i\_lrOffset.

Case 2

The input i\_ifTouchProbe is TRUE when the reference travel is started.There are two actions that are determined after the reference travel is started:

If the input i\_xRotativeSystem = FALSE, then the axis moves to the opposite direction instead of the direction defined by the input i\_etMode until a TouchProbe is FALSE. Then, the axis moves as described in Case 1.

If the input i\_xRotativeSystem = TRUE, then the axis moves in the direction that was defined by the input i\_etMode until the next edge of the TouchProbe is detected.

Case 3

The hardware limit switch is active in whose direction the axis is supposed to move (the direction is defined by the input i\_etMode).

The axis moves in the opposite direction instead of the direction that was defined by the input i\_etMode until the TouchProbe detect the edge. Afterwards the axis moves for i\_xRotativeSystem = FALSE like described in Case 2.

Case 4

The hardware limit switch opposite to the axis' direction of motion (defined by the input i\_etMode) is enabled. The axis moves until it is free and subsequently moves as described for Case 1.

If the axis meets the hardware limit switch during the reference travel, in whose direction the axis shall move (defined by the input i\_etMode), then the axis will behave like described in Case 3 after reaching the hardware limit switch
