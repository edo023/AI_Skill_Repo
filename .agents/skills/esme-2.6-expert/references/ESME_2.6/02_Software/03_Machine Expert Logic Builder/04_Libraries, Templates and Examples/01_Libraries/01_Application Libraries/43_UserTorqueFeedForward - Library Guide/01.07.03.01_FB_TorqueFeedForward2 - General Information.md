# FB\_TorqueFeedForward2 - General Information

|  |  |
| --- | --- |
| Type: | Function Block |
| Available as of: | V1.5.1.0 |
| Versions: | Installed version |

## Description

This function block supplies a torque feed forward value from the application to a drive. The function block converts the commanded torque into an equivalent axis value (based on motor data) and writes it to the feed forward parameter of the drive. In addition, the drive‑side feed forward control mode can be set, allowing the application to define how the drive applies the feed forward value within the control loops.

The function block has no inputs or outputs. The functionalities are provided with the methods and properties. A cyclic call of the function block is not permitted.

You can register a corresponding logger point to receive messages in the Application Logger, depending on the log level.

For correct operation of the torque feed forward function block, the following conditions must be fulfilled:

* The Sercos parameter P‑0‑1069.0.0 must be included in the IDN mapping of the drive object.
* The ActualWorkingMode of the drive must be physical.
* The Sercos ActualState must be phase 4 (operational).
* The function block must be successfully configured, this is achieved when the call of the Configure() method is completed without error.

During operation:

* The method SetRefControlMode() writes the requested ControlMode to the corresponding Sercos parameter of the drive.
* The method SetRefTorque() writes the desired torque feed forward value to the dedicated current feed forward parameter (P‑0‑1069.0.0) of the drive.

NOTE:

* The IDN mapping is configured either directly in the objects Device Editor (LMC / PacDrive) of the drive, or it must be set programmatically using the function FC\_ConfigureIdnMapping on the M660 Motion controller.
* After every Sercos phase ramp‑up (transition back to CP0 → CP4), the configuration must be repeated using Configure().
* For optimal drive performance, the torque feed forward value must be calculated every Sercos cycle and written to the real‑time parameter via SetRefTorque().
* Simultaneous execution of this function block with the legacy function blocks FB\_TorqueFeedForward and FB\_TorqueFeedForwardConfig is not permitted.

## **ApplicationLogger - Logger Point**

The function block implements a logger point that can be registered with the global Application Logger. When registered, the function block generates logger entries on the following events:

* Information regarding the logger point registration (log level = UserAction)
* Configuration status (log level = DebugMessage)
* Change of the required torque operation mode (log level = StatusMessage)

With the method [Register](IF_LoggerPointConfigurationRegister-4CD06FDB.html), the function block FB\_TorqueFeedForward2 is registered as a logger point in the global Application Logger. The values for the LoggerPoint parameters type and source are set to FB\_TorqueFeedForward2 and UTFF, respectively.

For more information, refer to the [*ApplicationLogger Library Guide*](../../../../../api/crossBook?lang=en-US&virtualBookName=APL2LG&topicID=LibraryGuideEIO000000_00_2C85D7FD).
