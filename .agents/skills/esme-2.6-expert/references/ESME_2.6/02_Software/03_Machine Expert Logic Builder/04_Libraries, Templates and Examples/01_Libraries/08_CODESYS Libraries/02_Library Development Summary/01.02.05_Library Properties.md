# Library Properties

Library properties serve for managing the meta data of a library. They are accessible via the [Project Information](projectinfo.html#projectinfo_project_information). As from CODESYS 3.5.5.0 these properties have the following meaning:

| Name | | Type | Value (Default) | Description |
| --- | --- | --- | --- | --- |
| Required | Company | Text |  | Serves for structuring (filter) in the “Add Library” dialog |
| Title | Text |  | Name of the library |
| Version | Version |  | Library version |
| Released | | Bool | False | A library should not be modified after having been released |
| Author | | Text |  | Author of the current library version |
| DefaultNamespace | | Text |  | World-wide unique prefix, for defining the scope of the symbols of the library |
| Description | | Text |  | Short description of the purpose of the library |
| Placeholder | | Text |  | Which [Placeholder](reuse.html#reuse_referencing_other_libraries__reuse_placeholderreference) should be used for referencing this library.  This library follows the rules for a [Common Library](library_types.html#library_types_common_libraries). |
| AutoResolveUnbound | | Bool | False | This library has no device dependencies and requires the automatic resolving of its placeholder |
| IsContainerLibrary | | Bool | False | This library follows the rules for a [Container Library](library_types.html#library_types_container_libraries) |
| IsInterfaceLibrary | | Bool | False | This library follows the rules for a [Interface Library](library_types.html#library_types_interface_libraries) |
| IsCommonLibraryContainer | | Bool | False | This library follows the rules for a [Container Library](library_types.html#library_types_container_libraries), but is referenced by a Placeholder |
| LanguageModelAttribute | | Text | qualified-access-only | The access on symbols of the library is only possible via [Namespace](namespaces.html#namespaces_namespace)/[Prefix](prefix.html#prefix_library_prefix) |
| IsEndUserLibrary | | Bool | False | This [Library](library_types.html#library_types_end_user_libraries) is especially designed for the needs of end users |
| UnitTestingDefine | | Text |  | This value can be used as a define in the application build properties to enable unit testing for the library in this application. The keywords PRIVATE, INTERNAL, PROTECTED and FINAL are ignored during compile. |
| DocFormat [[3]](#properties_library_properties__fn_properties_id10) | | Text | reStructuredText | Comments will be interpreted according the rules of [reStructuredText](reSTOverview.html#restoverview_formatting_commands_overview) |
| LibDocContent | | Text | CommentsAndDocs | DocsOnly: Only comments marked with a triple slash (///) are relevant.  CommentsAndDocs: Any comment style is relevant for the [LibDocScriptiing](docScripting.html#docscripting_script_your_documentation). |
| DocLanguages [[1]](#properties_library_properties__fn_properties_id8) | | Text |  | List of comma separated language ids (e.g, “de, fr”) |
| OnlineHelp [[1]](#properties_library_properties__fn_properties_id8) [[4]](#properties_library_properties__fn_properties_id11) | | Bool | False | When set to True, this library should be part of the CODESYS online help |
| SourceLibrary [[1]](#properties_library_properties__fn_properties_id8) | | Bool | False | When set to True, this library should be delivered not in \*.compiled-library format |
| ServerBuildException [[1]](#properties_library_properties__fn_properties_id8) | | Text | ok | Ignore Server Pipeline Errors |
| PreserveCompiledLibComments | | Bool | False | When the attribute is set to True, CODESYS does not remove comments from compiled libraries. |
| Precursor [[2]](#properties_library_properties__fn_properties_id9) | | Text |  | Reference to the last released library, which represents the incompatible immediate precursor.  The text string follows this format: “<Title>, <Version> (<Company>)”  For example: “SysTypes Interfaces, \* (System)” |
| ShowSmartCodingInfo | | Bool | False | Ensures that for libs which are inserted implicitly still “Smart Coding” is working. |
| Support32BitOnly | | Bool | False | The compiler generates an error if commanded generating code for a 64bit system. |
