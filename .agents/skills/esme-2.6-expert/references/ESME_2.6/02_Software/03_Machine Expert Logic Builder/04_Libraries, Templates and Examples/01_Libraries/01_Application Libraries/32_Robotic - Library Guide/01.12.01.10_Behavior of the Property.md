# Behavior of the Property lrMinVelOverrideVariation

## Overview

When lrMinVelOverrideVariation is greater than zero and the delta of the new VelOverride to its value is greater or equal to lrMinVelOverrideVariation, the VelOverride is set to the new value.

In case the delta is lower, the previous value is kept.

It is possible to set VelOverride to 0.0 and to 100.0 independent from the variation threshold.

This feature can be used in case VelOverride is controlled by some logic and very small changes should not become effective. This reduces the amount of calculations the controller has to perform, as otherwise any change of the VelOverride would force a recalculation of the motion profile.

## Examples

| Current value | New value | lrMinVelOverrideVariation | Result | Reason |
| --- | --- | --- | --- | --- |
| 5.0 | 5.5 | 1.0 | 5.0 | The delta between previous and new value is less than lrMinVelOverrideVariation. |
| 45.1 | 44.9 | 0.5 | 45.1 | The delta between previous and new value is less than lrMinVelOverrideVariation. |
| 99.5 | 100.0 | 2.0 | 100.0 | The value 100.0 can be set, independent from lrMinVelOverrideVariation. |
| 0.01 | 0.0 | 1.0 | 0.0 | The value 0.0 can be set, independent from lrMinVelOverrideVariation. |
| 33.3 | 35.7 | 1.0 | 35.7 | The delta between previous and new value is greater than lrMinVelOverrideVariation |
| 75.0 | 76.0 | 1.0 | 76.0 | The delta is equal to lrMinVelOverrideVariation; the new value is accepted. |
| 23.0 | 23.001 | 0.0 | 23.001 | lrMinVelOverrideVariation is 0.0, any new value of VelOverride is accepted |
