# DriverOpenP (FUN)

FUNCTION DriverOpenP : CAA.HANDLE

See [DriverOpenH](UzaP1MMIvsGx5wgQfvzFSZKw87g_driveropenh.html#uzap1mmivsgx5wgqfvzfszkw87g_driveropenh_driveropenh_fun) for details. In contrast to [DriverOpenH](UzaP1MMIvsGx5wgQfvzFSZKw87g_driveropenh.html#uzap1mmivsgx5wgqfvzfszkw87g_driveropenh_driveropenh_fun) no dynamic memory will be allocated. Memory specified by pMemory and szMemSize is used. Use [DriverGetSize](UzaP1MMIvsGx5wgQfvzFSZKw87g_drivergetsize.html#uzap1mmivsgx5wgqfvzfszkw87g_drivergetsize_drivergetsize_fun) to calculate the required memory size. Please note that required memory size can change with new runtime version.

**Note**

pMemory should be aligned to a 4 byte address on 32 bit systems and an 8 byte address on 64 bit systems. To ensure this alignment, you can declare the memory with ARRAY [0..n] of \_\_XWORD. Please keep in mind that the required memory size is different for 32 and 64-bit systems.
