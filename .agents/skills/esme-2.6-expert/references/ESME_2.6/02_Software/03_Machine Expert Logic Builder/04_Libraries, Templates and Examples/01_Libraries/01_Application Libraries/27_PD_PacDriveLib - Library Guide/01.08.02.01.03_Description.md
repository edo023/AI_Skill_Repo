# Description

Description

NOTE: For the most common applications, using this monitoring function block to increase the diagnostic coverage is not required. However, please refer to the documentation of the device to be monitored to establish whether this function block must be used in connection with the device in order to implement a safety function. For information on this issue, please refer to the circuit suggestion or to the explanation on implementing the STO safety function of the device in question.

This function block may also be used in cases where its use is not mandatory. In such cases, it can help to recognize wiring errors in the InverterEnable circuit when setting up the control cabinet or replacing devices. Wiring errors however are only recognized if all affected safety circuits are triggered separately.

The function block must not be used as the only checking measure for the InverterEnable circuit. Not all wiring errors can be recognized by the function block.

If the function block is active and the input i\_xStart = TRUE, the input i\_xInverterEnable is compared to the parameter Drive.InverterEnable. If the two signals i\_xInverterEnable and Drive.InverterEnable are different for a period of [Gc\_timAcceptableSignaleDifference](../Global_Elements/Global_Elements-2.htm#XREF_D_SE_0087806_1), an error is issued.

If a discrepancy between the InverterEnable signals has been recognized, this is reported via the diagnostic outputs of the function block. In such a case, the output q\_etDiag reports ET\_Diag.ExecutionAborted and the output q\_etDiagExt reports ET\_DiagExt.InverterEnableCheckFailed.

The user has to ensure an appropriate reaction to the detected error (e.g. disabling the position control).
