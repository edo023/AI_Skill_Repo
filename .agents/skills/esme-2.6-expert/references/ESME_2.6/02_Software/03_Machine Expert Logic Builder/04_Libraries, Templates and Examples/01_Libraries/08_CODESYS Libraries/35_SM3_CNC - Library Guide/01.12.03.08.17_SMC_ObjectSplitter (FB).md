# SMC\_ObjectSplitter (FB)

FUNCTION\_BLOCK SMC\_ObjectSplitter

This function can be used if a velocity optimizer (like [SMC\_LimitDynamics](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-limitdynamic.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_limitdynamic_smc_limitdynamics_fb) ) is applied to the queue. The limitation of [SMC\_LimitDynamics](8OGX-fND8b3kPGnX4Q3neKp1m4c_smc-limitdynamic.html#8ogx_fnd8b3kpgnx4q3nekp1m4c_smc_limitdynamic_smc_limitdynamics_fb) always affects a complete object. This means that, the smaller the objects, the better the result of the limitiation. Objects without geometrical expansion (like M-functions) or geometrical curvature (like straight lines) remain unchanged.

With eMode, it is defined, how the division is done:

> * | if FIX is set, every curved object is divided in | so many objects as defined in dSplittingParameter. |
> * | if LENGTH is set, every object is divided so, | that it has the length dSplittingParameter at most. |
