# ST\_PropertiesConnack

## Overview

|  |  |
| --- | --- |
| Type: | Structure |
| Available as of: | V2.4.4.0 |
| Inherits from: | - |

## Description

The structure ST\_PropertiesConnack specifies the properties sent by the MQTT broker inside the CONNACK message as a response to a CONNECT message.

## Structure Elements

| Name | Data type | Description |
| --- | --- | --- |
| udiSessionExpiryInterval | UDINT | Indicates the session expiry interval applied to the connection. If the MQTT broker does not send a value, the configuration applied in the connect properties of the client are considered. |
| uiReceiveMaximum | UINT | Indicates the number of QoS 1 and QoS 2 publications that the MQTT client is willing to process concurrently for the client.  Default value: 65535 |
| uiMaximumQoS | [ET\_QualityOfService](D-SE-0086508.html) | Indicates the maximum supported QoS level of the MQTT broker.  Default value: ET\_QualityOfService.QoS2 |
| xRetainAvailable | BOOL | When TRUE, the MQTT broker supports retain messages.  Default value: TRUE |
| udiMaximumPacketSize | UDINT | Indicates the maximum packet size that the MQTT broker can accept.  Default value: 0 |
| sAssignedClientIdentifier | STRING[255] | Indicates the client identifier assigned by the MQTT broker because no client identifier was configured in the connection settings of the client.  Default value: NULL  NOTE: The length of the value is limited to 255 characters. If it exceeds the maximum length, it is truncated and the output q\_xTruncatedResponseData is set to TRUE. |
| uiTopicAliasMaximum | UINT | Indicates the maximum value that the MQTT broker accepts as topic alias sent by the client.  Default value: 0 |
| sReasonString | STRING[255] | Indicates a human readable string designed for diagnostic purposes that the MQTT broker sends to the client.  NOTE: The length of the value is limited to 255 characters. If it exceeds the maximum length, it is truncated and the output q\_xTruncatedResponseData is set to TRUE. |
| xWildCardSubscriptionAvailable | BOOL | When TRUE, the MQTT broker supports wildcard subscriptions.  Default value: TRUE |
| xSubscriptionIdentifiersAvailable | BOOL | When TRUE, the MQTT broker supports subscription identifiers.  Default value: TRUE |
| xSharedSubscriptionAvailable | BOOL | When TRUE, the MQTT broker supports shared subscriptions.  Default value: TRUE |
| uiServerKeepAlive | UINT | Indicates the keep-alive value applied to the connection. If the MQTT broker does not send a value, the configuration applied in the connection settings of the client are considered.  NOTE: If the server does not respond to a PING request within a duration equal to one and a half times the keep-alive value ([uiKeepAlive](D-SE-0086514.html)), the client disconnects. |
| sResponseInformation | STRING[255] | Indicates the basis for creating a response topic.  Default value: NULL  NOTE: The length of the value is limited to 255 characters. If it exceeds the maximum length, it is truncated and the output q\_xTruncatedResponseData is set to TRUE. |
| sServerReference | STRING[255] | Indicates the reference to another server to be used by the client.  Default value: NULL  NOTE: The length of the value is limited to 255 characters. If it exceeds the maximum length, it is truncated and the output q\_xTruncatedResponseData is set to TRUE. |
| astUserProperties | ARRAY[1..GPL.Gc\_uiMaxNumberOfUserProperties] OF ST\_UserProperty | Indicates the user properties received from the MQTT broker.  NOTE: A user property with an empty name represents the end of the list. |
| sAuthenticationMethod | STRING[255] | Indicates the authentication method used and how to interpret the authentication data.  NOTE: The length of the value is limited to 255 characters. If it exceeds the maximum length, it is truncated and the output q\_xTruncatedResponseData is set to TRUE. |
| uiAuthenticationDataLength | UINT | Indicates the length of the authentication data. |
| abyAuthenticationData | ARRAY[0..254] OF BYTE | Indicates the authentication data. |
