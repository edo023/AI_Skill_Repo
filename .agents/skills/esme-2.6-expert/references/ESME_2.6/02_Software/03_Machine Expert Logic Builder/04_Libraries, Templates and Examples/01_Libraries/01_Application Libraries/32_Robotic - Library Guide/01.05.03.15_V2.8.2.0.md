## New Functions

Integration of SCARA (Selective Compliance Assembly Robot Arm)

* IF\_RobotConfiguration.Scara4Ax()

  + The RoboticModule can be configured as a SCARA robot with four axes including four degrees of freedom (IF\_RobotConfiguration.Scara4Ax()).
* Orientation

  ROB.ET\_RobotComponent.OrientationZ is introduced if the robot is configured as a SCARA.

  + IF\_RobotConfiguration.Scara4Ax()
  + IF\_RobotConfiguration.SchneiderElectricRobot() Robot S-Series

  The orientation is supported by functionalities of:

  + IF\_RobotConfiguration
  + IF\_RobotConfigurationAdvanced
  + IF\_RobotMotion
  + IF\_RobotJogging
  + IF\_RobotFeedback
* IF\_RobotMotion.MoveJ()

  + With the method MoveJ(), a motion job can be issued to move in a point-to-point motion to a target position in three-dimensional space. It is also possible to perform a modification of the robot arm configuration, for example from right to left arm configuration in case of a SCARA.
* IF\_RobotFeedbackScara4Ax

  + With IF\_RobotFeedbackScara4Ax, the Cartesian positions of the motors A to D in the mechanical robot coordinate system can be read.

## Enhancements

StartPointNotIdentical

The diagnostic message ET\_DiagExt.StartPointNotIdentical occurred on sending a IF\_RobotMotion.MoveS() motion command under the following conditions/situation:

* IF\_RobotFeedback.rstLastTarget is used for calculating the spline table.
* IF\_RobotConfigurationAdvanced.AdditionalTransformationTCP() is parametrized.
* IF\_AdditionalTransfromationTCP.Change() is called more than once without a path movement in between.
* Tracking is active during sending the IF\_RobotMotion.MoveS() motion command.

The diagnostic message no longer occurs in this situation.
