# ST\_Synchron - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Data structure |
| Available as of: | V1.1.4.0 |
| Inherits from: | - |
| Versions: | Current version |

## Description

This structure is used to configure a station that operates in synchronous mode.

## Structure Elements

| Variable | Data type | Description |
| --- | --- | --- |
| ifMaster | SystemConfigurationItf.IF\_Master | Interface of the master for this station. The following types of masters are supported:  Physical and virtual axes, SINCOS and incremental encoder. |
| lrMasterPeriod | LREAL | Period in which the master moves. |
| lrSynchronStart | LREAL | Path for acceleration to master velocity. |
| lrSynchronEnd | LREAL | Path for deceleration from master velocity to stop. |
| lencHelp | SystemConfiguration.L\_ENC | Logical auxiliary encoder. (Must be used) |
| lrStartDelayDistance | LREAL | Distance between the Touchprobe sensor and the transfer point to the train. |
| etLawOfMotion | [ET\_LawOfMotion](D-SE-0077735.html#D-SE-0077735) | Law of motion by means of which the train moves. The available laws are:  **ModAccTr**: modified acceleration trapezoid  **InclSin**: inclined sine line  **SystemCam**: system or user-defined profile. |
| diUserCamId | DINT | ProfileId of a user-defined Cam profile. Only effective if etLawOfMotion = SystemCam. |
| lrMinProductLength | LREAL | Minimum length in units of the master where the Touchprobe sensor must be covered so that a product is detected as valid. |
| lrLockOutDistance | LREAL | After a product is detected as valid, the Touchprobe sensor is switched off for the lrLockOutDistance. |
| etEventTrigger | [ET\_EventTrigger](D-SE-0077733.html#D-SE-0077733) | Enumeration which defines how the start signals for the synchronous station are generated. The following options are possible:  StartOnSignal: Start signals come from the Touchprobe sensor or from the bit xStart.  SyncToPhase: Start signals are automatically generated to the specified phase lrPhase of the respective cycle.  MasterSelectPos: Start signals are generated if the Touchprobe sensor on lrMasterSelectPos is covered. The start takes place on lrPhase. |
| lrPhase | LREAL | Phase in the master period on which the product arrives in the mode etEventTrigger = MasterSelectPos. In the mode etEventTrigger = SyncToPhase, it is the phase where the product is to be stored on the master belt. |
| lrMasterSelectPos | LREAL | The point in the master period where the Touchprobe input is queried. If a product is available at this point then a step is executed; if no products are available, then no step is executed. |
| stDepartureMove | [ST\_Movement](D-SE-0077763.html#D-SE-0077763) | Motion parameter or the departure from this station to the next. See ST\_Movement. The parameters are adopted when the next train departs. |
| uiSyncStepsPreStation | UINT | The steps which the train travels synchronously before it enters into the station and starts loading / unloading. |
| lrGearFactor | LREAL | Indicates the gear factor between encoder and trains. If a value greater than 1 is selected, the trains move faster than the encoder by the factor lrGearFactor. |
| lrEndOfSyncPhaseOffset | LREAL | Indicates an offset before the end of the synchronous phase of a step from where synchrony should be terminated and a compensating cam should be started. |
| xSafeArrival | BOOL | FALSE: Default value  TRUE: Reduces internal collision prevention threshold.  Refer to the hazard message below. |
| xOverwriteMasterVelocity | BOOL | FALSE: Default value  TRUE: User-defined velocity (ST\_Synchron.lrUserDefinedMasterVelocity) is used for synchronous master.  Refer to the hazard message below. |
| lrUserDefinedMasterVelocity | LREAL | Velocity for synchronous master of this station. This value is only adopted if ST\_Synchron.xOverwriteMasterVelocity = TRUE.  Refer to the hazard message below. |

| WARNING | |
| --- | --- |
|  | UNINTENDED EQUIPMENT OPERATION  * Define motion parameters to avoid collisions if xSafeArrival is set to TRUE. * Define the velocity for synchronous master of this station to avoid collisions when using lrUserDefinedMasterVelocity or when setting xOverwriteMasterVelocity to TRUE.  Failure to follow these instructions can result in death, serious injury, or equipment damage. |
