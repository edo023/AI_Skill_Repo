# i_xDrv1RdyStat, i_xDrv2RdyStat

i\_xDrv1RdyStat, i\_xDrv2RdyStat

These inputs correspond to ready status of both drives. The information can be obtained from bit 1 of status word of Altivar drives. This information has to be interlocked (logical AND) with an information that the drive is communicating with the controller.

Example of CANopen status verification for node 3:

VAR  
      GET\_STATE\_Node: CIA405.GET\_STATE;  
      xNodeOperational: BOOL;  
END\_VAR  
\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_  
(\* Check the CANopen node with NodeId 3 \*)  
GET\_STATE\_Node(  
      NETWORK:= 1,  
      ENABLE:= NOT GET\_STATE\_Node.CONFIRM AND NOT (GET\_STATE\_Node.ERROR > 0),  
      TIMEOUT:= 1000,  
      DEVICE:= 3);  
  
IF GET\_STATE\_Node.CONFIRM OR (GET\_STATE\_Node.ERROR > 0) THEN  
      xNodeOperational:= (GET\_STATE\_Node.STATE = 5);  
END\_IF

If the communication with any of the 2 drives is interrupted, the FB has to get the information in order to stop the movement.

The FB needs to have information about status of both drives. If one of the drives is not ready, the FB does not authorize start of the movement. If one of the ready bits turns FALSE during movement, both drives are stopped.
