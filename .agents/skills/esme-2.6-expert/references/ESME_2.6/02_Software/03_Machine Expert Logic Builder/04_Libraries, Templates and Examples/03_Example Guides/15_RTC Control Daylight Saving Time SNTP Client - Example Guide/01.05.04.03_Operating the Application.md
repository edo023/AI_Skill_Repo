# Operating the Application

## Visualization

The application example implements a visualization for monitoring and control purposes.

This section provides a brief description of each function. In addition, refer to the [visualization screen](D-SE-0077825.html#D-SE-0077825).

## Read Current RTC of the Controller

In the display Status Controller RTC on the visualization Visu\_RTC\_Control, the current value of the RTC (Real-Time Clock) of the controller is displayed as UTC (Coordinated Universal Time) and as local time. The local time is calculated by adding the biases which are defined with the selected timezone. For the local time, it is indicated whether the daylight saving time is active.

In addition, the status of the backup battery is displayed.

NOTE: If the backup battery is depleted, the RTC of the controller is not maintained during a power interruption of the controller.

## Select a Timezone

In the display Timezone Configuration on the visualization Visu\_RTC\_Control, you can select a predefined timezone. According to the selected timezone configuration, the parameter for the biases and the switching limits for the DST (Daylight Saving Time) are displayed.

In addition, you can adjust the configuration parameters for the selected timezone.

The selection of the timezone immediately affects the calculation of the local time as shown in the display Status Controller RTC.

## Set the RTC of the Controller

In the display Set Controller RTC > Clock on the visualization Visu\_RTC\_Control, you can select the date and time for the RTC of the controller.

NOTE: In case the time to set is within the doubled hour when switching from the DST to standard time, you need to select whether the time to timezone set is daylight saving time or not.

Click the Set RTC in order to set the RTC of the controller with the selected date and time.

## Set a Weekly Drift for the RTC of the Controller

In the display Set Controller RTC > Drift on the visualization Visu\_RTC\_Control, you can select the weekly drift by which the RTC of the controller shall be accelerated (positive value) or slowed down (negative value). In addition, you need to select the weekday and the time, for which the weekly drift is to be calculated.

Click Set RTC Drift in order to set the selected weekly drift for the RTC of the controller.

## Control the SNTP Client

In the display SNTP client in the visualization Visu\_RTC\_Control, the SNTP client can be configuerd, controlled and monitored.

| Step | Action |
| --- | --- |
| 1 | Enter the IP address of the time server in the corresponding field. |
| 2 | Click Get Timestamp from NTP or SNTP server to receive the time stamp from the time server. |
| 3 | Click Get Timestamp from NTP or SNTP server and set the RTC in order to syncronize the RTC of the controller with the time server.  **Result:** The status of the SNTP client can be monitored in the corresponding display. |

NOTE: In case the SNTP client detects an error during execution the error information are displayed and the function block can be restarted using the button Clear Error Detection.
