# ET\_EventTrigger - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Enumeration type |
| Available as of: | V1.1.4.0 |
| Versions: | Current version |

## Description

This enumeration contains the variables by means of which the start signals of a synchronous station are generated.

## Enumeration Elements

| Name | Value | Description |
| --- | --- | --- |
| StartOnSignal | 1 | Start signals are either generated by the connected Touchprobe sensor or via the bit xStart. With this type of start signals, the products must not have any phase position, only the distance of the Touchprobe sensor to the transfer point is relevant. If the products do not have any phase position or different distances, a compensation motion is carried out. |
| SyncOnPhase | 2 | Start signals are automatically generated. During each master period, a start signal is generated exactly at lrPhase. External inputs are irrelevant. This setting is particularly suitable in order to unload products from the trains in-phase. |
| MasterSelectPos | 3 | The start signals are generated via a Touchprobe input. The Touchprobe input is queried once during a master period, at the position lrMasterSelectPos. If the Touchprobe is occupied at this time, a step is executed. If the Touchprobe is not occupied, no step is executed and the trains stop. The products are stored directly at lrPhase, and no correction is carried out. If MasterSelectPos is selected, then the products must have the correct phase position before inserting them into the MultiBelt. |
