# Preconditions for Running the Application Example

## Overview

To run the application example, the following preconditions must be fulfilled:

* The OPC UA server for the controller is enabled and the security settings are performed.
* User management for the visualization is created.
* The application is downloaded and is running on the associated controller.
* A PC, laptop, or mobile device with a compatible web browser is connected to the same network as the controller so that the web visualization can be accessed.
* The digital certificate of the controller for secured communication between server and client has been declared as trusted. For information about the certificate management for the OPC UA client, refer to the document [*How To Manage Certificates for OPC UA Client, User Guide.*](../../../../../api/crossBook?lang=en-US&virtualBookName=HowCerOPC&topicID=D_SE_0100128)

## Enable the OPC UA Server

Before downloading the application to the Modicon M262 controller, you must enable the OPC UA server for the controller and configure the security settings.

| Step | Action |
| --- | --- |
| 1 | Double-click the MyController node in the Devices tree. |
| 2 | In the opened device editor, select the tab OPC UA Server Configuration. |
| 3 | Activate the check box OPC UA Server enabled. |
| 4 | Activate the check box Disable anonymous login. |
| 5 | Select Basic256Sha256 for the parameter Security settings > Security Policy. |
| 6 | Select SignAndEncrypt for the parameter Security settings > Message Security. |

For PacDrive LMC controllers, the OPC UA server is enabled inside the application by calling the function SystemInterface.FC\_OpcUaStart. Before downloading the project, assign the authentication information to the following global constants:

* GVL.Gc\_sServerUsername
* GVL.Gc\_sServerPassword

If required, you can configure further server-specific settings in the tab OPC UA Server Configuration of the device editor of the controller.
