# Functional Description

## Program - SR\_<name function template>

The function template provides a program which has been created in programming language ST (Structured Text). This program is added under the Application node within a folder with the name assigned within the Add Function From Template dialog box. The program has the same name as the folder with the prefix SR\_. A program-call is added automatically to the associated task.

The program implements the state machine for the DNS client with the following states:

| State | Description |
| --- | --- |
| Idle | If the command `xCmdExecute` is detected, the execution of the FB\_DnsClient is initiated. |
| Executing | This state is active as long as the FB\_DnsClient indicates q\_xBusy = TRUE. |
| Disabling | After the FB\_DnsClient was completed successfully, (q\_xDone = TRUE), the function block is disabled in this state. After disabling the state Idle becomes active and the function block is ready for next execution. |
| Error | This state is active when an error was detected (q\_xError = TRUE) during the execution of the function block. If the command `xCmdErrReset` is detected, the function block is reset and the state Idle becomes active. |

## Visualization

The function template implements a visualization to control and monitor provided functions. The name of the visualization corresponds to the name assigned within the Add Function From Template dialog box. It is available in the same folder as the program.
