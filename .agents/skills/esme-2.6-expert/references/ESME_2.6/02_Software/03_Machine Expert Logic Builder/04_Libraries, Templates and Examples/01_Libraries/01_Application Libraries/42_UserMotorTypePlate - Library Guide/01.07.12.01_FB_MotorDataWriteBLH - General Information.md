# FB\_MotorDataWriteBLH - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Function block |
| Available as of: | V1.1.1.5 |

## Task

This function block writes the motor data to the drive. Also refer to the chapter [Asynchronous Motor](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.3rdPartyMotors&topicID=D_SE_0088217).

NOTE: Execute this POU only when required for application needs. The encoder memory area has a limited number of write cycles.

NOTE: Incorrect machine encoder typeplate data has a direct influence on the behavior of the controller.

| DANGER | |
| --- | --- |
|  | UNINTENDED OPERATION OF THE MOTOR  * Verify assignment of machine encoder data to the motor before writing. * Ensure that no one is in the operating zone during the startup.  Failure to follow these instructions will result in death or serious injury. |

NOTE: A deactivated monitoring function can lead to a possible undetected overheating of the motor.

| DANGER | |
| --- | --- |
|  | OVERHEATING OF THE MOTOR  * Verify assignment of motor data to the motor before writing. * Use appropriate methods to verify thermal overload of the motor.  Failure to follow these instructions will result in death or serious injury. |

## Requirements

* Sercos must be in phase 2.
* No motor data must be stored in the selected drive (use FB\_MotorDataDelete, if necessary).
* The *[MotorIdentification](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM62Drive&topicID=D_SE_0071763)* parameter of the axis must be set to the correct value.

## Description

This POU receives a BLH file with motor data and writes it to the selected axis.

The motor data is stored to the selected storage location.

* In the memory of the encoder, if sufficient memory is available in the encoder.
* Stored in the servo amplifier if the encoder has no or insufficient memory or if no encoder is used.

The motor data is permanently stored. Thus, the motor data must be written only once.

The POU has a long processing time (> 1 s). Therefore, it must be executed in a task that is not time-critical and with an appropriate watchdog time.

| DANGER | |
| --- | --- |
|  | UNINTENDED OPERATION OF THE MOTOR  Ensure that the typeplate of the new drive or motor is consistent with the typeplate of replaced drive or motor.  Failure to follow these instructions will result in death or serious injury. |

NOTE: When replacing drives or motors, make sure that the correct motor data is stored in the drive or motor. In all likelihood, the motor data must be rewritten.

If motor data is to be modified (for example, because the motor was replaced), you must first delete the existing motor data with FB\_MotorDataDelete.

## CheckingMotorDataInDrive

The function block is being executed. Waiting until q\_xDone has the value TRUE.

## Disabled

| Cause | Solution |
| --- | --- |
| The POU is disabled. | Set the i\_xEnable input from FALSE to TRUE to enable the POU. |

## Done

The motor data has been written successfully.

## FileAccessNotPossible

| Cause | Solution |
| --- | --- |
| The file cannot be accessed. | Verify whether the file is used elsewhere in the project. |

## FunctionNotSupportedByThisDevice

| Cause | Solution |
| --- | --- |
| Invalid object type | Verify the selected object. |

## InvalidStorageLocation

| Cause | Solution |
| --- | --- |
| There is no Hiperface encoder connected to the physical machine encoder input. | Connect a Hiperface encoder to the physical machine encoder input. |
| The encoder does not provide sufficient memory or there is no encoder. | Connect an encoder with a Hiperface interface with a minimum of 2 Kb of physical memory. |

## MotorDataAlreadyStoredInDrive

| Cause | Solution |
| --- | --- |
| The drive already contains the motor data. | The existing data must be deleted before new data can be written. Motor data can be deleted using FB\_MotorDataDelete. |

## Prepare

The POU in the q\_xBusy execution is TRUE.

## ReinitializationFailed

| Cause | Solution |
| --- | --- |
| Reinitialization was not successful. | Verify the Sercos phase. |
| The status of the procedure command Reinit cannot be verified. | Increase the value of the property timDelayPCMDReinitCheckStatus. |

## SercosCommunicationNotPossible

| Cause | Solution |
| --- | --- |
| Sercos communication is not possible. | Verify the Sercos phase and the wiring. |

## WaitForExecute

The POU is active and ready for execution. q\_xReady is TRUE.

## WaitForSercosPhase2

The POU is active and is waiting for Sercos phase 2. q\_xReady is FALSE.
