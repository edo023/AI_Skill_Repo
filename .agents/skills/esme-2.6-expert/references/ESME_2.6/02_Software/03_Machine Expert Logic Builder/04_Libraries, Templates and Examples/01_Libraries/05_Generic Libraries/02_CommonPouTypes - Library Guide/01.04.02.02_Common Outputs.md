# Common Outputs

The table provides an overview of the general outputs implemented by the base function blocks provided. The outputs are of type BOOL and the default value is FALSE:

| Common output | Description |
| --- | --- |
| q\_xActive | The output q\_xActive indicates that the function block is enabled.  If the function block is deactivated with a falling edge at i\_xEnable, the q\_xActive output remains TRUE until the internal execution has been completed (for example, free resources, close connections). Thus, the function block must be called until the output q\_xActive becomes FALSE. |
| q\_xReady | The output q\_xReady indicates that the initialization has been completed and that the function block can be controlled through its inputs according to the functionality.  If invalid input values are detected during initialization, q\_xReady remains FALSE. |
| q\_xBusy | The output q\_xBusy indicates an ongoing function block execution. If the execution has been completed (successfully or not) or has been aborted, the output q\_xBusy is set to FALSE. |
| q\_xDone | The output q\_xDone indicates that the execution of the function block has been completed successfully. |
| q\_xAborted | The output q\_xAborted indicates that the execution of the function block has been aborted with the input i\_xAbort. |
| q\_xError | The output q\_xError indicates that an error has been detected. |
