# AlarmState (ENUM)

TYPE AlarmState :

This enumeration defines the possible states an alarm can have.

| InOut: | | Name | Initial | Comment | | --- | --- | --- | | NotDefined | 16#FF | Alarm is not active. | | Normal | 0 | Alarm is not active. | | Active | 2 | Alarm is active (alarm condition is true). | | WaitingForConfirmation | 3 | Alarm is no longer active, but has to be confirmed. | | ActiveAcknowledged | 4 | Alarm is still active, has to be acknowledged. | | InactiveUnacknowledged | 5 | Alarm is inactive, has to be acknowledged. | | Pending | 1 | An state indicating, that the alarm condition is true, but the alarm is not yet in the state “Active” | |
