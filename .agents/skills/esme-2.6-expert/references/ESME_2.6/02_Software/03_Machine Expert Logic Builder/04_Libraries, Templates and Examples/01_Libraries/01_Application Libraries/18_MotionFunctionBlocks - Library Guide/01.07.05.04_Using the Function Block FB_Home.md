# Using the Function Block FB\_Home

## Overview

Operating principle of the function block FB\_Home:

| Step | Description |
| --- | --- |
| 1 | Initialization:  Before the function block FB\_Home can be used, it must be initialized using the method [FB\_Home.Init](FB_Home-InitMethod-46178154.html). During initialization, the function block is assigned to a specific axis where the homing operations are executed. The block is operational only after a successful initialization. |
| 2 | Cyclic Execution:  The method [FB\_Home.Cycle](FB_Home-CycleMethod-45AB3661.html) must be called in every PLC cycle. It handles two essential tasks:  * Monitoring ongoing homing processes * Providing diagnostic information on the function block  The cyclic call helps to ensure that the internal states and process transitions are processed consistently and reliably. |
| 3 | Starting a Homing Mode:  The ifCmd property provides access to the command methods used to start one of the supported homing modes.  The following steps are executed, when a command method is invoked:   1. Verification whether the axis is ready for the selected homing mode. 2. If the conditions for the axis to be ready are fulfilled, the corresponding homing operation on the axis is started.  NOTE: Do not call the same or a second command method during an active homing process. |
| 4 | Process Monitoring:  The present status of the active or completed homing process can be retrieved with ifCmd. stCmdStatus. It provides information on the present status of the homing operation: whether the homing operation is in progress, has completed successfully, or whether an error has been detected. |

NOTE:

* The function block is limited to homing functionalities supported by a motion axis. Administrative or other motion related operations should be implemented using [FB\_AxisMotion](FB_AxisMotion-4108E0AC.html).
* The function block must not be called directly.
* The function block is operated only with its methods and properties. Execute the method calls from the same task.
* When using multitask or multicore configurations, ensure that a method call is not interrupted or overlapped by a method call from a different task. Avoid parallel or concurrent access to the function block.

## Program Example

The following program example shows how to create an instance of FB\_Home, how to initialize it, execute a homing command, and monitor the process including error handling.

Declaration

|  |
| --- |
| ``` VAR    fbMotionAxis01     : MFB.FB_AxisMotion;    fbHomeAxis01       : MFB.FB_Home;    xInitDone          : BOOL;    diState            : DINT;    xError             : BOOL;    etResult           : MFB.ET_Result;    sResultMsg         : STRING(255);    xErrorHomeCmd      : BOOL;    etResultHomeCmd    : MFB.ET_Result;    sResultMsgHomeCmd  : STRING(255);    xCmdHome           : BOOL;    xCmdErrorReset     : BOOL; END_VAR ``` |

Implementation

|  |
| --- |
| ``` IF NOT xInitDone THEN         diState := 0;         fbHomeAxis01.Init(                 i_ifAxis         := DRV_01_A.Axis,                 q_xError         => xError,                 q_etResult       => etResult,                 q_sResultMsg     => sResultMsg);         IF NOT xError THEN                 xInitDone := TRUE;                 diState := 10;         END_IF END_IF  // cyclic call fbHomeAxis01.Cycle(q_xError => xError, q_etResult => etResult, q_sResultMsg => sResultMsg); ``` |
| ``` CASE diState OF         0: // idle                 ;         10: // initialized - wait for command                 IF xCmdHome THEN                         xCmdHome   := FALSE;                         fbHomeAxis01.ifCmd.SetPosition(                                i_lrPosition    := 0,                                q_xError        => xErrorHomeCmd,                                q_etResult      => etResultHomeCmd,                                q_sResultMsg    => sResultMsgHomeCmd);                         IF NOT xErrorHomeCmd THEN                                diState := 20; // homing in progress                         ELSE                                diState := 100; // error handling                         END_IF                  END_IF          20: // home in progress - wait for result                  IF NOT fbHomeAxis01.ifCmd.stGetCmdState.xBusy THEN                        IF fbHomeAxis01.ifCmd.stGetCmdState.xDone THEN                                 diState := 10; // homing done                        ELSIF fbHomeAxis01.ifCmd.stGetCmdState.xError THEN                                xErrorHomeCmd := TRUE;                                diState       := 100; // error handling                        END_IF                  END_IF                  etResultHomeCmd         :=   fbHomeAxis01.ifCmd.stGetCmdState. etResult;                  sResultMsgHomeCmd       :=   fbHomeAxis01.ifCmd.stGetCmdState. sResultMsg;          100: // error handling  -  wait for reset                  IF xCmdErrorReset THEN                        xCmdErrorReset := FALSE;                        xErrorHomeCmd   := FALSE;                        fbHomeAxis01.ifAdministrative.ResetResult();                        etResultHomeCmd   := fbHomeAxis01.ifCmd. stGetCmdState.etResult;                        sResultMsgHomeCmd := fbHomeAxis01.ifCmd. stGetCmdState.sResultMsg;                        diState           := 10; // reset done                  END_IF END_CASE ``` |

## Implementation Details

* The function block FB\_Home must be initialized by the method [FB\_Home.Init](FB_Home-InitMethod-46178154.html) before triggering homing commands. The reference to the motion device is handed over during initialization. If the initialization does not succeed, a diagnostic message is reported by the method outputs.
* The homing modes can be triggered by the call of dedicated methods of the FB\_Home available through the interface [IF\_HomeCmd](IF_HomeCmd-44A15B86.html). The interface IF\_HomeCmd is returned by the property ifCmd of the function block FB\_Home.
* In order to trigger a homing command successfully the axis must be in a valid state. Otherwise, the command is rejected and the diagnostic outputs of the called method indicate an error. Invalid input parameters are indicated by the diagnostic output of the method and the command state.
* If the homing process takes more than one cycle it is processed by the method [FB\_Home.Cycle](FB_Home-CycleMethod-45AB3661.html#FB_Home-CycleMethod-45AB3661). The FB\_Home.Cycle method must be called cyclically in the user application. The diagnostic outputs of this method indicate, together with the Command state, errors detected while processing a homing command.
* The Command State can be retrieved by property available in IF\_HomeCmd. It returns the structure ST\_HomeCmdState indicating the last accepted homing command and its corresponding state.

  NOTE: The command state should be verified cyclically after calling the FB\_Home.Cycle method.
* The homing process is based on function blocks of the PLCopen MC part 1 library (refer to the [M262 Synchronized Motion Control Library Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=PLCO&topicID=D_SE_0086552)). In the event of the process being aborted by a concurrent function block call, FB\_Home reports by the Command State that the process is aborted.
* If an online change occurs the function block FB\_Home detects the event and aborts the command if the online change is not secure (for example HomingToDigitalInput is referenced as an input when the command is triggered). Therefore, the MC\_Stop is used with the motion parameters configured for the motion device.
