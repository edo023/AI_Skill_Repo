# Operating the Application

## General

The application example implements a web visualization for monitoring and control purposes.

This section provides a brief description of each function. In addition, refer to the [visualization screen](D-SE-0090023.html#D-SE-0090023).

## Open the Web Visualization

To open the web visualization, enter the following address in the address bar of your web browser:

*http://*<IP Address form the controller>*:8080\webvisu.htm*

NOTE: By default, the web visualization protocol is blocked by the firewall of the controller. Consult the Programming Guide specific to your controller for information about the configuration of the firewall.

## Obtain the IPv4 Address Using the DNS Client

For the direct connection to the MQTT server, the MQTT client requires the IPv4 address of the server. The IPv4 address can be obtained by using the implemented DNS client functionality. Therefore, switch to the DNS Client / Ping Service visualization screen and send a request to a reachable DNS server to get resolved IP address for the specified domain name.

## Establish a Connection to an MQTT server

Before you can establish a connection, you must specify the corresponding parameters in the MQTT Client - Connect screen.

* Depending on the network and the server you want to connect to, activate or deactivate the checkbox Connect via Proxy[1)](#fntarg_1).
* For connection through proxy server:
  + Enter the IP address of the proxy server (also refer to the DNS Client / Ping Service visualization)
  + Enter the IP address or the Hostname of the MQTT server
* For direct connection to the MQTT server:
  + Enter the IP address of the MQTT server (also refer to the DNS Client / Ping Service visualization)
* Adjust, if required, the Port number where the MQTT server is listening
* Enter your Client ID
* Enter Username and Password if required
* Adjust the Advanced connection parameter if required

In case of TLS encryption accepting only trusted certificates the server certificate has to be known and rated as trusted. For more information about certificate management on the controller, refer to [*How to Manage Certificates on the Controller - User Guide*](../../../../../api/crossBook?lang=en-US&virtualBookName=HowMgCer&topicID=D_SE_0095876).

To establish the connection, click Connect. The progress and the result of the operation are displayed in the Status area on the visualization screen. After the connection is successfully established, the connection status changes to Connected. The button Connect changes to Disconnect and can be used to close the connection.

The application example implements an auto reconnect function which can be activated / deactivated at any time.

## Publish

Before you can publish messages, you must specify the corresponding parameters.

* Enter the name for the Topic
* Select the level for Quality of Service (QoS)
* Activate the checkbox Retain if required
* Adjust the values for timeout and retransmission if required (only for QoS1)

The application implements a simulation mode for publishing automatically created messages in JSON format which are sent continuously with an adjustable time interval. The simulation mode is started by activating the check box Simulation mode.

For manual publishing of messages, enter the data to be published into the table in the Data area and click Publish.

The progress and the result of the operations are displayed in the Status area from the last operation on the visualization screen.

## Subscribe

Before you can subscribe to a topic and receive corresponding messages, you must specify the relevant parameters.

* Enter name for the Topic to be subscribed
* Select the level for Quality of Service (QoS)
* Adjust the values for timeout and retransmission if required (only for QoS1)

To subscribe to the specified topic, click Subscribe. The progress and the result of the operation are displayed in the Status area on the visualization screen. After the topic is successfully subscribed, the indicator Ready illuminates (green). The button Subscribe changes to Unsubscribe and can be used to cancel the subscription.

As long the subscription to a topic is active, the client receives published messages for that topic. If a new message has been received, the indicator New data available in the Received Data area illuminates (green). By clicking Get message, the message is copied to the process variable which is displayed in the field Message.

The application implements an automatic mode for copying the message whenever a new message is available. The automatic mode can be activated via check box Auto mode.

[1](#fnsrc_1) This example implements a solution for establishing a connection to an MQTT server through a proxy server. This solution is working for proxy servers which support the establishing of a tunneled connection to the remote server using the HTTP request CONNECT.
