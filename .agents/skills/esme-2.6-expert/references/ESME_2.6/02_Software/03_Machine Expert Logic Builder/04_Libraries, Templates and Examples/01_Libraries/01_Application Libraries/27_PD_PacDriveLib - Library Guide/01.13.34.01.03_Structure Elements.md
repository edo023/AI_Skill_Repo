# Structure Elements

Structure Elements

| Variable | Data type | Description |
| --- | --- | --- |
| diStepNo | DINT | Number in ascending sequence (1, 2, 3, or 10, 20, 30 ... ); 0 = End of the table |
| lrPos | LREAL | Target position or distance of the motion |
| etPosMode | [ET\_PosMode](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.SystemInterface&topicID=D_SE_0084933_1) | Type of positioning |
| lrVel | LREAL | Velocity in units/s. Value range: 0 ... MaxVel. |
| lrEndVel | LREAL | Velocity in units/s at the end of the positioning. Value range: 0 ... MaxVel. |
| lrAcc | LREAL | Acceleration in units/s2. Value range: >0 ... MaxAcc. |
| lrDec | LREAL | Deceleration in units/s2. Value range: >0 ... MaxAcc. |
| lrJerk | LREAL | Preset of the maximum jerk in units/s3 |
| diStepRepeat | DINT | Branching out only occurs after the first step when all repetitions are processed. In single step mode (i\_xSingleStepMode), only a start for the entire step is required. diStepRepeat = 0 means a single implementation of the step. |
| timDwellTime | TIME | Dwell time after positioning. Only after this period of time has elapsed, will there be a switchover of the next step. If positioning is not programmed, the dwell time starts immediately. Dwell time has no effect during rounding. |
| byStartCondition | BYTE | Start condition for this step |
| byOutput | BYTE | Bit pattern which is active at output q\_byOutput while the step is active |
| diJumpStep | DINT | If the jump condition (byJumpCond) is fulfilled, jumping to a step (diStepNo) is specified using diJumpStep. Jumps can be in all directions including within the step itself. The jump is executed at the end of the step. |
| byJumpCondition | BYTE | Jump condition for diJumpStep |
