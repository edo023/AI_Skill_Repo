# Description

Description

The POU monitors the status of the SERCOS loop. Each axis’ HomeOK signal is reset to FALSE if the ring is not in phase 4.

The POU distributes the jogging signals. The jogging signals are contained in the standard motion interface specified by the iq\_stMachineMotionItf input. The StandardMotionInterface also contains the i\_udiModuleSelect parameter which selects which axis is to receive the jog signal. The jog signals to the other axes are reset to false.

The functions of the POU are available as long as the machine is activated (iq\_stMachineItf.i\_xEnable). The POU indicates that it is enabled and working correctly with the q\_xActive and q\_xReady outputs. The q\_xActive output will not turn on if the sub modules (axes) were not incorporated into the standard module interface for the machine correctly. The inverter-enable and jog signals are reset to false when the function block is not active.

The q\_etDiag <> GD.ET\_Diag.Ok output indicates that the POU is in an exception state. An exception condition will also reset the q\_xReady output to FALSE. The exception could be caused by one of the following conditions:

o The quantity of sub modules (axes) contained by the standard module interface for the machine doesn’t agree with the count contained in the sub module add interface

o There is a configuration error with a sub module (axis) within the sub module add interface

The global exception list specified with the iq\_stExceptionList input and the global logging list specified with the iq\_stLogDataList input are used to log exceptions.

This is a function block that will require an instance of it to be declared.
