# Using the Results of a Teaching Procedure

## Overview

A teaching procedure returns the global pose of either the robot or another coordinate system, depending on which of the two is used as reference system by the procedure. If a teaching procedure is used in a project based on [FB\_EntitiesHandler](D-SE-0098031.html), then it is possible to update at runtime the base pose of a certain entity by calling again the proper Add method (for example, [AddRobot](IF_EntitiesHandler-AddRobotMethod-CDC90516.html)) and providing the global pose estimated by the teaching as a new base pose value.

After that, the definition of the tracking coordinate systems for each robot and/or each tracking system involved must also be updated. This can be done by calling ROB.ModifyCoordinateSystem2 if the robot is not moving.

If no [FB\_EntitiesHandler](D-SE-0098031.html) is used inside the project, it is usually only required to handle the definition of the relative transformations from the coordinate systems of the individual robots to tracking systems. This can be done by calling ModifyCoordinateSystem2.

The output q\_stRobotToOtherSystemPose of the method EstimateCartesianPose returns the relative cartesian transformation from the coordinate system of the robot to the coordinate system of the other system involved in the teaching procedure. If the other system is a tracking system, the value can be used to update the position and orientation of the tracking system with reference to the coordinate system of the robot using ModifyCoordinateSystem2.
