## Task

Initializes a diagnostic handler with an ID, a name, and allows to assign a parent diagnostic handler.

## Description

The method InitHandler is used to initialize a diagnostic handler with an ID, a name, and allows to assign a parent diagnostic handler. To simplify the identification of a diagnostic handler inside your machine, it is a good practice to use unique IDs and names.

NOTE: All diagnostic handlers assigned to the same parent must have a unique ID.

## Return Value

| Data type | Description |
| --- | --- |
| BOOL | Indicates TRUE if diagnostic handler has been initialized successfully. |

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_udiId | UDINT | Specifies the ID of the diagnostic handler. |
| i\_sName | STRING[80] | Specifies the name of the diagnostic handler. |
| i\_ifParent | [IF\_DiagHandlerAccess](IF_DiagHandlerAccess-44A5CB2A.html#IF_DiagHandlerAccess-44A5CB2A) | Specifies the parent of the diagnostic handler (optional).  If this input is not assigned or set to 0, the diagnostic handler has no parent. |

| Output | Data type | Description |
| --- | --- | --- |
| q\_xError | BOOL | Value is TRUE if an error was detected. For details, refer to q\_etResult and q\_sResultMsg. |
| q\_etResult | [ET\_Result](ET_Result-GeneralInformation-36C542C6.html) | Provides diagnostic and status information as a numeric value.  If q\_xError  = FALSE, q\_etResult provides status information. If q\_xError  = TRUE, q\_etResult provides diagnostic/error information. |
| q\_sResultMsg | STRING[80] | Provides additional diagnostic and status information as a text value. |
