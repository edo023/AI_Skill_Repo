# FB\_MotorCustomizedKeyWrite - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Function block |
| Available as of: | V.1.2.0.0 |

## Task

Writing the customized key of a motor.

## Description

This function block writes the customized key in the electronic motor typeplate.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_xEnable | BOOL | A rising edge FALSE -> TRUE activates the POU, a falling edge TRUE -> FALSE deactivates the POU.  A deactivated POU does not execute any actions. |
| i\_xExecute | BOOL | FALSE -> TRUE: The POU writes the customized key to the selected axis. During the writing procedure q\_xBusy = TRUE. As soon as the writing procedure is completed, q\_xDone changes to TRUE. |
| i\_ifDrive | [SystemConfigurationItf.IF\_Drive](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.SystemConfigurationItf&topicID=D_SE_0089154) | Input for the axis that is to be controlled. |
| i\_etStorageLocation | [ET\_StorageLocation](D-SE-0090526.html#D-SE-0090526) | Storage location to which the customized key is to be written (default value: drive). |
| i\_abyCustomizedKey | ARRAY [0..7] OF BYTE | Customized key to be written to the typeplate. Numbers must be provided in ASCII format. |
| i\_xReinit | BOOL | TRUE: After writing the customized key, the drive is reinitialized and the motor typeplate is read.  FALSE: The drive is not reinitialized and the motor typeplate is not read. |

| Output | Data type | Description |
| --- | --- | --- |
| q\_xActive | BOOL | TRUE: The POU is active and has to be executed further.  FALSE: The POU is inactive. |
| q\_xReady | BOOL | TRUE: The POU is ready to operate and can accept user commands.  FALSE: The function block is not ready to accept user commands. |
| q\_xBusy | BOOL | TRUE: The POU executes the issued user command.  FALSE: The POU is waiting for further user commands. |
| q\_xDone | BOOL | TRUE: The user command has been executed.  FALSE: The user command is being executed, or no user command has been issued. |
| q\_etDiag | [GD.ET\_Diag](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.GlobalDiagnostic&topicID=D_SE_0076228) | General library-independent statement on the diagnostic.  A value unequal to GD.ET\_Diag.Ok corresponds to a diagnostic message. |
| q\_etDiagExt | [ET\_DiagExt](D-SE-0090520.html#D-SE-0090520) | POU-specific output on the diagnostic.  q\_etDiag = GD.ET\_Diag.Ok -> status message  q\_etDiag <> GD.ET\_Diag.Ok -> diagnostic message |
| q\_sMsg | STRING[80] | Event-triggered message which gives more detailed information on the diagnostic state. |

## Properties

## Disabled

| Cause | Solution |
| --- | --- |
| The POU is disabled. | Set the i\_xEnable input from FALSE to TRUE to enable the POU. |

## Done

The motor customized key was successfully written.

## Executing

The POU is being executed.

## FunctionNotSupportedByThisDevice

| Cause | Solution |
| --- | --- |
| Invalid object type | Verify the selected object. |

## ReinitializationFailed

| Cause | Solution |
| --- | --- |
| Reinitialization was not successful. | Verify the Sercos phase. |
| The status of the procedure command Reinit cannot be verified. | Increase the value of the property timDelayPCMDReinitCheckStatus. |

## SercosCommunicationNotPossible

| Cause | Solution |
| --- | --- |
| Sercos communication is not possible. | Verify the Sercos phase and the wiring. |

## WaitForExecute

The POU is active and ready for execution. q\_xReady is TRUE.

## WaitForSercosPhase2

The POU is active and is waiting for Sercos phase 2. q\_xReady is FALSE.
