# Common Inputs

The table provides an overview of the general inputs implemented by the base function blocks provided. The inputs are of type BOOL:

| Common input | Description |
| --- | --- |
| i\_xEnable | A rising edge of the input i\_xEnable activates and initializes the function block and a falling edge deactivates the function block. A deactivated function block (q\_xActive = FALSE) does not execute actions and the outputs are set to default values.  If the execution of the function block is unsuccessful, a new rising edge is required to reactivate the function block. |
| i\_xExecute | A rising edge of the input i\_xExecute starts the execution of the function block.  Additional rising edges while the function is being executed are ignored.  A falling edge when the execution is finished resets the function block and the function block is ready for a new execution.  One of the outputs q\_xDone, q\_xAborted or q\_xError remains TRUE until the input i\_xExecute is set to FALSE. If the input is set to FALSE before the execution is finished, the output q\_xDone, q\_xAborted or q\_xError is set to TRUE for one cycle. |
| i\_xAbort | A rising edge of the input i\_xAbort aborts the execution of the function block.  Trying to execute the function block with a rising edge of the input i\_xExecute while i\_xAbort is TRUE does not start the execution: the output q\_xAborted remains TRUE. |
| i\_xErrorQuit | A rising edge of the input i\_xErrorQuit acknowledges the error state and sets the q\_xError output to FALSE. |
