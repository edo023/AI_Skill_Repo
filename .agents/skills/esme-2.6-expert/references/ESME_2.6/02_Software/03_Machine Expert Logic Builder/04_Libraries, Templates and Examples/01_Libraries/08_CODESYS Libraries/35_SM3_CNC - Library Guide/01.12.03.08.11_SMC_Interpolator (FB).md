# SMC\_Interpolator (FB)

FUNCTION\_BLOCK SMC\_Interpolator IMPLEMENTS [SMC\_IInterpolator](pSc41CfRieQozA2o_fxRBRpweGA_smc-iinterpolato.html#psc41cfrieqoza2o_fxrbrpwega_smc_iinterpolato_smc_iinterpolator_itf)

This function block is used to convert a continuous path described by [SMC\_GEOINFO](cAH0ifE6NckjZp5eF3Qpy-filyk_smc-geoinfo.html#cah0ife6nckjzp5ef3qpy_filyk_smc_geoinfo_smc_geoinfo_struct) objects into discrete path position points taking into account a defined velocity profile and time pattern. Afterwards, these position points will typically be transformed by the IEC-program (e.g. to drive-axis-positions) and sent to the drives.

**Note**

At the end of a path run, the input path queue is empty. If you want to process the same outline once again, you either have to transform the CNC program via decoder and path preprocessing modules to a [SMC\_OUTQUEUE](cAH0ifE6NckjZp5eF3Qpy-filyk_smc-outqueue.html#cah0ife6nckjzp5ef3qpy_filyk_smc_outqueue_smc_outqueue_fb) structure, or you have to use the function block [SMC\_RestoreQueue](feQc-ora8AEm1NYfJrbEBiXVSiM_smc-restorequeue.html#feqc_ora8aem1nyfjrbebixvsim_smc_restorequeue_smc_restorequeue_fun) (also part of SM3\_CNC). This is only possible if the path queue is big enough to hold the complete path.
