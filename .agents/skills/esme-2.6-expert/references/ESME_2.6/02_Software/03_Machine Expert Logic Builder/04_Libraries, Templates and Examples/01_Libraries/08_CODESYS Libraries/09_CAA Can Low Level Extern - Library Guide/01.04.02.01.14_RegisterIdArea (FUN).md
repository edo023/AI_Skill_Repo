# RegisterIdArea (FUN)

FUNCTION RegisterIdArea : [ERROR](gBaR_WSLC7pYiPIcZ7U3-TDNYt0_error.html#gbar_wslc7pyipicz7u3_tdnyt0_error_error_enum)

This function registers a CAN identifier area to an IdAreaReceiver created by [CreateIdAreaReceiver](UzaP1MMIvsGx5wgQfvzFSZKw87g_createidarearece.html#uzap1mmivsgx5wgqfvzfszkw87g_createidarearece_createidareareceiver_fun). If the parameter cobIdStart equals cobIdEnd only one identifier is registered. The values of the masks activate the evaluation of the value parameters. If the mask value is FALSE the value parameter is ignored.

Receivers can be also used for Tx loopback. If xTransmitMask is set to FALSE or xTransmitMask and xTransmitValue are set to TRUE CAN messages which are sent via [Write](UzaP1MMIvsGx5wgQfvzFSZKw87g_write.html#uzap1mmivsgx5wgqfvzfszkw87g_write_write_fun) will be received (applies to all transmit messages on the CAN interface). Use function [IsTransmitMessage](piu7Mxibn-WYn7nN5fzBEdtBTsE_istransmitmessag.html#piu7mxibn_wyn7nn5fzbedtbtse_istransmitmessag_istransmitmessage_fun) to distinguish between receive and transmit messages.

**Note**

Current implementation of this receiver type does not support 29 bit identifiers. Set x29BitIdValue always to FALSE and x29BitIdMask to TRUE. Otherwise an error will be returned.

**Example**

See [CreateIdAreaReceiver](UzaP1MMIvsGx5wgQfvzFSZKw87g_createidarearece.html#uzap1mmivsgx5wgqfvzfszkw87g_createidarearece_createidareareceiver_fun).
