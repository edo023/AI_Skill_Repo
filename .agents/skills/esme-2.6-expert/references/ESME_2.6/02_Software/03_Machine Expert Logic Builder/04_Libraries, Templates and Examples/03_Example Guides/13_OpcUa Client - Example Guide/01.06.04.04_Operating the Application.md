# Operating the Application

## General

The application example implements a web visualization for monitoring and control purposes.

This section provides a brief description of each function. In addition, refer to the [visualization screen](D-SE-0101419.html#D-SE-0101419).

## Open the Web Visualization

To open the web visualization, enter the following address in the address bar of your web browser:

* For Modicon M262 controllers:

  *https://<IP Address of the controller>:8089/webvisu.htm*
* For PacDrive LMC controllers:

  *https://<IP Address of the controller>/webvisu.htm*

When you are connecting to the controller web visualization, the login dialog box (Dialog\_Visu\_Login) is displayed. Only users with access rights to the runtime object WEB can connect to the web visualization. Refer to the user management of the controller.

## Establish a Connection to an OPC UA Server

Before you can establish a connection, you must specify the corresponding parameters.

* Enter the Server Endpoint URL to the controller where the OPC UA server is hosted.
* Enter the username and password for user authentication on the OPC UA server.
* Adjust the Advanced connection parameter if required.

To establish the connection, click Connect. The progress of the operation is displayed in the display Status. After the connection is successfully established, the namespace indexes and the node handles are retrieved. The status and the result of the single sequences is indicated by the corresponding indicators in the Status display.

NOTE: The application example also provides an OPC UA server implementation. The preset Server Endpoint URL references to the controller running the application example.

## Server Status

If a connection is successfully established, switch to the visualization screen Visu\_Main > Data exchange. The displayed server status is verified cyclically with a configurable interval. The progress and the result of the latest operation is displayed by the corresponding indicator lights on the visualization screen.

## Read Variables

To read the variables from the OPC UA server, click Read List. The progress and the result of the latest operation is displayed by the corresponding indicator lights on the visualization screen.

Additionally, the application implements an automatic mode for cyclic reading the variables from the OPC UA server. Start the automatic mode by activating the corresponding check box Auto mode. The interval for cyclic reading can be adjusted by the corresponding parameter next to the check box.

## Write Variables

To write the variables to the OPC UA server, click Write List. The progress and the result of the last operation is displayed by the corresponding indicator lights on the visualization screen.

Additionally, the application implements an automatic mode for cyclic writing the variables from the OPC UA server. Start the automatic mode by activating the corresponding check box Auto mode. The interval for cyclic writing can be adjusted by the corresponding parameter next to the check box.

## Subscribe

Before you subscribe, choose the synchronization mode to use by selecting the option Controller Sync or Firmware Sync.

To create the subscription and to monitor the items, click Subscribe. The progress and the result of the last operation is displayed by the corresponding indicator lights on the visualization screen.

For controller synchronization mode, click the button Operate to update the values of the monitored items. Additionally, the application implements an automatic mode for cyclic updating the variables. Start the automatic mode by selecting the option Auto mode. You can adjust the interval for cyclic updating with the corresponding parameter next to the option.

For firmware synchronization mode, the values of the monitored items are updated automatically by the underlying system. The indicator light illuminates if at least one new published value is available.

## Simulation

Display the simulation which exchanges values of the variables with the OPC UA server by activating the check box Simulation.
