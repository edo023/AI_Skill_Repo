# FB\_PulseWidthModulator Troubleshooting

## Troubleshooting

This table describes the possible issues and their solutions:

| Issue  Outputs of the function indicate the values | Cause | Solution |
| --- | --- | --- |
| q\_xError = TRUE  q\_etResult = InvalidInput | The value of i\_timPeriod is 0. | The value of i\_timPeriod must be greater than 0. |
| q\_xError = TRUE  q\_etResult = InvalidInput | The value of i\_lrDeadBand is out of range. | The value of i\_lrDeadBand must be in range 0.0...0.5. |
| q\_xError = TRUE  q\_etResult = InvalidInput | The value of i\_lrDutyCycle is out of range. | The value of i\_lrDutyCycle must be in range 0.0...1.0. |
| q\_xError = TRUE  q\_etResult = CycleTimeInvalid | The calling interval of the function block is greater than the time between two switching operations of the output. | The calling interval of the function block must not be greater than the time between two switching operations of the output. The time depends on i\_timPeriod, i\_lrDutyCycle, and i\_lrDeadBand. |
| q\_xError = TRUE  q\_etResult = InternalError | The execution of the function block ended with an indeterminable state. | Disable and enable the function block. |
