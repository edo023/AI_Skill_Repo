# Migration From Other Libraries

## General

The Mathematics library contains functions that are similar to functions from other libraries, such as GeoMath or PD\_PacDriveLib. An overview of the functions that can be replaced by functions from the Mathematics library can be found in the [Compatibility and Migration - User Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=CompMigr&topicID=ConversionTableForMathematicFunctio_3D6AFDE2).

## Migrating Functions from GeoMath

The interfaces of functions previously used in the GeoMath library have not been modified and can be migrated by adapting the namespace. The functions from the GeoMath library are included in the Mathematics library.

Procedure for migrating from the GeoMath library to the Mathematics library:

1. Add the Mathematics library to the Library Manager.
2. In the program code, replace the namespace **GEM** with **SE\_MATH**.
3. Remove the GeoMath library from the Library Manager.

## Migrating Functions from PD\_PacDriveLib

Almost all mathematical functions of the PD\_PacDriveLib library are available in the Mathematics library. The functions have an updated interface. The new, corresponding names can be found in the [Compatibility and Migration - User Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=CompMigr&topicID=ConversionTableForMathematicFunctio_3D6AFDE2).

Procedure for migrating from the PD\_PacDriveLib library to the Mathematics library:

1. Add the Mathematics library to the Library Manager.
2. Determine which function of the Mathematics library replaces the corresponding function from the PD\_PacDriveLib.
3. Replace the previous function name **PDL.FC\_xyz** by the corresponding function name **SE\_MATH.FC\_zyx**.
4. Adapt the function call (input names changed, diagnostic outputs changed).

   * If the input names of the function have changed, identify the new input names according to the present document.
5. Adapt exception handling

   * The outputs etDiag and etDiagExt have been removed. Now, there are three outputs for error handling:

     + q\_xError indicates the status of the function.
     + q\_etResult provides status information when the function is working and diagnostic information in the case of an error.
     + q\_sResultMsg provides additional information.
6. Remove the PD\_PacDriveLib library from the Library Manager.
