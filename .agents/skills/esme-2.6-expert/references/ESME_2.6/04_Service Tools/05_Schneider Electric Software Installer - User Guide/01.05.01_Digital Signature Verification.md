# Digital Signature Verification

## Verifying the Integrity of the Schneider Electric Software Installer

When the Schneider Electric Software Installer is started, each loaded dynamic-link library (DLL) is scanned to verify whether or not it is trusted. This is a built-in feature to help increase the trust level.

## Reporting Untrusted Components

If untrusted DLLs are detected, errors are reported:

* In the dialog box Integrity Verification.

  The start-up of the Schneider Electric Software Installer is suspended. You have the choice to continue or terminate the process.
* In the console, such as Logic Builder Shell.

  The process is terminated.

The result of the verification is documented in the log file of the Schneider Electric Software Installer.

## Verification Log Information

The digital signature verification results can be exported to the Verification Log (see dialog box Integrity Verification).

Untrusted digital signature verification results are written to:

* the Windows EventLog (`Source: "Schneider Electric Software"`).
* the log file that is stored by default on the Desktop when the Schneider Electric Software Installer is started without user interface. When the Schneider Electric Software Installer is started with user interface, you can browse to a folder of your choice to save the log file.

## How to Proceed when an Error is Detected

When an error is detected, the dialog box Integrity Verification is displayed when the Schneider Electric Software Installer is started. It provides the following options:

* The Export Verification Log... button allows you to save the log information to an XML file. Consult this file to get more information about the DLLs that were detected as untrusted.
* The Exit  button allows you to exit the installer.
* To continue using the installer, select the check box indicating that you have taken the notification about untrusted components into account and click the button Continue. Note that the notification will appear with every start of the installer.

Analyze each notification before you continue to use the installer. As there are many reasons for untrusted components, there are also different ways to proceed:

* If an expected reason is reported, you can continue to use the installer.
* If an unexpected reason triggered by an installed third-party product on the same PC is reported, you can continue to use the installer.
* If, however, an untrusted component is reported for an unexpected reason and for which you do not recognize the component, analyze the Verification Log to find out why a DLL was detected as untrusted before you continue to use the installer.
