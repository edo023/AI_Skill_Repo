# Digital Signature Verification

## Verifying the Integrity of the SQL Gateway Service and SQL Gateway Console

When the SQL Gateway service or the SQL Gateway Console is started, each loaded dynamic-link library (DLL) is scanned to verify whether or not it is trusted. This is a built-in security feature to help increase the trust level.

## Reporting Untrusted Components

If untrusted DLLs are detected, you are informed depending on the verification level:

* In the status bar.
* In the dialog box Integrity Verification.

## Verification Levels

The table lists the verification levels and the behavior of the SQL Gateway Console and the SQL Gateway service:

| Verification Level | SQL Gateway Console | SQL Gateway service |
| --- | --- | --- |
| `Error` (default) | The Integrity Verification dialog box is displayed (blocking start-up of the SQL Gateway Console).  Refer to the section [How to Proceed from the Integrity Verification Dialog Box](#DigiSignatureVerif-CD8EB964__HowToProceedFromTheIntegrityVerific-CD9CB945). | * Database connections are terminated. * No further database access is allowed. * Untrusted DLLs are reported to the SQL Gateway Console. |
| `Warning` | A notification is displayed in the status bar. | * Untrusted DLLs are reported to the SQL Gateway Console. * Database access is allowed. |
| `Silent` |

The verification level is applied per process and is not set globally. Once the verification level is set, it cannot be changed for a running software process.

## Configuring the Verification Level

Different options are provided for configuring the verification level.

* As the preferred option, edit the value of the `SignatureVerificationWarningLevel` in the configuration files:

  + SchneiderElectric.SqlGateway.exe.config (for the SQL Gateway Windows service application)
  + SchneiderElectric.SqlGatewayConsole.exe.config (for the SQL Gateway Console)

  ```
  <?xml version="1.0"?>
  <configuration>
      <appSettings>
          <!-- Default SignatureVerificationWarningLevel [Error, Warning, Silent]. It can be overriden by command line arguments. -->
          <add key="SignatureVerificationWarningLevel" value="Error"/>
      </appSettings>
  </configuration>
  ```

  By default, the `SignatureVerificationWarningLevel` is set to the value `Error`.
* With the command line argument

  ```
  "-- SignatureVerificationWarningLevel=[Error|Warning|Silent]"
  ```

  To specify the verification level argument, adapt the Start menu shortcut.

  NOTE: The verification level set with the command line argument overrides the value configured in the configuration file of the application.

  Perform the following steps:

  1. Stop the service.

     ```
     sc stop "SchneiderElectric SqlGateway Service"
     ```
  2. Delete the service.

     ```
     sc delete "SchneiderElectric SqlGateway Service"
     ```
  3. Create service with command line.

     ```
     sc create "SchneiderElectric SqlGateway Service" binPath="C:\Program Files (x86)\Schneider Electric\EcoStruxure Machine Expert\Tools\SQL Gateway\SchneiderElectric.SqlGateway.exe --SignatureVerificationWarningLevel=Silent" start=auto
     ```
  4. Configure service attributes.
  5. Start the service.

     ```
     sc start "SchneiderElectric SqlGateway Service"
     ```

## Verification Log Information

The digital signature verification results can be exported to the Verification Log (see dialog box Integrity Verification).

Untrusted digital signature verification results are written to the Windows EventLog (`Source: "Schneider Electric Software"`).

## Evaluating the Reasons for Untrusted Components

Analyze each notification of an untrusted DLL before you continue to work with the software. As there are many reasons for untrusted components, there are also different ways to proceed:

* If an expected reason is reported, you can continue to work with the software.
* If an unexpected reason triggered by an installed third-party product on the same PC is reported, you can continue to work with the software.
* If, however, an untrusted component is reported for an unexpected reason and for which you do not recognize the component, analyze the Verification Log to find out why a DLL was detected as untrusted before you continue to work with the software.

## How to Proceed from the Integrity Verification Dialog Box

When untrusted DLLs are detected, the Integrity Verification dialog box of the SQL Gateway Console provides the following options to proceed:

* To save the log information to an XML file, click the Export Verification Log... button. Consult this file to get more information about the DLLs that were detected as untrusted.
* To exit the software promptly, click the Exit  button. The SQL Gateway Console is closed and the SQL Gateway service is stopped.
* To continue using the SQL Gateway service and the SQL Gateway Console, select the check box indicating that you have taken the notification about untrusted components into account and click the button Continue with SQL Gateway.

  **Result**: Detected DLLs are acknowledged as trusted and database access is allowed for the active session. Every restart requires new confirmation.
