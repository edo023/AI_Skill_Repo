# Configuring an Unsecured Connection Between a Controller and the SQL Gateway

## Overview

If your controller does not support secured communication, you can configure unsecured communication between the controller and the SQL Gateway.

For configuring a secured connection between the controller and the SQL Gateway, execute the following steps that are described in this chapter:

* Activating TCP port for unsecured communication.
* As an optional step, define a list of Permitted Clients.

## Activating TCP Port for Unsecured Communications

To activate the TCP port for unsecured communications, proceed as follows:

| Step | Action |
| --- | --- |
| 1 | Select the Settings tab of the SQL Gateway. |
| 2 | Select the node Client Communication from the tree on the left-hand side. |
| 3 | In the Client Communication view on the right-hand side, select the option TCP Port. |
| 4 | Adapt the default port number 3457 to your individual configuration, if required. |

NOTE: With the default setting TCP Port 3457 selected and the option Permitted Clients Only not selected, communication on this port is not restricted. If your controller does not support secured TLS communication, it is a good practice to activate the option Permitted Clients Only and configure the IP addresses of trusted clients in the Permitted Clients (Unsecured) view.

## Configuring Controller Addresses as Permitted Clients

To define controllers that do not support secured TLS communication as permitted clients for the SQL Gateway, proceed as follows:

| Step | Action |
| --- | --- |
| 1 | Select the Settings tab of the SQL Gateway. |
| 2 | Select the node Client Communication > Permitted Clients (Unsecured) from the tree on the left-hand side. |
| 3 | Click Add Client.  **Result**: The Create Client dialog box opens. |
| 4 | Enter the IP Address and the Name of the controller you want to define as permitted client, and click the OK button.  **Result**: The dialog box closes and the controller is added as an entry of the list of **Permitted Clients (Unsecured)**. |

After you have defined controllers, you can edit the entries in the list.

To remove an entry from the list, click the Delete button.
