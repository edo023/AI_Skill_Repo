# Digital Signature Verification

## Verifying the Integrity of Your Software

When your software (such as EcoStruxure Machine Expert) is started, each loaded dynamic-link library (DLL) is scanned to verify whether or not it is trusted. This is a built-in security feature to help increase the trust level.

## Reporting Untrusted Components

If untrusted DLLs are detected, you are informed depending on the verification level:

* In the status bar.
* In the dialog box Integrity Verification.
* In the console, such as Logic Builder Shell.

The result of the verification is documented in the log file of your software.

## Verification Levels

The table lists the verification levels and the behavior in the software depending on whether it is started with our without user interface:

| Verification Level | Software started with user interface | Software started without user interface |
| --- | --- | --- |
| `Error` (default) | The dialog box Integrity Verification is displayed (blocking start-up of your software).  You have the choice to continue or terminate the process. | The error message is written to the console, such as Logic Builder Shell, and to the log file of your software.  The process is terminated. |
| `Warning` | A notification is displayed in the status bar and is documented in the log file of your software.  The process continues. | A notification is written to the console, such as Logic Builder Shell, and to the log file of your software.  The process continues. |
| `Silent` | The notification is documented in the log file of your software. |

The verification level is applied per process and is not set globally. Once the verification level is set, it cannot be changed for a running software process.

## Configuring the Verification Level

Different options are provided for configuring the verification level.

* As the preferred option, edit the value of the `SignatureVerificationWarningLevel` in the \*.exe.config configuration file of your Schneider Electric application that is available in the installation directory (*SchneiderElectric.<NAMEOFYOURAPPLICATION>.exe.config*).

  ```
  <?xml version="1.0"?>
  <configuration>
      <appSettings>
          <!-- Default SignatureVerificationWarningLevel [Error, Warning, Silent]. It can be overriden by command line arguments. -->
          <add key="SignatureVerificationWarningLevel" value="Error"/>
      </appSettings>
  </configuration>
  ```

  By default, the `SignatureVerificationWarningLevel` is set to the value `Error`.
* With the command line argument

  ```
  "-- SignatureVerificationWarningLevel=[Error|Warning|Silent]"
  ```

  To specify the verification level argument, run your software from console or adapt the Start menu shortcut.

NOTE: The verification level set with the command line argument overrides the value configured in the configuration file of the application.

## Verification Log Information

The digital signature verification results can be exported to the Verification Log (see dialog box Integrity Verification).

Untrusted digital signature verification results are written to:

* the Windows EventLog (`Source: "Schneider Electric Software"`).
* the log file of your software, such as EcoStruxure Machine Expert (see the *\TEMP\* directory).

## How to Proceed Depending on the Verification Level

Verification level = `Error`:

If the verification level `Error` applies, the dialog box Integrity Verification is displayed when your software is started. It provides the following options:

* The Export Verification Log... button allows you to save the log information to an XML file. Consult this file to get more information about the DLLs that were detected as untrusted.
* The Exit  button allows you to exit the software promptly.
* To continue using your software, select the check box indicating that you have taken the notification about untrusted components into account and click the button Continue. Note that the notification will appear with every start of your software.

Verification level = `Warning`:

If the verification level `Warning` applies, the startup process of your software continues and a notification is displayed in the statusbar. Clicking this notification opens the Integrity Verification dialog box which allows you to consult the Verification Log for more information about the DLLs that were detected as untrusted.

Analyze each notification before you continue to work with the software. As there are many reasons for untrusted components, there are also different ways to proceed:

* If an expected reason is reported, you can continue to work with the software.
* If an unexpected reason triggered by an installed third-party product on the same PC is reported, you can continue to work with the software.
* If, however, an untrusted component is reported for an unexpected reason and for which you do not recognize the component, analyze the Verification Log to find out why a DLL was detected as untrusted before you continue to work with the software.
