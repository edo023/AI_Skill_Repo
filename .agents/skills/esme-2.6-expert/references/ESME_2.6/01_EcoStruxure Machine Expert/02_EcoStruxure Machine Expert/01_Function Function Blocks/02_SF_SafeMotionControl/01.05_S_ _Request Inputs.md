# S\_\*\_Request Inputs

**NOTE:**

If not specified as a concrete formal parameter name, the term S\_\*\_Request represents a common placeholder for the "safety-related function request" inputs in this documentation. **\*** stands as a placeholder for the safety-related function, for example, STO or SOS. Accordingly, S\_\*\_SafetyActive is used for the related signaling function block outputs.

* [STO - Safe Torque Off function](STO.html#STO)
* [SOS - Safe Operation Stop function](SOS.html#SOS)
* [SS1 - Safe Stop 1 function](SS1.html#SS1)
* [SS2 - Safe Stop 2 function](SS2.html#SS2)
* [SLS1 to SLS4 - four separate Safely Limited Speed functions](SLS.html#SLS)
* [SDIpos - Safe Direction Positive function](SDI.html#SDI)
* [SDIneg - Safe Direction Negative function](SDI.html#SDI)

Each of these state-controlled inputs evaluates the state of one connected safety-related control device or sensor (activated/not activated, interrupted/not interrupted, open/closed, depending on the used control device/sensor type).

Data type: SAFEBOOL

Initial value depends on the input:

* SAFEFALSE for S\_STO\_Request.
* SAFETRUE for the other request inputs.

**Connection:** Connect this input to a variable representing the request for the safety-related function from the process to be controlled or to a constant.

Usually, these inputs are connected (via an assigned global input variable) to a safety-related control device or sensor connected to the Safety Logic Controller (via safety-related input devices). For example, a safety door, an emergency-stop control device or electrosensitive protective equipment, such as a light curtain.

**NOTE:**

If a safety-related function is not needed in your application, you can leave the respective input unconnected.

**Exception**: The S\_STO\_Request input **must not** remain unconnected. If not used, you have to switch the STO function permanently off by connecting a SAFETRUE constant to this input.

**NOTE:**

The S\_STO\_Request input provides the only possibility of requesting an emergency stop of the axis if the `HW_STO` parameter of the safety logic is set to `Deactivated`.

Also refer to the following.

| WARNING | |
| --- | --- |
|  | **NON-CONFORMANCE TO SAFETY FUNCTION REQUIREMENTS**   * Validate the use of the S\_STO\_Request input with respect to the setting of the `HW_STO` device parameter. * Validate your safety-related architecture and thoroughly test the application.   **Failure to follow these instructions can result in death, serious injury, or equipment damage.** |

If the function block is activated (input [Activate](act_MotionFB.html#act_MotionFB) = TRUE), the possible values at these inputs lead to the following function block behavior:

| Value | Description |
| --- | --- |
| SAFETRUE | Respective safety-related monitoring function is not requested. |
| SAFEFALSE | Respective safety-related monitoring function is requested because the connected safety-related control device or sensor is activated (or the wiring to the control device/sensor is interrupted or the control device/sensor is inoperable).  **Further Information:**  The supported safety-related functions are described in detail in the topic "[Safety Function (Basics)](sfmotionfb.html#sfmotionfb__SafetyFunctions_Basics)".  As soon as the requested safety-related function has achieved the defined safe-state successfully on the drive module, the related S\_\*\_SafetyActive output is switched to SAFETRUE to indicate the achieved defined safe-state. |

## Single-channel or two-channel Connection of the safety-related Control Device/Sensor?

Regardless of whether there is a single-channel or two-channel connection between the safety-related control device/sensor and the terminals of the Safety Logic Controller (or safety-related input device respectively), the S\_\*\_Request input is only connected to one signal of the control device/sensor.

When using a two-channel control device/sensor, the two-channel signal must be evaluated or monitored as follows:

* Either through a suitable upstream function block, for example, SF\_Equivalent function block, if the control device has two N/C contacts.
* Or the evaluation of equivalence or antivalence must be performed in the safety-related input device.
