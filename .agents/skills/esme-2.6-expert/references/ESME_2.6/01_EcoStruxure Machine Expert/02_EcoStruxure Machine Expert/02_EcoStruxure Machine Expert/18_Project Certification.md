## Why certify a project?

After commissioning and completing the safety-related project, an acceptance procedure must be performed. Once this procedure is passed, any additional project modifications lead to further project reviews and re-validations resulting in new efforts and costs.

To avoid unintentional repeated validations, Machine Expert – Safety provides the possibility to certify the safety-related project. A certified project is protected by password against particular modifications (see table below).

To edit an already certified project, the certification password has to be entered.

Certifying the project as well as removing the project certification is logged in the [system event log](eventanderrorlog.html#eventanderrorlog).

## How to certify a project

1. Select 'Project > Project Certification...'.

   The 'Project Certification' menu item is only available if the project has been compiled successfully.
2. In the 'Project Certification' dialog, define a certification password by entering it twice into both input fields and confirm the dialog.

   **NOTE:**

   A secure password is one that has not been shared or distributed to any unauthorized personnel and does not contain any personal or otherwise obvious information. Further, a mix of upper and lower case letters, numbers and special characters offer the best security possible. You should choose a password length of at least 6 characters. The password is case-sensitive and can be a mix of up to 10 characters.

The project is now certified and locked (allowed operations see table below). The certification is displayed in the status bar:

![](images/Statusbar_CertificationOn.gif)

* Certifying the project as well as removing the project certification is logged in the [system event log](eventanderrorlog.html#eventanderrorlog).
* Project certification is only possible if the project has been compiled successfully.

## How to remove the certification from a project

1. Select 'Project > Project Certification...'.

   The 'Project Certification' dialog appears.
2. Enter the certification password and confirm the dialog to unlock the project.

## Restrictions due to a certification password

The following table shows the operations which can be performed in a certified project.

| Operation | Allowed |
| --- | --- |
| Resetting the event log | No for project events  Yes for system events |
| Creating new POUs | No |
| Opening POUs | Yes |
| Opening worksheets | Yes |
| Modifying the POU structure | No |
| Modifying the code of a POU | No |
| Modifying the variables definitions | No |
| Modifying the device parameterization | No |
| Compiling the project | Yes |
| Variable status (displaying present values of variables) | Yes |
| Downloading the project | Yes |
| Switching to debug mode | Yes |
| Forcing variables in debug mode | Yes |
| Overwriting variables in debug mode | Yes |
| Editing the project documentation (project info) | No |
| Downloading files | Yes |
| Modifying the project by "Replace" and "Global Replace" functions | No |

## Certification password lost - Obtaining a one-time certification password

A one-time password can be obtained from the Machine Expert – Safety support to unlock the project certification.

1. Select 'Project > Project Certification...'.
2. In the 'Project Certification' dialog, enter any password containing 6-10 characters.
3. Hold the <Ctrl> key down while confirming the 'Project Certification' dialog with 'OK'.

   The 'One-time certification password' dialog appears displaying a password request code generated by Machine Expert – Safety.
4. Contact your local Schneider Electric representative to obtain a one-time password for the generated code.
5. Enter the received password in the 'One-time certification password' dialog to unlock the project.
