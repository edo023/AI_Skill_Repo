# Error: Safe output ... written by non-safe input

A code network contains a mix of directly connected safety-related and standard data types.

The direct connection of a standard data type to a safety-related output variable corresponds to an invalid data type conversion from standard to safety-related.

Safety-related variables and FB/function inputs can only be written by safety-related variables or FB/function outputs to avoid implicit invalid data type conversions.

**Further Information:**

Refer to the topic ["Mixing safety-related andstandard variables in one FBD/LD network"](MixingSafeAndNonSafeVariables.html#MixingSafeAndNonSafeVariables) for detailed information on networks containing both safety-related and standard data types.

Double-click the error in the message window to jump to the suspected worksheet (where the error is marked) and correct.
