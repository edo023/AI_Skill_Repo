# Filtering the Event Log

The 'Filter options' dialog appears after clicking the 'Filter' button in the [event log dialogs](dialogsEventLog.html#dialogsEventLog). It is used to filter the present event log in order to show only a particular subset of entries.

**NOTE:**

The system event log can only be filtered for the 'User'.

| **Filter option** | **Meaning** |
| --- | --- |
| Log Events | Only log-related events in the log are displayed (e.g., resets of the event log). |
| Project Events | Only project-related events in the log are displayed (e.g., insertion or deletion of libraries). |
| POU Events | Only POU-related events in the log are displayed (e.g., insertion or deletion of POUs). |
| Device parameterization Events | Only device parameterization-related events in the log are displayed (e.g., modifying the configuration of safety-related I/O devices). |
| SafePLC Events | Only error messages received from the Safety Logic Controller error stack are displayed. |
| Bus Navigator Events | Only Bus Navigator-related events are displayed (e.g., changing the Safety Logic Controller). |
| User | Only event log entries of this particular user are displayed. |
