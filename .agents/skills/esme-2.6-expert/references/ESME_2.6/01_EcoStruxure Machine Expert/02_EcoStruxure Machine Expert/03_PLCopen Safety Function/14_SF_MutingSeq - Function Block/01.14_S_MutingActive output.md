# S\_MutingActive output

Output for the status of the muting operation.

Data type: SAFEBOOL

**Connection:** Connect the S\_MutingActive output to an output terminal of your application (for a muting lamp, for example), either directly or via other safety-related functions/function blocks.

The signal (S\_MutingActive) may only control the process directly if this does not adversely affect the safety-related function.

| WARNING | |
| --- | --- |
|  | **UNINTENDED EQUIPMENT OPERATION**   * Verify that the S\_MutingActive signal only controls the process directly if this does not adversely affect the safety-related function. * Validate the overall safety-related function, including the start-up behavior of the process, and thoroughly test the application.   **Failure to follow these instructions can result in death, serious injury, or equipment damage.** |

| Value | Description |
| --- | --- |
| SAFETRUE | This indicates that muting is active. The following conditions must be met.  * The function block has been activated (input Activate = TRUE) * **and** the muting lamp signals working order (S\_MutingLamp = SAFETRUE) * **and** the muting has been activated by an interruption of the light beams of both sensors located **before** the safety-related equipment (in the correct sequence).  Refer to the [information concerning the material flow direction in the function block overview](sfmutingseq.html#sfmutingseq) in this context. * **and** the active muting has not yet been completed. The muting operation is complete if the first muting sensor located **behind** the safety-related equipment switches from TRUE back to FALSE. Refer to the [information concerning the material flow direction in the function block overview](sfmutingseq.html#sfmutingseq) in this context. * **and** the function block has not detected an error.  **NOTE:**  If muting is active (S\_MutingActive = SAFETRUE), the light beam of the safety-related equipment may be interrupted. The S\_AOPD\_Out output remains SAFETRUE. |
| SAFEFALSE | Muting is not active. This may be due to one of the following reasons:  * Muting has not been activated by the muting sensors * **or** the function block is not activated (input Activate = FALSE) * **or** the function block has detected an error.    + The muting lamp is non-functional (S\_MutingLamp = SAFEFALSE)   + or the [muting sequence is invalid](function_MutingSeq.html#function_MutingSeq__InvalidMutingSequences_Seq)   + or the monitoring time set at MaxMutingTime has elapsed  **NOTE:**  If muting is not active, an interruption of the light beam of the safety-related equipment will cause the S\_AOPD\_Out output to be switched to the defined safe state (SAFEFALSE). |
