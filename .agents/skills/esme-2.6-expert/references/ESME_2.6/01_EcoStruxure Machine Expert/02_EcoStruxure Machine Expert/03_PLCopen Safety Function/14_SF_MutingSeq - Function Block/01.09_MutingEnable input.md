# MutingEnable input

State-controlled input for enabling the muting operation.

Data type: BOOL

Initial value: FALSE

**Connection:** Connect this input to an enable signal for the muting operation connected to the Safety Logic Controller, either directly or via other functions/function blocks. It can be connected to the TRUE constant if your risk analysis for the application results that a separate or external enable signal is not necessary.

When the function block is activated (input Activate = TRUE), no start-up inhibit is active, no error messages are present, and the light grid is not interrupted (S\_AOPD\_In = SAFETRUE), the possible values at this input lead to the following function block behavior:

| Value | Description |
| --- | --- |
| FALSE | No enable for the muting operation.  Interruption of the light beam of one or more of the muting sensors leads to an immediate error (Error = TRUE) and the S\_AOPD\_Out output switches to the defined safe state (S\_AOPD\_Out = SAFEFALSE). |
| TRUE | Enable for the muting operation.  Interruption of the light beam of one or more of the muting sensors starts the muting operation. |

**NOTE:**

Set the MutingEnable input to TRUE in order to enable a muting operation. This measure reduces the risk of an unintentional muting process.
