# StartTest input

Edge-triggered input for the request signal to start the sensor test.

Data type: BOOL

Initial value: FALSE

**NOTE:**

A sensor test can only be carried out if output TestPossible = TRUE.

A successful sensor test is indicated by output TestExecuted = TRUE.

You are responsible for controlling the StartTest input and for carrying out tests. It is your responsibility to carry out tests periodically in accordance with the results of the risk analysis you performed, by switching the StartTest input from FALSE to TRUE while output TestPossible = TRUE. Use the TestExecuted output to evaluate the results.

| WARNING | |
| --- | --- |
|  | **NON-CONFORMANCE TO SAFETY FUNCTION REQUIREMENTS**   * Regularly check the safety-related sensor by periodically switching the StartTest input from FALSE to TRUE in accordance with the risk analysis. * Make certain that appropriate procedures and measures have been established to help avoid hazardous situations when applying TRUE to the StartTest input.   **Failure to follow these instructions can result in death, serious injury, or equipment damage.** |

**Connection:** Connect this input to the signal of a button connected to the Safety Logic Controller, either directly or via additional functions/function blocks. Constant TRUE or FALSE values are not permitted.

If the function block is activated (input Activate = TRUE), the possible values at this input lead to the following function block behavior:

| Value | Description |
| --- | --- |
| TRUE | The test of the connected sensor is not requested. |
| Edge FALSE > TRUE | The test of the connected sensor is requested.  The function block performs internal edge monitoring for the input. The request is only executed on a positive signal edge. A further static TRUE signal after a positive signal edge has no effect.  **NOTE:**  The S\_OSSD\_Out output remains SAFETRUE while the test is being carried out. |
