# DiscrepancyTime input

Specification of the permissible discrepancy time for the closing operation of the safety equipment.

Data type: TIME

Initial value: #0ms

If S\_GuardSwitch1 or S\_GuardSwitch2 switches to SAFETRUE, the respective other input (S\_GuardSwitch1 or S\_GuardSwitch2) must also switch to SAFETRUE within the time set at DiscrepancyTime for the switching operation to be considered as valid. Unless both inputs have switched to SAFETRUE after the time set at DiscrepancyTime has elapsed, an error message is generated (Error = TRUE). The S\_GuardMonitoring output remains SAFEFALSE.

**Connection**: Connect this input to a time literal, i.e., to a constant of the data type TIME.

**Background:** As a result of mechanical influences, signals may have a tolerance when closing the safety equipment (switching the contacts of the position switches). By specifying the DiscrepancyTime value, the time period is defined within which the switching operations at inputs S\_GuardSwitch1 and S\_GuardSwitch2 must occur to be recognized as simultaneous. This means within the set discrepancy time both inputs may have different states during the closing operation. After that time, the function block detects this as an error.

**NOTE:**

If the discrepancy time is exceeded when the safety equipment is being closed, an error message is generated (Error = TRUE). The S\_GuardMonitoring output remains SAFEFALSE.

**NOTE:**

If only one signal reports the status of the door (one position switch only), this must be connected in parallel to both inputs S\_GuardSwitch1 and S\_GuardSwitch2. In this case, a time of 0 seconds must be set at DiscrepancyTime.
