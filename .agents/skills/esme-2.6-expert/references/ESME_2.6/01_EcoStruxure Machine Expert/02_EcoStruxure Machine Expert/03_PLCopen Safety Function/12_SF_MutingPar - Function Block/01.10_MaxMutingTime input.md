# MaxMutingTime input

Specification of the maximum permissible time for the entire muting operation. If the muting operation is not completed within this time, the S\_AOPD\_Out output is switched to the defined safe state SAFEFALSE and the function block signals an error (Error switches to TRUE).

Data type: TIME

Initial value: #0ms

**Connection**: Connect this input to a time literal, i.e., to a constant of the data type TIME.

The following is valid for a material flow direction from left to right:

* This timer starts when at least one muting sensor of the first parallel sensor pair (in this case, input MutingSwitch11 or MutingSwitch12, sensor located **before** the safety equipment) provides a TRUE signal (i.e., there is an object inside the detection area of the sensor).
* The muting operation is complete when at least one muting sensor of the second parallel sensor pair (MutingSwitch21 or MutingSwitch22, sensor located **behind** the safety equipment) provides FALSE again.

The following is valid for a material flow direction from right to left:

* This timer starts when at least one muting sensor of the first parallel sensor pair (in this case, input MutingSwitch21 or MutingSwitch22, sensor located **before** the safety equipment) provides a TRUE signal (i.e., there is an object inside the detection area of the sensor).
* The muting operation is complete when at least one muting sensor of the second parallel sensor pair (MutingSwitch11 or MutingSwitch12, sensor located **behind** the safety equipment) provides FALSE again.

**Further Information:**

Also refer to [the example muting operation procedure shown in the functional description](function_mutingPar.html#function_mutingPar__function_mutingPar_Process) for this function block.
