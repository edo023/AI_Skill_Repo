# UnlockRequest input

State-controlled and edge-triggered input for the request signal to unlock the safety equipment (i.e., the guard locking).

Data type: BOOL

Initial value: FALSE

**Connection:** Connect this input to a button/key switch connected to the Safety Logic Controller, either directly or via other safety-related functions/function blocks. Constant TRUE or FALSE values are not permitted.

If the function block is activated (input Activate = TRUE), the possible values at this input lead to the following function block behavior:

| Value | Description |
| --- | --- |
| Edge FALSE > TRUE  followed by TRUE | There is a request to unlock the guard locking on the safety equipment.  The TRUE signal must be maintained for as long as the guard locking is unlocked via the S\_UnlockGuard output (e.g., opening the safety equipment). |
| SAFEFALSE | There is no request to unlock the guard locking on the safety equipment. |
