# S\_AOPD\_In input

State-controlled input for the status of the safety equipment (e.g., light grid).

Data type: SAFEBOOL

Initial value: SAFEFALSE

**Connection:** Connect this input to an optoelectronic safety equipment (e.g., a light grid) connected to the Safety Logic Controller, either directly or via other safety-related functions/function blocks. Constant SAFETRUE or SAFEFALSE values are not permitted.

If the function block is activated (input Activate = TRUE), the possible values at this input lead to the following function block behavior:

| Value | Description |
| --- | --- |
| SAFEFALSE | The light beam of the optoelectronic safety equipment is interrupted.  **NOTE:**  When muting is active (output S\_MutingActive = SAFETRUE), the S\_AOPD\_Out output remains in the non-safe state SAFETRUE (e.g., "machine running").  **NOTE:**  When muting is not active (output S\_MutingActive = SAFEFALSE), the S\_AOPD\_Out output switches to the defined safe state SAFEFALSE (e.g., "stop machine"). |
| SAFETRUE | The light beam of the optoelectronic safety equipment is not interrupted. |

## Single-channel or two-channel safety equipment?

Irrespective of whether there is a single-channel or two-channel connection between the safety equipment and the Safety Logic Controller, the S\_AOPD\_In input is only connected to one signal.

If a two-channel safety equipment is used, either the input channels of the safety-related input device must be parameterized accordingly or the SF\_Equivalent or SF\_Antivalent function blocks must be connected upstream for evaluating or monitoring the two-channel signal (e.g., SF\_Equivalent function block for equivalence monitoring).
