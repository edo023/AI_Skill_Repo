# S\_SafetyRequest output

Request signal for the connected safety-related periphery to execute a safety-related function.

Data type: SAFEBOOL

**Connection:** Connect this output to a safety-related single- or two-channel output of a safety-related device to control the downstream safety-related periphery.

| Value | Description |
| --- | --- |
| SAFETRUE | Execution of a safety-related function is not requested. |
| SAFEFALSE | Execution of a safety-related function is requested. |

## Single-channel or two-channel connection of the periphery?

Independent of whether the connection between the subsequent safety-related periphery and the safety-related device is using a single or two channels, the outputs of the safety-related device are only controlled by a signal at S\_SafetyRequest.

When the connection between subsequent safety-related periphery and the safety-related device is using two channels, the two-channel control is implemented by the safety-related device. The function block only provides one signal.
