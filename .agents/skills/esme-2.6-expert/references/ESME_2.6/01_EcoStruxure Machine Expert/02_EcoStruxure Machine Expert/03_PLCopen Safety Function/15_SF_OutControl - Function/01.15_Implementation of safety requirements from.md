# Implementation of safety requirements from applicable standards

The safety-related function block has been developed according to the safety requirements (from applicable standards) listed in this section. All other requirements from these standards must be observed when implementing the safety-related function.

This section describes either how the function block meets the requirements of the standards or what measures need to be taken to meet the requirements of the standards.

| Standards | Context/Requirement | Implementation |
| --- | --- | --- |
| EN ISO 13849-1 | Stop function | The function block optionally supports the request for an additional operation stop if the stop function was triggered by safety equipment or if the function block is deactivated. A stop request due to triggered safety equipment (signal at S\_SafeControl) and/or an operation stop (signal at ProcessControl) switches S\_OutControl to SAFEFALSE. |
| EN ISO 13849-1 | Start and restart  Fluctuations, failure and return of energy | As an option, the function block supports a start-up/restart inhibit following  * Restart of the Safety Logic Controller * Function block activation * Return of the signal at S\_OutControl  The function block optionally supports the request for an additional operation stop if the stop function was triggered by safety equipment or if the function block is deactivated.  A stop request due to triggered safety equipment (signal at S\_SafeControl) and/or an operation stop (signal at ProcessControl) switches S\_OutControl to SAFEFALSE. |
| EN ISO 13849-1 | Manual reset device | The Reset input supports the function of the manual reset device.  **NOTE:**  Resetting does not occur with a negative (falling) edge, as specified by standard ISO 13849-1, but with a positive (rising) edge. |
| EN ISO 12100-2 | start-up after failure of supply voltage/spontaneous restart | As an option, the function block supports a start-up/restart inhibit following  * Restart of the Safety Logic Controller (S\_AutoReset = SAFEFALSE) * Function block activation (S\_StartReset = SAFEFALSE) * Removal of the request for the safety-related function due to a signal change from SAFEFALSE to SAFETRUE at S\_SafeControl (e.g., by deactivation of an evaluated emergency-stop control device).  If Activate does not indicate the status of the safety-related devices, you must implement this function by other means.  You are responsible for planning and implementing the start-up/restart behavior according to your risk analysis.  To prevent an unintended start-up/restart, you may need to perform an additional operation stop following the removal of the request for the safety-related function. This will depend on both the results of the risk analysis and the signal path of the reset signal. |
| EN 60204 | Stop functions | The function block (S\_OutControl enable signal) executes stop category 0. |
| EN 60204 | Start | The S\_OutControl enable signal is only switched to SAFETRUE if the combination of input signals applies accordingly. |
| EN 60204 | Locking/sequential start-up | The function block optionally supports the request for an additional operation stop if the stop function was triggered by safety equipment or if the function block is deactivated.  A stop request due to triggered safety equipment (signal at S\_SafeControl) and/or an operation stop (signal at ProcessControl) switches S\_OutControl to SAFEFALSE. |
