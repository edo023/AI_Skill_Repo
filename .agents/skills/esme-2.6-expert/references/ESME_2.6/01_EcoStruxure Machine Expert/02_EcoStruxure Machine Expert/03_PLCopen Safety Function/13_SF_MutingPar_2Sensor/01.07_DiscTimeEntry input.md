# DiscTimeEntry input

Specification of the permissible discrepancy time for the switching operation of the muting sensors at the S\_MutingSwitch11 and S\_MutingSwitch12 inputs.

Data type: TIME

Initial value: #0ms

**Connection**: Connect this input to a time literal, i.e., to a constant of the data type TIME.

Once the first input has switched to SAFETRUE, the second input must follow within the time set at DiscTimeEntry. If both inputs switch to SAFETRUE within this time, muting is active (S\_MutingActive = SAFETRUE).

**NOTE:**

If muting is active (S\_MutingActive = SAFETRUE), the light beam of the safety equipment may be interrupted. The S\_AOPD\_Out output remains SAFETRUE.

The time set at DiscTimeEntry is exceeded if the second muting sensor input does not switch to SAFETRUE within the set duration. The S\_AOPD\_Out enable output is then switched to the defined safe state SAFEFALSE and the function block signals an error (Error = TRUE).

**Further Information:**

Also refer to [the example muting operation procedure shown in the functional description](function_MutingPar_2Sensor.html#function_MutingPar_2Sensor__function_mutingPar_Process) for this function block.

**Minimum value:** 0 s

**Maximum value:** 4 s
