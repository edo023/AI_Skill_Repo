# S\_ChannelA and S\_ChannelB inputs

State-controlled inputs for the status of the connected two-channel switch or sensor.

Data type: SAFEBOOL

Initial value: SAFEFALSE

In order to switch the S\_EquivalentOut output to SAFETRUE, the S\_ChannelA and S\_ChannelB input must both be SAFETRUE at the same time (taking the time set at DiscrepancyTime into account when switching to SAFETRUE).

**Connection:**  Connect these inputs to the two-channel switch or sensor connected to the Safety Logic Controller. Constant TRUE or FALSE values are not permitted.

**NOTE:**

**Closed-circuit current principle**: Connect the signal of an N/C contact to each of the two inputs if you want the S\_EquivalentOut output to display a SAFETRUE signal when the contacts are not activated (closed-circuit current principle).

**Operating current principle**: Connect the signal of an N/O contact to each of the two inputs if you want the S\_EquivalentOut output to display a SAFETRUE signal when the contacts are not activated (operating current principle).

Both inputs must always be connected to the same type of contact:

- either N/O contact to S\_ChannelA**and**S\_ChannelB

- or N/C contact to S\_ChannelA**and**S\_ChannelB

| Value | Description |
| --- | --- |
| SAFETRUE | Request to switch [S\_EquivalentOut](out_Equivalent.html#out_Equivalent) to SAFETRUE. |
| SAFEFALSE | Request to switch [S\_EquivalentOut](out_Equivalent.html#out_Equivalent) to SAFEFALSE. |

**NOTE:**

If both S\_ChannelA and S\_ChannelB inputs switch to the SAFETRUE value within the set discrepancy time, the S\_EquivalentOut output switches to SAFETRUE. The S\_EquivalentOut output remains SAFETRUE for as long as both inputs have the SAFETRUE value.

The discrepancy time is exceeded if different states are present at the inputs (S\_ChannelA and S\_ChannelB) once the time set at DiscrepancyTime has elapsed. This results in an error message (Error = TRUE). In such cases, the function block always switches to the defined safe state (S\_EquivalentOut = SAFEFALSE).
