# Error output

Output for error message.

Data type: BOOL

Every error message (Error = TRUE) causes the enable signal at the S\_TwoHandOut output to switch to SAFEFALSE and retain this state.

To leave the error state (Error = TRUE), both inputs S\_Button1 and S\_Button2 must show the SAFEFALSE state.

| WARNING | |
| --- | --- |
|  | **UNINTENDED START-UP**   * Make certain that appropriate procedures and measures have been established to help avoid hazardous situations in the event of an error message or when exiting the error state of the function block. * Ensure that no other persons can access the zone of operation when removing an error. * Use appropriate safety interlocks where personnel and/or equipment hazards exist.   **Failure to follow these instructions can result in death, serious injury, or equipment damage.** |

**Connection:** If required, the signal can be transmitted externally for diagnostic purposes (via a global I/O variable or other safety-related functions/function blocks).

| Value | Description |
| --- | --- |
| TRUE | The function block has detected an error:   * **Either** at least one input was in the SAFETRUE state during function block activation (with input Activate = TRUE) * **or** the change in state from SAFEFALSE to SAFETRUE at the inputs involved a time offset of more than 500 ms.  The enable signal at the S\_TwoHandOut output switches to SAFEFALSE. |
| FALSE | The function block is either inactive or has not detected an error. |
