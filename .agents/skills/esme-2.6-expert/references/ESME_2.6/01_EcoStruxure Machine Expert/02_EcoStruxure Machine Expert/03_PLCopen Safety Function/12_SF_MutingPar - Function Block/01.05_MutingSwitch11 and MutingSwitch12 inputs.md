# MutingSwitch11 and MutingSwitch12 inputs

State-controlled inputs for the first parallel muting sensor pair.

Data type: BOOL

Initial value: FALSE

**Connection:** Connect these inputs to the muting sensors 1 and 2 connected to the Safety Logic Controller. Constant TRUE or FALSE values are not allowed.

**NOTE:**

Depending on the result of the risk analysis, optical, mechanical or inductive sensors such as reflection light sensors, mechanical or inductive switches can be used as muting sensors. Optical sensors serve as examples in the help information.

The functional significance of the inputs differs according to the material flow direction:

* For a material flow direction from **left to right**, both of these sensors are positioned **before** the safety-related equipment.

  In this case, the status at the MutingSwitch11 and MutingSwitch12 input can **start the muting operation** and **activate muting**.
* For a material flow direction from **right to left**, both of these sensors are positioned **behind** the safety-related equipment.

  In this case, the status at the MutingSwitch11 and MutingSwitch12 inputs can **stop the muting operation**.

**Further Information:**

Also refer to [the detailed description and the example muting operation procedure shown in the functional description](function_mutingPar.html#function_mutingPar) for this function block.

If the function block is activated (input Activate = TRUE), the possible values at these inputs lead to the following function block behavior:

| Value | Description |
| --- | --- |
| TRUE | The light beam of the connected muting sensor is interrupted (i.e., there is an object inside the detection area of the sensor).  **NOTE:**  Once the first input of the sensor pair has switched to TRUE, the second input must follow within the time set at DiscTime11\_12. Otherwise this will be recognized as an error (refer also to the DiscTime11\_12 description). |
| FALSE | The light beam of the connected muting sensor is not interrupted (i.e., there is no object inside the detection area of the sensor). |

## Single-channel or two-channel muting sensor?

Irrespective of whether there is a single-channel or two-channel connection between the muting sensors and the Safety Logic Controller, inputs MutingSwitch11 and MutingSwitch12 are only connected to one signal.

If a two-channel muting sensor is used, either the input channels of the safety-related input device must be parameterized accordingly or the SF\_Equivalent or SF\_Antivalent function blocks must be connected upstream in order to evaluate or monitor the two-channel signal.
