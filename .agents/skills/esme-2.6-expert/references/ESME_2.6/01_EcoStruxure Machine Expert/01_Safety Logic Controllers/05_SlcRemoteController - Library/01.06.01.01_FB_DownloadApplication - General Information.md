# FB\_DownloadApplication - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Function block |
| Available as of: | V1.1.0.0 |

## Task

The function block is used to download an application in EcoStruxure Machine Expert to the Safety Logic Controller (SLC) by using the Sercos interface.

## Description

In case of an error-free build of the safety-related application in Machine Expert - Safety, the safety-related application can be exported as a .bin file via Project > Application Download.

For downloading the safety-related application to the SLC, you first have to copy the .bin file to the file system of the non-safety-related controller.

In the EcoStruxure Machine Expert application, set the path to the .bin file at the input i\_sFilepath and set the password of the SLC at the input i\_sPassword of the FB\_DownloadApplication.

The input i\_ifSafeLogic has to be assigned depending on the non-safety-related controller used. For a PacDrive controller, assign the safe logic object which you can find in the Devices tree. For an M262 controller, assign the property PacDriveCompatibility implemented by the safe logic object which you can find in the Devices tree.

If the memory key of the SLC does not yet have a valid password, set it before downloading the application by using the user command SetSafeKeyPassword of the function block [FB\_RemoteController](D-SE-0075082.html).

To execute the download of the application, set the input value at i\_xExecute to TRUE.

When FB\_DownloadApplication requests the confirmation of CRC (Cyclic Redundancy Check) at its diagnostic output WaitForCRCConfirm, set the input value at i\_xConfirmCRC to TRUE.

NOTE: When exporting the safety-related application from Machine Expert - Safety, note the CRC of the application displayed in the window Project Info. Open the window via Project > Project information, tab Project. Verify the CRC by comparing the value of the output q\_udiCRC of the function block FB\_DownloadApplication to the CRC of the application you noted previously. The CRC displayed in Machine Expert - Safety is a hexadecimal number.

After confirmation of the CRC, the application download is completed, and, after an automatic restart of the SLC, the downloaded safety-related application is available on the SLC.

NOTE: The Sercos state must be at least in phase 2 to be able to download the application. It is not necessary to bring the Machine Expert - Safety project into debug mode before the download of the application. By enabling the function block and setting the correct password, you are confirming the download. In addition, you have to confirm the CRC of the application. To download an application on a new or formatted memory key using this function block, the memory key password of the memory key has to be configured first. This can be done using Machine Expert - Safety or via the user command SetSafeKeyPassword of the function block [FB\_RemoteController](D-SE-0075082.html).

NOTE: To download a safety-related application using FB\_DownloadApplication, it is not necessary to set the SLC parameter RemoteControlAllowed to Yes-ATTENTION in Machine Expert - Safety.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_xEnable | BOOL | A rising edge (FALSE -> TRUE) activates the POU, a falling edge (TRUE -> FALSE) deactivates the POU.  A deactivated POU does not execute any actions. |
| i\_xExecute | BOOL | A rising edge (FALSE -> TRUE) starts the download of the safety-related application |
| i\_ifSafeLogic | SystemConfigurationItf.IF\_SafeLogic | Specifies the SLC to interact with.  The device has to exist in the Devices tree in EcoStruxure Machine Expert. |
| i\_sFilepath | STRING[80] | Specifies the path to the .bin file to download.  NOTE: The file path is relative to the root. |
| i\_sPassword | STRING[80] | Memory key password |
| i\_xConfirmCRC | BOOL | CRC confirmation input after comparing the CRC output at q\_diCRC with the CRC that matches the safety-related application.   * FALSE: No confirmation * Edge FALSE > TRUE: Confirmation. |

| Output | Data type | Description |
| --- | --- | --- |
| q\_xActive | BOOL | TRUE: The POU is active and has to be executed further.  FALSE: The POU is inactive. |
| q\_xReady | BOOL | TRUE: The POU is ready to operate and can accept user commands.  FALSE: The function block is not ready to accept user commands. |
| q\_xBusy | BOOL | TRUE: The POU executes the issued user command.  FALSE: The POU is waiting for further user commands. |
| q\_xDone | BOOL | TRUE: The user command has been executed.  FALSE: The user command is being executed, or none has been issued yet. |
| q\_udiCRC | UDINT | Indicates the CRC of the application which has been downloaded to the SLC. |
| q\_etDiag | [GD.ET\_Diag](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.GlobalDiagnostic&topicID=D_SE_0076228) | General, library-independent statement on the diagnostic.  A value unequal to GD.ET\_Diag.Ok corresponds to a diagnostic message. |
| q\_etDiagExt | [ET\_DiagExt](D-SE-0075070.html#D-SE-0075070) | POU-specific output for the diagnostic.  q\_etDiag = ET\_Diag.Ok -> status message  q\_etDiag <> ET\_Diag.Ok -> diagnostic message |
| q\_sMsg | STRING[80] | Event-triggered message which gives more detailed information on the diagnostic state. |

## Diagnostic Messages

| q\_etDiag | q\_etDiagExt | Description |
| --- | --- | --- |
| OK | Disabled | The POU is disabled. |
| OK | Initializing | The POU is being initialized. |
| OK | Running | The POU is busy processing a request or internal procedures. |
| OK | WaitForCRCConfirm | Waiting for confirmation of the CRC which has been returned by q\_udiCRC via the input parameter i\_xConfirmCRC. |
| OK | WaitForExecution | The POU is ready and waiting for execution or for a user input. |
| ExecutionAborted | CommandFailedToExecute | A command was successfully sent to the SLC but terminated unsuccessfully. |
| ExecutionAborted | InvalidFilePath | The path specified by the input variable i\_sFilepath is invalid. |
| InputParameterInvalid | InvalidFile | The file with the path specified at the input i\_sFilepath is invalid. |
| InputParameterInvalid | InvalidPassword | The password specified by the input variable i\_sPassword is invalid. |
| SercosConditionInvalid | CommunicationFailure | An error was detected while attempting to communicate with the SLC. |
| UnexpectedProgramBehavior | CommunicationFailure | An error was detected while attempting to communicate with the SLC. |

## CommandFailedToExecute

|  |  |
| --- | --- |
| Enumeration name: | CommandFailedToExecute |
| Enumeration value: | 31 |
| Description: | A command was successfully sent to the SLC but terminated unsuccessfully. |

| Cause | Solution |
| --- | --- |
| No, or otherwise incorrect SLC password at the input i\_sPassword. | Use the correct password at the input i\_sPassword. |
| Restart command was unsuccessful (for example "timeout" ...) | Manually trigger a restart of the SLC using the function block FB\_RemoteController or the rotary switch of the SLC. |
| Download of "sent application" not successful. SLC reports an incorrect .bin file format (for example incorrect CRC, incorrect version in header, ...) | .bin file is incorrect, export a new application from Machine Expert - Safety. |
| Unlock SLC by sending "unlock key" was unsuccessful. | Repeat application download. |
| .bin file handling was unsuccessful (for example "read from file" or "close file", ...) | Verify .bin file status. Repeat application download. |

## CommunicationFailure

|  |  |
| --- | --- |
| Enumeration name: | CommunicationFailure |
| Enumeration value: | 30 |
| Description: | An error was detected while attempting to communicate with the SLC. |

| Cause | Solution |
| --- | --- |
| The SLC replied to a request incorrectly. | Disable and enable the function block to clear the error state detected and retry the command. |
| Sercos is not in phase 2, phase 3 or phase 4. | Set Sercos to phase 2, phase 3 or phase 4. |
