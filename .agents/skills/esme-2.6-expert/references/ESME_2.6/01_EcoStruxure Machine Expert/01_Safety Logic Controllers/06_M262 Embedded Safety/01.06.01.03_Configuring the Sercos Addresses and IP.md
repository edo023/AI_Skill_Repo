# Configuring the Sercos Addresses and IP Address Assignment

## Example-based Settings

As you have created your project based on the SLC Remote Controller (M262) example, the relevant communication parameters are already configured as follows. Adapt the values, if required by your application.

Settings in the editor Schneider Electric Sercos III Parameters (for the SLC), and Device Parameters Parameters (for the bus couplers), parameter group Identification:

| Parameter | Meaning |
| --- | --- |
| IdentificationMode = Topology mode | The topological address of the Sercos slaves is relevant for the configuration.  Alternatively, you can set the parameter for each Sercos slave to Sercos mode, and then set ConfiguredSercosAddress to the value which is specified using the address switches at the device. Refer to the respective hardware guide(s) and the [*Sercos For Modicon M262 Motion Controller User Guide*](../../../../../api/crossBook?lang=en-US&virtualBookName=m262serc&topicID=D_SE_0096054) for details and information regarding the advantages of each mode. |
| ConfiguredToplogicalAddress = 1 for SLC and 2 for the TM5NS31 bus coupler | The value corresponds to the topological positions of the Sercos slaves in the example project. Adapt these settings if you have connected the devices in another order than described in the chapter [*Wiring of the Sercos Bus*](D-SE-0096235.html#D-SE-0096235__D-SE-0096235.3). |
| IPConfigMode = Automatic IP address assignment | The IP address of each Sercos slave is assigned automatically (by the master, starting from the IP address of the Sercos\_Master), based on the defined topology address. After the Sercos has reached at least phase 2 (CP2), the assigned device IP addresses are visible in the editor during online mode. |
