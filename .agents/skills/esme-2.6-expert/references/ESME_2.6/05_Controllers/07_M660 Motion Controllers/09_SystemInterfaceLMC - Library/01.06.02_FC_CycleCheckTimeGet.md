# FC\_CycleCheckTimeGet

## Overview

|  |  |
| --- | --- |
| Type: | Function |
| Available as of: | V1.0.0.0 |

## Description

Use the function FC\_CycleCheckGet of the library SystemInterfaceLMC to read the default values for cycle time monitoring.

The function is used in conjunction with the function [FC\_CycleCheckSet](FC_CycleCheckSet-45D7156A.html), which can be used to temporarily extend the time limit for cycle time monitoring so you can execute time-consuming operations without diagnostic messages and the watchdog being triggered.

The value for i\_diOverrunTime specifies the time until a cycle time error is detected by cycle time monitoring and the diagnostic message “8313 Excessive cycle time overrun” (diagnostic class 3) and the watchdog are triggered.

The value for i\_diWDogFactor specifies a factor applied to i\_diOverrunTime.

## Interface

| Input/Output | Data type | Description |
| --- | --- | --- |
| iq\_diOverrunTime | DINT | Default time until a cycle time error is detected by cycle time monitoring and the diagnostic message “8313 Excessive cycle time overrun” (diagnostic class 3) and the watchdog are triggered. |
| iq\_diWDogFactor | DINT | Default factor by which i\_diOverrunTime is multiplied to extend the time limit for cycle time monitoring. |

## Return Value

| Data type | Description |
| --- | --- |
| DINT | 0: OK  -1: Error detected. |
