# Diagnostics Concept

## Overview Diagnostics

Diagnostics refers to the detection of events in the system that can have an impact on the operation of the machine/process. Such events have different levels of severity. Their detection triggers a response adapted to the severity of the event and its potential consequences. The severity is determined by the diagnostic class assigned to an event.

Diagnostics is based on the following concepts:

| Term | Definition |
| --- | --- |
| Diagnostic code (DiagCode) | The diagnostic code is a unique 64 bit number which describes an event in the entire system. A diagnostic code cannot be assigned to two different events. For example, diagnostic code 8100 is assigned to “Motor overload” in a drive and cannot be assigned to any other event in the system. |
| Diagnostic class (DiagClass) | The diagnostic class identifies the severity of an event using classes from 0 to 4. 0 is the lowest severity, 4 is the highest severity. |
| Message | A unique message is assigned to each diagnostic code. |
| Extended message(DiagExtMsg) | An extended message describes a detected error to greater detail. There is no unique relationship between an extended message and a diagnostic code. |
| Diagnostic message(DiagMsg) | A diagnostic message comprises the complete diagnostics information, that is, diagnostic code, diagnostic class and message. |

## Diagnostic Classes

The system assigns a specific diagnostic class to each detected event. The diagnostic class determines the severity of the event that triggers the diagnostic message.

Diagnostic classes and their severity levels:

| Diagnostic class | Description | Severity level |
| --- | --- | --- |
| 4 | This diagnostic class indicates that an unrecoverable error has been detected which results in a stop of the system.  Diagnostic messages of diagnostic class 4 cannot be acknowledged, but require a power cycle of the controller after the cause of the error has been removed. | High |
| 3 | Unrecoverable error detected resulting in a stop of a device connected to the controller, for example, a drive or a power supply.  Remove the cause of the error and power cycle or reinitiialize the affected device. For details, refer to the corresponding [diagnostic messages](ListOfDiagnosticMessages-FD0C3E94.html) and user guides of the devices. | Medium high |
| 2 | This diagnostic class represents an alert. The affected device remains operative, but requires your attention. If you do not remove the detected error, a subsequent diagnostic message with a higher diagnostic class may be triggered. | Medium low |
| 1 | This diagnostic class is assigned to diagnostic messages providing you with information on the system. | Low |

## Accessing Diagnostic Messages

You can access diagnostic messages with the user function Diagnostic. This user function is available on the tab Parameter of the device editor of devices that provide diagnostics messages. For details, refer to [Diagnostic - User Function](M660_UserFunct_GroupSercosDiagnostic-44AAE71E.html). Use the corresponding parameters to manage diagnostics messages in your application.

## Configuration of Responses

In your application, the function FC\_DiagMessageConfigClass of the *SystemDiagnosis* library lets you configure the system response to diagnostic messages that are not assigned to a fixed diagnostic class. The minimum severity level (diagnostic class) you can configure corresponds to the default severity level of the diagnostic code.

For more information, refer to the *SystemDiagnosis* library guide.
