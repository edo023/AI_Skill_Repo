# Diagnostics-Specific Information on Edge I/O Modules

## Diagnostics

Under Sercos, diagnostics information from Edge I/O modules is read by the controller using the Sercos parameter IDN S-0-0390.

You can access diagnostic messages with the user function [Diagnostic](M660_UserFunct_GroupSercosDiagnostic-44AAE71E.html) on the tab Sercos Parameter of the device editor.

Edge I/O modules use a dedicated system for communicating diagnostics messages which is based on a Group Cycle Status (GCS) mechanism. For details, refer to the Modicon Edge I/O Diagnostic Data user guide.

The Edge I/O network interface module and the connected Edge I/O modules provide the controller with GCS status bits. The GCS are available as of Sercos communication phase CP4.

If an error is detected, the parameter DiagExtMsg indicates the a message in the following format: mod#NNN:0xGG:

* mod: Module error detected
* NNN: The position number of the affected module. The number of the module corresponds to its positions in the Devices tree, with the network interface module being number 0.

  + 0.0

    : The GCS value of the network interface module is invalid.
  + 0.1

    : The GCS value of the EmbeddedBusExtender of the network interface module is invalid.
  + Number: Module whose GCS value is invalid.
* 0xGG: Code of the diagnostic message from Edge I/O.

For details on the individual codes, refer to the Modicon Edge I/O Diagnostic Data user guide.

The controller monitors the GCS values during the transition to Sercos communication phase CP4. The phase transition is suspended if any of the bits is invalid until a timeout occurs. If the timeout is reached before the bits become valid, the Edge I/O network interface module releases the transition to Sercos communication phase CP4 and generates a diagnostic message with diagnostic class 3.

You can access additional diagnostics information on Edge I/O using the Edge I/O web interface. For details, refer to theModicon Edge I/O Configurator and Web Interface user guide.
