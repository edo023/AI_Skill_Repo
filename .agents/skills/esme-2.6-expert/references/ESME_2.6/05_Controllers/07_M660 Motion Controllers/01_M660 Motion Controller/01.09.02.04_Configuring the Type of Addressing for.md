# Configuring the Type of Addressing for Sercos Communication

## Overview

The address for Sercos communication is used to identify a subordinate Sercos device on the Sercos bus for transmission of the data in [Sercos telegrams](SercosTelegram-494E27C0.html). By specifying the address of a subordinate Sercos device and the type of addressing to be used, you link a logical device in EcoStruxure Machine Expert and in your application to a physical device on the Sercos bus. Each subordinate Sercos device on the Sercos bus must have a unique address.

If you use the addressing type SercosAddress and have assigned identical numbers to different devices, the system detects that the addresses are not unique and does not phase up to communication phase CP4. However, if you use the addressing type TopologyAddress and add an additional device in between other devices at a later point in time, the physical positions of the devices downstream the line or the ring change. This also changes their topology address. The same applies if you interchange the Sercos cables of devices. In both cases, this does not result in non-unique addresses, but data may be sent to unintended devices.

You can use the following addressing types to address a subordinate Sercos device on the Sercos bus, depending on the type of device:

* Topology address (default)
* Sercos address
* Application type
* Serial number of the controller
* Serial number of the motor

The type of addressing for a Sercos device is selected with parameter IdentificationMode. You can use different types of addressing on the same Sercos bus.

The addressing type TopologyAddress uses the physical position of the subordinate device on the Sercos bus. For example, the first subordinate device on the line has the address 1. When you add subordinate devices to the Sercos master, consecutive topology numbers are assigned in the order in which the subordinate devices are added.

The addressing type SercosAddress uses the numerical address assigned to a device to identify the device on the Sercos bus.

The addressing type ApplicationType uses a string value with a maximum length of 40 bytes assigned to a device to identify the device on the Sercos bus.

The addressing type SerialNumberController uses the serial number of the controller read from the controller and assigned to a device to identify the device on the Sercos bus.

The addressing type SerialNumberMotor uses the serial number of the motor read from the drive and assigned to a device to identify the device on the Sercos bus.

During phase-up, the master verifies correct addressing of the subordinate devices. If an address on the bus is not unique, phase up is not completed.

## Configuration of the Type of Addressing for Sercos Communication

| Step | Action |
| --- | --- |
| 1 | Open the tab Parameters of the Sercos device to be configured. |
| 2 | For addressing type TopologyAddress (default setting):   * Use the default setting of the parameter IdentificationMode (TopologyAddress). * Verify that the value of the parameter ConfiguredTopologyAddress corresponds to the physical position of the device on the Sercos bus. Correct the value, if required.   For addressing type SercosAddress:   * Set the parameter IdentificationMode to SercosAddress. * Set the value of the parameter ConfiguredSercosAddress to the address value of the subordinate device.   For addressing type ApplicationType:   * Set the parameter IdentificationMode to ApplicationType. * Set the value of the parameter ConfiguredSercosApplication to an application type to be used for addressing the device.   For addressing type SerialNumberMotor:   * Set the parameter IdentificationMode to SerialNumberMotor. * Set the value of the parameter ConfiguredSerialNumber to the serial number to be used for addressing the device.   For addressing type SerialNumberController:   * Set the parameter IdentificationMode to SerialNumberController. * Set the value of the parameter ConfiguredSerialNumber to the serial number to be used for addressing the device. |
