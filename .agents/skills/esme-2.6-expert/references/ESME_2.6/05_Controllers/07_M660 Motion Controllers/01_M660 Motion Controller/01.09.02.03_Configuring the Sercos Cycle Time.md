# Configuring the Sercos Cycle Time

## Overview

The Sercos cycle time is a key factor in the performance of your system. The correct cycle time depends on a range of factors including devices on the Sercos bus, amount of data for cyclic data exchange, cycle time of the MAST task and other tasks. The Sercos cycle time triggers the Real-Time Motion Process. The values for the axes are calculated in the Sercos cycle time.

Refer to [Sercos Communication Cycle](SercosCommunicationCycle-494E4399.html) for details on the communication cycle.

## Configuration of the Sercos Cycle Time

The cycle time can be set in communication state NRT and in communication phase CP0. The configured value becomes effective as of communication phase CP2.

| Step | Action |
| --- | --- |
| 1 | Open the tab Parameters of the Sercos master. |
| 2 | Set the parameter ConfiguredCycleTime of the user function CycleTimeConfig to one of the following values (nanoseconds):   * 1000000 (= 1 ms) * 2000000 (= 2 ms) * 4000000 (= 4 ms) |

For additional information on the parameter, refer to [SercosCycletimeConfig - User Function](M660_UserFunct_SercosCycletimeConfig-449D0B4A.html).
