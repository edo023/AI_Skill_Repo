# FpgaFirmware - User Function

## Description

The user function FpgaFirmware is available for the following device objects:

* [CommunicationDevice\_CN20\_CN21](CommunicationDevice_CN20_CN21CommDe-4C5BB44F.html)

  , communication device
* [CommunicationDevice\_CN22\_CN22](CommunicationDevice_CN22_CN23CommDe-4C5EBC80.html)

  , communication device
* [CommunicationDevice\_CN8\_CN9](CommunicationDevice_CN8_CN9-DeviceO-067E64B6.html), communication device
* [OptionCard\_CN20\_CN21](OptionCard_CN_CN-DeviceObjectInTheD-067DD19F.html), option card
* [OptionModule\_CN••\_CN••](OptionModule_CN_CN-DeviceObjectInTh-067DB53D.html), option module

The FPGA firmware determines the type of communication technology available at an [interface point](InterfacePoint-UserFunction-068F78C0.html#InterfacePoint-UserFunction-068F78C0), for example, Profinet or Sercos.

The user function FpgaFirmware provides parameters for reading the values of the parameters determining the type of communication technology assigned to a communication device, and for setting the type of firmware update.

When the controller starts, the system compares the versions of the firmware in the FPGA of the communication devices to the versions of the firmware on the SD card. If a firmware on the SD card is newer than the firmware of the FPGA of the corresponding communication device, an update is started, depending on the value of the parameter UpdateMode.

During a firmware update, the **ERR/WD** LED of the controller flashes orange. After the firmware update has completed, the controller automatically restarts.

NOTE: You cannot connect to the controller as long as an FPGA firmware update is running. The update takes approximately seven minutes.

## Parameter Revision

|  |  |
| --- | --- |
| Data type | STRING(80) |
| Traceable | No |

The parameter Revision allows you to read the version and the build number of the firmware of the communication device, delimited by a dot (for example, 2.4).

## Parameter Feature

|  |  |
| --- | --- |
| Data type | Enumeration of DINT |
| Traceable | Yes |

The parameter Feature allows you to read the type of firmware of the communication device.

The parameter provides the following values:

| Value | Description |
| --- | --- |
| None | No firmware with network technology assigned to communication device. |
| Profinet | The communication device uses a firmware with Profinet network technology. |
| Sercos | The communication device uses a firmware with Sercos network technology. |

## Parameter ConfiguredRevision

|  |  |
| --- | --- |
| Data type | STRING(80) |
| Traceable | Yes |

The parameter ConfiguredRevision allows you to read and write the firmware version to be used for the communication device if the type of update is set to Fixed with UpdateMode.

## Parameter UpdateMode

|  |  |
| --- | --- |
| Data type | Enumeration of DINT |
| Traceable | Yes |

The parameter UpdateMode allows you to read and write the type of update of the firmware of the communication device.

The parameter provides the following values:

| Value | Description |
| --- | --- |
| Automatic | The firmware for the communication device is updated automatically. |
| Fixed | The firmware for the communication device is set to a firmware version specified with ConfiguredRevision. |
| Disabled | The firmware update for the communication device is disabled. |
