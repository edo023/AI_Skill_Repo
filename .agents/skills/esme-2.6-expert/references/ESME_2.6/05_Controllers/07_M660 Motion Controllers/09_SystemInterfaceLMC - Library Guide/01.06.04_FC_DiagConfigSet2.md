# FC\_DiagConfigSet2

## Overview

|  |  |
| --- | --- |
| Type: | Function |
| Available as of: | V1.0.0.0 |

## Description

Use the function FC\_DiagConfigSet2 of the library SystemInterfaceLMC to modify the diagnostic configuration of diagnostic messages.

The system response to the diagnostic message specified with i\_diDiagCode is set to the diagnostic class specified with i\_diDiagClass.

The response of the device specified with i\_stLogAddr is set to i\_diDiagSubClass.

NOTE: If the device is reset (restart) or if Sercos is reset to communication phase CP0, the diagnostic configuration is also reset.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_diDiagCode | DINT | Number of the diagnostic message to which the system response is to be modified. |
| i\_diDiagClass | DINT | New diagnostic class for the diagnostic message. |
| i\_stLogAddr | [DeviceTypeInterfaces.IF\_Device\_Type](IF_Device_Type-56CAF679.html#IF_Device_Type-56CAF679) | Identifies the device. |
| i\_diDiagSubClass | DINT | New diagnostic subclass for the diagnostic message. |

## Return Value

| Data type | Description |
| --- | --- |
| DINT | 0: OK  -1: Invalid diagnostic code  -2: Invalid diagnostic class  -4: Invalid diagnostic class (only the classes 1, 2 or 3 can be assigned to the diagnostic codes of a drive with the response AB, BD1, BD2, and CD).  -7: Invalid subclass  -461: In the present Sercos communication phase, writing parameters via the service channel is not supported.  -462: Addressed device does not support this function.  -464: Data could not be transferred to the device. |
