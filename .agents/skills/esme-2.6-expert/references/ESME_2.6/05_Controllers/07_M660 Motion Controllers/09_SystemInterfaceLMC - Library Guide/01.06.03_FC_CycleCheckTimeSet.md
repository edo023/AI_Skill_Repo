# FC\_CycleCheckTimeSet

## Overview

|  |  |
| --- | --- |
| Type: | Function |
| Available as of: | V1.0.0.0 |

## Description

Use the function FC\_CycleCheckSet of the library SystemInterfaceLMC to temporarily extend the time limit for cycle time monitoring so you can execute time-consuming operations without diagnostic messages and the watchdog being triggered.

| WARNING | |
| --- | --- |
|  | UNINTENDED EQUIPMENT OPERATION  * Use this function only during commissioning and testing. * Perform a risk assessment before using this function.  Failure to follow these instructions can result in death, serious injury, or equipment damage. |

The value for i\_diOverrunTime specifies the time until a cycle time error is detected by cycle time monitoring and the diagnostic message “8313 Excessive cycle time overrun” (diagnostic class 3) and the watchdog are triggered.

The value for i\_diWDogFactor specifies a factor applied to i\_diOverrunTime.

The total time extension for cycle time monitoring is calculated as:

i\_diOverrunTime x i\_diWDogFactor.

The values for both parameters must be greater than 0.

To extend the time limits for cycle time monitoring across multiple cycles, retain the original value of 1 for i\_diWDogFactor and increase the value for i\_diOverrunTime. For example, if the cycle time is 5 ms, set the value of i\_diOverrunTime to 5000. To extend the monitoring time to 25 ms, set the value of i\_diOverrunTime to 25000.

Do not restore the original time limits in the same cycle in which the code to be tested is executed. Instead, restore the original values at the beginning of the subsequent cycle.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_diOverrunTime | DINT | Time until a cycle time error is detected by cycle time monitoring and the diagnostic message “8313 Excessive cycle time overrun” (diagnostic class 3) and the watchdog are triggered. |
| i\_diWDogFactor | DINT | Factor by which i\_diOverrunTime is multiplied to extend the time limit for cycle time monitoring. |

## Return Value

| Data type | Description |
| --- | --- |
| DINT | 0: OK  -1: Error detected.  -2: i\_diOverrunTime is not greater than 0.  -3: i\_diWDogFactor is not greater than 0. |

## Code Example

The following code snippet is an example of how the function can be used.

```
// variable to handle cycle time monitoring extension, not all variables are listed
xTestTimeSetNew : BOOL := TRUE;
xTestTimeSetOld : BOOL := FALSE;
diOverrunTime : DINT;
diWDogFactor : DINT;
c_diOverrunTime : DINT := 100000; // 100ms
c_diWDogFactor : DINT := 1;
resultGet : DINT;
resultSet : DINT;

// second action in cycle 2, restore original values
IF xTestTimeSetOld THEN
  xTestTimeSetOld := FALSE;
resultSet := FC_CycleCheckTimeSet(i_diOverrunTime:=diOverrunTime,i_diWDogFactor:=diWDogFactor);
END_IF

// first action, save original values, set new ones, execute time-consuming code
IF xTestTimeSetNew THEN
  xTestTimeSetNew := FALSE;
  xTestTimeSetOld := TRUE;
  resultGet := FC_CycleCheckTimeGet(diOverrunTime,diWDogFactor);
  resultSet := FC_CycleCheckTimeSet(i_diOverrunTime:=c_diOverrunTime, i_diWDogFactor:=c_diWDogFactor);

// TIME CONSUMING CODE (UP TO 100MS)
END_IF
```
