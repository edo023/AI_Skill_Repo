# System Limits

The following system limits apply:

| Item | Limit |
| --- | --- |
| Shortest Sercos cycle time | 1 ms |
| Maximum number of physical axes | Up to 64  The maximum number of axes corresponds to the number of drives, regardless of the working mode. Axes with the working mode Disabled or Optional count toward the maximum number of axes. |
| Maximum number of motion jobs started simultaneously in a single cycle | 12 |

The controller supports parallel execution of multiple motion jobs. However, starting too many jobs by setting the Execute flag in the same cycle can exceed the available processing time within this cycle. This is due to the fact that starting a motion job creates more load than running it.

It is a best practice to execute motion jobs sequentially instead of starting them concurrently. As a rule, you can assume that starting a motion job requires an average processing time of approximately 9 µs. This has to be added to the regular processing time per motion job. Since starting and handling motion jobs is only one of several tasks handled during the Real-Time Process (RTP) cycle, the total number of motion jobs that can be started simultaneously also depends on the overall system load and configuration.
