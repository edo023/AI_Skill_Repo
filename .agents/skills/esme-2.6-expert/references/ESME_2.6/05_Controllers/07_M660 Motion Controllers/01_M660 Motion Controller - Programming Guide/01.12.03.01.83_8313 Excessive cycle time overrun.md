# 8313 Excessive cycle time overrun

Diagnostic class: 4

The cycle time is monitored by the system. The running task is stopped if the cycle time is exceeded. Stopping the task is controlled by the system, not by the task. Operations being executed are not taken into account.

Therefore, it cannot be ruled out that the task is performing, for example, a read/write operation on the file system, and blocks another system resource due to the exclusive access. This resource is not released when the task is stopped. If other tasks try to access this resource, subsequent errors may occur.

A cycle time error of a task is detected if the watchdog time is exceeded (tab Configuration, Applications tree Applications tree > Application > Task configuration > MAST, items Time and Sensitivity.

You can acknowledge the diagnostic message by resetting the application. The detected error can only be removed by restarting the controller (Online > Reset Cold).

| Cause | Solution |
| --- | --- |
| Calling or processing a parameter or function block takes too long. | * Perform a cold reset of the controller (select Online > Reset Cold). * Verify and correct, if necessary, the application. |
| The time interval in the task configuration is too short. | * Increase the time interval in the task configuration. * If possible, execute operations in a new or different task. |
