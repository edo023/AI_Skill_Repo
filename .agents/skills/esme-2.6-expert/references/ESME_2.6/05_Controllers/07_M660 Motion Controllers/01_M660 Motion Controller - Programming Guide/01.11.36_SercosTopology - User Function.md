# SercosTopology - User Function

## Description

The user function SercosIdentification is available for the following device obects:

* [SercosMaster](SercosMaster-DeviceObjectInTheDevic-0684DFF3.html), Sercos master

The user function SercosTopology lets you read settings relating to the Sercos topology.

## Parameter LastDeviceP1

|  |  |
| --- | --- |
| Data type | STRING(80) |
| Traceable | No |

The parameter LastDeviceP1 lets you read the name of the last physical Sercos device on port 1 (**CN8**).

## Parameter LastDeviceP2

|  |  |
| --- | --- |
| Data type | STRING(80) |
| Traceable | No |

The parameter LastDeviceP2 lets you read the name of the last physical Sercos device on port 2 (**CN8**).

## Parameter ScannedDevices

|  |  |
| --- | --- |
| Data type | DINT |
| Traceable | Yes |

The parameter ScannedDevices lets you read the number scanned physical devices on the Sercos bus.

## Parameter Topology

|  |  |
| --- | --- |
| Data type | Enumeration of DINT |
| Traceable | Yes |

The parameter Topology lets you read the Sercos topology.

| Value | Meaning |
| --- | --- |
| no link | No Sercos device connected to port 1 and port 2. |
| line p1 | The Sercos devices are connected to port 1 (single line). |
| line p2 | The Sercos devices are connected to port 2 (single line). |
| double line | The Sercos devices are connected to ports 1 and 2 (double line). |
| ring | The Sercos devices are connected to ports 1 and 2 (ring). |
| defect ring | Ring interrupted, for example, due to disconnection of cable. This condition persists for a short period of time until the automatic healing is completed and the topology has changed to line. |
