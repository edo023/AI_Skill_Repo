# SercosIdentification - User Function

## Description

The user function SercosIdentification is available for the following device objects:

* [DRV\_Lexium62 (Lexium 62 Drive)](DRV_Lexium62Lexium62Drive-DeviceObj-0684F462.html), drive
* [DRV\_Lexium62Integrated (Lexium ILM62 Drive)](DRV_Lexium62IntegratedLexiumILM62Dr-0684ED06.html), drive
* [PSM\_Lexium62 (Lexium62 Power Supply)](PSM_Lexium62Lexium62PowerSupply-Dev-0790BF9D.html)
* [NTSNSC1200](NTSNSC1200-DeviceObjectInTheDevices-068506CC.html)

The user function SercosIdentification provides parameters relating to the type of identification and addressing of a device on the Sercos bus.

## Parameter ConfiguredTopologyAddress

|  |  |
| --- | --- |
| Data type | UINT(1...512) |
| Traceable | Yes |

The parameter ConfiguredTopologyAddress lets you configure the topology address of a device. The topology address corresponds to the physical position of the device on the Sercos bus.

The topology address is used for addressing a Sercos device if the parameter [IdentificationMode](#M660_UserFunct_SercosIdentification-44A406D8__M660_Param_IdentificationMode-44A4FEE1) is set to TopologyAddress.

The configured topology address is applied when the Sercos communication phase transitions from CP1 to CP2.

Modifying the cabling of a device on the Sercos bus changes the topology address the devices on the Sercos bus.

## Parameter VendorDeviceID

|  |  |
| --- | --- |
| Data type | STRING(80) |
| Traceable | No |

The parameter VendorDeviceID lets you read the vendor device ID of the device.

The vendor device ID is a unique number assigned by the vendor to each device and lets you identify a Sercos device on the Sercos bus.

NOTE: The parameter value is transferred from the subordinate device to the master over the Sercos service channel. If the device is virtual or if Sercos is not in communication phase CP2 or higher, the value is an empty string.

## Parameter IdentificationMode

|  |  |
| --- | --- |
| Data type | Enumeration of DINT |
| Traceable | Yes |

The parameter IdentificationMode lets you configure the type of addressing of a device on the Sercos bus.

The parameter provides the following values:

| Value | Meaning |
| --- | --- |
| SerialNumberController | The device is addressed using the configured controller [serial number](#M660_UserFunct_SercosIdentification-44A406D8__M660_Param_ConfiguredSerialNumber-44A4FE17) of the device. |
| TopologyAddress (default) | The device is addressed using the configured [topology address](#M660_UserFunct_SercosIdentification-44A406D8__M660_Param_TopologyAddress-44A4FD57) of the device. |
| SerialNumberMotor | The device is addressed using the configured motor [serial number](#M660_UserFunct_SercosIdentification-44A406D8__M660_Param_ConfiguredSerialNumberMoto-44A5002C) of the motor. |
| ApplicationType | The device is addressed using the configured [application type](#M660_UserFunct_SercosIdentification-44A406D8__M660_Param_ConfiguredApplicationType-44A4F824) of the device. |
| SercosAddress | The device is addressed using the configured [Sercos address](#M660_UserFunct_SercosIdentification-44A406D8__M660_Param_ConfiguredSercosAddress-44A4FC66) of the device. |

## Parameter IdentifyDevice

|  |  |
| --- | --- |
| Data type | BOOL |
| Traceable | Yes |

The parameter IdentifyDevice lets you enable and disable the visual device identification of the physical device in the machine using the flashing LED S3.

This parameter is effective in the Sercos communication phases CP2, CP3 and CP4.

The parameter provides the following values:

| Value | Description |
| --- | --- |
| FALSE (default value) | Device identification is disabled. |
| TRUE | Device identification is enabled. The corresponding LED of the addressed device flashes. |

NOTE: The parameter value is transferred from the master to the subordinate device over the service channel. Typically, this takes about 10 ms, but the transmission can take up to 1 second, depending on the load on the service channel.
