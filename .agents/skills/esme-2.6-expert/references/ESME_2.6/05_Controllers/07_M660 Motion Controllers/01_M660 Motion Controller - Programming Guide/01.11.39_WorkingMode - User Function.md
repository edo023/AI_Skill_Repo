# WorkingMode - User Function

## Description

The user function SercosIdentification is available for the following device objects:

* [DRV\_Lexium62 (Lexium 62 Drive)](DRV_Lexium62Lexium62Drive-DeviceObj-0684F462.html), drive
* [DRV\_Lexium62Integrated (Lexium ILM62 Drive)](DRV_Lexium62IntegratedLexiumILM62Dr-0684ED06.html), drive
* [PSM\_Lexium62 (Lexium62 Power Supply)](PSM_Lexium62Lexium62PowerSupply-Dev-0790BF9D.html), power supply
* [NTSNSC1200](NTSNSC1200-DeviceObjectInTheDevices-068506CC.html), Edge I/O NTSNSC1200 network interface module

The user function WorkingMode comprises parameters for setting and reading the working mode Sercos devices.

## Parameter DesiredMode

|  |  |
| --- | --- |
| Data type | Enumeration of DINT |
| Traceable | Yes |

The parameter DesiredMode lets you set the working mode of the device.

The parameter allows you to configure a device as a virtual device. A virtual device is treated like a device that is physically present and connected. However, the controller does not perform any communication with a virtual device.

If the value of the parameter is modified in Sercos communication phase CP4, the new value becomes effective only after the next full Sercos phase-up.

| Value | Meaning |
| --- | --- |
| Virtual (default) | The working mode Virtual is intended for use during development, for example, for testing an application before the drive is connected to the controller. |
| Physical | The working mode Physical is intended for regular machine operation when the device is connected. |
| Deactivated | In the working mode Deactivated, calculation of reference values is deactivated to reduce the cycle load. |
| Optional | In the working mode Optional, the device is searched for on the Sercos bus. If it is found, it is used like a physical device (corresponding to parameter value Physical). If it is not found, it is used like a virtual device (corresponding to parameter value Virtual). |

## Parameter ActualState

|  |  |
| --- | --- |
| Data type | Enumeration of DINT |
| Traceable | Yes |

The parameter ActualState lets you can read the active working mode of the device as set with the parameter [DesiredMode](#M660_UserFunct_GroupWorkingMode-44A54DB0__M660_Param_DesiredMode-44A5E285).

| Value | Meaning |
| --- | --- |
| Virtual | In this working mode, the device operates as a virtual device. |
| Physical | In this working mode, the device operates as a physical device. |
| Deactivated | In this working mode, the device operates as a deactivated device. |
| Error | An error has been detected. |
