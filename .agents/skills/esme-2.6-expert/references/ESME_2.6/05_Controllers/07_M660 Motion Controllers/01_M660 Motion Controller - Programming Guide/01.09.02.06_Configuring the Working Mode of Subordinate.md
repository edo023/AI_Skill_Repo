# Configuring the Working Mode of Subordinate Devices

## Working Modes

Subordinate devices can have the following working modes:

* **Physical**
* **Virtual**
* **Deactivated**
* **Optional**

The working mode **Physical** is intended for regular machine operation when the device is connected.

The working mode **Virtual** is intended for use during development, for example, for testing an application before the drive is connected to the controller. By default, subordinate devices added to the Sercos master tree are virtual. The system treats the virtual device like a device that is physically present and connected. However, it neither searches for a virtual device nor does it perform any communication with it.

NOTE: The working mode **Virtual** is not intended to be used as an operational axis. To set up a virtual axis (for example, for a virtual shaft as an abstract representation of the machine velocity, or as an intermediate axis for complex reference value chains of master and subordinate axes), use FB\_ControlledAxis (refer to the MotionInterface library guide for details).

While the axis of a device in working mode **Virtual** behaves like a virtual axis, the virtual device causes an additional performance impact. Further, it is counted toward the maximum number of Sercos devices, which is not the case if you declare a virtual axis by using FB\_ControlledAxis.

In the working mode **Deactivated**, calculation of reference values is deactivated to reduce the cycle load.

In the working mode **Optional**, the device is searched for on the Sercos bus. If it is found, it is used like a physical device (corresponding to the working mode **Physical**). If it is not found, it is used like a virtual device (corresponding to the working mode **Virtual**).

## Configuring the Working Mode of a Subordinate Device

| Step | Action |
| --- | --- |
| 1 | Open the tab Parameters of the device to be configured. |
| 2 | Set the parameter DesiredMode of the user function WorkingMode to Virtual (default value), Physical, Deactivated or Optional as required. |

For details, refer to [WorkingMode - User Function](M660_UserFunct_GroupWorkingMode-44A54DB0.html).
