# Default Firewall Configuration

## Overview

The following applications/ports are opened by the default firewall configuration:

| Protocol | Port Number | Application | Default Rule State | Default Service State |
| --- | --- | --- | --- | --- |
| ICMP | - | Network management, for example, ping | Active | Active |
| TCP | 1105 | NetManage | Active | Active |
| TCP | 4840 | OPC UA | Active | Inactive |
| TCP | 8080 | WebVisu | Active | Inactive |
| TCP | 11740 | NetManage | Active | Active |
| UDP | 1202 | Network variables | Active | Inactive |
| UDP | 1203 | Network variables | Active | Inactive |
| UDP | 1740...1743 | CoDeSys services | Active | Active |
| UDP | 27127 | NetManage | Active | Active |
| TCP | >= 65000 | File transfer of the FTP server | Active | Active |
| HTTPS | 443 | WebVisu, IEC application libraries | Active | Inactive |

NOTE:

* Rules can be set to inactive by commenting out in the configuration file.
* Services marked as inactive in service state do not run by default, but must be started by you. The corresponding ports are opened in the firewall so that the service is available without additional firewall configuration.
