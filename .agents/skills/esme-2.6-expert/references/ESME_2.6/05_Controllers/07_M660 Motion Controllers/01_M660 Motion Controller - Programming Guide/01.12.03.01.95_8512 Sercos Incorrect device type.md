# 8512 Sercos: Incorrect device type

Diagnostic class: 3

The device type of the Sercos device found at position (x) on the Sercos bus does not match the expected type (type).

| Cause | Solution |
| --- | --- |
| Incorrect assignment of a Sercos device detected during transition from Sercos communication phase CP2 to CP3. | Verify the identification mode and the type of identification (user function [SercosIdentification](M660_UserFunct_SercosIdentification-44A406D8.html#M660_UserFunct_SercosIdentification-44A406D8) and its parameters). |
| * Ext. Diagnosis NoPS connected  Unable to assign a power supply to the drive (refer to parameter [ConfiguredPowerSupply](M660_UserFunct_DrivePowerSupply-44AB50C0.html#M660_UserFunct_DrivePowerSupply-44AB50C0__M660_Para_ConfiguredPowerSupply-44AB4CD2) and [PowerSupply](M660_UserFunct_DrivePowerSupply-44AB50C0.html#M660_UserFunct_DrivePowerSupply-44AB50C0__M660_Para_PowerSupply-00414736)). * The parameter [ConfiguredPowerSupply](M660_UserFunct_DrivePowerSupply-44AB50C0.html#M660_UserFunct_DrivePowerSupply-44AB50C0__M660_Para_ConfiguredPowerSupply-44AB4CD2) is not empty, but no PowerSupply (for example, LXM62PSM) is contained in the project. | Add an appropriate PowerSupply to the project (LXM62PSM). |
| The parameter [ConfiguredPowerSupply](M660_UserFunct_DrivePowerSupply-44AB50C0.html#M660_UserFunct_DrivePowerSupply-44AB50C0__M660_Para_ConfiguredPowerSupply-44AB4CD2) contains a name which does not correspond to a [PowerSupply](M660_UserFunct_DrivePowerSupply-44AB50C0.html#M660_UserFunct_DrivePowerSupply-44AB50C0__M660_Para_PowerSupply-00414736) (for example, LXM62PSM) in the project. | * Add another power supply with the correct name to the project. * Correct the value of the parameter [ConfiguredPowerSupply](M660_UserFunct_DrivePowerSupply-44AB50C0.html#M660_UserFunct_DrivePowerSupply-44AB50C0__M660_Para_ConfiguredPowerSupply-44AB4CD2). |
| Inconsistent feature configuration in a Lexium 62 Double Drive, for example:   * Drive\_A = Standard / 1 * Drive\_B = Standard Plus / 2   The values in the parameter DeviceVariantSet must be identical. | Verify the settings of the feature configuration for both drive instances. |
