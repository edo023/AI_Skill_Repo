# OptionCard\_CN••\_CN•• - Device Object in the Devices Tree

## Overview

The device object OptionCard\_CN••\_CN•• is a communication device that represents an option card installed in one of the PCI Express Card slots of the controller. If the option card has communication interfaces (RJ45 connectors), this communication device provides the basic communication functionality, for example, for TSN.

Option cards are available with one dual-port connector (OptionCardCommSingle) or two dual-port connectors (OptionCardCommDouble). Examples:

* OptionCard\_CN20\_CN21 (OptionCardCommSingle) represents the interface point for the physical connectors **CN20** and **CN21**.
* OptionCard\_CN20\_CN23 (OptionCardCommDouble) represents the physical connectors **CN20**, **CN21**, **CN22** and **CN23**. Each pair of dual-port connectors (**CN20/CN21** and **CN22/CN23**) provides its own communication technology and has dedicated FPGAs. An option card of type OptionCard\_CN20\_CN21 (OptionCardCommSingle) provides the user functions of the tab Parameters.

Below an option card of the type OptionCard\_CN20\_CN23 (OptionCardCommDouble), the child device objects CommunicationDevice\_CN20\_CN21 and CommunicationDevice\_CN22\_CN23can be added. This device object provides the same functionality as the device object [CommunicationDevice\_CN8\_CN9](CommunicationDevice_CN8_CN9-DeviceO-067E64B6.html).
