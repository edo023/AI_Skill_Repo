# RAM Disk

## RAM Disk

M660 controllers provide a RAM disk with a size of 104 MB.

A RAM disk is a portion of the system RAM that is configured to behave like a storage disk. Access to a RAM disk is faster than to a storage disk, making it suitable for fast file operations. In addition, SD cards have a limited number of program/erase (P/E) cycles. Continuous cyclic write access to the SD card reduces the number of P/E cycles. Using the RAM can contribute to prolonging the service life of the SD card.

However, since the RAM disk resides in the volatile memory, the data is lost when the system is powered off or reboots.

| NOTICE | |
| --- | --- |
|  | LOSS OF DATA  Ensure that data which is essential for your system and which is only contained on the RAM disk is saved to a permanent memory before the controller is powered off.  Failure to follow these instructions can result in equipment damage. |

The RAM disk is accessible via the Files tab of the device editor of the controller, displayed as $ram0$.

In your application, access the RAM using the CAA File library. It is available under the device name $ram0$. In your code, reference it using the string $$ram0$$.
