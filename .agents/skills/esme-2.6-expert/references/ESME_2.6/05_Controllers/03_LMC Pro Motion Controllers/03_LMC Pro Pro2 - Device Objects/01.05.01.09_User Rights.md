# User Rights

## Introduction

User rights contain the following elements: User, Group, Object, Operation, User Rights, Access Rights. These elements allow you to manage user accounts and user access rights to control the access on the global projects.

* A User is a person or a service with specific User Rights.
* A Group is a Persona or a Function. It is predefined or added. Each Group provides accesses using Object.
* An Object is composed of predefined accesses using Operation.
* An Operation is a basic, operational action (for example, edit, delete, etc.).
* User Rights are the possible Access Rights: VIEW, MODIFY, EXECUTE and ADD\_REMOVE for the dedicated operation.

## Default User Management Settings

By default, the following settings are enabled:

* User management password policy
* User login lock
* Check password expiration

These default settings only take effect when no related settings are stored in section CmpUserMgr of sysc3.cfg file.

Due to password expiration policy, the user might be prompted to re-enter a new password when there are changes made to the [PLC Time](D-SE-0006802.html) that result in password expiration.

For more information, refer to [Change Runtime Password Policy Dialog Box](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083386#ChangeRuntimePasswordPolicyDialogBo_9C3C10B5).

## Access Rights

For each Group linked with an Object, User Rights are predefined with specifics Access Rights.

This table indicates the Access Rights:

| Access Rights | Description (depends on the object) |
| --- | --- |
| VIEW | Allow to read only parameters and applications. |
| MODIFY | Allow to write, modify and download parameters and applications. |
| ADD\_REMOVE | Allow to add and remove files, scripts and folders. |
| EXECUTE | Allow to execute and start applications and scripts. |

## Import / Export User Rights

Import User Rights and Export User Rights commands allow you to import/export [Device user management (\*.dum2) files](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0083876). The importation of exported user rights into an LMC controller is only possible if these user rights have been exported from an LMC controller.

## Symbol Rights

The Symbol Rights tab allows you to configure user group access to the symbol sets. It consists of a customizable set of symbols allowing to separate functions and associate them with a user right. If supported by the target device, you can combine different symbol sets from the symbols of the application in the symbol configuration editor. The information about the symbol sets is downloaded to the controller. Then you can define the user group that has access to each symbol set.

## Troubleshooting

To gain access to a controller that has user access-rights enabled and for which you do not have the password(s), perform an Update Firmware operation. You can clear the User Rights by updating the firmware using the offline mechanism in Controller Assistant. This effectively removes the existing application from the controller memory, but restores the ability to access the controller.
