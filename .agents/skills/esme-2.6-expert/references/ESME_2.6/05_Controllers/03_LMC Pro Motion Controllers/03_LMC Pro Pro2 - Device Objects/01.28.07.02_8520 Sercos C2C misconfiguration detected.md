# 8520 Sercos C2C misconfiguration detected

## Description

[Refer to Diagnostic class (standard): 3](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Diagnostic&topicID=D_SE_0063169)

During the configuration of the C2C network, a misconfiguration is detected.

The extended diagnostic provides information on the kind of misconfiguration:

| Ext. diagnosis | Meaning |
| --- | --- |
| `duplicate ID` | There are at least two enabled producer objects (C2C encoder Output, C2C Data Output) which have the same ID.  For details, refer to diagnostic message [8521 Sercos C2C configuration](D-SE-0064012.html#D-SE-0064012). |
| `SercosPhaseUp` | An error condition is active that can only be solved by the Sercos phase up of the superordinate network.  For details, refer to the diagnostic messages displayed in the message logger. |
| `No Cascading` | Both C2C Slave object and C2C Master object are enabled at the same time. Cascading subnetworks is not allowed in a C2C network. A PacDrive LMC cannot be C2C Master and C2C Slave at the same time. |
| `Type mismatch` | There are at least two data objects with the same ID but different types (for example, Encoder/Data). For details, refer to diagnostic message [8521 Sercos C2C configuration](D-SE-0064012.html#D-SE-0064012). |
| `Len mismatch` | There are at least two data objects with the same ID but different data size.  For details, refer to diagnostic message [8521 Sercos C2C configuration](D-SE-0064012.html#D-SE-0064012). |
| `DataT mismatch` | There are at least two data objects with the same ID but different data types.  For details, refer to diagnostic message [8521 Sercos C2C configuration](D-SE-0064012.html#D-SE-0064012). |
| `MaxDataSize` | The maximum data size for inputs and outputs was exceeded. Refer to C2C\Architecture\Boundary conditions.  For details, refer to diagnostic message [8521 Sercos C2C configuration](D-SE-0064012.html#D-SE-0064012). |
