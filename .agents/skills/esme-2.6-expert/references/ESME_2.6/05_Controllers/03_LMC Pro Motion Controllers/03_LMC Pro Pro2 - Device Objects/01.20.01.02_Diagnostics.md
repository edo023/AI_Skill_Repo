# Diagnostics

## General

To be able to perform a diagnostic, the methods GetMasterDiag and GetSlaveDiag as well as their structures ST\_MasterDiag and ST\_SlaveDiag are used, which will be described in the following.

## Methods GetMasterDiag and GetSlaveDiag

The two methods GetMasterDiag and GetSlaveDiag, which are defined in the interface ICmpIoDrvProfibusDiag of the function block IoDrvPROFIBUSDPV1Master, provide access to the structures, which are visible in the "Status" tab of the PROFIBUS DPV1 master.

```
Method GetMasterDiag
METHOD GetMasterDiag : DINT
VAR_IN_OUT
   stMasterDiag : ST_MasterDiag;
END_VAR
```

The structure ST\_MasterDiag is defined in the library IoDrvPROFIBUSDPV1Master in the DataStructs directory. To be able to use the structure, a variable of this type has to be defined.

| Paramter | Meaning |
| --- | --- |
| CommunicationState | Current network status of the communication channel  Value UNKNOWN / 0:  Value NOT\_CONFIGURED / 1.  Value STOP / 2:  Value IDLE / 3:  Value OPERATE / 4: |
| ErrorCount | Total number of detected errors since the last power-up or the last restart. |
| SlaveState | Value 0: not defined  Value 1: OK  Value 2: There is an error in at least one slave.  Value 3: An advisory is pending in at least one slave. |
| NumConfigSlaves | Number of configured slaves in the network |
| NumActiveSlaves | Number of active slaves in the network |
| NumDiagSlaves | Number of slaves with diagnostic in the network |

```
Method GetSlaveDiag
METHOD GetSlaveDiag : DINT
VAR_INPUT
   byStationAddress : BYTE;
END_VAR
VAR_IN_OUT
   stSlaveDiag : ST_SlaveDiag;
END_VAR
```

The structure ST\_SlaveDiag is defined in the library IoDrvPROFIBUSDPV1Master in the DataStructs directory. To be able to use the structure, a variable of this type has to be defined.

Status1

| Bit | Meaning |
| --- | --- |
| D7 | Master\_Lock: slave is being parameterized by a different master |
| D6 | Prm\_Fault: telegram error of the last parameter |
| D5 | Inv.\_Sl\_Res.: implausible answer of the slave |
| D4 | Not\_Supp.: indeterminable command from slave detected |
| D3 | Ext\_Diag: section EXT\_Diag is used for the extended diagnostic |
| D2 | Cfg\_Fault: slave is parameterized incorrectly |
| D1 | Sta.\_Not\_Rdy: slave not ready |
| D0 | Sta.\_Non\_Exist: slave not responding |

## Result of the Diagnostic

Should an error have occurred, it is listed under the "Status" register of the respective device dialog box.

NOTE: Diagnostic parameters may be traced in all tasks **except for** the SystemMotionTask since this will cause a delay of the retrieval of the diagnostic from the NetC component and thus of the RTP.
