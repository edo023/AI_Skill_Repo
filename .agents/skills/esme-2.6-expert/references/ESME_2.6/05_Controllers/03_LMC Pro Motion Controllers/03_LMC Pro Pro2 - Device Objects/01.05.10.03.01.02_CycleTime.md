# CycleTime

## General

|  |  |
| --- | --- |
| Type | ER |
| Devices supporting the parameter | PacDrive LMC x00C, PacDrive LMC x01C |
| Traceable | Yes |

## Functional Description

Defines the Sercos bus cycle time in ns.

In **V1.31.10.0**, only 1 ms CycleTime is permissible.

For versions **V1.32.X.0**, the cycle time can be set to either 1, 2 or 4 ms. The values must be entered in ns (1000000 ns, 2000000 ns, 4000000 ns).

If any deviating numerical values are entered, they are corrected automatically as follows:

| Entering | Cycle time |
| --- | --- |
| From 0000000 to 1999999 ns | Is turned into 1000000 ns |
| From 2000000 to 3999999 ns | Is turned into 2000000 ns |
| From 4000000 to 9999999 ns | Is turned into 4000000 ns |

The CPU load (AvailableLoad) is a determining factor when choosing the cycle time.

NOTE: If the CycleTime is changed, the controller must be reset. In **V1.32.6.0**, download of the application is followed by a message indicating that a cold start-up of the controller has to be executed in order to apply the values.

Proceed as follows: Transfer the program to the controller (download) and then execute a cold start-up of the controller.

The maximum number of real Sercos slave devices depends on the:

* PacDrive controller type
* Sercos slave device type
* Configured functions (DriveIO or DriveTouchprobe)
