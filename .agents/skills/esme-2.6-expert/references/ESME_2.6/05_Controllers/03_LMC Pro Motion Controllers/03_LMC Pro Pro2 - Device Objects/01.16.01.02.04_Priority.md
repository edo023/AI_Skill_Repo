# Priority

## General

|  |  |
| --- | --- |
| Type | EF |
| Devices supporting the parameter | Cam switch group |
| Traceable | No |

## Functional Description

Displays the priority at which the tasks are processed.

NOTE: The priority of the Cam switch group objects that were set using the Priority or IECTaskPriority parameters cannot be the same for two Cam switch group objects.

For special requirements, you can set the priority of the cam switch group one priority level below or above the real-time process (RTP). The tasks of the RTP include processing the master encoder and the logical encoder, processing motion orders with reference to value calculation, processing the measuring inputs, recording the trace values, and of course exchanging real-time data over the real-time bus (Sercos). A change to the parameter takes effect immediately.

| Value | Data type | Meaning |
| --- | --- | --- |
| IEC-Priority / 0 | DINT | This priority setting is equivalent to the priority setting for tasks in the task configuration. This setting must be made in parameter IECTaskPriority. |
| lower RTP / 1 | DINT | The cam switch group is not affected by tasks, communication drivers, or diagnoses. |
| higher RTP / 2 | DINT | The cam switch group is not affected by tasks, communication drivers, diagnoses, or the real-time process. |

NOTE: This priority setting manipulates system behavior. It may lead to undesirable side effects.

The "higher RTP / 2" priority setting should only be set in consultation with your Schneider Electric representative.
