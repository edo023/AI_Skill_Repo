# NetID

## General

|  |  |
| --- | --- |
| Type | EF |
| Devices supporting the parameter | PacDrive LMC x00C,  PacDrive LMC x01C |
| Traceable | No |

## Functional Description

By using this parameter, you can define the offset for the two address ranges to be calculated.

| Value | Data type | Meaning |
| --- | --- | --- |
| 172.20.0.0 (default) | STRING(15) | This parameter defines the offset for the two address ranges to be calculated.  The parameter must have the following format: [a1].[a2].[a3].[a4]. The third position ([a3]) must be divisible by 4 without remainder. This condition results from the subnet mask.  Access type: read/write |

While the value for this parameter is being entered, it is checked whether the third position ([a3]) is divisible by 4 without remainder.
