# FC\_OpcUaStart

## Overview

|  |  |

| Type: | Function |
| Available as of: | V1.51.0.0 |
| Inherits from: | - |

## Task

Start the OPC UA server.

NOTE: By default the OPC UA server is deactivated.

## Description

The OPC UA server is initialized and started with the parameters of the ServerConfig.ini file which is located on the compact flash card. By the first start procedure, a certificate is generated from the certificate parameters of the ServerConfig.ini file.

NOTE: The run of the function FC\_OpcUaStart() takes place at the same time as the start procedure of the server and requires at least 60 ms.

NOTE: If no valid user name and password is assigned to the OPC UA server start function, then the default user name and the default password is used.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_sUser | STRING[255] | User name |
| i\_sPassword | STRING[255] | Password |

## Return Value

| Data type | Description |
| --- | --- |
| DINT | 0: The OPC UA server is started.  1: The standard- user name and password is active.  -1: The OPC UA server is already running.  -4: An invalid combination of user name and password, for example, one part empty. The standard- user name and password is used. |

## Example

```
diResult : DINT;
diResult := Systeminterface.FC_OpcUaStart('User', 'Password');
diResult := Systeminterface.FC_OpcUaStart(i_sUser := sUser, i_sPassword := sPassword);
```
