# FC\_ControllerStopReset

## Overview

|  |  |
| --- | --- |
| Type: | Function |
| Available as of: | SystemInterface\_1.32.6.0 |
| Versions: | Current version |

## Task

Deactivate the command "Stop", for one drive or all drives.

## Description

The command "Stop" for one or all the drives (i\_stAxisId=Gc\_stLogAddrAllTypes) is deactivated.

The controller (for example, PacDrive LMC600) differentiates between three sources for stop requests:

* Diagnostics
* State machine of the controller
* User

You can initiate the "Stop" command via the function FC\_ControllerStopSet or the parameter *[ControllerStopSet](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM52Drive&topicID=D_SE_0071452)*.

The command "Stop" is only deactivated when the partial requests were revoked. You can read the present result by using the parameter ControllerStopSet. An active master-and-slave job is cancelled. The affected drives are stopped. The deceleration is taken from the respective parameter *[ControllerStopDec](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM52Drive&topicID=D_SE_0071532)*. The job processing remains in the "Stop" condition until the stop requirements have been reset. Provided jobs are not affected. The new status is processed in the next real-time cycle. Read *[AxisState](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM52Drive&topicID=D_SE_0071486)* in order to verify the present situation.

Also refer to FC\_ControllerStopSet and *[ControllerStopSet](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM52Drive&topicID=D_SE_0071452)*.
