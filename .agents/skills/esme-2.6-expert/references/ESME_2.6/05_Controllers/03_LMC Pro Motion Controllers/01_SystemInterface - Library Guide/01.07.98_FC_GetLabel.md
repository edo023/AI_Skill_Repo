# FC\_GetLabel

## Overview

|  |  |
| --- | --- |
| Type: | Function |
| Available as of: | SystemInterface\_1.34.1.0 |
| Support for: | - |
| Versions: | Current version |

## Task

Read out label (designation) of a memory medium.

## Description

This function reads out the label of a memory medium. If a device has no label, then an empty string is returned.

NOTE: A USB device has a name, a number, and a label.

The name is specified by the system and looks, for a USB device, as follows: "/usb2msd16a".

The name also contains the number of the USB device (is counted according to the inserting sequence), in this case 0.

The label is the user-defined designation, for example "USB stick" or "DataStore 2 GB". It is only to identify each device precisely among others (possible equal USB devices).

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_sVolumeName | STRING[80] | Name of the device whose label shall be read out |

| Input/Output | Data type | Description |
| --- | --- | --- |
| iq\_sLabel | STRING[11] | Label of the device |

## Return Value

| Data type | Description |
| --- | --- |
| DINT | 0: The label has been read out successfully  -1: Error when reading out the label  -318: Invalid parameter |

## Example

**Requirement**

The designation of a USB device shall be determined.

**Declaration**

```
iq_sLabel : STRING := "";
diStatus : DINT := 0;
```

**Program**

```
diStatus := FC_GetLabel ( '/usb2msd16a', iq_sLabel );
```

**Result**

If diStatus = 0, then iq\_sLabel has a valid designation for the desired device.
