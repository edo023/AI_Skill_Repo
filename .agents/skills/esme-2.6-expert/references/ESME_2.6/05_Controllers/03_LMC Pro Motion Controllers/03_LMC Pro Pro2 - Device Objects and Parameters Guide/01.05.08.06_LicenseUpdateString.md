# LicenseUpdateString

## General

|  |  |
| --- | --- |
| Type | EF |
| Devices supporting the parameter | PacDrive LMC x00C,  PacDrive LMC x01C |
| Traceable | No |

## Functional Description

This parameter has three possible values that are each dependent on the state of the LicenseUpdateMode parameter.

| Value of LicenseUpdateMode | Contents of LicenseUpdateString |
| --- | --- |
| off / 0 | The parameter has no meaning and is blocked to input. |
| request key / 1 | In this state, `LicenseUpdateString` contains the request key that has to be forwarded to request license points. |
| authorization key / 2 | For this state, `LicenseUpdateString` is used to enter the authorization key. |

## State of The LicenseUpdate Parameter

After you enter the authorization key, you must restart the system. While booting, the system verifies whether the key is correct. The new number of points is available after the system has booted. If the key has errors, the previous number of points is retained and you are again prompted to enter a correct key. To obtain the authorization key, follow these five steps:

1. Use LicenseUpdateMode to create a request key
2. Send the request key to Schneider Electric
3. Receive the authorization key
4. Set the LicenseUpdateMode parameter to authorization key / 2
5. Enter the authorization key into the LicenseUpdateString parameter
6. Reboot the PacDrive controller
