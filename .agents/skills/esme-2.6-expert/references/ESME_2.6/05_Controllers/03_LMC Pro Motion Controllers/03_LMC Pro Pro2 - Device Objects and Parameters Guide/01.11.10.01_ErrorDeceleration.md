# ErrorDeceleration

## General

|  |  |
| --- | --- |
| Type | EF |
| Offline editable | Yes |
| Devices supporting the parameter | SinCos Encoder Input  Incremental Encoder Input  Encoder network (Synchronization encoder output)  C2C Encoder Input |
| Traceable | Yes |

## Functional Description

A synchronization error on coupled machines is detected in the following cases:

* 8601 "Master Encoder signal out of range" (SinCos)
* 8782 "Master Encoder signal out of range"(Incremental Encoder Input)
* [8522 C2C encoder error detected](D-SE-0064013.html#D-SE-0064013)(C2C Encoder Input)

The parameter ErrorDeceleration can bring the system to a standstill with a defined velocity deceleration. When a track error is detected, the velocity is set to 0 with the defined deceleration.

If 0 is entered in ErrorDeceleration, then no velocity deceleration is created by the system.

Encoder behavior when ErrorDeceleration = 0 and track error has been detected:

* SinCos encoder input (physical master encoder): The velocity remains constant.
* Incremental Encoder Input: The velocity always follows the received incremental encoder input signals. If the connection to the encoder is interrupted, the velocity is lowered to 0.
* Synchronization encoder output ([Encoder network](D-SE-0076315.html#D-SE-0076315)): The last known velocity is maintained.
* C2C Encoder Input: The last known velocity is maintained.
