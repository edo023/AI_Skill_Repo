# General Information

The parameters explained in this document are characterized by parameter types. The following table provides information on the different parameter types:

| Type | Offline editable | Properties | Read value | Write value |
| --- | --- | --- | --- | --- |
| ER | Yes | Input with user initialization  Transfer only after reset | Quick memory access | Not used, as transfer takes place only after reset |
| ED | Yes | Input with user initialization  Transfer directly after the modification | Quick memory access | Quick memory access |
| ED | No | Input with automatic initialization to default value  Transfer directly after the modification Can only be modified online by the Logic Builder | Quick memory access | Quick memory access |
| EF | Yes | Input with user initialization  Transfer directly after the modification | Quick memory access | Functional access (internal calculation required) |
| EF | No | Input with automatic initialization Transfer directly after the modification | Quick memory access | Functional access (internal calculation required) |
| ES | No | Input Transfer directly after the modification Can only be modified online by the Logic Builder | Quick memory access | Communication via Sercos  Delay of the caller (typically 10-100 ms) |
| AK | No | Output Constant value | Quick memory access | Not possible |
| AD | No | Output Dynamic value | Quick memory access | Not possible |
| AF | No | Output  Dynamic value | Functional access (internal calculation required) | Not possible |
| AS | No | Output  Dynamic value | Communication via Sercos  Delay of the caller (typically 10...100 ms) | Not possible |
| NOTE: If the parameter is displayed in the Logic Builder in a black font, it is editable offline. If it is displayed in a gray font, it is only editable online. | | | | |

NOTE: In the case of parameters read or written via Sercos, the typical time delay is 10...100 ms. However, this value can be delayed by 100 ms or more, for example, when using trace logging or Error Quit trace logging or Error Quit.  
Tracing these parameters causes a delay of the trace task.
