# FTP Client

## Introduction

The controller provides an FTP client. The FTP client can establish an unencrypted FTP connection as well as an encrypted FTPS connection (FTP over TSL) to an FTP server.

Connections can be established using the library **FtpRemoteFileHandling**.

You can use the function block FB\_FtpClient for unencrypted connections.

Alternatively, you can use the function block FB\_FtpSecureClient for encrypted connections (FTP over TSL).

For details, refer to the [FtpRemoteFileHandling Library Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=ftplib&topicID=).

NOTE: Schneider Electric adheres to industry best practices in the development and implementation of control systems. This includes a "Defense-in-Depth" approach to secure an Industrial Control System. This approach places the controllers behind one or more firewalls to restrict access to authorized personnel and protocols only.

## Certificate Management

Certificates obtained from an FTP server can be managed using the Security Screen.

When you initially connect to an FTP server, the controller as the FTP client receives a certificate that you need to trust to establish the connection. You can view the certificate on the tab Security Screen in the folder Untrusted Certificates in order to verify whether you want to trust it.

For details on declaring a certificate as trusted, refer to the [How to Manage Certificates on the Controller User Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=HowMgCer&topicID=).
