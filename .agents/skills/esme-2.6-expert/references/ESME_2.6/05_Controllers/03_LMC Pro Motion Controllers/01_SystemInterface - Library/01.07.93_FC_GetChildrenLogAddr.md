# FC\_GetChildrenLogAddr

## Overview

|  |  |
| --- | --- |
| Type: | Function |
| Available as of: | SystemInterface\_1.32.6.0 |
| Versions: | Current version |

## Task

Determine the logical addresses of the subordinate "child objects" in the PLC configuration (for example, which real Sercos slaves are configured on the bus).

## Description

The logical addresses of the subordinate "child objects" of an object i\_stParentId in the PLC configuration are determined. This can be done for every object in the PLC configuration. The logical addresses of the existing "child objects" are copied to the transferred array iq\_astChildrenLogAdr. The number of existing "child objects" is output as the return value of the function FC\_GetChildrenLogAddr(). The number of fields described in the array iq\_astChildrenLogAdr transferred equals the number of "child objects".

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_stParentId | ST\_LogicalAddress | Object whose "child object" has to be determined |

| Input/Output | Data type | Description |
| --- | --- | --- |
| iq\_astChildrenLogAdr | ARRAY [1..1024] OF ST\_LogicalAddress | Logical addresses of the existing "child objects" |

## Return Value

| Data type | Description |
| --- | --- |
| DINT | >0: Number of existing child objects  0: No child objects found  -1: Invalid logical address < i\_stParentId > |
