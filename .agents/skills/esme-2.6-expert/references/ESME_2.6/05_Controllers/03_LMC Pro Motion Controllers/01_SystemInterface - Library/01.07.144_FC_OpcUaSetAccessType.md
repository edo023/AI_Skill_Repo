# FC\_OpcUaSetAccessType

## Overview

|  |  |
| --- | --- |
| Type: | Function |
| Available as of: | V1.56.15.12 |
| Inherits from: | - |
| Implements: | - |

## Task

Configure the access type of the OPC UA server.

## Description

The access type of the OPC UA server is configured.

The OPC UA server can access variables via different methods. The default access method is IecVarAcc (ET\_OpcUaAccessType.IECVARACC). Some variables can be accessed faster via direct memory access (ET\_OpcUaAccessType.MEMORYACC). If the access type is set to memory access, the OPC UA server uses memory access where possible.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_etOpcUaAccessType | [ET\_OpcUaAccessType](D-SE-0084923.html#D-SE-0084923) | OPC UA server configuration |

## Return Value

| Data type | Description |
| --- | --- |
| DINT | 0: New access type is set.  -1: Invalid access type specified, access type was not changed. |

## Example

```
etNewAccessType : ET_OpcUaAccessType;
diResult : DINT;
etNewAccessType := ET_OpcUaAccessType.MEMORYACC
diResult := Systeminterface.FC_OpcUaSetAccessType(i_etOpcUaAccessType := etNewAccessType);
```
