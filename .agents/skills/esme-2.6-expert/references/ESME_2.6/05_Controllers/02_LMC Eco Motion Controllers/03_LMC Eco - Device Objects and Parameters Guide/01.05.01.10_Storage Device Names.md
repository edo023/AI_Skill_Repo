# Storage Device Names

## Placeholder Device Names

Storage devices, such as memory cards (CF or SD cards) or USB keys, are automatically mounted to the system.

The device names are available using placeholders.

Available placeholders:

* $ram0$: Access to the files on the RAM storage
* $ide0$: Access to the files on the memory card (boot partition)
* $USBDevice1$: Access to the files on the first found USB device (/msd0a)
* $USBDevice2$: Access to the files on the second found USB device (/msd16a)

NOTE: The device names for USB keys may change depending on whether the USB key was connected before or after the boot process of the controller.

## Placeholders in the Online File Explorer

In the online file explorer, placeholders are displayed in a blue color.

The placeholder device names for USB keys are only visible if the USB key is connected.

The placeholder $ide0$ is not visible in the online file explorer. The files are displayed directly in the root view.

## Placeholders in Applications

In your application, placeholder device names need to be escaped using the character $.

Example:

```
VAR
	hFile : POINTER TO BYTE;
	result : UDINT;
END_VAR

// Open a file using the placeholder for the USB device
hFile := SysFileOpen('$$USBDevice1$$/test.txt', ACCESS_MODE.AM_READ, ADR(result));
IF result = 0 THEN
	// File operations here
	SysFileClose(hFile);
END_IF
```

## Device Name Structure for Compatibility

The path to the memory storages is named according to the following schema:

```
/msd<Device><Partition>
```

`<Device>` is replaced by a number per device:

* `0` for the first found device
* `16` for the second found device
* `32` for the third found device
* and so on

`<Partition>` is replaced by a single character for each partition of the device:

* `a` for the first partition
* `b` for the second partition
* `c` for the third partition
* and so on

Example:

The third partition of the second found device, for example, a USB key, is mounted to `/msd16c`.

For compatibility reasons, the partition containing the operation system ("boot partition") is renamed to `ide0:`.

NOTE: The device names for USB keys may change depending on whether the USB key was connected before or after the boot process of the controller.
