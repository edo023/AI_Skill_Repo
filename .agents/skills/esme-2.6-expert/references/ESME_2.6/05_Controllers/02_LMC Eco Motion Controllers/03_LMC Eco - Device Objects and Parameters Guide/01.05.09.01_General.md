# General

## General Information

The FDR (Fast Device Replacement) function facilitates the replacement of devices, for example, if a device needs to be uninstalled for maintenance and a different device of the same type is installed. If the FDR function is enabled, the PacDrive LMC x00C or PacDrive LMC x01C controller verifies the firmware of the connected Sercos devices.

If a replacement device has a firmware version different from the firmware version stored on the memory card of the controller, the FDR function automatically updates the firmware of the replacement device to the firmware version stored on the memory card of the controller. Depending on the firmware version of the replacement device, the FDR function may perform a downgrade.

The FDR function on the controller uses the firmware versions of the devices contained in the controller image. Refer to the [Controller Assistant User Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0031742) for details on managing firmware and controller images.

## Functional Description

FDR is performed for a set of Schneider Electric devices which are contained in the configuration of the EcoStruxure Machine Expert project in the controller.

The parameter WorkingState of the device has to have the value real/1 for FDR to be performed.

Depending on the configuration, the FDR function is performed during phasing up of Sercos (transition from Communication Phase 1 to Communication Phase 2) or during booting. The parameter State of the Sercos interface contains the communication phase.

Information on FDR is displayed in the controller display and logged by the message logger.

## Limitations

The FDR function can be performed for Sercos devices of Schneider Electric. Third-party devices are not supported.

If FDR is to be used, the devices may not be addressed via the serial number of the motor (`SerialNumberMotor / 1`).

You can only use one firmware version for a given TM5/TM7 safety module type. In Controller Assistant, if you select a version for a module type in a group (SLC 100 / 200 Safety Modules or SLC 300 / 400 Safety Modules), this version is also used for modules of the same type contained in a different group. Refer to the [Controller Assistant User Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=SoMProg&topicID=D_SE_0031742) for details on managing firmware and controller images.
