# Overview User Function NetXEthernet

## Introduction

The following controllers provide an additional standard Ethernet interface:

* PacDrive LMC Pro/Pro2 (embedded)
* PacDrive LMC Eco (option module)

The additional Ethernet interface is configured via the user function NetXEthernet of the Ethernet object in the Devices tree.

Connecting via the Ethernet interface:

* To establish a connection via the additional Ethernet interface with Logic Builder in combination with the connection mode Fast TCP, enter the IP address of the additional port.
* To establish a connection via the additional Ethernet interface with Logic Builder in combination with the connection mode Nodename (or IP Address), use the IP address of the default port.
* To establish a connection via the additional Ethernet interface with Vijeo-Designer in combination with the connection mode Nodename, use the node name of the default port, and ensure that the IP addresses of the network adapters of the PC and the HMIs are configured in accordance with the additional network.

## Limitations

The additional Ethernet interface cannot be used for the encoder network and cannot be used to perform a download of the controller application.

## Parameters

The Ethernet object is configured via the following parameters:

* Common parameters, that are also used by other user functions:

  + LogAdr
  + ObjectType
  + stLogicalAddress

* Specific parameters for the user function NetXEthernet (subnode of UserFunctions):

  + Enable
  + State
  + IP Address
  + Subnetmask
  + NetX\_Connector
  + Valid
