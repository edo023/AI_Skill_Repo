# SlaveCycleErrorLimit

## General

|  |  |
| --- | --- |
| Type | EF |
| Devices supporting the parameter | Encoder network (Sync. Module) |
| Traceable | Yes |

The error tolerance for synchronization is defined using the parameter SlaveCycleErrorLimit as in MasterCycleErrorLimit.

It defines how many cycles must pass before the synchronization master issues diagnostic message 8335 "EncNet synchronization not possible" should it stop receiving status telegrams from a slave. A corresponding alert 8336 "EncNet Sync disturbance detected" is issued after half of those cycles have passed.

NOTE: How you set the error tolerance depends on your network conditions. Contact your local Schneider Electric service representative.

## Example

SlaveCycleTime: 4ms

SlaveCycleErrorLimit: 10

-> The diagnostic message 8336 "EncNet sync fault detected" is triggered after 20.

-> The diagnostic message 8335 "EncNet synchronization not possible" wird is triggered after 40.
