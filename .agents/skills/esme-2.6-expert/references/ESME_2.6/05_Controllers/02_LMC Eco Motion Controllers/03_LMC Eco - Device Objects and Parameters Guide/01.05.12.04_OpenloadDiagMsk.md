## Functional Description

The parameter is used to enter the bit field for activating the diagnostic message `8788 Wiring error detected` with ext. diagnostic message `Openload` for field outputs.

Each bit is assigned to an output corresponding to the bit number. The n bits in the parameter OpenloadDiagMsk are assigned to the bit in the parameter *[OpenloadDiagMsg](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM62Drive&topicID=D_SE_0089341)* in the n outputs. Modifications in the parameter OpenloadDiagMsk are automatically written to the corresponding outputs in the parameter OpenloadDiagMsg and vice-versa. The last modification is active. Also see parameter *[OverloadDiagMsk](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LXM62Drive&topicID=D_SE_0071986)*.

| Bit | Meaning |
| --- | --- |
| 0 | The output is not monitored. |
| 1 | The output is monitored and the diagnostic message is sent if monitoring with the DiagMask parameter is also activated. |

NOTE: If the output is not connected while monitoring is activated, the diagnostic message `8788 Wiring error (Openload)` is sent immediately.
