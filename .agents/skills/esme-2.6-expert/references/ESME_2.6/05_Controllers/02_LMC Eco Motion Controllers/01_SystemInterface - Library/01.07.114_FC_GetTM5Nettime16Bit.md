# FC\_GetTM5Nettime16Bit

## Overview

|  |  |
| --- | --- |
| Type: | Function |
| Available as of: | V1.54.3.1 |

## Description

This function lets you correlate the timing in the Real Time Process (RTP) of the controller and the cycle of a TM5 bus interface.

The function uses a time offset i\_wRTPOffset that is relative to the start of the ongoing RTP cycle. It calculates an absolute timestamp which is valid in the time clock of the given device.

Calculation: iq\_wTM5Nettime = [start of RTP cycle] + i\_wRTPOffset

If the function returns a value less than 0, the value of iq\_wTM5Nettime is invalid.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_stLogAddrTM5 | ST\_LogicalAddress | Identifies the device. |
| i\_wRTPOffset | WORD | Time offset relative to start of the RTP cycle in which the function is executed.  Value range: 0...32767  Unit: µs |

| Input/Output | Data type | Description |
| --- | --- | --- |
| iq\_wTM5Nettime | WORD | Calculated timestamp |

## Return Value

| Data type | Description |
| --- | --- |
| DINT | 0: No error detected  -1: The device addressed with i\_stLogAddrTM5 was not found.  -2: Communication with device addressed with i\_stLogAddrTM5 was not possible.  -3: The device addressed with i\_stLogAddrTM5 is not present on the bus.  -4: The value for i\_wRTPOffset was too low.  -5: The value for i\_wRTPOffset was too high. |
