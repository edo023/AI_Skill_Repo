# FC\_ReadCarriersPhysicalId

## Overview

|  |  |
| --- | --- |
| Type: | Function |
| Available as of: | SystemInterface\_1.78.9.4 |
| Versions: | Current version |

## Task

Read the physical IDs of the carriers on a Lexium™ MC multi carrier track.

## Description

The function reads the physical IDs of the carriers on a Lexium™ MC multi carrier track. On a track, you can use two types of carriers: Smart Carriers with a pre-defined ID that is hard-coded in the hardware and Basic Carriers without pre-defined ID.

With the function FC\_ReadCarriersPhysicalId, the controller firmware reads out the IDs of all carriers in the system and indicates the values in the carrier device parameter CarrierIdentification.PhysicalId. For all carriers that cannot be detected as Smart Carriers, the ID indicated is 0.

The following preconditions must be met for the execution of the function FC\_ReadCarriersPhysicalId:

* All segments of the track with the WorkingState = real / 1 indicate the SegmentEnable.State = Enabled / 3.
* All carriers on the track with the WorkingState = real / 1 indicate the CarrierState.Enabled = false / 0.

If the preconditions are not met, the function returns the value -2.

For more information on the parameters [CarrierIdentification.PhysicalId](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=PhysicalId_419E653F), [SercosIdentification.WorkingState](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=WorkingState_B495D426), [SegmentEnable.State](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=State_9BF24E09), [CarrierIdentification.WorkingState](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=WorkingState_B4974475) and [CarrierState.Enabled](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=Enabled_9C690CAD), refer to the Lexium™ MC multi carrier Device Objects and Parameters Guide.
