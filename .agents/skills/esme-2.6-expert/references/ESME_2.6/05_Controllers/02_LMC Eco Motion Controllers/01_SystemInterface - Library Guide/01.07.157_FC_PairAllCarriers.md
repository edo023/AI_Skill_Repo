# FC\_PairAllCarriers

## Overview

|  |  |
| --- | --- |
| Type: | Function |
| Available as of: | SystemInterface\_1.78.9.4 |
| Versions: | Current version |

## Task

Pair the logical carriers (carrier device tree objects) with the physical carriers on a Lexium™ MC multi carrier track.

## Description

On a Lexium™ MC multi carrier track, you can use two types of carriers: Smart Carriers with a predefined ID that is hard-coded in the hardware and Basic Carriers without predefined ID.

When the IDs (PhysicalId) of the Smart Carriers have been read out with the function [FC\_ReadCarriersPhysicalId](ReadCarriersPhysicalId-Gen-420EB61F.html#ReadCarriersPhysicalId-Gen-420EB61F), the device object parameter CarrierIdentification.ConfiguredLogicalId can be modified according to the requirements. After that, a new pairing of the logical carriers to the physical carriers is required.

The following preconditions must be met for the execution of the function FC\_PairAllCarriers:

* All carriers are disabled. If this is not the case, the function returns the value -2.
* The real-time connections of all carriers must have the same configuration. If this is not the case, the function returns the value -3.

The function FC\_PairAllCarriers first unpairs the logical carriers from the physical carriers. Then it starts pairing the LogicalId to the modified ConfiguredLogicalId.

  

NOTE: The modified ConfiguredLogicalId must be within the range of LogicalIds for the carriers on the selected track.

For more information on the parameters [CarrierIdentification.PhysicalId](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=PhysicalId_419E653F), [CarrierIdentification.ConfiguredLogicalId](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=ConfigLogicId_9C6C7FF5) and [CarrierIdentification.LogicalId](../../../../../api/crossBook?lang=en-US&virtualBookName=MCRDOaPG&topicID=LogicId_9C6BFB2C) refer to the Lexium™ MC multi carrier Device Objects and Parameters Guide.

## Return Value

| Data type | Description |
| --- | --- |
| DINT | 0: OK  -2: At least one carrier is not disabled.  -3: At least one carrier connection has a different real-time data configuration.  -4: Two or more logical carriers have the same ConfiguredLogicalId.  -5: The ConfiguredLogicalId of the carrier is invalid.  -6: The logical carrier was not found.  -7: The logical address of the track is invalid. |
