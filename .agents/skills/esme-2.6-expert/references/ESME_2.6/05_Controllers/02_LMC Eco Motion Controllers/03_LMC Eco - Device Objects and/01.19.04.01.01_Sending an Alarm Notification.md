# Sending an Alarm Notification

## General

The following example code explains sending an alarm:

```
PROGRAM SR_PBSlave_SendAlarm
VAR
   ifPBSlave : IF_PROFIBUS_DPV1_Slave := PROFIBUS_DPV1_Slave;
   abyAlarmData : ARRAY[0..(Gc_PROFIBUSDPSlave_C1AlarmDataMaxSize-1)] OF BYTE;
   udiResult : UDINT := 0;
   xSent : BOOL := FALSE;
END_VAR
abyAlarmData[0] := 1;
abyAlarmData[1] := 2;
abyAlarmData[2] := 3;
abyAlarmData[3] := 4;
abyAlarmData[4] := 5;
IF (xSent = FALSE) THEN
   udiResult := ifPBSlave.SendAlarmNotification(
      i_bySlotNumber := 1, i_byAlarmType := Gc_PROFIBUSDPSlave_C1AlarmType_ProcessAlarm,
      // Can be Gc_PROFIBUSDPSlave_C1AlarmType_* or 32-126 for manufacturer-specific alarms.
      i_bySequenceNumber := 0,  // Value/Range: 0-31.
      i_byAddAcknowledge := Gc_PROFIBUSDPSlave_C1AlarmAddAck_Yes,
      // Must be Gc_PROFIBUSDPSlave_C1AlarmAddAck_*.
      i_byAlarmSpecifier := Gc_PROFIBUSDPSlave_C1AlarmSpecifier_00,
      // Must be Gc_PROFIBUSDPSlave_C1AlarmSpecifier_*.
      i_byAlarmDataLength := 5,  // Value/Range: 0-59.
      i_rabyAlarmData := abyAlarmData,  // This area contains alarm specific data.
      i_udiResponseTimeout := 500  // The timeout for the response in ms
   );
xSent :=  TRUE;
END_IF
```

## Possible Actual Values

| Actual values | Description |
| --- | --- |
| i\_byAlarmType: |  |
| Gc\_PROFIBUSDPSlave\_C1AlarmType\_DiagnosticAlarm | Indicates a certain event, such as a short circuit or overtemperature. |
| Gc\_PROFIBUSDPSlave\_C1AlarmType\_ProcessAlarm | Indicates a certain event in the monitored process, such as reaching a critical limit value. |
| Gc\_PROFIBUSDPSlave\_C1AlarmType\_PullAlarm | Indicates that a certain module has been disconnected. |
| Gc\_PROFIBUSDPSlave\_C1AlarmType\_PlugAlarm | Indicates that a certain module has been connected. |
| Gc\_PROFIBUSDPSlave\_C1AlarmType\_StatusAlarm | Indicates the state change of a certain module, such as "run", "stop" or "ready". |
| Gc\_PROFIBUSDPSlave\_C1AlarmType\_UpdateAlarm | Indicates the value of a parameter in a module which has been changed by means of local operation or remote access. |
| i\_bySequenceNumber: |  |
| Value/Range: 0-31. |  |
| i\_byAddAcknowledge: |  |
| Gc\_PROFIBUSDPSlave\_C1AlarmAddAck\_No | This alarm does not have to be acknowledged separately by the user. |
| Gc\_PROFIBUSDPSlave\_C1AlarmAddAck\_Yes | This alarm has to be acknowledged separately by the user. |
| i\_byAlarmSpecifier: |  |
| Gc\_PROFIBUSDPSlave\_C1AlarmSpecifier\_00 | Not differentiated further. |
| Gc\_PROFIBUSDPSlave\_C1AlarmSpecifier\_01 | An error has occurred and is displayed. The slot is disturbed. |
| Gc\_PROFIBUSDPSlave\_C1AlarmSpecifier\_10 | An error has occurred and is not displayed. The slot is ok. |
| Gc\_PROFIBUSDPSlave\_C1AlarmSpecifier\_11 | An error has occurred and is not displayed. The slot continues to be disturbed. |
| i\_byAlarmDataLength: |  |
| Value/Range: 0-59. |  |
