# Migration Notes for Users of Hardwired Encoder Coupling or Encoder Network

## Description

C2C network can be a functional replacement for the [encoder network](D-SE-0070719.html#D-SE-0070719).

The main benefits of the C2C network compared to hardware encoder coupling and encoder network are:

| C2C network | Encoder network |
| --- | --- |
| No additional hardware is needed. | Hardwired encoder coupling (BT-4/Enc1 is needed.)  Encoder network (Managed switch is needed.) |
| No hardwired encoder connection is needed. | Hardwired encoder coupling |
| A deterministic real-time communication channel is used. | Encoder network (non real-time/ non deterministic Ethernet is used.) |
| The processor load is reduced. | Encoder network |
| It is possible to distribute several encoder signals. | Encoder network  Hardwired encoder coupling |
| The producer can be on master or slave side. | Encoder network  Hardwired encoder coupling |
| The jitter between the synchronized systems is minimized. | Encoder network |

NOTE: The application design and programming of the C2C network is different compared to hardwired encoder coupling or encoder network.

Your existing applications need to be adapted and validated accordingly.

Read this documentation to learn how to use the C2C network.
