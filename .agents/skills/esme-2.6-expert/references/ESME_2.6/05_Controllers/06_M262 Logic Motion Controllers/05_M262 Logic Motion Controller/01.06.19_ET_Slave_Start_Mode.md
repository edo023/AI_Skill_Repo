# ET\_Slave\_Start\_Mode

## Overview

|  |  |
| --- | --- |
| Type: | List type |
| Available as of: | V1.1.75.6318 |

## Description

This enumeration is used as input option of MC\_CamIn. It selects if, at start, the cam output position is used as subordinate axis position or if the subordinate axis position does not change.

When the cam gets active after a positive edge at the input Execute of MC\_CamIn, this enumeration defines how to handle a position delta between the first calculated position of the cam law and the position the subordinate axis has when the cam gets active.

NOTE: Discrepancies between the physical position of the cam and the position in the cam definition may provoke position jumps if you use the subordinate axis start mode Absolute. If there is a position difference between the position of the subordinate axis and its calculated start position and this start position can be reached in spite of the position difference, this movement may be made in the form of a sudden position jump.

| WARNING | |
| --- | --- |
|  | UNINTENDED EQUIPMENT OPERATION  Verify the physical position of the subordinate axis at the start of the cam and verify that it matches the position in the cam definition.  Failure to follow these instructions can result in death, serious injury, or equipment damage. |

## Enumeration Elements

| Name | Value (INT) | Description |
| --- | --- | --- |
| Absolute | 0 | For starting the cam, the position of the subordinate axis is directly set to the first calculated Y value. The subordinate axis position is calculated on the basis of the cam definition and “master as seen by the subordinate axis”. As opposed to the subordinate axis start modes Relative and RampIn where there are no offsets and superimposed movements. The reference velocity and the acceleration are calculated on the basis of the cam definition.  If there is a position difference between the position of the subordinate axis and its calculated start position (Y value) for the cam, and if this start position cannot be reached within one task scan, an error is detected. However, if this start position can be reached in spite of the position difference, this movement may be made in the form of a sudden position jump. |
| Relative | 1 | After the master as seen by the subordinate axis for the start is determined according to ET\_MasterStartMode, the cam position is calculated. For the duration of the cam running, the cam is shifted by the difference between this calculated cam position at start and the axis position at this time. This prevents a position jump at the start of the cam while respecting the shape of the cam. |
| RampIn | 2 | It is assumed that the absolute subordinate axis position is equal to the cam Y coordinate for the cam to be in sync.  At the beginning, the cam starts like subordinate axis relative, meaning f(X start) is correlated with the absolute subordinate axis position when the cam starts. Then a ramp-in movement is performed which offsets the subordinate axis to align the coordinate system of the axis position with the coordinate system of Y. |
