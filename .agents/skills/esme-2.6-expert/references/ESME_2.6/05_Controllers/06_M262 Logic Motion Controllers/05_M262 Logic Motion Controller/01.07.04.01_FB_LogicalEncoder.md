| Implements: | IF\_AuxiliaryMotionCommand, IF\_LogicalEncoder |

## Description

A logical encoder is a software device that allows you to create encoder functionality. It implements the interface IF\_Axis. A logical encoder mirrors the movements of its connected parent axis. The parent axis is the axis that corresponds to the parent device of the logical encoder. Drives, physical or logical encoders can be used as parent devices.

The logical encoder can be used as a master axis for synchronized motion tasks, for example, in conjunction with the function blocks MC\_CamIn, MC\_GearIn or MC\_CustomJob. This way, the connected drive or encoder axis can be used for other purposes.

Movements can be superimposed on the logical encoder if you connect a superimposed axis. A logical encoder itself can serve as the superimposed axis.

Permissible axis types for the logical encoder are linear without limits and modulo as provided by the interface IF\_Axis. You can modify the axis type of a logical encoder as long as it is not the master axis of a synchronized motion job.

If neither a parent axis nor a superimposed axis is connected, the logical encoder is in the PLCopen state StandStill. Once the logical encoder is connected, it transitions to the PLCopen state SynchronizedMovement.

Reference an instance of a logical encoder by its name, for example, fbMyLogicalEncoder.

A logical encoder has no PowerOn property. It is considered to be powered off.

A logical encoder is treated as being homed. Executing the function block MC\_SetPosition does not modify the flag xHomed of the logical encoder axis.

If the parent device of the logical encoder is disabled or an error is detected, the state of the logical encoder does not change. It follows the parent device regardless of its state.

A logical encoder itself can be the parent device of another logical encoder, that is, logical encoders can be nested. If this results in a loop, an error is detected (RefPosGeneratorInvalidCascading).

The logical encoder can only be used in conjunction with multi-axis motion function blocks. If you use it with a single-axis motion function block such as MC\_MoveAbsolute, an error is detected (NotSupportedWithLogicalEncoder). The logical encoder can only be used for a single absolute synchronized motion job with MC\_CamIn and MC\_CustomJob. This does not apply to MC\_GearIn since this is based on velocity.

**Performance considerations**

A logical encoder has an impact on the performance of the controller. One virtual axis corresponds to approximately six encoder axes in terms of load.

**Connection to and disconnection from a parent device**

When the logical encoder is connected to the parent device, its movement values are not modified and remain as they were before the connection. The logical encoder only follows the movements of the parent device when the parent device moves. The logical encoder can be connected in the PLCopen states StandStill and SynchronizedMotion, with or without following movements, for example, as a cam.

When the logical encoder is disconnected from the parent device, the movement values are retained and the axis is set to the PLCopen state StandStill. The logical encoder can be disconnected even if it serves as a master axis for a synchronized movement. The only effect of disconnecting it is that the velocity of the logical encoder is set to zero.

**Logical encoder and MC\_SetPosition**

The function block MC\_SetPosition can be executed for a logical encoder. If the axis type of the logical encoder is LinearWithoutLimits, there are no restrictions in terms of position. If the axis type of the logical encoder is Modulo, the position must be within the modulo range. If it is not, an error is detected (PositionOutsideModulo).

**Logical encoder and MC\_PhasingAbsolute**

The function block MC\_PhasingAbsolute can be started using the logical encoder as master axis if MC\_CamIn is already running on the logical encoder as a master axis. If this condition is not met, an error is detected (NoCamInJobOnSlaveAxis). If the logical encoder is not connected to a parent device,  an error is detected (LogicalEncoderHasNoMasterConnected).

**Logical encoder and MC\_CamIn**

If the logical encoder is used in conjunction with the function block MC\_CamIn, the start modes Absolute, Relative and KeepEncoderPosition are available:

* Absolute

  If the logical encoder has not been used in the context of the cam before, the position of the parent device is set as logical encoder position. If the logical encoder has been used in the context of the cam before (for example, a sequence of cams), the logical encoder position remains unmodified.
* Relative

  The position of the logical encoder is set to the start point of the cam. If the cam moves backwards, the logical encoder is set to the upper  X limit of the cam, otherwise to the lower.
* KeepEncoderPosition

  The cam starts from the position of the logical encoder. If you attempt to use a master axis other than a logical encoder with this start mode with MC\_CamIn, an error is detected (MasterStartModeOnlySupportedWithLogicalEncoder).

If you use a logical encoder as a master axis of the function block MC\_CamIn, the inputs MasterScaling MasterOffset of MC\_CamIn cannot be used. You must use the default values for these inputs.

If you use a logical encoder as a master axis for the function block MC\_CamIn, the movement values in the structure stMotionOfMaster of the interface IF\_Axis must be the movement values of the logical encoder.

**Logical encoder and superimposed movements**

Movements can be superimposed on the logical encoder, and a logical encoder can be the superimposed axis itself. If the axis and the superimposed axis connected with the method ConnectSuperimposed form a loop, an error is detected (RefPosGeneratorInvalidCascading).

The logical encoder does not use the absolute position of the superimposed axis, but adds the cyclic increment of the superimposed axis to its cyclic movement. Because of that, MC\_SetPosition and modulo are not taken into account.

The property stMotionOfSuperimposed is a reference to the structure ST\_MovementValues and allows you to get the position, velocity and acceleration of the superimposed axis.

**Devices tree object**

A logical encoder can also be added via the Devices tree. To do so, first add the device LogicalEncoderPool under the controller. You can then add one or multiple Logical\_Encoder objects under the LogicalEncoderPool. Before such a Logical\_Encoder object can be used, connect it to a parent device with Logical\_Encoder.Axis.ConnectMasterAxis (the same restrictions as for FB\_LogicalEncoder.ConnectMasterAxis apply). Use Logical\_Encoder.Axis.DisconnectMasterAxis to disconnect the logical encoder from its parent device.
