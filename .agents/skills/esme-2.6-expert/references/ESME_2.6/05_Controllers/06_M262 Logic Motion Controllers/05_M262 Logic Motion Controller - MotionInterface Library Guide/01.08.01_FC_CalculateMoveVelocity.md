# FC\_CalculateMoveVelocity

## Overview

|  |  |
| --- | --- |
| Type: | Function |
| Available as of: | V2.30.0.0 |

## Description

This function calculates the motion profile for a movement from an initial movement state to a target velocity on the basis of the movement values provided via the inputs. The values include the start position, the start velocity, the start acceleration, the acceleration and deceleration during the movement, the jerk and the target velocity. The calculated motion profile can consist of various movement phases.

The calculated motion profile is returned via the output q\_stTrajectory in the structure [ST\_Trajectory](ST_Trajectory-40FFC26B.html#ST_Trajectory-40FFC26B) as an array of motion profile points in the structure [ST\_TrajectoryPoint](ST_TrajectoryPoint-4100368A.html#ST_TrajectoryPoint-4100368A).

The motion profile can be acceleration-based or jerk- based (enumeration [ET\_TrajectoryType](ET_TrajectoryType-40FF1B22.html) ).

If an acceleration-based motion profile is used, the motion profile consists of various movement phases with constant acceleration within a given movement phase. The acceleration can change between movement phases.

If a jerk-based motion profile is used, the motion profile consists of various movement phases with identical jerk within a given movement phase. The jerk can change between movement phases. It can be zero in individual movement phases, that is, infinite jerk.

The structure ST\_Trajectory contains the type of the motion profile (acceleration-based or jerk-based), the number of movement phases of the motion profile and the reference to the array of motion profile points in the structure ST\_TrajectoryPoint. The number of motion profile points corresponds to the number of movement phases plus one point.

The array of ST\_TrajectoryPoint contains the movement values between the movement phases plus the initial and final movement state. If there are n movement phases, this results in n+1 motion profile points. The initial motion profile point represents the initial movement state of the initial movement phase. For an acceleration-based motion profile, the initial acceleration value of this movement phase is used for the complete movement phase. For a jerk-based motion profile, the initial jerk value of this movement phase is used for the complete movement phase. Along with the movement state, the motion profile points also contain the time (in seconds) when the motion profile point is reached. Thus, the last motion profile point represents the end position, velocity, acceleration and time.
