Commissioning an Automation System

Setting

In order to use the PacDrive automation system, you have to control the hardware using a PC with EcoStruxure Machine Expert. Minimum hardware for the example in this document is a logic motion controller, a power supply, a servo drive, a motor, and a touch probe.

Hardware

Choosing and assembling the hardware (controller, power supply, servo drive) is beyond the scope of this document. Refer to the technical documentation for the hardware you are using.

The test-/training-hardware-assembly used for this document consists of the following components:

oLogic motion controller: PacDrive LMC 300C

oPower supply: Lexium LXM62PSD

oServo drive: Lexium LXM62D

oMotor: BSH0701T01A2A

oA typical Touch Probe sensor

If you use different components to follow this example, you need to adapt the instructions accordingly.

New Project from Template

| Step | Action |
| --- | --- |
| 1 | From the Windows Start menu start Logic Builder Vx.y Motion. |
| 2 | Create a new project from template. To do this, click File > New Project > From Project Template > Template Full. |

Identifying the Motion Controller

If you have just one test-system, the motion controller identifies itself by being the only one present.

If many controllers are attached to the same network, identify the controller by initiating optical and acoustical signaling of the controller. Also, the controller shows its IP-address on the display.

To identify the controller by initiating optical and acoustical signaling of the controller, perform the following steps:

| Step | Action |
| --- | --- |
| 1 | Right-click on LMC\_PacDrive > Controllerselection > LMC 300C. |
| 2 | Click Signal optical or Signal optical and acoustical. |

IP-V4-Settings

In order to connect the controller and EcoStruxure Machine Expert via Ethernet with IP V4, verify that IP-V4-settings match. You can accomplish this by [configuring the IP-V4-settings of the network adapter in Windows](#XREF_D_SE_0066416_8) or by [configuring the IP-settings of the controller.](#XREF_D_SE_0066416_9)

Configuring the IP-V4-Settings of the Network Adapter in Windows

| Step | Action |
| --- | --- |
| 1 | Start Windows Control Panel > Network and Sharing Center > Change adapter settings > LAN-Connection. |
| 2 | Select Internet Protocol Version 4 (TCP/IPv4) and click Properties. |
| 3 | Set the IP address to a value in the range of the IP address mask for the controller and set a suitable subnet mask. No gateway address and no DNS server address are needed if you only connect the PC to the controller and if there are no other network connections. |
| 4 | Confirm the settings and an Ethernet connection to the controller is established. |

Configuring the IP-Settings of the Controller

You can modify IP-settings of the controller even if they do not match the Windows-IP-settings.

| Step | Action |
| --- | --- |
| 1 | Select the controller in the list of controllers. |
| 2 | Right-click the controller entry and execute the command Process communication settings... from the contextual menu.  Result: The Process communication settings... dialog box opens with the settings of the controller.  NOTE: Some, if not most, controllers provide a parameter (such as RemoteAccess) that helps prevent changing communication parameters of the controller. |
| 3 | Configure the communication parameters:  oBoot Mode  oFIXED: A fixed IP address is used according to the values entered below (IP address, Subnet mask, Gateway).  oBOOTP: The IP address is received dynamically by BOOTP (bootstrap protocol). The values below are ignored.  oDHCP: The IP address is received dynamically by DHCP (dynamic host configuration protocol). The values below are ignored.  NOTE: Not all devices support BOOTP and/or DHCP.  oIP address  When configuring IP addresses, refer to the safety message below.  This box contains the IP address of the controller. It is a unique address that consists of four numbers in the range of 0...255 separated by periods. The IP address has to be unique in this (sub)network.  oSubnet mask  The subnet mask specifies the network segment to which the controller belongs. It is an address that consists of four numbers in the range of 0...255 separated by periods. Generally, only the values 0 and 255 are used for subnet mask numbers. However, other numeric values are possible. The value of the subnet mask is generally the same for all controllers in the network.  oGateway  The gateway address is the address of a local IP router that is located on the same network as the controller. The IP router passes the data to destinations outside of the local network. It is an address that consists of four numbers in the range of 0...255 separated by periods. The value of the gateway is generally the same for all controllers in the network.  oTo save the communication settings in the controller even if it is restarted, activate the option Save settings permanently. |
| 4 | Click OK to transfer the settings to the controller. |

Carefully manage the IP addresses because each device on the network requires a unique address. Having multiple devices with the same IP address can cause unintended operation of your network and associated equipment.

Power Supply Settings

Most of the default settings do not need to be modified; however, a few settings need to be adjusted.

| Step | Action |
| --- | --- |
| 1 | Click Devices tree > PSM\_PowerSupply > Parameters > Common and state. |
| 2 | Set PhaseCheckMode to Check only one phase. |
| 3 | Set MainsVoltageMode to 230V. |
| 4 | Set BrakingResistorMode to 230V Mode. |
