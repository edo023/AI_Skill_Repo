# Applications / Ports (Default Firewall Configuration)

## Overview

The following applications / ports are opened by the default firewall configuration:

| Protocol | Port Number | Application | Default Rule State | Default Service State |
| --- | --- | --- | --- | --- |
| ICMP | - | Network management, for example, ping | Active | Active |
| TCP | 21 | Commands of the FTP server | Active | Active |
| TCP | 1105 | NetManage | Active | Active |
| TCP | 4840 | OPC UA | Active | Inactive |
| TCP | 5000 | MArea | Active | Inactive |
| TCP | 8080 | WebVisu | Active | Inactive |
| TCP | 11740 | NetManage | Active | Active |
| UDP | 1202 | Network variables | Active | Inactive |
| UDP | 1203 | Network variables | Active | Inactive |
| UDP | 1740...1743 | CoDeSys services | Active | Active |
| UDP | 27127 | NetManage | Active | Active |
| TCP | >= 65000 | File transfer of the FTP server | Active | Active |
| TCP | - | Port forwarding for TM5CSLC•00 | Inactive | Active |
| UDP | - | Port forwarding for TM5CSLC•00 | Inactive | Active |

NOTE: Refer to the CommonToolbox Library Guide for functions to configure the firewall of a PacDrive LMC controller to allow for access to a TM5CSLC•00 safety logic controller via EcoStruxure Machine Expert - Safety.

NOTE:

* Rules can be set to inactive by commenting out in the configuration file.
* Rules marked as “inactive rule state” (in table above) are commented out by default.
* Services marked as inactive in service state are not running by default, but must be started by you. The corresponding ports are opened in the firewall so that the service is available without additional firewall configuration.
