# Configuring the Fallback Behavior

![](images/logo.png)

Configuring the Fallback Behavior

Introduction

This functionality is available for TM2 expansion modules with outputs.

You can configure the fallback behavior, if the expansion module is connected to a distributed device:

oOTB device

oTM3 bus coupler

The fallback is triggered when there is a communication error between:

othe controller and the distributed device.

othe distributed device and the expansion module.

Fallback Configuration

To configure the fallback behavior, double-click the expansion module node in the Devices tree. In the I/O Configuration tab, select the Mode:

oMaintain: The output remains in its current state.

oFallback Value: The output is set to the configured value, either 0 or 1. The default ForceValue is 0.

You must configure the fallback behavior for each output.
