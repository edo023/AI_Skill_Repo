# Analog Input Filter

## General

The module is equipped with a parameterizable input filter.

The minimum cycle time must be >400 μs. In the case of shorter cycle times, the filter function is disabled.

## Filter Stages (InputFilter)

Depending on the filter stage, the input value is valuated more or less highly. After an input ramp limit is set, the evaluation is performed.

Formula for the evaluation of the input value: valuenew = valueold - (valueold/filter stage) + (input value/filter stage)

## Adjustable Filter Stages

| Characteristic number | Filter stage |
| --- | --- |
| 0 | Filter disabled |
| 1 | Filter stage 2 |
| 2 | Filter stage 4 |
| 3 | Filter stage 8 |
| 4 | Filter stage 16 |
| 5 | Filter stage 32 |
| 6 | Filter stage 64 |
| 7 | Filter stage 128 |

## Input Ramp Limit (InputLimitationAI)

An input ramp limit can only be set in connection with a filter. In this case, the input ramp limit is set before the filter.

The difference of the input value deviation is checked for exceeding the specified limit.

If so, the updated input value equals the old value ± the limit value.

The input ramp limit serves to suppress disturbing pulses (spikes).

## Filter Stages and Limit Values

The filtering of the analog inputs is parameterized by this tab. The minimum cycle time shall be >500 μs.

If the input filter is activated, the channels are scanned at ms-intervals. The time delay between the channels is 200 μs. The conversion is asynchronous from the network cycle.
