# Functional Description

Functional Description

Enables the zero track detection.

A rising edge of this parameter (from false / 0 -> true / 1) activates the zero track detection on the incremental encoder input. Zero track detection evaluates the level of the zero track of the incremental encoder.

| Value | Data type | Meaning |
| --- | --- | --- |
| false / 0 | BOOLEAN | Zero track detection disabled. |
| true / 1 | BOOLEAN | Zero track detection enabled. |

NOTE: When a zero track occurs, the module immediately sets the internal increment counter to 0. In the next Sercos cycle, [Position](EncoderInput-14.htm#XREF_D_SE_0069589_1) and [Velocity](EncoderInput-16.htm#XREF_D_SE_0069592_1) are corrected.

This is not relevant when you are using the zero track as a reference. When you are using the zero track for a highly cyclic activity, the slight deviation in velocity can cause the position to deviate.

The Position results from the counter status.

Velocity is taken from the previous velocity since the present value is not available. As a result, the position derived from the velocity is slightly inaccurate.
