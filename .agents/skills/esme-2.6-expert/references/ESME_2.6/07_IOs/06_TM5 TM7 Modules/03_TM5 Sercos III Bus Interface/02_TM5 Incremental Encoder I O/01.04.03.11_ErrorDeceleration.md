# ErrorDeceleration

![](images/logo.png)

ErrorDeceleration

General

|  |  |
| --- | --- |
| Type | EF |
| Devices supporting the parameter | SinCos Encoder Input,  Incremental Encoder Input,  Encoder network (Synchronization encoder output),  C2C Encoder Input |
| Traceable | No |

Functional Description

A synchronization error detection on coupled machines is recognized in the following cases:

o8601 Master Encoder signal out of range (SinCos master encoder)

o8782 Master Encoder signal out of range (incremental encoder input)

o8522 C2C encoder error detected (C2C encoder input)

The parameter ErrorDeceleration can bring the system to a standstill with a defined velocity deceleration. When a track error is detected, the velocity is set to 0 with the defined deceleration.

If 0 is entered in ErrorDeceleration, then no velocity deceleration is created by the system.

Encoder Behavior

Encoder behavior when ErrorDeceleration = 0 and a track error has been detected:

oSinCos Encoder Input (physical master encoder): The velocity remains constant.

oIncremental Encoder Input: The velocity follows the received incremental encoder input signals. If the connection to the encoder is interrupted, the velocity is reduced to 0.

oSynchronization encoder output (encoder network): The last known velocity is maintained.

oC2C Encoder Input: The last known velocity is maintained.
