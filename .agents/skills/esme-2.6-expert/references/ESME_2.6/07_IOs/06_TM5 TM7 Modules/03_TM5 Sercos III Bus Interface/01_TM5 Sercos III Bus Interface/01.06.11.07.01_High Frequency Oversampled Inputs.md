# High Frequency Oversampled Inputs

## General Information

Refer to [Configuration Profile TM5SDM8DTS\_OIOO\Oversampled Inputs](D-SE-0069001.html#D-SE-0069001) as the high frequency oversampled input functionality has similar features.

The only difference is the data size which can be transferred from the module to the controller per cycle.

The configuration profile TM5SDM8DTS\_OIOO can receive eight bits per cycle. Every bit represents 125 μs. With the high frequency oversampled inputs (TM5SDM8DTS\_HFOIDO), every bit represents 31.25 μs. Therefore 4 bytes = 32 bits are transferred from the module to the controller per cycle.

## High Frequency Oversampling Inputs Buffer Handling on the Module

To enable the data transmission from the TM5SDM8DTS high frequency oversampled inputs to the controller, the OversampleEnable bit in [OversampleControl](D-SE-0069001.html#D-SE-0069001) has to be set to 1.

Every 1/32 cycle, the module latches the input level of the physical input channel to one buffer entry.

After setting the OversampleEnable bit in the OversampleControl byte, the module copies its position inside its circular buffer (OversampleInputCycle) and the four 8-bit patterns from the following 32 buffer entries to the real-time channel.

## OversampleInput0xSample1\_8...OversampleInput0xSample25\_32

The four 8-bit patterns can be read from the application via the [input channels](D-SE-0069001.html#D-SE-0069001) OversampleInput0xSample1\_8...OversampleInput0xSample25\_32.

The parameters have the following physical channel mapping on the module (refer to [General Information on TM5SDM8DTS](D-SE-0070065.html#D-SE-0070065)):

| Parameter | Channel |
| --- | --- |
| OversampleInput01Sample1\_8...OversampleInput01Sample25\_32 | 1 |
| OversampleInput02Sample1\_8...OversampleInput02Sample25\_32 | 2 |
| OversampleInput03Sample1\_8...OversampleInput03Sample25\_32 | 5 |
| OversampleInput04Sample1\_8...OversampleInput04Sample25\_32 | 6 |
