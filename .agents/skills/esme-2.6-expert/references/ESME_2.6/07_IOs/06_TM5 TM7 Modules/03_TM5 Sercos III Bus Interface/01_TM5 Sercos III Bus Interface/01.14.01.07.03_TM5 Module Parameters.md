# TM5 Module Parameters

## Parameters for Phase 2

| Parameter | Meaning | Default | Data type |
| --- | --- | --- | --- |
| PropritaryParameter-Set\_S-0-1503.0.20 | Data used for configuration of TM5 module.  NOTE: Not to be edited. For internal use only. | – | ARRAY [0...67] OF BYTE |

## Safety-Related Configuration

| Parameter | Meaning | Default | Data type |
| --- | --- | --- | --- |
| SafeLogicID | Unique number of the SLC (Safety Logic Controller) | 1 | WORD |
| SafeModuleID | Unique number of the safety-related module | 2 | WORD |

## Electronic Label

|  |  |  |  |
| --- | --- | --- | --- |
| Parameter | Meaning | Default | Data type |
| FW\_Version | The parameter  contains the firmware version of the device. The parameter corresponds to the Sercos IDN S-0-1300.x.09.  If the device is virtual, the value is an empty string. | - | String |
| HW\_Version | The parameter  contains the hardware version of the device. The parameter corresponds to the Sercos IDN S-0-1300.x.08.  If the device is virtual, the value is an empty string. | - | String |
| VendorDeviceID | The parameter  contains the device ID assigned by the vendor. The parameter corresponds to the Sercos IDN S-0-1300.x.05.  This parameter is only available if the module is used with a PacDrive controller. It is not available if the module is used with an M262 controller.  If the device is virtual, the value is an empty string. | - | String |

NOTE: The parameter values are available as of Sercos communication phase 2. The parameter values are transferred from the secondary device to the primary device through the Sercos service channel. This means that if the working mode is “simulated” or if Sercos is not in communication phase 2 or greater, the parameter values are empty strings.
