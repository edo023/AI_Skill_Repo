# DIAGNOSTICS Page

This page is available when you are connected to a network interface module.

If you are ![](images/DD00780836.png) offline, a message prompts you to go ![](images/DD00780835.png) online.

When ![](images/DD00780835.png) online, this page displays:

* On the left part of the page, a tree view of the modules in your cluster. You can see from the tree list the status of each module in your cluster:

  ![](images/DD00770464.png) OK

  ![](images/DD00770465.png) advisory

  ![](images/DD00770466.png) error
* In the middle of the page, the status of the selected module in the tree view is displayed with the following information:
  + Diagnostic ID: Displays the diagnostic identification number. Click the diagnostic ID to be redirected to the [Modicon Edge I/O - Diagnostic Data - User Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=EdgeIO_Diag_UG&topicID=), where you can access details and troubleshooting if needed.
  + Function: Displays the function name MainFunction + SubFunction for the diagnostic ID.
  + Current status: Displays the status (Error and Advisory).
  + Timestamp: When the NTP time is available, displays the timestamp in DD/MM/YYYY HH:MM:SS.ms format (with µs precision). If NTP time is not available, the system uses the build time of the firmware as a fall-back option for time-stamping.
  + Message: Displays the message associated with the diagnostic.

The Get Details button provides access to the complete list of diagnostics for the selected module.

For more details, click the ID of the diagnostic to be redirected to the [Modicon Edge I/O - Diagnostic Data - User Guide](../../../../../api/crossBook?lang=en-US&virtualBookName=EdgeIO_Diag_UG&topicID=).
