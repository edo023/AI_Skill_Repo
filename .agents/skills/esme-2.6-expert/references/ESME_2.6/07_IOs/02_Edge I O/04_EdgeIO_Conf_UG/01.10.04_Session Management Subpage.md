# Session Management Subpage

The Session Management subpage allows you to manage the security session parameters:

| Parameter | Description |
| --- | --- |
| Minimum Inactivity Period (min) | The time before the session is locked due to inactivity. The time (in min) must be between 5 and 30 minutes (15 per default).  NOTE: Modification of this parameter only takes effect upon next login. |
| Password Attempts Timer (min) | The time after which the incorrect login attempts counter resets. The time (in min) must be between 1 and 5 minutes (3 per default). |
| Maximum Login Attempts | Defines a maximum number of login attempts permitted during the Password Attempts Timer (min) time; other attempts are rejected until the timer is finished. The number of attempts must be between 3 and 8 (5 per default). |

NOTE:

* Only users with the SECADM role can modify these parameters. Other users have a read-only access.
* If you are managing the security settings through the CAE software, this subpage is in read-only mode. Users with the Security Management permission can modify these parameters in the CAE software.
