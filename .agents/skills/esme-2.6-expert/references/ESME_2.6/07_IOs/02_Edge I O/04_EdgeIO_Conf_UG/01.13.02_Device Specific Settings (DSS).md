# Device Specific Settings (DSS)

This table describes the MODELS > Specific Settings parameters for EtherNet/IP and Modbus TCP Network Interface Modules:

| Key | Type | Default Value | Description |
| --- | --- | --- | --- |
| DPWS Enabled | BOOL | TRUE | Allows you to enable or disable the protocol DPWS. |
| EIP Enabled | BOOL | FALSE | Allows you to enable or disable EtherNet/IP for each communication port. |
| Modbus TCP Server Enabled | BOOL | FALSE | Allows you to enable or disable the protocol for each communication port. |
| NTP Client Enabled | BOOL | FALSE | Allows you to enable or disable the protocol for each communication port. |
| RSTP Enabled | BOOL | TRUE | Allows you to enable or disable the protocol for each communication port. |
| FDR Enabled | BOOL | TRUE | Allows you to enable or disable this protocol and the TFTP protocol for each communication port.  NOTE: Requires configuration of the network interface module in standard mode. |
| Explicit Data Exchange Static Configuration Read Enabled | BOOL | FALSE | Allow you to enable or disable the messaging from Modbus TCP to OPC UA for Read Static Configuration. |
| Explicit Data Exchange Dynamic Configuration R/W Enabled | BOOL | FALSE | Allow you to enable or disable the messaging from Modbus TCP to OPC UA for Read and Write Dynamic Configuration. |
| Explicit Data Exchange Diagnostic Read Enabled | BOOL | FALSE | Allow you to enable or disable the messaging from Modbus TCP to OPC UA for Read Diagnostics and Expert Diagnostics. |
| OPCUA SecurityPolicy | INTEGER | 3 | None = 0, Basic256Sha256 = 3   * None: No encryption and no signature. * Basic256Sha256: Encryption and signature possible. |
| OPCUA SecurityMode | INTEGER | 2 | None = 0, Sign = 1, Sign and Encrypt = 2  This parameter depends on the OPCUA SecurityPolicy:   * If OPCUA SecurityPolicy is None, OPCUA SecurityMode is None. * If OPCUA SecurityPolicy is basic256sha256, OPCUA SecurityMode is Sign or Sign and Encrypt. |
| OPCUA AuthenticationMode | INTEGER | 1 | Anonymous = 0, Username = 1   * Anonymous: No username required to connect to the server. * Username: The username configured on the device is required. |
