# Network Interface Subpage

The Network Interface subpage allows you to modify the network interface module settings by clicking the Edit Settings button.

NOTE:

* Only users with the SECADM role can modify these parameters.
* If you are managing the security settings through the CAE software, this subpage is in read-only mode. Users with the Security Management permission can modify these parameters in the CAE software.

The available settings depend on the network interface module and/or the mode you are using.

This table lists the existing settings:

| Setting | Default Value | Description |
| --- | --- | --- |
| DPWS | Enabled | Allows you to enable or disable the protocol DPWS. |
| EtherNet/IP | Disabled | Allows you to enable or disable EtherNet/IP for each communication port. |
| Explicit Data Exchange Diagnostic Read | Disabled | Allows you to enable or disable the messaging from Modbus TCP to OPC UA for Read Diagnostics and Expert Diagnostics. |
| Explicit Data Exchange Dynamic Configuration R/W | Disabled | Allows you to enable or disable the messaging from Modbus TCP to OPC UA for Read and Write Dynamic Configuration. |
| Explicit Data Exchange Static Configuration Read | Disabled | Allows you to enable or disable the messaging from Modbus TCP to OPC UA for Read Static Configuration. |
| FDR | Enabled | Allows you to enable or disable this protocol and the TFTP protocol for each communication port. |
| Modbus TCP Server | Disabled | Allows you to enable or disable the protocol for each communication port. |
| NTP Client | Disabled | Allows you to enable or disable the protocol for each communication port. |
| OPCUA AuthenticationMode | Username | Anonymous: No username required to connect to the server.  Username: The username configured on the device is required. |
| OPCUA SecurityPolicy | Basic256Sha256 | None: No encryption and no signature.  Basic256Sha256: Encryption and signature possible. |
| OPCUA SecurityMode | Sign and encrypt | This parameter depends on the OPCUA SecurityPolicy:   * If OPCUA SecurityPolicy is None, OPCUA SecurityMode is None. * If OPCUA SecurityPolicy is basic256sha256, OPCUA SecurityMode is Sign, or Sign and Encrypt. |
| RSTP | Enabled | Allows you to enable or disable the protocol. |
