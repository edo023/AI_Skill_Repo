# Reflex Output Sub-Function

## Description

The Frequency Meter, Period Meter, Ratio Meter, Single Phase Counting, Single Event Counting and Dual Phase Counting functions can manage up to 4 reflex outputs that can be triggered during the operation of the module.

A reflex output is enabled when assigning a physical output to the Reflex 0: Location...Reflex 3: Location parameters.

There are two ways to operate the configured reflex outputs:

* With the [compare function](CompareFunction-284AA1DA.html). To enable the compare function, define a trigger condition for the Reflex 0: Condition...Reflex 3: Condition parameter:
  + Reflex 0 Condition: Counter Within Threshold...Reflex 3 Condition: Counter Within Threshold
  + Reflex 0 Condition: Capture Within Threshold...Reflex 3 Condition: Capture Within Threshold
  + Reflex 0 Condition: Counter Cross Threshold...Reflex 3 Condition: Counter Cross Threshold
* With the ForceReflexOutput command.

When using the compare function, the outputs are independent of the I/O bus cycle and have an update rate depending on the comparison trigger but less than 20 μs.

There are 4 configurable thresholds available that can define up to five counting zones.

## Implicit Data

The following table presents the input implicit data of the Reflex Output sub-function:

| Name  *Parameter Name* | Value | Data Type  Size in Bytes  R/W | Description |
| --- | --- | --- | --- |
| ReflexState | 0...255 | BYTE  1  R/- | For each physical output configured in the Reflex 0: Location...Reflex 3: Location parameters, each ReflexState bit returns the result of the comparison between the counter value and the configured Reflex 0: Condition...Reflex 3: Condition parameters (bit field).  Bit 0...3 = TRUE when the comparison result for Reflex 0: Location...Reflex 3: Location is TRUE.  NOTE: ReflexState bits are valid for Reflex 0: Condition: Counter Within Threshold...Reflex 3: Condition: Counter Within Threshold or Reflex 0: Condition: Capture Within Threshold...Reflex 3: Condition: Capture Within Threshold. Other bits are reserved. |
| ThresholdState | 0...255 | BYTE  1  R/- | Each ThresholdState bit returns the result of the comparison between the counter value and the threshold parameters (bit field).  Bit 0...3 = TRUE when the counter value is greater than Threshold 0 Value...Threshold 3 Value.  NOTE: Other bits are reserved. |
| OutputState | 0...255 | BYTE  1  R/- | Each bit sets the voltage of the configured physical output Reflex 0: Location...Reflex 3: Location to 24 Vdc when the EnableReflexOutput bit 0...3 is TRUE and the ReflexState bit 0...3 is TRUE.  Bit 0...3 = TRUE when the voltage of the configured physical output Reflex 0: Location...Reflex 3: Location is 24 Vdc.  NOTE: Other bits are reserved. |

NOTE: The implicit data is updated at each I/O bus cycle.

The following table presents the output implicit data of the Reflex Output sub-function:

| Name  *Parameter Name* | Value | Data Type  Size in Bytes  R/W | Description |
| --- | --- | --- | --- |
| EnableReflexOutput | 0...255 | BYTE  1  R/W | Each bit allows OutputState 0...3 to operate the physical output configured in Reflex 0: Location...Reflex 3: Location.  Bit 0...3 = physical output configured in Reflex 0: Location...Reflex 3: Location.  NOTE: Bits used inside EnableReflexOutput and bits used inside ForceReflexOutput operate the same physical output and must be mutually exclusive. Other bits are reserved. |
| ForceReflexOutput | 0...255 | BYTE  1  R/W | Each bit allows to force the voltage of the configured physical output Reflex 0: Location...Reflex 3: Location to 24 Vdc.   * Set to TRUE to force the voltage of the configured physical output Reflex 0: Location...Reflex 3: Location to 24 Vdc. * Set to FALSE to unforce the configured physical output Reflex 0: Location...Reflex 3: Location.   NOTE: Bits used inside EnableReflexOutput and bits used inside ForceReflexOutput operate the same physical output and must be mutually exclusive. Other bits are reserved. |
