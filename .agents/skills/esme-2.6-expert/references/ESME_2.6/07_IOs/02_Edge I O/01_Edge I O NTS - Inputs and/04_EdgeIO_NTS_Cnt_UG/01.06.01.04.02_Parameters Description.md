# Parameters Description

## Configurable Parameters

The following table presents the configurable parameters for the NTSEHC0120H module:

| Displayed Name  *Parameter Name* | Value | Data Type | Description |
| --- | --- | --- | --- |
| Device Mode  DeviceMode | 0: Normal\*  1: Optional  2: Virtual reserved | ENUM | Allows you to select the device mode:  * Normal: The module is part of the software configuration and is physically connected in the cluster. * Optional: The module is part of the software configuration. A dummy module or the configured module must be physically installed in the cluster. Whether either module is present does not cause a configuration error to be detected. * Virtual reserved: The module is part of the software configuration. A dummy module must be physically installed in the cluster. If the virtual module is physically installed in the cluster, a configuration error is detected. |
| Rearming Output Mode  RearmOutputMode | 0: Latched Off  1: Auto Recovery\* | ENUM | Allows you to select the rearming mode for an output channel which are latched off due to a detected error.  Two modes are available:  * Latched Off: The channel is rearmed if the cause of the detected error is no longer present, and a rising edge on RearmOutputCmd is applied. * Auto Recovery: the output channel is rearmed automatically if the cause of the detected error is no longer present for a predefined delay. |
| \* Parameter default value | | | |

The following table presents the input configurable parameters for the channels of the NTSEHC0120H module:

| Displayed Name  *Parameter Name* | Value | Data Type | Description |
| --- | --- | --- | --- |
| Latch  Latch | 0: No\*  1: Rising Edge - Automatic Acknowledge  2: Falling Edge - Automatic Acknowledge  3: Both Edges - Automatic Acknowledge  4: Rising Edge - Manual Acknowledge  5: Falling Edge - Manual Acknowledge  6: Both Edges - Manual Acknowledge | ENUM | Allows incoming pulses with a pulse width shorter than the network interface module scan time to be captured and recorded. For more information, refer to [Input Latch](Latch-CBCD55ED.html). |
| Filter  InputFilter | 0: 0  1: 0.0005  2: 0.001  3: 0.002  4: 0.005  5: 0.01  6: 0.05  7: 0.1  8: 0.25  9: 0.5  10: 1  11: 2  12: 4\*  13: 12 | ENUM | Allows reducing the effect of bounce on the input. Changes to the signal are only detected if the pulse width of the input signal is longer than the filter time (ms). |
| \* Parameter default value | | | |

The following table presents the output configurable parameters for the channels of the NTSEHC0120H module:

| Displayed Name  *Parameter Name* | Value | Data Type | Description |
| --- | --- | --- | --- |
| Fallback Mode  OutputFallbackMode | 0: Fallback Value\*  1: Maintain | ENUM | Allows you to select the behavior for the output in case of a communication interruption:  * Fallback Value: Sets the output at the configured Predefined Fallback Value value. * Maintain: The output remains in its state. |
| Predefined Fallback Value  PredefinedFallbackValue | 0: 0\*  1: 1 | ENUM | Determines the state for the output in case of a communication interruption and Fallback Mode parameter is set to Fallback Value. |
| \* Parameter default value | | | |

## Implicit Data

The following table presents the input implicit data for the NTSEHC0120H module:

| *Parameter Name* | Value | Data Type  Size in Bytes  R/W | Description |
| --- | --- | --- | --- |
| GCS | 0...255 | BYTE  R/- | Group Cyclic Status  Bit 0: Data quality  Bit 1: General module status  Bit 2: I/O status  Bit 3: Receive status  Bit 4: Output status  Bit 5: Advisory status  Bit 6: N/A  Bit 7: Data freshness  NOTE: For more information, refer to [Group Cyclic Status (GCS)](../../../../../api/crossBook?lang=en-US&virtualBookName=EdgeIO_Diag_UG&topicID=GroupCyclicStatusDefinition_F906694D). |
| ChannelHealthIn0\_7(1) | 0...255 | BYTE  1  R/- | Status of input channel 0...7 (Bit field)  Bit 0...7 = Value of input channel 0...7   * Bit = FALSE: Channel is invalid or not present. * Bit = TRUE: Channel is valid or disabled. |
| IValue0\_7 | 0...255 | BYTE  1  R/- | Value of the input channels (Bit field)  Bit 0...7 = Value of input channel 0...7  NOTE: Unused bits are set to 0. |
| ChannelHealthOut0\_7 | 0...255 | BYTE  1  R/- | Status of output channel 0...7 (Bit field)  Bit 0...7 = Value of output channel 0...7   * Bit = FALSE: Channel is invalid or not present. * Bit = TRUE: Channel is valid or disabled. |
| ModuleState | 0...255 | BYTE  1  R/- | Status of module detected error (Bit field)  Bit 0: HSC Output fallback status  Bit 1: HSC Short circuit group 0 status  Bit 2: N/A  Bit 3: HSC Internal field power supply status  NOTE: Bit 4...7 are reserved.   * Bit = FALSE: No error detected * Bit = TRUE: Error detected |
| (1) This parameter is not part of the implicit data if the optimized I/O profile is selected. | | | |

The following table presents the output implicit data for the NTSEHC0120H module:

| *Parameter Name* | Value | Data Type  Size in Bytes  R/W | Description |
| --- | --- | --- | --- |
| LatchAck0\_7 | 0...255 | BYTE  1  R/W | When Latch is set to Manual Acknowledge, resets the latch value of the input on the channel 0...7.  Bit 0...7 = Value of channel 0...7 |
| QValue0\_7 | 0...255 | BYTE  1  R/W | Value of the output channels (Bit field).  Bit 0...7 = Value of output channel 0...7  NOTE: Unused bits are reserved. |
