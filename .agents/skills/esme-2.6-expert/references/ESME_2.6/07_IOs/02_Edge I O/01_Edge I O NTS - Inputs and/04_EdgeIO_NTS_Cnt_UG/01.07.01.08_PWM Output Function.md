# PWM Output Function

## Description

The PWM Output produces a periodic signal at the PWM Ouput Location output while the enable condition is TRUE.

The frequency and the duty cycle are defined by the implicit data *PWMOutputFrequency and PWMOutputDuty*.

NOTE: If the PWMOutputFrequency or PWMOutputDuty are modified while the PWM Output is active, the PWM Output cycle finishes before applying the modified parameters.

With the PWM Output function:

* The [enable condition](EnableFunction-284A99B2.html) is controlled with a software command or a physical input.
* The [preset condition](PresetSubFunction-FA7D6B9D.html) is controlled with a software command or a physical input.

The following table describes the features of the PWM Output function:

| Item | Description |
| --- | --- |
| Inputs | The PWM Output function requires a single fast output PWM Location to operate.  An optional input can be assigned to EN Location and SYNC Location. |
| Output Frequency | 0.01 Hz...20 kHz |
| Duty cycle | 0.1...99.9%, with limits depending on the output frequency. |

## Implicit Data

The following table presents the input implicit data of the PWM Output function:

| Name  *Parameter Name* | Value | Data Type  Size in Bytes  R/W | Description |
| --- | --- | --- | --- |
| OperationalState | 0...255 | BYTE  1  R/- | Operational state of the PWM Output function.  Bit 0 (Run), this bit indicates if the counter is active.   * TRUE when the enable condition is TRUE. * FALSE when the enable condition is FALSE.   Bit 1 (Valid), this bit indicates if the PWM Output parameters are valid.   * TRUE when the frequency and duty cycle are within the valid ranges. * FALSE, when:    + Bit 0 (Run) is FALSE.   + The user input PWM Output frequency or duty cycle is not within the valid range. The PWM Output function remains active and uses the last valid value.   Bit 3 (Preset Flag), this bit indicates if the preset condition bit was set to TRUE.   * TRUE on the rising edge of the preset condition. * FALSE on the rising edge of the Acknowledge Preset bit.  NOTE: After a configuration download, it may be necessary to acknowledge this Preset Flag.   NOTE: Other bits are reserved. |

The following table presents the output implicit data of the PWM Output function:

| Name  *Parameter Name* | Value | Data Type  Size in Bytes | Description |
| --- | --- | --- | --- |
| OperationalCommand | 0...65,535 | INT  2  R/W | Bit 0 (Enable): When TRUE, the EN Location physical input can set the enable condition to TRUE.  Bit 1 (Enable Preset): When TRUE, the SYNC Location physical input can set the preset condition to TRUE.  Bit 7 (Force Enable): When TRUE, the enable condition is TRUE.  Bit 8 (Force Preset): On a rising edge, the preset condition is TRUE.  Bit 11 (Acknowledge Preset): On a rising edge, sets the Preset Flag to FALSE.  NOTE: Other bits are reserved. |
| PWMOutputFrequency | 0...2,000,000 | INT32  4  R/W | Sets the pulse frequency for PWM output in percentiles.  For example, PWMOutputFrequency = 174,654 is a PWM output pulse frequency of 1,746.54 Hz.  0 is not a valid value. |
| PWMOutputDuty | 0...1,000 | INT  2  R/W | Sets the pulse duty cycle for PWM output with the following limits:   * 0.01 Hz ≤ PWMOutputFrequency ≤ 200 Hz, duty cycle: 0.1 %...99.9 %. * 200 Hz ≤ PWMOutputFrequency ≤ 1 kHz, duty cycle: 1 %...99 %. * 1 kHz < PWMOutputFrequency ≤ 10 kHz, duty cycle: 5 %...95 %. * 10 kHz < PWMOutputFrequency ≤ 20 kHz, duty cycle: 10 %...90 %.   For example, PWMOutputDuty = 123 is a pulse duty cycle of 12.3 %.  Duty step: 0.1 %  0 and 1,000 are not valid values. |
