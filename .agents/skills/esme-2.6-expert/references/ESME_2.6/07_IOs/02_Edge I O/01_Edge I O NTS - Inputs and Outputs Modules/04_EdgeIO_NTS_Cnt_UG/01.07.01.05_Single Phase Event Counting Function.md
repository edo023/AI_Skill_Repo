# Single Phase Event Counting Function

## Description

The Single Phase Event Counting function counts the number pulses that occur during a given period of time (Time base) on input A.

With the Single Phase Event Counting function:

* The [enable condition](EnableFunction-284A99B2.html) is controlled with a software command.
* The [preset condition](PresetSubFunction-FA7D6B9D.html) is controlled with a software command or a physical input.

The following diagram and table describes the Single Phase Event Counting principle:

![](images/DD00864519.png)

NOTE: The pulse is counted on the rising edge of the input A.

| Stage | Action |
| --- | --- |
| 1 | While the enable condition is TRUE, the counter accumulates the number of pulses on input A during the Time base duration. |
| 2 | At the end of Time base period, the counter value is set to the number of pulses on input A and the OperationalState bit 1 (Valid) is set to TRUE.  The counting restarts for a new period of time.  On the rising edge of the preset condition:   * The accumulated value of pulses on input A is reset to 0. * The counter value is not updated. * The counting restarts for a new period of time.   NOTE: When the enable condition is FALSE, you can reset the internal timer relative to the time base on the rising edge of the preset condition. |
| 3 | The counting restarts for a new period of time. |
| 4 | In this example, a reflex output is configured with the Counter Within Threshold condition:   * The Reflex Condition: Counter Within Threshold value is 2. * Threshold 0 value is 4. * Threshold 1 value is the maximum value.   At the end of the Time base period, the reflex output is set to TRUE because the counter value is between threshold 0 and the maximum value. For more information about reflex outputs, refer to [Reflex Output Sub-Function](TPC_EDGEIOCountingReflexOutputSubFu-E349278D.html). |
| 5 | The counting restarts for a new period of time. |

The following table describes the features of the Single Phase Event Counting function:

| Item | Description |
| --- | --- |
| Inputs | The Single Phase Event Counting function requires a single fast input A Location to operate.  An optional input can be assigned to SYNC Location. |
| Counter register | 32 bits |
| Reflex Output | Up to 4 reflex outputs can be configured. For more information about reflex outputs, refer to [Reflex Output Sub-Function](TPC_EDGEIOCountingReflexOutputSubFu-E349278D.html). |
| Counter Value Update Rate | At the end of each Time base period. |
| Reflex Output State Update Rate | At the end of each Time base period. |

## Implicit Data

The following table presents the input implicit data of the Single Phase Event Counting function:

| Name  *Parameter Name* | Value | Data Type  Size in Bytes  R/W | Description |
| --- | --- | --- | --- |
| CounterValue | 0...4,294,967,295 | INT32  4  R/- | Counter value.  NOTE: The implicit data CounterValue is updated at each I/O Bus cycle. |
| OperationalState | 0...255 | BYTE  1  R/- | Operational state of the single phase event counting function.  Bit 0 (Run), this bit indicates if the counter is active.   * TRUE when the enable condition is TRUE. * FALSE when the enable condition is FALSE.   Bit 1 (Valid), this bit indicates if the measurement is valid.   * TRUE when the measurement is within the CounterValue range. * FALSE, when:    + Bit 0 (Run) is FALSE.   + The measurement is not within the CounterValue range.   Bit 3 (Preset Flag), this bit indicates if the preset condition bit was set to TRUE.   * TRUE on the rising edge of the preset condition. * FALSE on the rising edge of the Acknowledge Preset Flag bit.   NOTE: Other bits are reserved. |

The following table presents the output implicit data of the Single Phase Event Counting function:

| Name  *Parameter Name* | Value | Data Type  Size in Bytes  R/W | Description |
| --- | --- | --- | --- |
| OperationalCommand | 0...65,535 | INT  2  R/W | Bit 1(Enable Preset): When TRUE, the SYNC Location physical input can set the preset condition to TRUE.  Bit 5 (Enable Compare): When TRUE, the compare function is active.  Bit 6 (Suspend Compare): When TRUE, the compare function is suspended.  Bit 7 (Force Enable): When TRUE, the enable condition is TRUE.  Bit 8 (Force Preset): On a rising edge, sets the preset condition to TRUE.  Bit 11 (Acknowledge Preset Flag): On a rising edge, sets the Preset Flag to FALSE.  NOTE: Other bits are reserved. |
