# Functional Description

Functional Description

Displays the version name of the bootloader of the device. The value is available as of Sercos phase 2. The bootloader version consists of the following parts: V<Generation>.<Family>.<Revision>.<Build>.

The version number part from generation and family of the bootloader must be less or equal to the corresponding version number part of the firmware.

If the device in the Devices tree is virtual (non-real), then the parameter value is empty ('').

NOTE: The parameter is currently not supported by the listed devices (empty string "). The parameter is available to establish a uniform electronic type plate for the devices.

NOTE: The parameter value is transferred from the slave to the master via Sercos. This means that if Sercos is not in phase 4 (operating phase), an empty string '' is indicated here. If Sercos is in phase 4, then the parameter value is transferred over Sercos.
