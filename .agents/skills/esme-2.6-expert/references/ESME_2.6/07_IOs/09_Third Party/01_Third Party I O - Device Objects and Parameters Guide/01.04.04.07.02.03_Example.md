Example: DiagCodes in the Sercos IDN 390 'Diagnosis Number' for TM5CSLCx00FS or TM5NS31

Safe Logic Controller TM5CSLCx00FS:

o0xC10F1000 "S3/BC(ERROR): General error": General error

o0x010F1801 "S3/BC(ERROR): General error (manufacturer-specific)": Invalid bus controller firmware file

o0x010F1802 "S3/BC(ERROR): General error (manufacturer-specific)": Invalid bus controller configuration data

o0x010F1803 "S3/BC(ERROR): General error (manufacturer-specific)": Invalid IOM configuration data -> Slot

o0x010F1804 "S3/BC(ERROR): General error (manufacturer-specific)": Flash read failed

o0x010F1805 "S3/BC(ERROR): General error (manufacturer-specific)": Out of bus controller resources

o0x010F1806 "S3/BC(ERROR): General error (manufacturer-specific)": Unrecoverable error

o0xC10F6320 "S3/BC(ERROR) :S3/BC(ERROR): Invalid Reg CFG Data -> Slot

Bus Interface TM5NS31: I/O status codes:

o0xC10F8100 "S3/BC(ERROR): Monitoring.Communications": Sercos communication error

o0xC10FB000 "S3/BC(ERROR): Local bus (System bus of the modular station)": General IOM module error -> Slot

o0xC10EB001 "S3/BC(WARNING): Local bus slave missing": Configured IOM missing on local bus -> Slot

o0xC10FB002 "S3/BC(ERROR): Incorrect local bus slave present": IOM mismatch -> Slot

o0xC10FB004 "S3/BC(ERROR): Local bus devices amount error": IOM unsupported -> Slot

o0xC10FB010 "S3/BC(ERROR): Device errors": IOM Failed -> Slot

NOTE: The parameter value is transferred from the slave to the master via the parameter channel of the Sercos by every access. Typically, this takes about 10 ms. By a high capacity of the parameter channel, times up to 1 s can occur. If the Sercos bus is in phase 0 or 1, then a standard value is indicated here. If the Sercos bus is in phase 3 or 4, then the parameter value is transferred and indicated. In the Sercos phase 2, the parameter can be read through the application.
