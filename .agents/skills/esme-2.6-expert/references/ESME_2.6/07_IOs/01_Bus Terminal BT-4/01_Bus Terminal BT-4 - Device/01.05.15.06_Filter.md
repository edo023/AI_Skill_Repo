# Filter

![](images/logo.png)

Filter

General

|  |  |
| --- | --- |
| Type | EF |
| Devices supporting the parameter | SinCos Encoder Input,  Incremental Encoder Input |
| Traceable | Yes |

Functional Description

Displays the filter value in ms.

The input value is rounded down internally to the next power of two. The reasonable maximum value depends on the parameter [CycleTime](../../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Parameter.LMCEco&topicID=D_SE_0073362_1).

CycleTime 1 ms -> maximum value 128

CycleTime 2 ms -> maximum value 256

CycleTime 4 ms -> maximum value 512

Example

Filter = 40 ms, CycleTime = 2 ms

-> internal time constant of the filter = 32 ms

-> maximum value 256 was not reached

NOTE: The filter is a low-pass filter and affects the actual velocity. If the filter is set too high, it can lead to oscillation of the system depending on the velocity path. If you modify the [FeedConstant](../IncrementalEncoderOutput/IncrementalEncoderOutput-7.htm#XREF_D_SE_0076271_1), [Direction](../IncrementalEncoderOutput/IncrementalEncoderOutput-9.htm#XREF_D_SE_0076273_1), and Filter parameters when the master encoder is running, it leads to a continuous loss of position.
