# Functional Description

Functional Description

Enables the zero track detection.

A rising edge of this parameter (switch-over from false / 0 -> true / 1) activates the zero track detection on the incremental encoder input. Zero Track detection evaluates the level of the incremental encoder's zero track.

| Value | Data type | Meaning |
| --- | --- | --- |
| false / 0 | BOOL | Zero track detection disabled. |
| true / 1 | BOOL | Zero track detection enabled. |

NOTE: When a zero track occurs, the HW immediately sets the internal increment counter to 0. In the next Sercos cycle, [Position](../IncrementalEncoderOutput/IncrementalEncoderOutput-10.htm#XREF_D_SE_0076274_1) and [Velocity](../MasterEnc_SinCos/MasterEnc_SinCos-12.htm#XREF_D_SE_0076246_1) are corrected.

This can be ignored when you are using the zero track as a reference. When you are using the zero track for a highly cyclic activity, the slight deviation in velocity can cause the position to deviate.

The position results from the current counter status.

Velocity is taken from the previous velocity since the current value is not available. As a result, the position derived from the velocity is slightly inaccurate.
