# Error: Formal parameter '...':'...' is already assigned.

In an ST code worksheet, a formal parameter is used in more than one assignment statement.

The error message specifies the affected ST statement (instance call) and the formal parameter: '`statement`':'`parameter`'.

Error example:

`TON_S_1(IN := SAFETRUE, PT := SAFETIME#3s);`

`TON_S_1.IN := SAFETRUE; (* second assignment of the IN formal parameter *)`

`TON_Out := TON_S_1.Q;`

Remedy:

* Delete the second (superfluous) assignment statement and recompile the worksheet. This way, all formal parameter assigments are within the instance call.

  Example:

  `TON_S_1(IN := SAFETRUE, PT := SAFETIME#3s);`

  `TON_Out := TON_S_1.Q;`

  Or
* First, assign the formal parameters (before the instance call), and then call the instance without assignment statements.

  `TON_S_1.IN := SAFETRUE;`

  `TON_S_1.PT := SAFETIME#3s;`

  `TON_S_1();`

  `TON_Out := TON_S_1.Q;`
