# Error (ST parser): '...' expected

The ST parser detected a syntactical error in an ST code construct. The code element mentioned in the error message is missing and the code position where the element is expected is underlined red in the code editor.

The following code elements can be demanded in the error message:

| Error message | Meaning / Remedy |
| --- | --- |
| ';' expected. | Insert a semicolon completing a statement line at the end of the marked line. |
| '(' or ':=' expected | An opening parentheses operator '(', used to specify the priority of operands within a statement is missing, or an assignment statement ':=' in the underlined expression. |
| ')' expected. | A closing parentheses operator ')', used to specify the priority of operands within a statement is missing in the underlined expression. |
| ':=' expected. | An assignment operator ':=' is missing in the underlined code line.  An assignment is the simplest ST statement type. It copies the value of the expression on the right to the variable on the left. Both the variable on the left and the value of the expression on the right must have the same data type. |
| ',' expected. | A comma, separating the passed parameters within a function/FB call is missing. |

|  |  |
| --- | --- |
| 'END\_FOR' expected. | A 'FOR' iteration statement must be closed by an 'END\_FOR' element.  Correct statement syntax:  FOR a:=1 TO 10 BY 3 DO  incr:= start + a;  **END\_FOR;** |
| ']' expected. | A closing bracket is missing in the underlined FOR iteration statement. The control variable must be enclosed by brackets:  FOR a:=1 TO 10 BY 3 DO  f[a**]** :=b;  END\_FOR; |
| 'TO' expected.  'DO' expected. | The 'TO' or 'DO' keyword to complete a FOR iteration statement is missing.  Correct statement syntax:  FOR a:=1 **TO** 10 BY 3 **DO**  incr:= start + a;  END\_FOR; |

|  |  |
| --- | --- |
| 'THEN' expected. | A selection statement (IF or ELSEIF) is not complete without the 'THEN' element.  Correct statement syntax:  **IF** a < b **THEN** c:=1;  **ELSIF** a=b **THEN** c:=2;  **ELSE** c:=3;  **END\_IF;** |
| 'END\_IF' expected. | An 'IF' selection statement must be closed by an 'END\_IF' element. Correct statement syntax see line above. |
| 'END\_CASE' expected. | A 'CASE' selection statement must be closed by an 'END\_CASE' element.  Correct statement syntax:  CASE f OF  1: a:=3;  2..5: a:=4;  6: a:=2;  b:=1;  ELSE a:=0;  **END\_CASE;** |
| 'OF' expected. | The 'OF' keyword is missing to complete a CASE selection statement. This keyword introduces the list of matches to be evaluated. |

**NOTE:**

The Edit Wizard prevents entering syntactical errors. Using the Edit Wizard, syntax templates prepared with place holders can be inserted.

**Further Information:**

Refer to the topic ["ST Code Elements"](elementsintheSTeditor.html#elementsintheSTeditor) for information on the ST syntax.
