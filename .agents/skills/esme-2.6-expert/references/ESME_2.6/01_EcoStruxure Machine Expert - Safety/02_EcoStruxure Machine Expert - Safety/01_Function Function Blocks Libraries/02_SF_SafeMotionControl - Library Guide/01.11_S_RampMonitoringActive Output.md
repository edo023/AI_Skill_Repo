# S\_RampMonitoringActive Output

Output for signaling the active ramp monitoring for the requested safety-related monitoring function(s).

Ramp monitoring can be parameterized for the following safety-related monitoring functions:

* [SS1 - Safe Stop 1 function](SS1.html#SS1)
* [SS2 - Safe Stop 2 function](SS2.html#SS2)
* [SLS1 to SLS4 - four separate Safely Limited Speed functions](SLS.html#SLS)

Data type: SAFEBOOL

**Connection:** If required, the signal can be connected to a variable of the SAFEBOOL data type or to other safety-related functions/function blocks for further processing or evaluation.

| Value | Description |
| --- | --- |
| SAFETRUE | * Ramp monitoring is active for at least one requested safety-related monitoring function, and * the function block is activated.  If ramp monitoring is active, the deceleration curve is monitored during the t2 interval and must follow the parameterized ramp. Otherwise, the fallback function defined for the corresponding safety-related function is activated. |
| SAFEFALSE | * Ramp monitoring is inactive for the requested safety-related monitoring function(s), or * the function block is not activated.  If ramp monitoring is not active, the deceleration curve is not monitored during the t2 interval. Even acceleration is allowed in the ramp-down phase. The safety-related function must be executed correctly the latest before t2 elapses. Otherwise, the fallback function defined for the corresponding safety-related function is activated. |
