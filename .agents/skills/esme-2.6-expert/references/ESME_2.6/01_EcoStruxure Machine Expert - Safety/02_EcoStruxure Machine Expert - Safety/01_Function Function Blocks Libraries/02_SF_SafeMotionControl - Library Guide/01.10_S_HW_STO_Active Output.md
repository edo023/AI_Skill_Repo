# S\_HW\_STO\_Active Output

Output for signaling that the **STO** safety-related function has been requested via the hard-wired signal link.

Data type: SAFEBOOL

**NOTE:**

This output is only relevant if the `HW_STO` device parameter of the safety logic is set to `Activated`. Only with this setting, the use of the hard-wired STO request is possible (see section "[STO hard-wired](function_MotionFB.html#function_MotionFB__STOhardwired)").

**Connection:** If required, the signal can be connected to a variable of the SAFEBOOL data type or to other safety-related functions/function blocks for further processing or evaluation.

| Value | Description |
| --- | --- |
| SAFETRUE | This status has the following reason:  * STO function is requested via the hard-wired direct signal link of the safety logic, and * the function block is activated. |
| SAFEFALSE | This status is due to one of the following reasons:  * STO function is requested by software via the S\_STO\_Request input parameter of the function block. * STO function is not requested. * The function block is not activated. |

| WARNING | |
| --- | --- |
|  | **NON-CONFORMANCE TO SAFETY FUNCTION REQUIREMENTS**   * Validate the use of the S\_STO\_Request input with respect to the setting of the `HW_STO` device parameter. * Validate your safety-related architecture and thoroughly test the application.   **Failure to follow these instructions can result in death, serious injury, or equipment damage.** |
