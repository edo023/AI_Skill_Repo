# ResetRequest output

Output for signaling "reset is required".

This output indicates whether a reset by the operator is required.

Data type: BOOL

Initial value: FALSE

**Connection:** Connect the ResetRequest output to an output terminal of your application (e.g., for an indicating device to signal "reset is required"), either directly or via other standard functions/function blocks.

| Value | Description |
| --- | --- |
| TRUE | A reset is required:   * to remove an active restart inhibit * **or**  to reset an error.  This is the case if the function block is in one of the following diagnostic states.   * 8422 * C420 * C440 * C450  Refer to the topic ["Diagnostic codes"](codes_EnableSwitch_SE.html#codes_EnableSwitch_SE) for details about these states as well as the FB/environmental conditions. |
| FALSE | No reset is required. |
