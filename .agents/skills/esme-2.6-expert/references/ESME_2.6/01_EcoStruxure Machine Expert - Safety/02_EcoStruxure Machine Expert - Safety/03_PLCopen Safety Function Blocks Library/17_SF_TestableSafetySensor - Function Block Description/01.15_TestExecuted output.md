# TestExecuted output

Output for signaling the status of the sensor test.

Data type: BOOL

A TRUE signal at the TestExecuted output results that the sensor test has been completed without errors.

**NOTE:**

The enable signal at the S\_OSSD\_Out output can be SAFETRUE even if there is no TRUE signal at TestExecuted.

The automatic test has to be performed with positive results for the safety-related sensor to function correctly.

**Connection:** Connect the TestExecuted output to a global I/O variable or via additional functions/function blocks in order to transmit the signal externally for diagnostic purposes (e.g., controlling a signal lamp).

| Value | Description |
| --- | --- |
| TRUE | The automatic sensor test has been performed successfully.  **NOTE:**  After a successful test, TestExecuted retains the TRUE status until a new sensor test is requested at the StartTest input. |
| FALSE | The automatic sensor test   * has not yet been performed. * **or** is currently in progress * **or** has been carried out with errors.  Refer to the hazard message below this table. |

| WARNING | |
| --- | --- |
|  | **NON-CONFORMANCE TO SAFETY FUNCTION REQUIREMENTS**   * Regularly check the safety-related sensor by periodically switching the StartTest input from FALSE to TRUE in accordance with the risk analysis. * Make certain that appropriate procedures and measures have been established to help avoid hazardous situations when applying TRUE to the StartTest input.   **Failure to follow these instructions can result in death, serious injury, or equipment damage.** |
