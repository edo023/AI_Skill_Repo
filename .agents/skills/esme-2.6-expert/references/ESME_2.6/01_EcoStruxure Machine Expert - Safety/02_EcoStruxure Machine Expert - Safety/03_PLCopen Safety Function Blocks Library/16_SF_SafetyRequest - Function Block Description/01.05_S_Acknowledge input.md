# S\_Acknowledge input

State-controlled input for the status feedback of the connected safety-related periphery (confirmation message).

Data type: SAFEBOOL

Initial value: SAFEFALSE

**Connection**: Connect this input to the feedback signal (confirmation signal) of the safety-related periphery, either directly or via other safety-related functions/function blocks. Constant SAFETRUE or SAFEFALSE values are not permitted.

If the function block is activated (Activate = TRUE), the possible values at this input lead to the following function block behavior:

| Value | Description |
| --- | --- |
| TRUE | The connected safety-related periphery signals execution of the safety-related function. |
| FALSE | Feedback of the connected safety-related periphery about execution of the safety-related function is not present. |

## Single-channel or two-channel feedback signal from the periphery?

Irrespective of whether there is a single-channel or two-channel feedback signal between the safety-related periphery and the Safety Logic Controller, the S\_Acknowledge input is only connected to one signal.

A two-channel signal must be evaluated or monitored as follows:

* Either through a suitable upstream function block, e.g., SF\_Equivalent, if the connected periphery has two N/C contacts.
* Or the evaluation of equivalence or antivalence must be performed in the safety-related input device.
