# S\_MutingLamp input

State-controlled input for the feedback signal from the muting lamp.

Data type: SAFEBOOL

Initial value: SAFEFALSE

The signal at this input indicates whether the connected muting lamp is functioning correctly or not.

**Connection:** Connect this input to the feedback signal of a muting lamp connected to the Safety Logic Controller, either directly or via other safety-related functions/function blocks. It can be connected to the SAFETRUE constant if your risk analysis for the application indicates that a muting lamp with feedback signal is not necessary.

| Value | Description |
| --- | --- |
| SAFETRUE | Muting lamp is in working order (irrespective of whether the muting lamp is being controlled or not). |
| SAFEFALSE | Muting lamp is non-functional. |

## Single-channel or two-channel muting lamp?

Irrespective of whether the muting lamp provides a single-channel or two-channel feedback signal, the S\_MutingLamp input is only connected to one signal.

If a two-channel muting lamp is used, either the input channels of the safety-related input device must be parameterized accordingly or the SF\_Equivalent or SF\_Antivalent function blocks must be connected upstream for evaluating or monitoring the two-channel signal (e.g., SF\_Equivalent function block for equivalence monitoring).
