# SafetyDemand output

Output for signaling "safety-related function requested" (request for the stop function at the S\_EnableSwitchOut output of the function block).

This output indicates whether the safety chain is interrupted and as a result, the attention of the operator is required. The operator must verify the safety equipment and ensure that the safety chain is closed again.

Data type: BOOL

Initial value: FALSE

**Connection:** Connect the SafetyDemand output to an output terminal of your application, either directly or via other standard functions/function blocks.

| Value | Description |
| --- | --- |
| TRUE | The safety-related function is requested (enable switch is in switching stage 2), or the function block is in one of the following diagnostic states.   * 8802 * 8812 * 8822 * 8832 (status "Wait for equivalence". In case of a following equivalent error, SafetyDemand switches back to FALSE.)  Refer to the topic ["Diagnostic codes"](codes_EnableSwitch_SE.html#codes_EnableSwitch_SE) for details about these states as well as the FB/environmental conditions. |
| FALSE | The safety-related function is not requested. |
