# ET\_SafeOSState - General Information

## Overview

|  |  |
| --- | --- |
| Type: | List type |
| Available as of: | V1.0.0.0 |

## Description

This enumeration lists the possible SafeOSState values describing the state of the safety-related application.

## Enumeration Elements

| Name | Value (hex) | Description |
| --- | --- | --- |
| Invalid | 00 | Indicates an invalid SafeOSState value. |
| On | 0F | Indicates that the Safety Logic Controller is powered on, but no valid program is stored in the Safety Logic Controller memory. No program data and no system data are available. |
| Loading | 33 | Indicates the state Loading. |
| SafeStop | 55 | Indicates that the program is loaded but not executed (defined safe stop). |
| SafeRun | 66 | Indicates that the program is executed (defined safe run). |
| DebugHalt | 99 | Indicates single cycle operations in debug mode. |
| DebugStop | AA | Indicates that the program is loaded, but not executed. The debug mode is active. |
| DebugRun | CC | Indicates that the program is executed in debug mode. |
| NoExecution | F0 | Indicates that the program is stored on the Safety Logic Controller and is automatically executed in a few execution cycles. |
| DebugOffline | FF | Indicates the state DebugOffline. |
