# FB\_RemoteController - General Information

## Overview

|  |  |
| --- | --- |
| Type: | Function block |
| Available as of: | V1.0.0.0 |

## Task

The function block is used to read the status of an SLC and execute maintenance operations.

## Description

The function block allows you to query the status of an SLC and execute several maintenance operations.

For correct operation, set the inputs i\_ifSafeLogic and i\_sPassword first and enable the function block via the input i\_xEnable afterwards. If a command has to be sent to the SLC, then set the input i\_etUserCommand and execute the request via a rising edge on i\_xExecute.

If the memory key of the SLC does not yet have a password, you must first set a password via the command ET\_UserCommand.SetSafeKeyPassword and the input i\_sNewPassword.

NOTE: If you want to set the password for a memory key, Sercos must be in communication phase 2, 3 or 4.

The function block has to be called until q\_xDone or q\_xError changes its value to TRUE (and q\_xActive changes to FALSE). If no error was detected, then the status of the SLC can be evaluated by using the structure at the output q\_stStatus. Errors detected during the process of the command request can be analyzed with q\_etDiag and q\_etDiagExt.

If an error is detected, no further commands are processed by the function block and it can only be reset by disabling and then enabling it again via the input i\_xEnable.

## Interface

| Input | Data type | Description |
| --- | --- | --- |
| i\_xEnable | BOOL | A rising edge (FALSE -> TRUE) activates the POU, a falling edge (TRUE -> FALSE) deactivates the POU, clearing the errors and resetting the POU to its default state.  A deactivated POU does not execute any actions. |
| i\_xExecute | BOOL | A rising edge (FALSE -> TRUE) starts to process the command specified by the input i\_etUserCommand on the SLC specified by the input i\_ifSafeLogic. |
| i\_etUserCommand | [ET\_UserCommand](D-SE-0075078.html#D-SE-0075078) | Specifies the action that has to be executed by the SLC. |
| i\_ifSafeLogic | SystemConfigurationItf.IF\_SafeLogic | Specifies the SLC to interact with.  The device has to exist in the Devices tree in EcoStruxure Machine Expert. |
| i\_sPassword | STRING[80] | Password of the SLC to execute the command on. If i\_etUserCommand indicates that only the status is requested, then the password is not relevant. |
| i\_sNewPassword | STRING[80] | Specifies a new Memory Key password to be stored on the SLC. |

| Output | Data type | Description |
| --- | --- | --- |
| q\_xActive | BOOL | TRUE: The POU is active and has to be executed further.  FALSE: The POU is inactive. |
| q\_xReady | BOOL | TRUE: The POU is ready to operate and can accept user commands.  FALSE: The function block is not ready to accept user commands. |
| q\_xBusy | BOOL | TRUE: The POU executes the issued user command.  FALSE: The POU is waiting for further user commands. |
| q\_xDone | BOOL | TRUE: The user command has been executed.  FALSE: The user command is being executed, or none has been issued yet. |
| q\_stStatus | [ST\_StatusResponse](D-SE-0075098.html#D-SE-0075098) | Last status reported by the SLC. |
| q\_etDiag | [GD.ET\_Diag](../../../../../api/crossBook?lang=en-US&virtualBookName=PD.Lib.GlobalDiagnostic&topicID=D_SE_0076228) | General, library-independent statement on the diagnostic.  A value unequal to GD.ET\_Diag.Ok corresponds to a diagnostic message. |
| q\_etDiagExt | [ET\_DiagExt](D-SE-0075070.html#D-SE-0075070) | POU-specific output on the diagnostic.  q\_etDiag = GD.ET\_Diag.Ok -> status message  q\_etDiag <> GD.ET\_Diag.Ok -> diagnostic message |
| q\_sMsg | STRING[80] | Event-triggered message that provides more detailed information on the diagnostic state. |
| q\_stStatusExtended | [ST\_StatusExtendedResponse](ST_StatusExtendedResponse-GeneralIn-55650698.html) | Last extended status reported by TM5CSLC300FS or TM5CSLC400FS. |

## CommandFailedToExecute

|  |  |
| --- | --- |
| Enumeration name: | CommandFailedToExecute |
| Enumeration value: | 31 |
| Description: | A command was successfully sent to the SLC but terminated unsuccessfully. |

| Cause | Solution |
| --- | --- |
| No, or otherwise incorrect SLC password at the input i\_sPassword. | Verify the password at the input i\_sPassword and correct it if necessary. If the password has not already been set, set the password and try again. |
| The parameter RemoteControlAllowed is not set to "Yes-ATTENTION" in the active SLC program. | Set the parameter RemoteControlAllowed to "Yes-ATTENTION". |
| The requested command preconditions are not met. For example, a module replacement acknowledge was issued even though there are no replaced modules to be acknowledged. | Only issue commands allowed given the state of the SLC. |
| Incorrect order of commands executed. | Execute the commands in the correct order.  For example: first confirm the memory key, then the modules. |
